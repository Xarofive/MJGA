<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Задача 4.4: Напоминание о гидратации</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Потренироваться объединять условия через <code>||</code> и соблюдать порядок проверок: сначала валидность, потом бизнес-логика.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>
<p style="margin-bottom: 10px;">Фитнес-приложение решает, нужно ли брать бутылку воды на прогулку по температуре и времени на улице.</p>
<p style="margin: 0;"><strong>Ваша задача:</strong> вернуть рекомендацию по двум параметрам: <code>temp</code> и <code>minutesOutside</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Оператор ||</h4>
<p><code>||</code> означает «или»: достаточно, чтобы выполнилось хотя бы одно условие.</p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>if (temp &gt; 30 || minutesOutside &gt; 90) {
    return "bring bottle";
}</code></pre>
<p style="margin-bottom: 0;">Невалидные данные проверяйте первыми, иначе они могут попасть в обычную ветку.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<p><strong>Создайте метод с точной сигнатурой:</strong></p>
<pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px; overflow-x: auto;"><code>static String hydrationReminder(int temp, int minutesOutside)</code></pre>
<ul>
	<li><code>"invalid"</code> — температура ниже -30°C или время отрицательное;</li>
	<li><code>"bring bottle"</code> — температура выше 30°C или время больше 90 минут;</li>
	<li><code>"optional"</code> — температура от 20°C до 30°C включительно или время от 30 до 90 минут включительно;</li>
	<li><code>"no need"</code> — все остальные допустимые случаи.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px; overflow-x: auto;"><code>hydrationReminder(35, 20)  -> "bring bottle"
hydrationReminder(18, 120) -> "bring bottle"
hydrationReminder(25, 40)  -> "optional"
hydrationReminder(15, 10)  -> "no need"
hydrationReminder(-35, 20) -> "invalid"</code></pre>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>
<p style="margin-bottom: 0;">Почему для <code>"bring bottle"</code> нужен <code>||</code>, а не <code>&amp;&amp;</code>? Нужно ли ждать одновременно жару и долгую прогулку?</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура метода</h2>
<pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px; overflow-x: auto;"><code>static String hydrationReminder(int temp, int minutesOutside) {
    // 1. invalid
    // 2. bring bottle
    // 3. optional
    // 4. no need
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>
<div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px; margin: 16px 0;">
<ul style="margin-bottom: 0;">
	<li>☐ Невалидные данные проверяются первыми</li>
	<li>☐ Для обязательной воды используется логика <code>temp &gt; 30 || minutesOutside &gt; 90</code></li>
	<li>☐ <code>30</code> градусов и <code>90</code> минут относятся к <code>"optional"</code></li>
	<li>☐ Для <code>"optional"</code> учтены оба диапазона</li>
	<li>☐ Метод возвращает строку и ничего не печатает</li>
</ul>
</div>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
