<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Задача 4.6: Адаптивное качество стриминга</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Потренироваться писать вложенную бизнес-логику без потери читаемости.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>
<p style="margin-bottom: 10px;">Видеосервис выбирает качество потока по скорости интернета и состоянию сети. При перегрузке сети качество ограничивается.</p>
<p style="margin: 0;"><strong>Ваша задача:</strong> вернуть качество стриминга по параметрам <code>speed</code> и <code>congested</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Разделение веток логики</h4>
<p>Когда правила сильно отличаются для двух режимов, удобно сначала разделить эти режимы, а потом проверять диапазоны внутри каждого.</p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>if (congested) {
    // правила для перегруженной сети
} else {
    // правила для нормальной сети
}</code></pre>
<p style="margin-bottom: 0;">Невалидную скорость нужно проверить до выбора режима.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<p><strong>Создайте метод с точной сигнатурой:</strong></p>
<pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px; overflow-x: auto;"><code>static String streamQuality(int speed, boolean congested)</code></pre>
<ul>
	<li><code>"invalid"</code> — скорость <code>&lt;= 0</code>;</li>
	<li>если сеть перегружена: <code>"low"</code> при скорости &lt; 15, иначе <code>"medium"</code>;</li>
	<li>если сеть не перегружена: <code>"low"</code> при скорости &lt; 8;</li>
	<li><code>"medium"</code> — от 8 до 14 включительно;</li>
	<li><code>"high"</code> — от 15 до 24 включительно;</li>
	<li><code>"ultra"</code> — от 25 и выше.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px; overflow-x: auto;"><code>streamQuality(30, false) -> "ultra"
streamQuality(18, true)  -> "medium"
streamQuality(5, false)  -> "low"
streamQuality(9, false)  -> "medium"
streamQuality(20, false) -> "high"
streamQuality(0, false)  -> "invalid"</code></pre>

<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>
<p style="margin-bottom: 0;">Почему при перегруженной сети нет вариантов <code>"high"</code> и <code>"ultra"</code>, даже если скорость высокая?</p>
</div>

    // 1. invalid
    // 2. congested branch
    // 3. normal network branch
}</code></pre>
