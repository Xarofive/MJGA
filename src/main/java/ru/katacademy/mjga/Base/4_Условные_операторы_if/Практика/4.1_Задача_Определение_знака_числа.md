<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Задача 4.1: Определение знака числа</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Научиться использовать цепочку <code>if</code> / <code>else if</code> / <code>else</code> для классификации числа.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>
<p style="margin-bottom: 10px;">В финансовых расчётах, аналитике и обработке датчиков часто нужно понять, значение положительное, отрицательное или равно нулю.</p>
<p style="margin: 0;"><strong>Ваша задача:</strong> создать метод <code>describeSign</code>, который возвращает текстовое описание знака числа.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Цепочка условий</h4>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>if (condition) {
    return "first";
} else if (anotherCondition) {
    return "second";
} else {
    return "fallback";
}</code></pre>
<p style="margin-bottom: 0;">Условия проверяются сверху вниз. Выполняется первый подходящий блок, остальные пропускаются.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<p><strong>Создайте метод с точной сигнатурой:</strong></p>
<pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px; overflow-x: auto;"><code>static String describeSign(int n)</code></pre>
<ul>
	<li>если <code>n &gt; 0</code>, вернуть <code>"positive"</code>;</li>
	<li>если <code>n == 0</code>, вернуть <code>"zero"</code>;</li>
	<li>если <code>n &lt; 0</code>, вернуть <code>"negative"</code>;</li>
	<li>обязательно используйте цепочку <code>if</code> / <code>else if</code> / <code>else</code>;</li>
	<li>отправьте только метод, без класса <code>Main</code> и без <code>import</code>.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px; overflow-x: auto;"><code>describeSign(9)  -> "positive"
describeSign(-5) -> "negative"
describeSign(0)  -> "zero"
describeSign(1)  -> "positive"</code></pre>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>
<p style="margin-bottom: 0;">Почему ноль нужно обработать отдельно? Почему его нельзя отнести ни к положительным, ни к отрицательным числам?</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура метода</h2>
<pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px; overflow-x: auto;"><code>static String describeSign(int n) {
    if (/* положительное */) {
        return "positive";
    } else if (/* ноль */) {
        return "zero";
    } else {
        return "negative";
    }
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>
<div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px; margin: 16px 0;">
<ul style="margin-bottom: 0;">
	<li>☐ Метод называется <code>describeSign</code></li>
	<li>☐ Возвращаемый тип — <code>String</code></li>
	<li>☐ Есть отдельная проверка для <code>n == 0</code></li>
	<li>☐ Строки совпадают точно: <code>"positive"</code>, <code>"zero"</code>, <code>"negative"</code></li>
	<li>☐ Нет класса <code>Main</code>, импортов и вывода в консоль</li>
</ul>
</div>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">

<ul style="margin-bottom: 0;">
	<li>Не используйте циклы и другие операторы выбора</li>
	<li>Не используйте <code>System.out.println</code></li>
	<li>Не возвращайте русские слова вместо требуемых строк</li>
