<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔢 Безопасное преобразование строки в число</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться превращать пользовательский текст в <code>int</code> и сообщать о проблемах через понятные исключения, не теряя контекст исходного ввода.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;">
<p style="margin: 0;">📖 <strong>Пререквизит:</strong> перед решением повторите <code>String.trim()</code>, <code>Integer.parseInt</code>, <code>try/catch</code>, <code>IllegalArgumentException</code> и <code>NumberFormatException</code>.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>

<p style="margin-bottom: 10px;">Пользователь вводит возраст, количество товаров или номер страницы. Для программы это сначала обычная строка: <code>"42"</code>, <code>" -7 "</code> или ошибочный текст вроде <code>"12px"</code>.</p>

<p style="margin: 0;"><strong>Ваша задача:</strong> написать метод, который очищает строку, преобразует её в число и выбрасывает разные понятные ошибки для <code>null</code>, пустого ввода и нечислового значения.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Преобразование через Integer.parseInt</h4>

<p><code>Integer.parseInt(text)</code> преобразует строку в целое число. Если строка не является корректным числом, Java выбрасывает <code>NumberFormatException</code>.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">int value = Integer.parseInt("42"); // 42</code></pre>

<p style="margin-bottom: 0;">Перед преобразованием используйте <code>trim()</code>, чтобы ввод <code>" 42 "</code> стал <code>"42"</code>.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🔍 Почему не возвращать -1 или 0 при ошибке</h4>

<p style="margin-bottom: 0;">Числа <code>-1</code> и <code>0</code> могут быть валидными данными. Исключение лучше: оно явно говорит, что произошла ошибка, и позволяет передать причину.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте метод безопасного преобразования строки в число:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура метода</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>static int safeParseInt(String text)</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Если <code>text == null</code>, выбросьте <code>IllegalArgumentException</code> с текстом <code>"input is null"</code></li>
	<li>Удалите пробелы по краям через <code>trim()</code></li>
	<li>Если после очистки строка пустая, выбросьте <code>IllegalArgumentException</code> с текстом <code>"empty input"</code></li>
	<li>Если строка корректная, верните число</li>
	<li>Если <code>Integer.parseInt</code> выбросил <code>NumberFormatException</code>, выбросьте <code>IllegalArgumentException</code> с текстом, который содержит исходную строку</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Не используйте вывод на экран</li>
	<li>Не создавайте дополнительные методы</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
	<li>Не заменяйте ошибку специальным числом вроде <code>-1</code> или <code>0</code></li>
</ul>
</div>

<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>

<p>Почему <code>null</code> нужно проверять до вызова <code>trim()</code>? Что произойдёт, если поменять порядок?</p>

<p style="margin-bottom: 0;"><em>Подсказка: у <code>null</code> нет методов, поэтому <code>text.trim()</code> сразу приведёт к другой ошибке.</em></p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>

<pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px; overflow-x: auto;">
<code class="language-java">static int safeParseInt(String text) {
    // 1. Проверьте null
    // 2. Очистите строку через trim()
    // 3. Проверьте пустую строку
    // 4. Выполните Integer.parseInt внутри try/catch
    // 5. При NumberFormatException выбросьте IllegalArgumentException с исходным вводом
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>

<div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px; margin: 16px 0;">
<ul style="margin-bottom: 0;">
	<li>☐ Метод называется точно <code>safeParseInt</code></li>
	<li>☐ Сигнатура: <code>static int safeParseInt(String text)</code></li>
	<li>☐ <code>null</code> обрабатывается до <code>trim()</code></li>
	<li>☐ Пустая строка после <code>trim()</code> обрабатывается отдельно</li>
	<li>☐ Для некорректного числа выбрасывается <code>IllegalArgumentException</code></li>
	<li>☐ Сообщение об ошибке содержит исходную проблемную строку</li>
	<li>☐ В методе нет печати в консоль</li>
