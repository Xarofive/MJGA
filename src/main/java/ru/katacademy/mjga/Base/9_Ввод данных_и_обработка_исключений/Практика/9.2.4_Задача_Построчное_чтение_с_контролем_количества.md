<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📖 Построчное чтение с контролем количества</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться читать строки из <code>BufferedReader</code>, пропускать пустой ввод и контролировать, что данных хватило для результата нужного размера.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;">
<p style="margin: 0;">📖 <strong>Пререквизит:</strong> повторите <code>BufferedReader.readLine()</code>, <code>String.trim()</code>, массивы фиксированной длины, <code>IOException</code> и сигнатуру метода с <code>throws IOException</code>.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>

<p style="margin-bottom: 10px;">Метод читает текстовый источник, где важны только непустые строки. Нужно получить ровно <code>n</code> строк: если данных меньше, метод должен сообщить, сколько требовалось и сколько получилось.</p>

<p style="margin: 0;"><strong>Ваша задача:</strong> прочитать ровно <code>n</code> непустых строк из переданного <code>BufferedReader</code>, не закрывая источник и не печатая ничего в консоль.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">readLine и конец данных</h4>

<p><code>BufferedReader.readLine()</code> возвращает строку без символа перевода строки. Если данных больше нет, метод возвращает <code>null</code>.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">String line = reader.readLine();
if (line == null) {
    // источник закончился
}</code></pre>

<p style="margin-bottom: 0;">Так как чтение может выбросить <code>IOException</code>, метод должен объявить <code>throws IOException</code>.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🧹 Что считается пустой строкой</h4>

<p style="margin-bottom: 0;">Строка <code>""</code>, строка <code>"   "</code> и строка с табами после <code>trim()</code> становятся пустыми. Такие строки нужно пропускать и не считать найденными.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте метод построчного чтения с контролем количества:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура метода</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>static String[] readNLines(BufferedReader reader, int n) throws IOException</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Если <code>reader == null</code>, выбросьте <code>IllegalArgumentException("reader is null")</code></li>
	<li>Если <code>n &lt; 0</code>, выбросьте <code>IllegalArgumentException("negative count")</code></li>
	<li>Верните массив из ровно <code>n</code> непустых строк</li>
	<li>Перед проверкой пустоты применяйте <code>trim()</code></li>
	<li>Пустые строки и строки из пробелов пропускайте</li>
	<li>Если данных не хватило, выбросьте <code>IllegalArgumentException</code>, сообщение содержит требуемое и фактическое количество</li>
	<li>Ошибку чтения <code>IOException</code> не перехватывайте, а передавайте наружу через <code>throws IOException</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Не используйте <code>ArrayList</code> и другие коллекции</li>
	<li>Не закрывайте <code>reader</code> внутри метода</li>
	<li>Не используйте вывод на экран</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
</ul>
</div>

<div style="margin-left: 20px; color: #2c3e50;">Ввод <code>" alpha \n\n beta \n"</code>, <code>n = 2</code><br />

<div style="background: #fffde7; border: 1px solid #fbc02d; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57f17; margin-top: 0;">🔧 Мини-проверка локально</h4>

<p><code>StringReader</code> создаёт источник данных из строки в памяти. Это удобно для проверки без ввода с клавиатуры.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">BufferedReader reader = new BufferedReader(new StringReader(" alpha \n\nbeta\n"));
String[] lines = readNLines(reader, 2);</code></pre>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>

<p>Почему метод не должен закрывать <code>reader</code>, если он получил его как параметр?</p>

<p style="margin-bottom: 0;"><em>Подсказка: тот, кто создал ресурс, обычно и управляет его жизненным циклом.</em></p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>

<pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px; overflow-x: auto;">
<code class="language-java">static String[] readNLines(BufferedReader reader, int n) throws IOException {
    // 1. Проверьте reader и n
    // 2. Создайте String[] длиной n
    // 3. Читайте строки до конца или пока не собрано n строк
    // 4. trim() + пропуск пустых строк
    // 5. Если строк не хватило, выбросьте IllegalArgumentException с need/got
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>

<div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px; margin: 16px 0;">
<ul style="margin-bottom: 0;">
	<li>☐ Метод имеет <code>throws IOException</code></li>
	<li>☐ Результат имеет длину ровно <code>n</code></li>
	<li>☐ Пустые строки пропускаются</li>
	<li>☐ Строки очищаются через <code>trim()</code></li>
	<li>☐ При нехватке данных выбрасывается исключение с числами</li>
	<li>☐ Коллекции не используются</li>
	<li>☐ <code>reader</code> не закрывается внутри метода</li>
	<li>☐ В методе нет печати в консоль</li>
</ul>
</div>
