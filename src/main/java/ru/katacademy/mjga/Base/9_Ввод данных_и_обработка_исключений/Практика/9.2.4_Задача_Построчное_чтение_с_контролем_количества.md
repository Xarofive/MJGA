<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📖 Построчное чтение с контролем количества</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Научиться читать строки из <code>BufferedReader</code>, пропускать пустые строки и контролировать, что данных достаточно.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>
<p style="margin-bottom: 10px;">Многие форматы требуют фиксированного количества непустых строк: заголовки, настройки, блоки данных. Если строк меньше, программа должна вернуть понятную ошибку, а не молча продолжить с неполным результатом.</p>
<p style="margin: 0;"><strong>Задача:</strong> прочитать ровно <code>n</code> непустых строк из переданного reader.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><code>BufferedReader.readLine()</code> возвращает строку без символа перевода строки. Если данных больше нет, метод возвращает <code>null</code>.</p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code class="language-java">String line = reader.readLine();
if (line == null) {
    // конец данных
}</code></pre>
<p style="margin-bottom: 0;">Метод чтения может выбросить <code>IOException</code>, поэтому в сигнатуре задачи нужно указать <code>throws IOException</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте метод <code>readNLines</code>:</strong></p>
<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>static String[] readNLines(BufferedReader reader, int n) throws IOException</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования</h4>
<ul style="margin: 10px 0;">
	<li>Если <code>reader == null</code>, выбросьте <code>IllegalArgumentException("reader is null")</code></li>
	<li>Если <code>n &lt; 0</code>, выбросьте <code>IllegalArgumentException("negative count")</code></li>
	<li>Верните массив из ровно <code>n</code> непустых строк</li>
	<li>Перед проверкой пустоты применяйте <code>trim()</code></li>
	<li>Пустые строки и строки из пробелов пропускайте</li>
	<li>Если данных не хватило, выбросьте <code>IllegalArgumentException</code>, сообщение содержит требуемое и фактическое количество</li>
	<li>Ошибку чтения <code>IOException</code> не перехватывайте, а передавайте наружу через <code>throws IOException</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>
<ul style="margin: 10px 0;">
	<li>Не используйте <code>ArrayList</code> и другие коллекции</li>
	<li>Не закрывайте <code>reader</code> внутри метода</li>
	<li>Не используйте вывод на экран</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Примеры работы</h2>

<div style="background: linear-gradient(135deg, #e3f2fd 0%, #bbdefb 100%); border-radius: 10px; padding: 20px; margin: 20px 0; border: 1px solid #64b5f6;">
<p>Ввод <code>"first\nsecond\nthird\n"</code>, <code>n = 3</code> → <code>{"first", "second", "third"}</code></p>
<p>Ввод <code>" alpha \n\n beta \n"</code>, <code>n = 2</code> → <code>{"alpha", "beta"}</code></p>
<p>Ввод <code>"one\n \ntwo\n"</code>, <code>n = 3</code> → <code>IllegalArgumentException</code>, сообщение содержит <code>need 3</code> и <code>got 2</code></p>
<p style="margin-bottom: 0;">Ввод <code>"anything\n"</code>, <code>n = 0</code> → пустой массив</p>
</div>

<div style="background: #fffde7; border: 1px solid #fbc02d; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57f17; margin-top: 0;">🔧 Мини-проверка локально</h4>
<p><code>StringReader</code> создаёт источник данных из строки в памяти. Это удобно для проверки: вместо ввода с клавиатуры вы заранее задаёте текст, который будет читать <code>BufferedReader</code>.</p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code class="language-java">BufferedReader reader = new BufferedReader(new StringReader(" alpha \n\nbeta\n"));
String[] lines = readNLines(reader, 2);</code></pre>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки</h4>
<p><strong>Ошибка 1:</strong> добавить пустую строку в результат.</p>
<p><strong>Ошибка 2:</strong> создать массив больше нужного размера и вернуть лишние <code>null</code>.</p>
<p><strong>Ошибка 3:</strong> закрыть reader, который был передан извне.</p>
<p style="margin-bottom: 0;"><strong>Ошибка 4:</strong> проглотить <code>IOException</code> и вернуть неполные данные.</p>
</div>
