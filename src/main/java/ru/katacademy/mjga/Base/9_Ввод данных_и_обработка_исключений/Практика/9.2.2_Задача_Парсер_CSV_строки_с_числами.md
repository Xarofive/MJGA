<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📊 Парсер CSV-строки с числами</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться разбирать структурированную строку, валидировать каждый элемент и возвращать массив чисел без частично корректных результатов.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;">
<p style="margin: 0;">📖 <strong>Пререквизит:</strong> повторите <code>String.split</code>, <code>trim()</code>, массивы, индексы с нуля, <code>Integer.parseInt</code> и обработку <code>NumberFormatException</code>.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>

<p style="margin-bottom: 10px;">В настройках или импорте данных приходит строка вида <code>"10, -5, 0"</code>. В ней могут быть лишние пробелы, пустые элементы или нечисловые значения.</p>

<p style="margin: 0;"><strong>Ваша задача:</strong> превратить CSV-строку в <code>int[]</code>, а при ошибке указать, какой элемент сломал обработку.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Разбиение строки и массив результата</h4>

<p><code>split(",")</code> делит строку по запятым, а <code>trim()</code> очищает каждый элемент:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">String[] parts = input.split(",");
int[] numbers = new int[parts.length];</code></pre>

<p style="margin-bottom: 0;">Если второй элемент некорректен, его индекс равен <code>1</code>, потому что индексация начинается с нуля.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🧭 Ошибка должна вести к месту проблемы</h4>

<p style="margin-bottom: 0;">Сообщение об ошибке должно содержать индекс проблемного элемента. Для нечислового значения дополнительно включайте само значение, например <code>abc</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте метод для парсинга CSV-строки с целыми числами:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура метода</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>static int[] parseCsvInts(String input)</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Если <code>input == null</code>, выбросьте <code>IllegalArgumentException("input is null")</code></li>
	<li>Если строка пустая или состоит только из пробелов, выбросьте <code>IllegalArgumentException("empty input")</code></li>
	<li>Разделитель — запятая</li>
	<li>Пробелы вокруг каждого элемента нужно удалить через <code>trim()</code></li>
	<li>Если элемент пустой после <code>trim()</code>, выбросьте <code>IllegalArgumentException</code>, сообщение содержит индекс элемента</li>
	<li>Если элемент не является числом, выбросьте <code>IllegalArgumentException</code>, сообщение содержит индекс и проблемное значение</li>
	<li>При успехе верните массив <code>int[]</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Не используйте <code>ArrayList</code> и другие коллекции</li>
	<li>Не используйте вывод на экран</li>
	<li>Не создавайте дополнительные методы</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
</ul>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>

<p>Почему нельзя вернуть частично заполненный массив, если первые элементы уже распарсились, а один из следующих оказался ошибочным?</p>

<p style="margin-bottom: 0;"><em>Подсказка: частичный результат может выглядеть как успешные данные и скрыть проблему импорта.</em></p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>

<pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px; overflow-x: auto;">
<code class="language-java">static int[] parseCsvInts(String input) {
    // 1. Проверьте null
    // 2. Очистите всю строку и проверьте пустоту
    // 3. Разбейте строку через split(",")
    // 4. Создайте int[] нужной длины
    // 5. Для каждого элемента: trim(), проверка пустоты, parseInt
    // 6. При ошибке выбросьте IllegalArgumentException с индексом и значением
}</code></pre>
