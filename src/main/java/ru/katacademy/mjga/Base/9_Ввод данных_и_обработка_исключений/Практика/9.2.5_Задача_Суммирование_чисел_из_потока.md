<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔢 Суммирование чисел из потока</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться читать числовые строки из <code>BufferedReader</code>, суммировать их и выдавать ошибку с контекстом проблемной строки.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;">
<p style="margin: 0;">📖 <strong>Пререквизит:</strong> повторите <code>BufferedReader.readLine()</code>, цикл чтения до <code>null</code>, <code>Integer.parseInt</code>, <code>trim()</code>, <code>IOException</code> и <code>IllegalArgumentException</code>.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>

<p style="margin-bottom: 10px;">Текстовый источник содержит числа по одному на строку. Между ними могут быть пустые строки. Нужно просуммировать все числа, а если встретился мусор вроде <code>abc</code>, остановиться с понятной ошибкой.</p>

<p style="margin: 0;"><strong>Ваша задача:</strong> написать метод, который читает поток до конца, пропускает пустые строки, суммирует числа и не скрывает ошибки чтения.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Цикл чтения до конца потока</h4>

<p>Обычно строки читают в цикле, пока <code>readLine()</code> не вернёт <code>null</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">String line;
while ((line = reader.readLine()) != null) {
    String cleaned = line.trim();
}</code></pre>

<p style="margin-bottom: 0;">Каждую непустую строку можно преобразовать через <code>Integer.parseInt(cleaned)</code>.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Ноль как результат и отсутствие чисел</h4>

<p style="margin-bottom: 0;">Сумма может быть равна <code>0</code>, если числа взаимно компенсировались. Но если чисел вообще не было, это другая ситуация: нужно выбросить <code>IllegalArgumentException("no numbers")</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте метод суммирования целых чисел из текстового потока:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура метода</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>static int sumInts(BufferedReader reader) throws IOException</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Если <code>reader == null</code>, выбросьте <code>IllegalArgumentException("reader is null")</code></li>
	<li>Читайте строки до конца потока</li>
	<li>Пропускайте пустые строки и строки только из пробелов</li>
	<li>Каждую непустую строку парсите через <code>Integer.parseInt</code></li>
	<li>Суммируйте все успешно прочитанные числа</li>
	<li>Если строка не является числом, выбросьте <code>IllegalArgumentException</code>, сообщение содержит проблемную строку</li>
	<li>Если не найдено ни одного числа, выбросьте <code>IllegalArgumentException("no numbers")</code></li>
	<li>Ошибку чтения <code>IOException</code> передавайте наружу через <code>throws IOException</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Не используйте вывод на экран</li>
	<li>Не закрывайте <code>reader</code> внутри метода</li>
	<li>Не создавайте дополнительные методы</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
</ul>
</div>
