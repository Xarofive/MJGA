<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🧮 Калькулятор арифметических операций</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться различать типы ошибок: недопустимый аргумент обрабатывать через <code>IllegalArgumentException</code>, а математическую ошибку — через <code>ArithmeticException</code>.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;">
<p style="margin: 0;">📖 <strong>Пререквизит:</strong> повторите цепочки <code>if / else if / else</code>, сравнение строк через <code>equals</code>, целочисленное деление и базовые исключения Java.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3>

<p style="margin-bottom: 10px;">Метод получает два числа и строковый оператор. Если оператор известен, нужно вернуть результат. Если оператор неизвестен или деление невозможно, программа должна сообщить о проблеме правильно выбранным исключением.</p>

<p style="margin: 0;"><strong>Ваша задача:</strong> реализовать мини-калькулятор для четырёх операций без <code>switch</code> и без вывода в консоль.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Сравнение строк</h4>

<p>Строки в Java сравнивают через <code>equals</code>, а не через <code>==</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">if ("+".equals(op)) {
    return a + b;
}</code></pre>

<p style="margin-bottom: 0;">Такой порядок безопасен даже тогда, когда <code>op == null</code>.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🧯 Деление на ноль</h4>

<p style="margin-bottom: 0;">Для операции <code>"/"</code> сначала проверьте <code>b == 0</code>, и только потом выполняйте <code>a / b</code>. Иначе ошибка возникнет раньше, чем вы сформируете нужное сообщение.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте метод-калькулятор для целых чисел:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура метода</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>static int calcOp(int a, int b, String op)</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li><code>"+"</code> — вернуть <code>a + b</code></li>
	<li><code>"-"</code> — вернуть <code>a - b</code></li>
	<li><code>"*"</code> — вернуть <code>a * b</code></li>
	<li><code>"/"</code> — вернуть целочисленное деление <code>a / b</code></li>
	<li>Если <code>op == null</code> или операция неизвестна, выбросьте <code>IllegalArgumentException</code> с текстом, содержащим операцию</li>
	<li>Если операция <code>"/"</code> и <code>b == 0</code>, выбросьте <code>ArithmeticException("divide by zero")</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Не используйте <code>switch</code></li>
	<li>Не используйте вывод на экран</li>
	<li>Не создавайте дополнительные методы</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
</ul>
</div>

<h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4>

<p>Почему неизвестная операция и деление на ноль должны давать разные типы исключений?</p>

<p style="margin-bottom: 0;"><em>Подсказка: в первом случае неправильный аргумент, во втором — математически невозможное действие.</em></p>
</div>

    // 3. Для неизвестного op выбросьте IllegalArgumentException
    // 4. Не используйте switch
