<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🐞 Debug: breakpoints, шаги и переменные</h1>

<div style="background: linear-gradient(to right, #e3f2fd, #bbdefb); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #1565c0; margin-top: 0;">📝 TL;DR</h3>

<p style="margin: 10px 0;">Debug &mdash; это запуск программы с возможностью остановиться на выбранной строке и посмотреть, что происходит внутри кода.</p>

<p style="margin: 10px 0;">Breakpoint ставится кликом слева от строки кода. После запуска через <strong>Debug</strong> программа остановится на этой строке, и IDEA покажет значения переменных.</p>

<p style="margin: 10px 0;">Главные кнопки: <strong>Step Over</strong> выполняет текущую строку, <strong>Step Into</strong> заходит внутрь метода, <strong>Resume Program</strong> продолжает выполнение до следующей остановки.</p>
</div>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">🎯 Чему вы научитесь</h3>

<ul style="margin: 10px 0;">
	<li>Ставить и убирать breakpoint</li>
	<li>Запускать программу в режиме Debug</li>
	<li>Смотреть значения локальных переменных</li>
	<li>Выполнять код по шагам</li>
	<li>Понимать, почему <code>if</code> выбрал конкретную ветку</li>
	<li>Отслеживать изменение переменной в цикле</li>
	<li>Проверять состояние объекта после вызова метода</li>
</ul>
</div>

<div style="background: #fff3e0; border: 1px solid #ffb74d; border-radius: 8px; padding: 14px; margin: 16px 0;">

<h2 style="color: #34495e; margin-top: 28px;">1. Что такое breakpoint</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Точка остановки</h4>

<p style="margin-bottom: 10px;">Breakpoint &mdash; это место, где программа должна остановиться во время Debug. Вы ставите его кликом в серой области слева от строки кода.</p>

<p style="margin: 0;">Когда выполнение дойдёт до этой строки, IDEA остановит программу до выполнения строки. В этот момент можно смотреть переменные, стек вызовов и выполнять код по шагам.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">public class Main {
    public static void main(String[] args) {
        int price = 100;
        int discount = 15; // поставьте breakpoint здесь
        int finalPrice = price - discount;

        System.out.println(finalPrice);
    }
}</code></pre>
</div>

<h2 style="color: #34495e; margin-top: 28px;">2. Как запустить Debug</h2>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">✅ Порядок действий</h4>

<ol style="margin: 10px 0;">
	<li>Поставьте breakpoint на строку, которую хотите проверить.</li>
	<li>Нажмите зелёную кнопку рядом с <code>main</code>.</li>
	<li>Выберите <strong>Debug 'Main.main()'</strong>, а не Run.</li>
	<li>Дождитесь, пока программа остановится на breakpoint.</li>
	<li>Посмотрите значения переменных в нижнем окне <strong>Debug</strong>.</li>
</ol>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">💡 Run против Debug</h4>

<p><strong>Run</strong> просто выполняет программу от начала до конца.</p>

<p style="margin-bottom: 0;"><strong>Debug</strong> выполняет программу с остановками и позволяет смотреть состояние программы во время выполнения.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">3. Основные кнопки Debug</h2>

<table style="border-collapse: collapse; width: 100%; margin: 20px 0;">
	<thead>
		<tr style="background: #f0f0f0;">
			<th style="border: 1px solid #ddd; padding: 10px; text-align: left;">Действие</th>
			<th style="border: 1px solid #ddd; padding: 10px; text-align: left;">Что делает</th>
			<th style="border: 1px solid #ddd; padding: 10px; text-align: left;">Когда использовать</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td style="border: 1px solid #ddd; padding: 10px;"><code>Step Over</code></td>
			<td style="border: 1px solid #ddd; padding: 10px;">Выполняет текущую строку и переходит к следующей</td>
			<td style="border: 1px solid #ddd; padding: 10px;">Когда хотите идти по коду сверху вниз</td>
		</tr>
		<tr>
			<td style="border: 1px solid #ddd; padding: 10px;"><code>Step Into</code></td>
			<td style="border: 1px solid #ddd; padding: 10px;">Заходит внутрь метода, который вызывается на строке</td>
			<td style="border: 1px solid #ddd; padding: 10px;">Когда ошибка может быть внутри метода</td>
		</tr>
		<tr>
			<td style="border: 1px solid #ddd; padding: 10px;"><code>Step Out</code></td>
			<td style="border: 1px solid #ddd; padding: 10px;">Выходит из текущего метода обратно к месту вызова</td>
			<td style="border: 1px solid #ddd; padding: 10px;">Когда зашли в метод и уже поняли, что там происходит</td>
		</tr>
		<tr>
			<td style="border: 1px solid #ddd; padding: 10px;"><code>Resume Program</code></td>
			<td style="border: 1px solid #ddd; padding: 10px;">Продолжает выполнение до следующего breakpoint или конца программы</td>
			<td style="border: 1px solid #ddd; padding: 10px;">Когда текущий участок уже проверен</td>
		</tr>
	</tbody>
</table>

<h2 style="color: #34495e; margin-top: 28px;">4. Пример: почему if пошёл не туда</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">public class Main {
    static String accessStatus(int age, boolean hasTicket) {
        if (age &gt; 18 &amp;&amp; hasTicket) {
            return "allowed";
        }

        return "denied";
    }

    public static void main(String[] args) {
        String result = accessStatus(18, true);
        System.out.println(result);
    }
}</code></pre>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">🔍 Что проверять в Debug</h4>

<p style="margin-bottom: 10px;">Поставьте breakpoint на строку с <code>if</code> и запустите Debug. В переменных будет видно: <code>age = 18</code>, <code>hasTicket = true</code>.</p>

<p style="margin-bottom: 0;">Условие <code>age &gt; 18</code> ложно для возраста 18. Если по правилу 18 уже разрешено, условие нужно заменить на <code>age &gt;= 18</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">5. Пример: переменная в цикле</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">public class Main {
    static int sumEven(int[] numbers) {
        int sum = 0;

        for (int number : numbers) {
            if (number % 2 == 0) {
                sum = number; // ошибка: старое значение суммы теряется
            }
        }

        return sum;
    }

    public static void main(String[] args) {
        int result = sumEven(new int[] {2, 4, 5, 6});
        System.out.println(result);
    }
}</code></pre>
</div>

<div style="background: #e8eaf6; border: 1px solid #3f51b5; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #283593; margin-top: 0;">🔍 Как увидеть ошибку</h4>

<p style="margin-bottom: 10px;">Поставьте breakpoint внутри <code>if</code>. Нажимайте <strong>Step Over</strong> и следите за <code>sum</code>.</p>

<p style="margin: 0;">После числа 2 сумма станет 2, после числа 4 станет 4, после числа 6 станет 6. Значит, код не накапливает сумму. Нужно написать <code>sum = sum + number</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">6. Пример: состояние объекта</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">class Counter {
    int value;

    void increment() {
        value++;
    }
}

public class Main {
    public static void main(String[] args) {
        Counter counter = new Counter();

        counter.increment();
        counter.increment();

        System.out.println(counter.value);
    }
}</code></pre>
</div>

<p>Поставьте breakpoint на первый вызов <code>counter.increment()</code>. Нажмите <strong>Step Into</strong>, чтобы зайти внутрь метода и увидеть, как меняется поле <code>value</code>.</p>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🧠 Практическое правило</h4>

<p style="margin-bottom: 0;">Если ошибка связана с результатом метода, используйте <strong>Step Into</strong>. Если метод уже понятен и вы хотите пройти дальше, используйте <strong>Step Over</strong>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">7. Мини-чек-лист Debug</h2>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 15px; margin: 20px 0;">
<ul style="margin: 10px 0;">
	<li>Поставьте breakpoint до подозрительной строки</li>
	<li>Запускайте через <strong>Debug</strong>, а не через <strong>Run</strong></li>
	<li>Смотрите значения переменных до и после выполнения строки</li>
	<li>Для циклов проверяйте несколько итераций, а не только первую</li>
	<li>Для методов используйте <strong>Step Into</strong></li>
	<li>Для объектов раскрывайте объект в панели Variables и смотрите поля</li>
	<li>После проверки убирайте ненужные breakpoints</li>
</ul>
</div>
