# Информационный портрет студента после Java Base

Если конструкция есть только в проверяющей обвязке, она не считается изученной студентом, пока не объяснена в теории или не требуется в задании.

## 2. Какие навыки студент должен иметь после Base

### Что умеет писать руками

- Полный файл `Main.java` с `public class Main` и `public static void main(String[] args)`.
- Консольный вывод через `System.out.println` и `System.out.print`.
- Объявления переменных четырех изученных примитивных типов `int`, `double`, `boolean`, `char` и ссылочного типа `String`.
- Арифметические выражения, конкатенацию строк, простые формулы.
- Собственные `static` методы с параметрами и `return`.
- Условия `if`, `else if`, `else`, логические выражения с `&&`, `||`, `!`.
- Циклы `for`, `while`, `do-while`; `break` и `continue`.
- Классический и современный `switch`, включая `case ->`, `switch expression`, `yield`.
- Массивы: создание, чтение, запись, перебор, копирование, фильтрация, объединение, удаление диапазона.
- Базовую обработку строк: `length`, `charAt`, `substring`, `trim`, `split`, `join`, `toLowerCase`, `toUpperCase`, `contains`, `StringBuilder`.
- Unicode-aware приемы: `codePointCount`, `codePoints`, `offsetByCodePoints`, `Character.toChars`, `Character.charCount`.
- Ввод через `Scanner` и `BufferedReader`, вывод через `PrintWriter` в задачах с потоками.
- Парсинг строк в числа через `Integer.parseInt(text.trim())`.
- Обработку ошибок ввода через `try/catch`, `throw new IllegalArgumentException`, `throws IOException`.
- `ArrayList<String>`, `ArrayList<Integer>`: `add`, `get`, `set`, `remove`, `size`, перебор.
- `HashMap<K,V>`: `put`, `get`, `containsKey`, `remove`, `size`, `getOrDefault`, `keySet`.
- Простые классы: поля, объектные методы, создание через `new`, обращение через точку, несколько объектов одного класса.
- Работа со ссылками: присваивание ссылок, `null`, сравнение ссылок через `==`.

### Что умеет читать в чужом коде

- Базовую структуру Java-программы и порядок выполнения.
- Сигнатуры методов: имя, параметры, тип результата, `void`, `return`.
- Линейный код с переменными и вычислениями.
- Вложенные условия, ранние `return`, guard clauses.
- Циклы со счетчиком, циклы до условия, пропуски через `continue`, ранний выход через `break`.
- Код обработки массивов и строк с индексами.
- Стандартные ошибки индексации и обработки пустого/`null` входа.
- Код с `Scanner`, `BufferedReader`, `PrintWriter`.
- Код с `ArrayList` и `HashMap`, включая сортировку ключей через `Collections.sort`.
- Простые классы с состоянием и методами, которые меняют поля.
- Stack trace на уровне “тип исключения + сообщение + первая строка моего кода”.

### Какие типовые ошибки должен уметь находить

- Нет `main`, неверное имя класса `Main`, забыты фигурные скобки или `;`.
- Перепутаны `print` и `println`, лишняя строка или неверное экранирование.
- Неверный тип переменной, строка в кавычках вместо числа/boolean или наоборот.
- `=` вместо `==` в логике.
- Неверный порядок условий, забытый `else`, неверно обработанная граница.
- Бесконечный цикл из-за отсутствия обновления переменной.
- Ошибка на границах цикла: `i <= array.length` вместо `i < array.length`.
- Забытый `break` в классическом `switch`.
- Метод печатает вместо того, чтобы вернуть значение.
- Метод объявлен `void`, хотя должен вернуть результат; нет `return`.
- Смешивание `Scanner.nextInt()` и `nextLine()` без учета перевода строки.
- Парсинг числа без `trim`.
- Скрытие ошибки пустым `catch`.
- `ArrayIndexOutOfBoundsException`, `IndexOutOfBoundsException`, `NullPointerException`, `NumberFormatException`.
- Изменение исходного массива там, где требуется вернуть новую копию.
- Использование `String.length()` как количества видимых Unicode-символов.
- Разрезание строки по UTF-16 индексу внутри surrogate pair.
- `ArrayList<int>` вместо `ArrayList<Integer>`.
- Зависимость от порядка обхода `HashMap`.
- Вызов метода у `null` или ожидание, что `second = first` скопирует объект.
- Использование `static` для состояния, которое должно принадлежать конкретному объекту.

### Какие задачи должен решать без подсказок

- Напечатать точный текст в нужном формате.
- Объявить переменные нужных типов и выполнить простой расчет.
- Написать метод `sum`, `multiply`, `discount`, `average`, `buildGreeting`.
- Написать классификатор через `if/else` по числам, флагам и строкам.
- Посчитать сумму/произведение/степень/сумму цифр циклом.
- Найти первый подходящий элемент, пропустить неподходящие, остановить цикл по условию.
- Просуммировать элементы массива, посчитать четные/нечетные, отфильтровать массив.

## 3. Java-конструкции, которые можно считать разрешенными в Core

Можно использовать без повторного объяснения синтаксиса, но с коротким напоминанием при усложнении задачи:

- `int`, `double`, `boolean`, `char` и `String`. Важно: это не вся система типов Java; `byte`, `short`, `long`, `float` в Base не разобраны.
- Строковые литералы, экранирование, `+` для строк.
- `System.out.println`, `System.out.print`.
- `public class Main`, `public static void main(String[] args)`.
- `static` методы учебного уровня, параметры, `void`, тип результата, `return`.
- `if`, `else if`, `else`, ранний `return`.
- `==`, `!=`, `>`, `>=`, `<`, `<=`, `&&`, `||`, `!`.
- `for`, `while`, `do-while`, `break`, `continue`.
- Классический `switch`, `case`, `default`, `break`.
- Современный `switch`: `case ->`, несколько значений в `case`, `switch expression`, `yield`.
- Массивы `int[]`, `String[]`, доступ по индексу, `length`, перебор.
- `Arrays.toString`, `Arrays.copyOf`, `Arrays.copyOfRange`, `System.arraycopy`.
- Базовые операции со строками: `length`, `charAt`, `substring`, `trim`, `split`, `String.join`, `toLowerCase`, `toUpperCase`, `contains`, `equals`, `equalsIgnoreCase`.
- Unicode-приемы из модуля 8: `codePointCount`, `codePoints`, `offsetByCodePoints`.
- `StringBuilder`.
- `Scanner` для простого учебного ввода.
- `BufferedReader`, `InputStreamReader`, `StringReader`, `PrintWriter` для построчного и тестируемого ввода/вывода.
- `Integer.parseInt`, `Double.parseDouble` только если требуется аналогично `Integer.parseInt`; `Double.parseDouble` встречается меньше, лучше напомнить.
- `try/catch`, `throw new IllegalArgumentException`, `throws IOException`.
- `ArrayList<String>`, `ArrayList<Integer>`, `HashMap<String, Integer>`, `HashMap<String, String>`, вложенный `HashMap` в стиле модуля 11.
- `Collections.sort` для сортировки списка ключей.
- Простые классы, поля, объектные методы, `new`, `null`, ссылки, сравнение ссылок через `==`.
- IntelliJ IDEA: открыть проект, запустить `main`, поставить breakpoint, пройти код по шагам, прочитать stack trace.

`Math` нельзя считать полностью разрешенной библиотекой. В материалах явно объяснен `Math.min` в модулях 7-8, `Math.pow` в задаче 5.3 запрещен, `Math.abs` встречается в проверяющем коде. В Core можно использовать `Math.min` после короткого напоминания; остальные методы `Math` лучше объяснять по месту.

## 5. Темы, которые НЕ надо считать пройденными

Не считать пройденными в смысле “можно использовать без объяснения”:

- Продвинутое ООП: инкапсуляция через `private`, конструкторы, перегрузка конструкторов, наследование, переопределение, `super`, интерфейсы, абстрактные классы, полиморфизм.
- Остальные примитивные типы Java и их детали: `byte`, `short`, `long`, `float`, диапазоны значений, переполнение, суффиксы литералов `L`/`f`, явные/неявные преобразования между числовыми типами.
- `enum`.
- Полноценный Collections Framework: `List`, `Map`, `Set` как интерфейсы, `HashSet`, `LinkedList`, `TreeMap`, итераторы, сложность операций за пределами базового объяснения `HashMap`.
- Generics как самостоятельная тема.
- Stream API и лямбда-выражения.
- Работа с файлами и ресурсами через `Files`, `Path`, `File`, try-with-resources на практике.
- Maven/Gradle как система сборки.
- Unit-тестирование как навык написания тестов.
- Git.
- SQL, базы данных.
- Многопоточность.
- Сетевое программирование/API.
- Аннотации.
- Пакеты и структура больших Java-проектов как отдельная тема. `import` есть, но `package` и архитектура пакетов не разобраны как учебная цель.

При этом нельзя писать, что “ООП не изучалось вообще”: модуль 12 полноценно вводит базовые классы, объекты, поля, методы, состояние, `new`, ссылки и `null`. Нельзя писать, что “исключения не изучались вообще”: модуль 9 полноценно вводит базовые `try/catch`, `throw`, `throws`, checked/unchecked exceptions.

## 6. Пробелы, которые могут помешать старту Core

- Методы: студент писал много маленьких методов, но декомпозиция большой задачи на несколько собственных методов ограничена. В Core стоит быстро повторить “один метод - одна ответственность”, имена методов, параметры и возвращаемые значения.
- Массивы: тема глубокая, но риск остается в индексах, пустых массивах, копировании и мутации. Перед коллекциями объектов лучше дать короткую диагностическую практику.
- Строки: курс дает сильный Unicode-блок, но он сложный. Возможен разрыв между “знаю `String`” и “понимаю UTF-16/code points”. В Core использовать Unicode-детали только там, где они нужны, и повторить перед текстовыми задачами.
- Циклы: есть практика по всем видам циклов, но `while` с ручным обновлением и `continue` чаще дают ошибки. В Core не усложнять ввод/коллекции до проверки уверенности с циклами.
- Типы: в Base есть только 4 примитива (`int`, `double`, `boolean`, `char`) и `String`; нет `byte`, `short`, `long`, `float`, диапазонов и преобразований. В начале Core нужно дать полную карту примитивов Java, иначе нельзя уверенно обсуждать память, переполнение, numeric promotion, `long`-идентификаторы и `float`/`double`.
- Исключения: базово изучены, но `try-with-resources`, иерархия исключений и проектирование ошибок не закреплены. В Core нельзя сразу требовать “правильную exception architecture”.
- Коллекции: `ArrayList` и `HashMap` изучены прикладно, но нет общего Collections Framework. Если Core начнет с `List`/`Map` интерфейсов, их надо объяснить.
- ООП: базовые классы есть, но нет инкапсуляции и конструкторов. Core не должен молча использовать `private` поля, getters/setters, constructors, inheritance.
- IDE/debug: материал есть в конце курса, но навык требует практики. На старте Core полезно дать задачу “найти баг через debugger”.
- Git отсутствует. Если Core предполагает работу в репозитории, Git нужно вводить отдельно.
- Терминал есть только на уровне проверки JDK. Командная строка как рабочий инструмент не закреплена.
- Декомпозиция задачи: студент решал алгоритмические задачи, но в основном с заданной сигнатурой и ограниченным контекстом. В Core нужно постепенно увеличивать самостоятельность постановки решения.

## 7. Итоговый портрет студента после Java Base

Студент после Base умеет писать небольшие Java-программы и отдельные методы, работать с переменными, условиями, циклами, массивами, строками, вводом, базовыми исключениями, `ArrayList`, `HashMap` и простыми классами.

Студент после Base знает базовую структуру Java-кода, назначение JDK/JVM, работу `main`, четыре примитивных типа (`int`, `double`, `boolean`, `char`), `String`, отличие примитивов от ссылок, базовую модель объекта, `null`, основные runtime-ошибки и способы найти место ошибки через debugger/stack trace.

Студент после Base еще НЕ знает на уровне уверенного применения полную систему примитивов Java (`byte`, `short`, `long`, `float`), диапазоны и преобразования числовых типов, продвинутое ООП, конструкторы, модификаторы доступа, наследование, интерфейсы, `enum`, общий Collections Framework, generics как тему, Stream API, файлы, Maven/Gradle, Git, SQL и unit-тестирование.

В Core можно опираться на базовый синтаксис, методы, условия, циклы, массивы, строки, простой ввод, базовые исключения, `ArrayList`/`HashMap`, простые классы и понимание ссылок.

В Core нельзя молча использовать `byte`, `short`, `long`, `float`, числовые диапазоны и преобразования, `private` поля, constructors, getters/setters, `List`/`Map` как интерфейсы, `Set`, `enum`, inheritance, interfaces, lambdas, streams, file API, Maven/Gradle, Git-команды и JUnit. Эти темы нужно вводить явно.
