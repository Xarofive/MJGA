<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🛡️ Удаление диапазона без изменения исходного массива</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Закрепить удаление диапазона и отдельно проверить, что результат не разделяет изменяемое состояние с исходным массивом.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Сценарий</h2>
<p>Метод должен удалить диапазон, но при этом быть безопасным: даже если границы неверные и фактически удалять нечего, возвращать нужно независимую копию, а не исходный объект.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<ul>
	<li>Массивы передаются в методы как ссылки на объекты.</li>
	<li>Возврат исходного массива может привести к неожиданной мутации снаружи.</li>
	<li>При <code>array == null</code> копировать нечего, поэтому возвращается <code>null</code>.</li>
	<li>Диапазон <code>[from, to)</code> удаляет элементы от <code>from</code> до <code>to - 1</code>.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><strong>Реализуйте метод:</strong></p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">static int[] removeRangeSafe(int[] array, int from, int to)</code></pre>
<ul>
	<li>Метод удаляет диапазон <code>[from, to)</code>.</li>
	<li>При некорректных границах возвращает независимую копию исходного массива.</li>
	<li>При корректных границах возвращает новый массив без удаляемого диапазона.</li>
	<li>Возвращённый массив не должен быть тем же объектом, что и <code>array</code>.</li>
	<li>Если <code>array == null</code>, верните <code>null</code>.</li>
	<li>Решение должно содержать только метод без вывода в консоль.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<ul>
	<li><code>removeRangeSafe(new int[]{0, 1, 2, 3, 4}, 1, 4)</code> → <code>[0, 4]</code></li>
	<li><code>removeRangeSafe(new int[]{1, 2, 3}, -1, 2)</code> → <code>[1, 2, 3]</code>, но это новый массив</li>
	<li><code>removeRangeSafe(null, 0, 1)</code> → <code>null</code></li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🤔 Момент для размышления</h2>
<p>Почему при некорректных границах нельзя просто вернуть <code>array</code>, если содержимое результата такое же?</p>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>
<ol>
	<li>Сначала обработайте <code>null</code>.</li>
	<li>Проверьте границы диапазона.</li>
	<li>Если границы неверные, верните копию исходного массива.</li>
	<li>Если границы верные, создайте новый массив нужной длины.</li>
	<li>Скопируйте элементы до и после удаляемого диапазона.</li>
	<li>Верните новый массив.</li>
</ol>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">

	<li><label><input type="checkbox"> Метод называется <code>removeRangeSafe</code>.</label></li>

	<li><label><input type="checkbox"> При корректных границах удаляется диапазон <code>[from, to)</code>.</label></li>
	<li><label><input type="checkbox"> В методе нет вывода в консоль.</label></li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Технические ограничения</h2>
<ul>
	<li>Не возвращайте <code>array</code>, кроме случая <code>null</code>.</li>
	<li>Не изменяйте исходный массив.</li>
	<li>Не используйте вывод в консоль.</li>
	<li>Откажитесь от включительного понимания индекса <code>to</code>.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🐛 Отладка: если что-то не работает</h2>
<ul>
	<li>Если тест проверяет ссылку и падает, убедитесь, что возвращается новый массив.</li>
	<li>Если падает <code>NullPointerException</code>, обработайте <code>null</code> самым первым условием.</li>
	<li>Если удаляется неверный фрагмент, проверьте правило <code>[from, to)</code>.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">💪 Дополнительный вызов (после сдачи задачи)</h2>
<p>Добавьте отдельную проверку, что <code>result != array</code> для корректных и некорректных границ.</p>

<h2 style="color: #34495e; margin-top: 28px;">🎨 Где это пригодится</h2>
<p>Безопасные копии нужны в библиотечных методах, бизнес-логике и API, где вызывающий код не должен получить доступ к внутреннему изменяемому состоянию. Это базовая идея защитного программирования.</p>
