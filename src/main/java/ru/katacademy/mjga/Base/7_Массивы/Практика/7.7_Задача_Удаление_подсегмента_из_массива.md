<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">✂️ Удаление диапазона из массива</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Научиться удалять диапазон элементов, создавая новый массив без выбранного участка.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Сценарий</h2>
<p>Нужно убрать из массива непрерывный фрагмент: например, удалить несколько элементов из середины списка. Так как длина массива в Java не меняется, результат создаётся как новый массив.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<ul>
	<li>Диапазон <code>[from, to)</code> включает <code>from</code>, но не включает <code>to</code>.</li>
	<li>Количество удаляемых элементов равно <code>to - from</code>.</li>
	<li>При некорректных границах нужно вернуть копию исходного массива.</li>
	<li>Порядок оставшихся элементов должен сохраниться.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><strong>Реализуйте метод:</strong></p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">static int[] removeRange(int[] array, int from, int to)</code></pre>
<ul>
	<li><code>from</code> — начальный индекс удаляемого диапазона, включительно.</li>
	<li><code>to</code> — конечный индекс, не включительно.</li>
	<li>Если <code>from &gt;= to</code>, <code>from &lt; 0</code> или <code>to &gt; array.length</code>, верните копию исходного массива.</li>
	<li>Иначе верните новый массив без элементов с индексами от <code>from</code> до <code>to - 1</code>.</li>
	<li>Порядок оставшихся элементов должен сохраниться.</li>
	<li>Решение должно содержать только метод без вывода в консоль.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<ul>
	<li><code>removeRange(new int[]{0, 1, 2, 3, 4, 5, 6}, 2, 5)</code> → <code>[0, 1, 5, 6]</code></li>
	<li><code>removeRange(new int[]{1, 2, 3}, 0, 2)</code> → <code>[3]</code></li>
	<li><code>removeRange(new int[]{1, 2, 3}, 1, 1)</code> → <code>[1, 2, 3]</code></li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🤔 Момент для размышления</h2>
<p>Почему <code>to</code> удобнее считать не включительным индексом, а границей после последнего удаляемого элемента?</p>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>
<ol>
	<li>Проверьте корректность границ.</li>
	<li>При ошибочных границах верните копию исходного массива.</li>
	<li>Создайте результат длиной <code>array.length - (to - from)</code>.</li>
	<li>Скопируйте элементы до <code>from</code>.</li>
	<li>Скопируйте элементы после <code>to</code>.</li>
	<li>Верните результат.</li>
</ol>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>

	<li><label><input type="checkbox"> Метод называется <code>removeRange</code>.</label></li>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Технические ограничения</h2>
<ul>
	<li>Не считайте <code>to</code> включительным индексом.</li>
	<li>Не изменяйте исходный массив.</li>
	<li>Не возвращайте массив неправильной длины с лишними нулями.</li>
	<li>Откажитесь от вывода в консоль.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🐛 Отладка: если что-то не работает</h2>
<ul>
	<li>Если удаляется лишний элемент, проверьте правило <code>[from, to)</code>.</li>
	<li>Если возникает выход за границы, сначала проверяйте <code>from</code> и <code>to</code>.</li>
	<li>Если порядок нарушен, проверьте индексы при копировании второй части.</li>
</ul>

<p>Удаление диапазона нужно при обработке списков, страниц данных, временных рядов, выделенных фрагментов и любых структур, где нужно исключить часть элементов, сохранив остальное.</p>
