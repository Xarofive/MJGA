<h1 style="text-align: center; color: #2c3e50;">new, ссылки и объекты в памяти</h1>

<p style="text-align: center; color: #666; margin-bottom: 30px;">Разбираем самый частый источник ошибок у новичков: объектная переменная хранит ссылку, а не сам объект.</p>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 TL;DR &mdash; Суть за 30 секунд</h3>
<ul>
	<li><code>new</code> создаёт новый объект.</li>
	<li>Переменная объектного типа хранит ссылку на объект.</li>
	<li>Присваивание <code>second = first</code> копирует ссылку, а не объект.</li>
	<li>Две переменные могут указывать на один и тот же объект.</li>
	<li><code>null</code> означает, что ссылки на объект нет.</li>
	<li>Вызов поля или метода через <code>null</code> приводит к <code>NullPointerException</code>.</li>
	<li><code>==</code> для объектов проверяет, одна ли это ссылка.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Что делает new и зачем он нужен</h2>

<p>Когда вы пишете <code>new ClassName()</code>, Java создаёт новый объект в памяти и возвращает ссылку на него.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Lamp lamp = new Lamp();</code></pre>

<p>Эта строка состоит из нескольких действий:</p>
<ol>
	<li>создать объект класса <code>Lamp</code>;</li>
	<li>выделить место под его поля;</li>
	<li>заполнить поля значениями по умолчанию;</li>
	<li>вернуть ссылку на объект;</li>
	<li>сохранить ссылку в переменную <code>lamp</code>.</li>
</ol>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 15px; margin: 20px 0;">
<p style="margin: 0;"><strong>💡 Аналогия:</strong> объект &mdash; квартира, ссылка &mdash; адрес квартиры. Переменная хранит адрес, по которому Java находит объект.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Ссылки против примитивов</h2>

<p>Примитивы копируют значение:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">int a = 10;
int b = a;
b = 20;

System.out.println(a); // 10
System.out.println(b); // 20</code></pre>

<p>Объектные переменные копируют ссылку:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Counter first = new Counter();
Counter second = first;

second.value = 10;

System.out.println(first.value);  // 10
System.out.println(second.value); // 10</code></pre>

<p><code>first</code> и <code>second</code> указывают на один объект. Поэтому изменение через одну переменную видно через другую.</p>

<h2 style="color: #34495e; margin-top: 30px;">Основные операции со ссылками</h2>

<h3 style="color: #34495e; margin-top: 20px;">Операция 1: создать новый объект</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Counter counter = new Counter();</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Операция 2: скопировать ссылку</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Counter alias = counter;</code></pre>

<p>Новый объект не создаётся. Появляется ещё одна переменная для доступа к тому же объекту.</p>

<h3 style="color: #34495e; margin-top: 20px;">Операция 3: проверить, один ли это объект</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">boolean same = counter == alias;</code></pre>

<p><code>==</code> отвечает на вопрос: “эти переменные указывают на один и тот же объект?”</p>

<h2 style="color: #34495e; margin-top: 30px;">null: ссылки нет</h2>

<p><code>null</code> &mdash; специальное значение для объектных переменных. Оно означает: переменная сейчас ни на какой объект не указывает.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Student student = null;</code></pre>

<p>Через <code>null</code> нельзя обращаться к полям и методам:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">student.name = "Alice"; // NullPointerException</code></pre>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #856404; margin-top: 0;">📌 Практическое правило</h4>
<p style="margin-bottom: 0;">Если переменная может быть <code>null</code>, перед обращением к полям и методам нужно либо проверить её, либо организовать код так, чтобы <code>null</code> туда не попадал.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Объекты в методах</h2>

<p>Когда объект передаётся в метод, копируется ссылка. Метод получает доступ к тому же объекту и может изменить его состояние.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Counter {
    int value;
}

static void increment(Counter counter) {
    counter.value++;
}

Counter counter = new Counter();
increment(counter);

System.out.println(counter.value); // 1</code></pre>

<p>Метод <code>increment</code> получил копию ссылки, но объект за этой ссылкой тот же самый.</p>

<h2 style="color: #34495e; margin-top: 30px;">Реальный кейс: один объект в разных частях программы</h2>

<h3 style="color: #34495e; margin-top: 20px;">Постановка задачи</h3>
<p>Есть счётчик событий. Его передают в несколько методов. Все методы должны увеличивать один общий счётчик, а не создавать свои отдельные копии.</p>

<h3 style="color: #34495e; margin-top: 20px;">Решение</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Counter {
    int value;

    void increment() {
        value++;
    }
}

static void processClick(Counter counter) {
    counter.increment();
}

static void processSubmit(Counter counter) {
    counter.increment();
}

Counter events = new Counter();

processClick(events);
processSubmit(events);

System.out.println(events.value); // 2</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Разбор решения</h3>
<ul>
	<li><code>events</code> хранит ссылку на один объект <code>Counter</code>.</li>
	<li>Оба метода получают ссылку на тот же объект.</li>
	<li>Каждый метод увеличивает поле <code>value</code> общего объекта.</li>
	<li>Итоговое значение равно <code>2</code>.</li>
</ul>

<h2 style="color: #34495e; margin-top: 30px;">Частые ошибки и как их избежать</h2>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 1: Ожидать копию объекта</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">Counter second = first;</code></pre>
<p style="margin-bottom: 0;">Эта строка копирует ссылку. Новый объект не создаётся.</p>
</div>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 2: Вызов метода через null</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">Counter counter = null;
counter.increment(); // NullPointerException</code></pre>
<p style="margin-bottom: 0;">Создайте объект или проверьте переменную на <code>null</code>.</p>
</div>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 3: Сравнивать содержимое через ==</h4>
<p style="margin-bottom: 0;"><code>==</code> сравнивает ссылки. Два разных объекта с одинаковыми полями всё равно дадут <code>false</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Мини-квиз для самопроверки</h2>

<div style="background: #f3e5f5; border: 1px solid #ce93d8; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #6a1b9a; margin-top: 0;">🧩 Вопрос 1: Что выведется?</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">Counter first = new Counter();
Counter second = first;
second.value = 7;

System.out.println(first.value);</code></pre>
<p style="margin-bottom: 0;">Ответ: <code>7</code>. Обе переменные указывают на один объект.</p>
</div>

<div style="background: #f3e5f5; border: 1px solid #ce93d8; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #6a1b9a; margin-top: 0;">🧩 Вопрос 2: Что вернёт сравнение?</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">Counter first = new Counter();
Counter second = new Counter();

System.out.println(first == second);</code></pre>
<p style="margin-bottom: 0;">Ответ: <code>false</code>. Это два разных объекта.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">📋 Шпаргалка: new, ссылки, null</h2>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">// новый объект
Counter first = new Counter();

// копия ссылки, не копия объекта
Counter second = first;

// новый независимый объект
Counter third = new Counter();

// сравнение ссылок
boolean same = first == second; // true

// ссылки нет
Counter empty = null;</code></pre>
