<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🧹 Нормализация пробелов вручную</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Закрепить нормализацию пробелов без регулярных выражений, используя посимвольный проход и <code>StringBuilder</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Сценарий</h2>
<p>Иногда регулярные выражения использовать нельзя или не хочется: нужно вручную пройти по строке, распознать пробельные символы и собрать аккуратный результат.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<ul>
	<li><code>Character.isWhitespace</code> распознаёт пробелы, табы и переводы строк.</li>
	<li><code>StringBuilder</code> подходит для пошаговой сборки строки.</li>
	<li>Нужно помнить, был ли перед текущим символом пробел.</li>
	<li>В начале и конце результата пробелов быть не должно.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><strong>Реализуйте метод:</strong></p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">static String normalizeSpacesManual(String s)</code></pre>
<ul>
	<li>Верните строку, где группы whitespace заменены одним пробелом.</li>
	<li>В начале и конце результата пробелов быть не должно.</li>
	<li>Используйте <code>Character.isWhitespace</code>.</li>
	<li>Используйте <code>StringBuilder</code>.</li>
	<li>Не используйте <code>split</code>, <code>replaceAll</code> и регулярные выражения.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<ul>
	<li><code>normalizeSpacesManual("  hello   world  ")</code> → <code>"hello world"</code></li>
	<li><code>normalizeSpacesManual("\tJava\nBase\t")</code> → <code>"Java Base"</code></li>
	<li><code>normalizeSpacesManual("")</code> → <code>""</code></li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🤔 Момент для размышления</h2>
<p>Какой флаг или состояние поможет понять, нужно ли добавить один пробел перед следующим словом?</p>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>
<ol>
	<li>Создайте <code>StringBuilder</code>.</li>
	<li>Идите по символам строки.</li>
	<li>Если символ пробельный, отметьте, что перед следующим словом может понадобиться пробел.</li>
	<li>Если символ не пробельный, при необходимости добавьте один пробел, затем сам символ.</li>
	<li>Верните собранную строку.</li>
</ol>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>

<ul style="margin-bottom: 0;">
	<li><label><input type="checkbox"> Используется <code>Character.isWhitespace</code>.</label></li>
	<li><label><input type="checkbox"> Используется <code>StringBuilder</code>.</label></li>
	<li><label><input type="checkbox"> Нет регулярных выражений.</label></li>
	<li><label><input type="checkbox"> Нет пробелов по краям результата.</label></li>

</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Технические ограничения</h2>
<ul>
	<li>Не используйте <code>split</code>, <code>replaceAll</code>, <code>replaceFirst</code> и регулярные выражения.</li>
	<li>Не собирайте строку через <code>+=</code> в цикле.</li>
	<li>Не оставляйте пробел в начале или конце результата.</li>
	<li>Откажитесь от проверки только символа <code>' '</code>: whitespace шире обычного пробела.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🐛 Отладка: если что-то не работает</h2>
<ul>
	<li>Если в начале есть пробел, добавляйте разделитель только когда результат уже не пустой.</li>
	<li>Если табы остаются, используйте <code>Character.isWhitespace</code>.</li>
	<li>Если между словами пропадает пробел, проверьте состояние “видели пробел перед словом”.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🎨 Где это пригодится</h2>
<p>Ручная нормализация полезна, когда нужна тонкая настройка обработки текста: парсеры, редакторы, подготовка данных, анализ логов и задачи, где регулярные выражения слишком тяжёлые или запрещены.</p>
