<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔗 Эффективная конкатенация чисел в строку</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Научиться собирать строку из чисел через <code>StringBuilder</code>, не создавая лишние промежуточные строки в цикле.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Сценарий</h2>
<p>Нужно превратить массив чисел в строку для вывода, лога или передачи дальше: числа должны идти через запятую, но после последнего числа запятая не нужна.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<ul>
	<li>Строки в Java неизменяемые.</li>
	<li>Конкатенация через <code>+=</code> в цикле создаёт лишние промежуточные строки.</li>
	<li><code>StringBuilder</code> эффективнее для пошаговой сборки.</li>
	<li>Разделитель удобно добавлять перед элементом, кроме первого, или после элемента, кроме последнего.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><strong>Реализуйте метод:</strong></p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">static String joinNumbers(int[] nums)</code></pre>
<ul>
	<li>Верните строку, где числа разделены запятыми без пробелов.</li>
	<li>Для пустого массива верните пустую строку.</li>
	<li>После последнего числа запятая не ставится.</li>
	<li>Используйте <code>StringBuilder</code>.</li>
	<li>Не используйте конкатенацию строк через <code>+</code> или <code>+=</code> внутри цикла.</li>
	<li>Решение должно содержать только метод без вывода в консоль.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<ul>
	<li><code>joinNumbers(new int[]{1, 2, 3})</code> → <code>"1,2,3"</code></li>
	<li><code>joinNumbers(new int[]{})</code> → <code>""</code></li>
	<li><code>joinNumbers(new int[]{42})</code> → <code>"42"</code></li>
	<li><code>joinNumbers(new int[]{-5, 0, 10})</code> → <code>"-5,0,10"</code></li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🤔 Момент для размышления</h2>
<p>Как написать условие для запятой так, чтобы она не появилась после последнего числа?</p>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>
<ol>
	<li>Создайте <code>StringBuilder</code>.</li>
	<li>Пройдите по массиву чисел.</li>
	<li>Добавляйте запятую только там, где она нужна.</li>
	<li>Добавляйте число через <code>append</code>.</li>
	<li>Верните <code>builder.toString()</code>.</li>
</ol>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>
<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<ul style="margin-bottom: 0;">
	<li><label><input type="checkbox"> Метод называется <code>joinNumbers</code>.</label></li>
	<li><label><input type="checkbox"> Используется <code>StringBuilder</code>.</label></li>
	<li><label><input type="checkbox"> Внутри цикла нет строкового <code>+=</code>.</label></li>
	<li><label><input type="checkbox"> После последнего числа нет запятой.</label></li>
	<li><label><input type="checkbox"> Пустой массив даёт пустую строку.</label></li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Технические ограничения</h2>
<ul>
	<li>Не используйте <code>+</code> или <code>+=</code> для сборки строки внутри цикла.</li>
	<li>Не ставьте пробелы после запятых.</li>
	<li>Не печатайте результат в консоль.</li>
	<li>Откажитесь от удаления последней запятой как единственной логики, если можете поставить разделитель аккуратно сразу.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🐛 Отладка: если что-то не работает</h2>
<ul>
	<li>Если в конце есть запятая, проверьте условие добавления разделителя.</li>
	<li>Если пустой массив возвращает запятую или пробел, проверьте цикл и начальное состояние builder.</li>
	<li>Если автопроверка ругается на стиль, уберите строковую конкатенацию внутри цикла.</li>
</ul>
