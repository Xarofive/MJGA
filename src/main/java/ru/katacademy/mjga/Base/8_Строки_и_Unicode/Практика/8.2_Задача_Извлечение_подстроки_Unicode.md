<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🌍 Извлечение подстроки с учётом Unicode</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Научиться извлекать подстроку по Unicode code points, не разрывая эмодзи и другие символы.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Сценарий</h2>
<p>Пользовательский текст может содержать латиницу, кириллицу, иероглифы и эмодзи. Нужно взять фрагмент по “человеческим” символам, а не по UTF-16 индексам Java.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<ul>
	<li><code>substring</code> работает с UTF-16 индексами.</li>
	<li>Параметры задачи заданы в code points.</li>
	<li><code>offsetByCodePoints</code> переводит позицию в code points в индекс строки.</li>
	<li>Если длина фрагмента выходит за конец строки, нужно вернуть текст до конца.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><strong>Реализуйте метод:</strong></p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">static String substringCodePoints(String s, int start, int length)</code></pre>
<ul>
	<li><code>start</code> и <code>length</code> заданы в code points, не в UTF-16 индексах.</li>
	<li>Если <code>start + length</code> выходит за конец строки, верните до конца строки.</li>
	<li>Если <code>length == 0</code>, верните пустую строку.</li>
	<li>Используйте <code>codePointCount</code> и <code>offsetByCodePoints</code>.</li>
	<li>Решение должно содержать только метод без вывода в консоль.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<ul>
	<li><code>substringCodePoints("Hello, World!", 7, 5)</code> → <code>"World"</code></li>
	<li><code>substringCodePoints("Hello😊世界", 5, 3)</code> → <code>"😊世界"</code></li>
	<li><code>substringCodePoints("Привет", 3, 10)</code> → <code>"вет"</code></li>
	<li><code>substringCodePoints("Test", 2, 0)</code> → <code>""</code></li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🤔 Момент для размышления</h2>
<p>Почему прямой вызов <code>s.substring(start, start + length)</code> может сломать строку с эмодзи?</p>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>
<ol>
	<li>Посчитайте общее количество code points.</li>
	<li>Определите конечную позицию с учётом выхода за конец строки.</li>
	<li>Преобразуйте начальную и конечную позиции через <code>offsetByCodePoints</code>.</li>
	<li>Верните <code>substring</code> по полученным UTF-16 индексам.</li>
</ol>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>
<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<ul style="margin-bottom: 0;">
	<li><label><input type="checkbox"> Метод имеет точную сигнатуру.</label></li>
	<li><label><input type="checkbox"> Границы считаются в code points.</label></li>
	<li><label><input type="checkbox"> Используется <code>offsetByCodePoints</code>.</label></li>
	<li><label><input type="checkbox"> Эмодзи не разрываются.</label></li>
	<li><label><input type="checkbox"> Метод не выводит текст в консоль.</label></li>

</div>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Технические ограничения</h2>
<ul>
	<li>Не используйте напрямую <code>s.substring(start, start + length)</code>.</li>
	<li>Не считайте <code>start</code> и <code>length</code> обычными индексами <code>char</code>.</li>
	<li>Не печатайте результат.</li>
	<li>Откажитесь от ручной обработки только конкретных примеров.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🐛 Отладка: если что-то не работает</h2>
<ul>
	<li>Если эмодзи превратился в непонятный символ, вы разрезали surrogate pair.</li>
	<li>Если фрагмент слишком короткий, проверьте расчёт конечной позиции.</li>
	<li>Если выход за конец строки падает ошибкой, ограничьте конец общим количеством code points.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">💪 Дополнительный вызов (после сдачи задачи)</h2>
<p>Добавьте обработку случая, когда <code>start</code> больше количества code points: возвращайте пустую строку.</p>

<h2 style="color: #34495e; margin-top: 28px;">🎨 Где это пригодится</h2>
<p>Unicode-безопасное извлечение подстрок нужно для превью сообщений, обрезки имён, лимитов профиля, текстовых редакторов и любых сервисов, где пользовательский текст может содержать эмодзи и разные языки.</p>
