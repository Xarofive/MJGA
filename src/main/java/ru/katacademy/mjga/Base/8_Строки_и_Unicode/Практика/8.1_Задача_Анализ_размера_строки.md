<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔤 Анализ размера строки: байты и символы</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>
<p style="margin: 0;">Научиться различать размер строки в UTF-8 байтах и количество Unicode code points в Java.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Сценарий</h2>
<p>Одна и та же строка может занимать разное количество байт в памяти или при передаче по сети. Нужно вернуть сразу два значения: сколько строка занимает в UTF-8 и сколько в ней Unicode code points.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>
<ul>
	<li>ASCII-символ обычно занимает 1 байт в UTF-8.</li>
	<li>Кириллица в UTF-8 обычно занимает 2 байта на символ.</li>
	<li>Некоторые символы, например эмодзи, могут занимать больше одного <code>char</code> в Java.</li>
	<li><code>codePointCount</code> считает Unicode code points, а не UTF-16 ячейки.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p><strong>Реализуйте метод:</strong></p>
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">static int[] byteCodePointCount(String s)</code></pre>
<ul>
	<li>Возвращает массив из двух чисел.</li>
	<li><code>result[0]</code> — количество байт в UTF-8.</li>
	<li><code>result[1]</code> — количество Unicode code points.</li>
	<li>Для байт используйте <code>s.getBytes(StandardCharsets.UTF_8).length</code>.</li>
	<li>Для code points используйте <code>s.codePointCount(0, s.length())</code>.</li>
	<li>Решение должно содержать только метод без вывода в консоль.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2>
<ul>
	<li><code>byteCodePointCount("")</code> → <code>[0, 0]</code></li>
	<li><code>byteCodePointCount("Hello")</code> → <code>[5, 5]</code></li>
	<li><code>byteCodePointCount("Привет")</code> → <code>[12, 6]</code></li>
	<li><code>byteCodePointCount("Java🚀")</code> → <code>[8, 5]</code></li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🤔 Момент для размышления</h2>
<p>Почему у строки <code>"Java🚀"</code> длина в UTF-8 байтах больше, чем количество code points?</p>

<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2>
<ol>
	<li>Посчитайте UTF-8 байты строки.</li>
	<li>Посчитайте количество code points.</li>
	<li>Создайте массив длиной 2.</li>
	<li>Положите байты в первый элемент, code points во второй.</li>
	<li>Верните массив.</li>
</ol>

<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2>
<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<ul style="margin-bottom: 0;">
	<li><label><input type="checkbox"> Метод называется <code>byteCodePointCount</code>.</label></li>
	<li><label><input type="checkbox"> Возвращается <code>int[]</code> длиной 2.</label></li>
	<li><label><input type="checkbox"> UTF-8 байты считаются через <code>StandardCharsets.UTF_8</code>.</label></li>
	<li><label><input type="checkbox"> Code points считаются через <code>codePointCount</code>.</label></li>
	<li><label><input type="checkbox"> Метод не печатает результат в консоль.</label></li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Технические ограничения</h2>
<ul>
	<li>Не используйте <code>s.length()</code> как количество символов для этой задачи.</li>
	<li>Не используйте кодировку по умолчанию без <code>StandardCharsets.UTF_8</code>.</li>
	<li>Не выводите результат в консоль.</li>
	<li>Откажитесь от ручных поправок для кириллицы или эмодзи.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">🐛 Отладка: если что-то не работает</h2>
<ul>
	<li>Если для <code>"Привет"</code> получилось <code>[6, 6]</code>, вы считаете символы, а не UTF-8 байты.</li>
	<li>Если для эмодзи результат странный, проверьте использование <code>codePointCount</code>.</li>
	<li>Если не компилируется <code>StandardCharsets</code>, добавьте нужный импорт.</li>
</ul>

<h2 style="color: #34495e; margin-top: 28px;">💪 Дополнительный вызов (после сдачи задачи)</h2>
<p>Проверьте метод на строке с несколькими эмодзи и сравните <code>s.length()</code> с <code>codePointCount</code>.</p>

<h2 style="color: #34495e; margin-top: 28px;">🎨 Где это пригодится</h2>
<p>Подсчёт байт и code points нужен в API, базах данных, сетевых протоколах, лимитах сообщений, логах и интерфейсах, где важно понимать реальный размер текста и не ломать Unicode-символы.</p>
