<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Задача 6.6: FizzBuzz с Guard Clauses</h1>
<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;"><h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3><p style="margin: 0;">Использовать серию независимых <code>if</code> с <code>continue</code>, чтобы избежать вложенности.</p></div>
<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;"><h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3><p style="margin-bottom: 10px;">Классическая задача FizzBuzz показывает, как важен порядок проверок и ранний переход к следующей итерации.</p><p style="margin: 0;"><strong>Ваша задача:</strong> вывести FizzBuzz для чисел от 1 до 30 без <code>else</code>.</p></div>
<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2><div style="background: #f0f4f8; border-radius: 8px; padding: 16px;"><h4 style="color: #455a64; margin-top: 0;">Guard clauses в цикле</h4><pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code>if (i % 15 == 0) {
    System.out.println("FizzBuzz");
    continue;
}</code></pre><p style="margin-bottom: 0;">После <code>continue</code> остальные проверки текущего числа не выполняются.</p></div>
<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2><ul><li>Используйте классический цикл <code>for</code> от 1 до 30 включительно.</li><li>Сначала проверяйте делимость на 15 и выводите <code>FizzBuzz</code>.</li><li>Затем проверяйте делимость на 3 и выводите <code>Fizz</code>.</li><li>Затем проверяйте делимость на 5 и выводите <code>Buzz</code>.</li><li>После каждого такого вывода используйте <code>continue</code>.</li><li>Если ни одно условие не подошло, выводите само число.</li><li>Не используйте <code>else</code>, вложенные <code>if</code> и <code>switch</code>.</li></ul>
<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2><p>Первые 15 строк вывода:</p><pre style="background: #2c3e50; color: #ecf0f1; padding: 16px; border-radius: 8px;"><code>1
2
Fizz
4
Buzz
Fizz
7
8
Fizz

Fizz

14
FizzBuzz</code></pre>
<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;"><h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4><p style="margin-bottom: 0;">Почему проверка делимости на 15 должна идти раньше проверок на 3 и 5?</p></div>
<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2><pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px;"><code>for (int i = 1; i &lt;= 30; i++) {
    if (i % 15 == 0) { ... continue; }
    if (i % 3 == 0) { ... continue; }
    if (i % 5 == 0) { ... continue; }
    System.out.println(i);
}</code></pre>
<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2><div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px;"><ul style="margin-bottom: 0;"><li>☐ Вывод содержит ровно 30 строк</li><li>☐ Условие на 15 проверяется первым</li><li>☐ Каждый guard-блок заканчивается <code>continue</code></li><li>☐ Нет <code>else</code> и <code>switch</code></li></ul></div>
<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;"><h4 style="color: #f57c00; margin-top: 0;">⚠️ Технические ограничения</h4><ul style="margin-bottom: 0;"><li>Не используйте <code>else</code>.</li><li>Не используйте вложенные <code>if</code>.</li><li>Не пропускайте <code>continue</code> после специальных выводов.</li></ul></div>
<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;"><h4 style="color: #c62828; margin-top: 0;">🐛 Отладка: если что-то не работает</h4><p><strong>15 выводится как Fizz?</strong> Проверка <code>i % 15 == 0</code> должна быть первой.</p><p><strong>После Fizz печатается число?</strong> Нужен <code>continue</code>.</p><p style="margin-bottom: 0;"><strong>Строк не 30?</strong> Проверьте диапазон цикла.</p></div>
