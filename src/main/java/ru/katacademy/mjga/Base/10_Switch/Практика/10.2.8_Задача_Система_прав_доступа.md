<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔐 Система прав доступа</h1>
<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;"><h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3><p style="margin: 0;">Использовать <code>switch expression</code> для маршрутизации ролей и синонимов ролей к уровню доступа.</p></div>
<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;"><p style="margin: 0;">📖 <strong>Пререквизит:</strong> повторите <code>String.trim()</code>, <code>toLowerCase()</code>, <code>switch</code> по строкам и объединение нескольких значений в одном <code>case</code>.</p></div>
<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;"><h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3><p style="margin-bottom: 10px;">Пользовательская роль приходит строкой: иногда с разным регистром, иногда с пробелами по краям, иногда как синоним основной роли.</p><p style="margin: 0;"><strong>Ваша задача:</strong> нормализовать роль и вернуть уровень доступа.</p></div>
<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2><div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;"><h4 style="color: #455a64; margin-top: 0;">Нормализация строки</h4><pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code class="language-java">String normalized = role.trim().toLowerCase();</code></pre><p style="margin-bottom: 0;">Так <code>" ADMIN "</code>, <code>"Admin"</code> и <code>"admin"</code> станут одной строкой.</p></div>
<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;"><h4 style="color: #e65100; margin-top: 0;">👥 Несколько значений в одном case</h4><p style="margin-bottom: 0;">В <code>switch expression</code> можно перечислить синонимы роли: <code>case "admin", "administrator", "root" -> "all";</code></p></div>
<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2><div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;"><p style="margin-top: 0;"><strong>Реализуйте метод:</strong></p><div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>public static String accessLevel(String role)</code></div><h4 style="color: #e65100; margin-top: 16px;">📝 Правила</h4><ul style="margin: 10px 0;"><li><code>"admin"</code>, <code>"administrator"</code>, <code>"root"</code> → <code>"all"</code></li><li><code>"editor"</code>, <code>"moderator"</code> → <code>"write"</code></li><li><code>"viewer"</code>, <code>"reader"</code>, <code>"guest"</code> → <code>"read"</code></li><li><code>"banned"</code>, <code>"blocked"</code> → <code>"none"</code></li><li>Любая другая роль → <code>"unknown"</code></li><li>Регистр и пробелы по краям не должны влиять на результат</li></ul><h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4><ul style="margin: 10px 0;"><li>Не используйте массивы, коллекции или <code>Map</code></li><li>Не выводите результат в консоль</li><li>Не создавайте дополнительные методы</li></ul></div>
<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2><div style="background: linear-gradient(135deg, #e3f2fd 0%, #bbdefb 100%); border-radius: 10px; padding: 20px; margin: 20px 0; border: 1px solid #64b5f6;"><div style="background: white; border-radius: 6px; padding: 15px; box-shadow: 0 2px 5px rgba(0,0,0,0.1);"><code>accessLevel("admin")</code> → <code>"all"</code><br><code>accessLevel("Moderator")</code> → <code>"write"</code><br><code>accessLevel(" guest ")</code> → <code>"read"</code><br><code>accessLevel("blocked")</code> → <code>"none"</code><br><code>accessLevel("owner")</code> → <code>"unknown"</code></div></div>
<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;"><h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4><p style="margin-bottom: 0;">Почему нормализацию роли лучше выполнить один раз до <code>switch</code>, а не повторять в каждом <code>case</code>?</p></div>
<h2 style="color: #34495e; margin-top: 28px;">📝 Структура программы</h2><pre style="background: #263238; color: #aed581; padding: 20px; border-radius: 8px; overflow-x: auto;"><code class="language-java">public static String accessLevel(String role) {
    String normalized = role.trim().toLowerCase();
    return switch (normalized) {
        // admin aliases -> all
        // editor aliases -> write
        // viewer aliases -> read
        // banned aliases -> none
        // default -> unknown
    };
}</code></pre>
<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2><div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px; margin: 16px 0;"><ul style="margin-bottom: 0;"><li>☐ Метод называется <code>accessLevel</code></li><li>☐ Роль очищается через <code>trim()</code></li><li>☐ Роль приводится к нижнему регистру</li><li>☐ Все синонимы ролей обработаны</li><li>☐ Используется <code>switch expression</code></li><li>☐ Нет массивов, коллекций и вывода</li></ul></div>
<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;"><h4 style="color: #f57c00; margin-top: 0;">⚠️ Технические ограничения</h4><ul style="margin-bottom: 0;"><li>Не используйте <code>Map</code> или коллекции</li><li>Не сравнивайте роли без нормализации</li><li>Не возвращайте <code>null</code> для неизвестной роли</li><li>Не печатайте результат</li></ul></div>
