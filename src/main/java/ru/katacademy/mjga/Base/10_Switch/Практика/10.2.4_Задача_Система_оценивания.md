<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🎓 Система оценивания</h1>
<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;"><h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3><p style="margin: 0;">Понять границу применения <code>switch</code>: диапазоны сначала превращаются в категорию, а затем обрабатываются через <code>switch expression</code>.</p></div>
<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 12px; margin: 16px 0;"><p style="margin: 0;">📖 <strong>Пререквизит:</strong> повторите диапазоны через <code>if</code>, целочисленное деление, <code>switch expression</code> и объединение нескольких <code>case</code>.</p></div>
<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;"><h3 style="color: #2d3436; margin-top: 0;">🎯 Сценарий</h3><p style="margin-bottom: 10px;">В журнале хранится балл от 0 до 100. Нужно вернуть буквенную оценку: A, B, C, D или F. Значения вне диапазона должны быть явно помечены как некорректные.</p><p style="margin: 0;"><strong>Ваша задача:</strong> проверить валидность балла, вычислить группу и выбрать оценку через <code>switch</code>.</p></div>
<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2><div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;"><h4 style="color: #455a64; margin-top: 0;">Диапазоны не пишутся прямо в case</h4><p>В Java нельзя написать <code>case score &gt;= 90 -></code>. Сначала нужно превратить балл в группу:</p><pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code class="language-java">int bucket = score / 10;</code></pre><p style="margin-bottom: 0;">Для <code>100</code> получится <code>10</code>, поэтому его нужно учесть вместе с <code>9</code>.</p></div>
<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;"><h4 style="color: #e65100; margin-top: 0;">🚧 Сначала валидность, потом switch</h4><p style="margin-bottom: 0;">Значения меньше <code>0</code> и больше <code>100</code> лучше обработать до вычисления группы, чтобы не смешивать ошибочные данные с реальными оценками.</p></div>
<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2><div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;"><p style="margin-top: 0;"><strong>Реализуйте метод:</strong></p><div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>public static String grade(int score)</code></div><h4 style="color: #e65100; margin-top: 16px;">📝 Правила</h4><ul style="margin: 10px 0;"><li><code>90..100</code> → <code>"A"</code></li><li><code>80..89</code> → <code>"B"</code></li><li><code>70..79</code> → <code>"C"</code></li><li><code>60..69</code> → <code>"D"</code></li><li><code>0..59</code> → <code>"F"</code></li><li>Меньше <code>0</code> или больше <code>100</code> → <code>"invalid"</code></li><li>После проверки валидности используйте <code>switch expression</code></li></ul><h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4><ul style="margin: 10px 0;"><li>Не используйте вывод в консоль</li><li>Не используйте массивы или коллекции</li><li>Не создавайте дополнительные методы</li></ul></div>
<h2 style="color: #34495e; margin-top: 28px;">🎯 Результат программы</h2><div style="background: linear-gradient(135deg, #e3f2fd 0%, #bbdefb 100%); border-radius: 10px; padding: 20px; margin: 20px 0; border: 1px solid #64b5f6;"><div style="background: white; border-radius: 6px; padding: 15px; box-shadow: 0 2px 5px rgba(0,0,0,0.1);"><code>grade(100)</code> → <code>"A"</code><br><code>grade(89)</code> → <code>"B"</code><br><code>grade(60)</code> → <code>"D"</code><br><code>grade(0)</code> → <code>"F"</code><br><code>grade(101)</code> → <code>"invalid"</code></div></div>
<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;"><h4 style="color: #e65100; margin-top: 0;">🤔 Момент для размышления</h4><p style="margin-bottom: 0;">Почему <code>case 10, 9 -> "A"</code> нужен, хотя оценка A начинается с 90?</p></div>

    if (score &lt; 0 || score &gt; 100) {
        return "invalid";
    }
    int bucket = score / 10;
    return switch (bucket) {
        // case 10, 9 -> "A"
        // ...
    };
}</code></pre>
<h2 style="color: #34495e; margin-top: 28px;">✅ Чек-лист проверки</h2><div style="background: #f8f9fa; border-left: 4px solid #28a745; padding: 16px; margin: 16px 0;"><ul style="margin-bottom: 0;"><li>☐ Метод называется <code>grade</code></li><li>☐ Значения вне <code>0..100</code> возвращают <code>"invalid"</code></li><li>☐ Используется <code>bucket = score / 10</code></li><li>☐ <code>100</code> попадает в <code>"A"</code></li><li>☐ Используется <code>switch expression</code></li><li>☐ Нет вывода в консоль</li></ul></div>
<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;"><h4 style="color: #f57c00; margin-top: 0;">⚠️ Технические ограничения</h4><ul style="margin-bottom: 0;"><li>Не пишите диапазоны прямо в <code>case</code></li><li>Не используйте массивы и коллекции</li><li>Не возвращайте пустую строку для неверного балла</li></ul></div>
