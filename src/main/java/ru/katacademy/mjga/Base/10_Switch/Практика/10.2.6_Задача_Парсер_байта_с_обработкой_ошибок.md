<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Парсер байта с обработкой ошибок</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">Цель задачи</h3>
<p style="margin: 0;">Совместить обработку исключений из модуля 9 и финальную классификацию результата через <code>switch</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">Контекст</h2>
<p>Байт в этой задаче — целое число от <code>0</code> до <code>255</code>. Ввод приходит строкой, поэтому возможны две разные ошибки: строка вообще не число или число вне диапазона.</p>

<h2 style="color: #34495e; margin-top: 28px;">Техническое задание</h2>
<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;">Создайте метод:</p>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 12px;"><code class="language-java">public static String parseByteStatus(String input)</code></pre>
<p><strong>Верните:</strong></p>
<ul>
	<li><code>"OK: value"</code>, если строка содержит число от <code>0</code> до <code>255</code>. Вместо <code>value</code> подставьте число.</li>
	<li><code>"RANGE ERROR"</code>, если строка является целым числом, но оно меньше <code>0</code> или больше <code>255</code>.</li>
	<li><code>"FORMAT ERROR"</code>, если строку нельзя преобразовать в целое число.</li>
</ul>
<p>Используйте <code>Integer.parseInt</code>, <code>try/catch</code> и <code>switch expression</code> по статусу.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">Подсказка по архитектуре</h2>
<p>Сначала получите статус: <code>"ok"</code>, <code>"range"</code> или <code>"format"</code>. Затем используйте <code>switch</code>, чтобы превратить статус в итоговую строку.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">return switch (status) {
    case "ok" -> "OK: " + value;
    case "range" -> "RANGE ERROR";
    default -> "FORMAT ERROR";
};</code></pre>
