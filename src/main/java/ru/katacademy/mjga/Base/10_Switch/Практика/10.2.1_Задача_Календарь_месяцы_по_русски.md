<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📅 Календарь: месяцы по-русски</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать <code>switch expression</code> в Java для выбора значения по числу и обрабатывать некорректный ввод через ветку <code>default</code>.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В приложениях часто нужно преобразовать числовой код в понятный текст: номер месяца в название, код статуса в описание, номер пункта меню в действие. Такие преобразования встречаются в календарях, отчётах, интерфейсах и обработке пользовательского ввода.</p>

<p style="margin: 0;">Когда вариантов немного и они заранее известны, <code>switch</code> делает код понятнее длинной цепочки <code>if-else</code>: сразу видно, какое значение какому результату соответствует.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Switch expression</h4>

<p>В современной Java <code>switch</code> может быть выражением: он не просто выполняет код, а сразу возвращает значение.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">return switch (number) {
    case 1 -> "one";
    case 2 -> "two";
    default -> "unknown";
};</code></pre>

<p style="margin-bottom: 0;">После закрывающей фигурной скобки нужна точка с запятой, потому что весь <code>switch</code> находится внутри оператора <code>return</code>.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Ветка default</h4>

<p><code>default</code> используется для значений, которые не попали ни в один <code>case</code>. В этой задаче это все числа меньше <code>1</code> и больше <code>12</code>.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">default -> "unknown";</code></pre>

<p style="margin-bottom: 0;">Так метод остаётся предсказуемым даже при некорректном номере месяца.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔀 Стрелочный синтаксис case</h4>

<p>Форма <code>case ... -></code> короче и безопаснее старого синтаксиса с двоеточием:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">case 1 -> "январь";</code></pre>

<p style="margin-bottom: 0;">В такой форме не нужен <code>break</code>, потому что выполнение не переходит в следующий <code>case</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🧰 Ключевые инструменты</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<ul style="margin: 0;">
	<li><code>public static</code> — объявление метода, который можно вызвать без создания объекта.</li>
	<li><code>int</code> — тип параметра с номером месяца.</li>
	<li><code>String</code> — тип возвращаемого названия месяца.</li>
	<li><code>return</code> — возврат результата из метода.</li>
	<li><code>switch (month)</code> — выбор результата по значению параметра.</li>
	<li><code>case ... -></code> — ветка для конкретного номера месяца.</li>
	<li><code>default</code> — обработка некорректного номера.</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Использовать старый switch без необходимости</strong><br />
❌ <code>case 1: return "январь";</code><br />
✅ <code>case 1 -> "январь";</code></p>

<p><strong>Ошибка 2: Забыть точку с запятой после switch expression</strong><br />
❌ <code>}</code><br />
✅ <code>};</code></p>

<p><strong>Ошибка 3: Вернуть пустую строку для неверного месяца</strong><br />
❌ <code>default -> "";</code><br />
✅ <code>default -> "unknown";</code></p>

<p><strong>Ошибка 4: Использовать массив или коллекцию</strong><br />
❌ <code>String[] months = {...};</code><br />
✅ Используйте <code>switch</code> с отдельными <code>case</code></p>

<p><strong>Ошибка 5: Неправильный регистр</strong><br />
❌ <code>"Январь"</code><br />
✅ <code>"январь"</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Забытые граничные случаи</strong><br />
❌ Обработать только числа <code>1..12</code> и не добавить <code>default</code><br />
✅ Добавьте <code>default</code> для <code>0</code>, отрицательных чисел и чисел больше <code>12</code></p>
</div>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка по архитектуре</h4>

<p>В большом приложении такой метод может быть частью класса-утилиты для форматирования дат. Остальной код передаёт номер месяца и получает готовую строку для интерфейса, отчёта или сообщения пользователю.</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px;"><code class="language-java">return switch (month) {
    case 1 -> "январь";
    case 2 -> "февраль";
    // ...
    default -> "unknown";
};</code></pre>

<p style="margin-bottom: 0;">Так логика выбора находится в одном месте, а вызывающий код не повторяет одинаковые проверки по всему проекту.</p>
