Java Base. Модуль 3. Методы

Цель подкаста:
Показать, что методы нужны не для красоты, а чтобы разделять программу на понятные действия.

Уровень:
Новичок, который уже понимает переменные и простые вычисления.

Общая идея модуля:
Метод - это именованное действие. Он может получать данные на вход, выполнять работу и иногда возвращать результат.

Темы для дебатов:
- Метод - это способ сократить код или способ назвать смысл действия?
- Когда лучше оставить код в main, а когда вынести в метод?
- Параметры делают метод гибким или усложняют чтение?
- Что понятнее новичку: "метод печатает результат" или "метод возвращает значение"?
- Почему имя метода иногда важнее его длины?
- Ошибка в методе страшнее или наоборот легче находится, потому что код разделен?

Вопросы, которые стоит раскрыть простыми словами:
- Чем метод похож на маленькую команду внутри программы?
- Что значит "передать аргумент"?
- Почему параметр живет только внутри метода?
- Чем отличается метод, который делает действие, от метода, который считает и возвращает ответ?
- Почему не стоит писать один огромный метод на всю задачу?

Хорошие аналогии:
- Метод как отдельный пункт инструкции.
- Параметры как входные данные для действия.
- Возвращаемое значение как ответ, который метод отдает обратно.

Чего избегать:
- Не обсуждать перегрузку, рекурсию и объектные методы.
- Не уходить в стек вызовов.
- Не спорить о функциональном программировании.
