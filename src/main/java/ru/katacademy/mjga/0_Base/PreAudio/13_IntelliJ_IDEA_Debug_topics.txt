Java Base. Модуль 13. IntelliJ IDEA, Debug и ошибки

Цель подкаста:
Сделать разговор о рабочем инструменте: как запускать код, читать ошибки и спокойно искать проблему.

Уровень:
Новичок, который уже писал учебные программы, но еще не уверен в самостоятельной отладке.

Общая идея модуля:
IDE помогает видеть проект, запускать программу, читать вывод и отлаживать код по шагам. Ошибка - это информация, которую нужно научиться читать.

Темы для дебатов:
- IDE помогает новичку или прячет от него важные детали?
- Запуск кнопкой Run - это магия или обычный рабочий инструмент?
- Debug нужен только при сложных ошибках или его стоит учить рано?
- Stack trace пугает или помогает найти место проблемы?
- Лучше угадывать ошибку по коду или смотреть значения переменных в Debug?
- Самостоятельная отладка - это технический навык или привычка не паниковать?

Вопросы, которые стоит раскрыть простыми словами:
- Где в IDE смотреть результат работы программы?
- Что значит "поставить breakpoint"?
- Почему выполнение по шагам помогает понять код?
- Как читать ошибку сверху вниз и находить полезную строку?
- Почему сообщение об ошибке не надо сразу закрывать?

Хорошие аналогии:
- Debug как режим замедленной съемки.
- Breakpoint как закладка "остановись здесь".
- Stack trace как маршрут, по которому программа пришла к ошибке.

Чего избегать:
- Не обсуждать сложные настройки IDE.
- Не уходить в профилирование и remote debug.
- Не превращать выпуск в инструкцию по всем кнопкам IntelliJ IDEA.
