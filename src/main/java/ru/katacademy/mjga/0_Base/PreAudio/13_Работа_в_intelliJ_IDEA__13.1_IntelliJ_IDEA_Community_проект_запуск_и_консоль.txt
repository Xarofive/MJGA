🧭 IntelliJ IDEA Community: проект, запуск и консоль

📝 TL;DR

IntelliJ IDEA Community Edition достаточно для Java Core: в ней можно открывать учебный проект, писать классы, запускать main, работать с Gradle/Maven и использовать полноценный Java Debugger.

Для запуска программы нужен класс с методом public static void main(String[] args). Запустить его можно зелёной кнопкой рядом с методом или через контекстное меню Run.

Вывод программы, текст ошибок и ввод через Scanner отображаются в нижнем окне Run. Если программа зависла или ждёт ввод, её можно остановить красной кнопкой Stop.

🎯 Чему вы научитесь

- Открывать учебный Java-проект в IntelliJ IDEA Community

- Находить нужный класс и метод main

- Запускать программу через кнопку Run

- Понимать, где находится вывод программы

- Отличать ошибку компиляции от ошибки во время выполнения

- Понимать частые сообщения вроде main method not found

📚 Предпосылки

- Вы уже писали простые Java-классы

- Вы знаете, что такое метод main

- Вы запускали код хотя бы через онлайн-редактор или терминал

- На компьютере установлена IntelliJ IDEA Community Edition

1. Community Edition подходит для курса

💡 Что важно знать

У IntelliJ IDEA есть две основные версии: Community и Ultimate. Для этого курса достаточно Community Edition.

Community Edition умеет работать с обычными Java-программами, Gradle/Maven-проектами, запуском классов, тестами и Debugger. Ultimate нужна в основном для расширенной работы с enterprise-фреймворками, базами данных и веб-инструментами, но для Java Core это не требуется.

2. Как открыть проект

📚 Порядок действий

- Откройте IntelliJ IDEA.

- Нажмите Open.

- Выберите папку проекта, а не отдельный файл.

- Дождитесь, пока IDEA проиндексирует проект и загрузит Gradle/Maven, если они есть.

- Откройте дерево файлов слева в окне Project.

Если проект открылся правильно, слева видны папки, Java-файлы и структура проекта. Если открыт только один файл, запуск и подсказки могут работать неполно.

3. Где находится код

project
└── src
└── main
└── java
└── Main.java

В учебных Java-проектах код часто лежит в папке src/main/java. Внутри могут быть пакеты, а в пакетах — классы.

🔍 Что искать в классе

Для запуска обычной программы нужен метод main:

public class Main {
public static void main(String[] args) {
System.out.println("Hello, Java!");
}
}

Рядом с классом или методом IDEA показывает зелёную кнопку запуска. Если её нет, проверьте сигнатуру метода main.

4. Как запустить программу

✅ Базовый запуск

- Откройте класс с методом main.

- Нажмите зелёный треугольник слева от метода или класса.

- Выберите Run 'Main.main()'.

- Посмотрите результат в нижнем окне Run.

public class Main {
public static void main(String[] args) {
int price = 100;
int discount = 15;
int finalPrice = price - discount;

System.out.println(finalPrice);
}
}

После запуска в консоли появится:

5. Run tool window: консоль и ввод

⌨️ Если программа ждёт ввод

Код со Scanner может остановиться и ждать, пока вы введёте данные в консоль:

import java.util.Scanner;

public class Main {
public static void main(String[] args) {
Scanner scanner = new Scanner(System.in);

int age = scanner.nextInt();
System.out.println("Age: " + age);
}
}

После запуска кликните в нижнюю консоль, введите число и нажмите Enter.

6. Частые проблемы при запуске

⚠️ Ошибки и что они означают

main method not found

IDEA не нашла метод запуска. Проверьте, что метод написан как public static void main(String[] args).

Cannot resolve symbol

IDEA не знает такое имя переменной, класса или метода. Часто причина — опечатка или переменная объявлена в другой области видимости.

Class is never used

Это не ошибка запуска, а предупреждение. Оно означает, что класс пока нигде не используется.

Process finished with exit code 0

Программа завершилась успешно. Это нормальное сообщение, а не ошибка.

7. Мини-чек-лист перед дебагом

- Проект открыт как папка проекта, а не как отдельный файл

- Класс содержит корректный метод main

- Вы запускаете именно тот класс, который хотите проверить

- Консоль находится в нижнем окне Run

- Если программа ждёт ввод, курсор стоит в консоли

- Если программа не останавливается, используйте кнопку Stop

🧩 Что дальше

Запуск показывает результат, но не объясняет, почему программа пришла именно к нему. Для этого нужен Debug: остановка на нужной строке, просмотр переменных и выполнение кода по шагам.
