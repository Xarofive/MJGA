🧯 Ошибки, stack trace и самостоятельная отладка

📝 TL;DR

Когда Java-программа падает, в консоли появляется stack trace — список вызовов, который помогает найти место ошибки.

В первую очередь ищите название исключения, сообщение ошибки и первую строку из вашего кода. В IntelliJ IDEA по строкам stack trace часто можно кликнуть, чтобы перейти к нужной строке файла.

Debugger полезен не только для неправильных результатов, но и для падений: поставьте breakpoint перед подозрительным местом и проверьте значения переменных до ошибки.

🎯 Чему вы научитесь

- Отличать ошибку компиляции от исключения во время выполнения

- Читать базовый stack trace

- Находить строку, где программа упала

- Разбирать NullPointerException

- Использовать Debug перед местом падения

- Составлять короткий план самостоятельной отладки

📚 Предпосылки

- Вы умеете запускать программу в IntelliJ IDEA

- Вы умеете ставить breakpoint и запускать Debug

- Вы знакомы с объектами, ссылками и null

1. Две большие категории ошибок

| Тип
| Когда появляется
| Пример

| Ошибка компиляции
| До запуска программы
| Cannot resolve symbol, пропущена ;

| Ошибка выполнения
| Во время работы программы
| NullPointerException, ArrayIndexOutOfBoundsException

💡 Как IDEA помогает

Ошибки компиляции IDEA обычно подчёркивает прямо в редакторе. Ошибки выполнения появляются в консоли после запуска программы.

2. Что такое stack trace

class Student {
String name;

String upperName() {
return name.toUpperCase();
}
}

public class Main {
public static void main(String[] args) {
Student student = new Student();
System.out.println(student.upperName());
}
}

Поле name не заполнено, поэтому оно равно null. При вызове name.toUpperCase() программа упадёт.

📌 Пример stack trace

Exception in thread "main" java.lang.NullPointerException
at Student.upperName(Main.java:5)
at Main.main(Main.java:12)

Здесь важно: тип ошибки — NullPointerException, строка падения — Main.java:5, путь вызовов идёт через Main.main.

3. Как читать stack trace

📚 Алгоритм чтения

- Найдите название исключения: например, NullPointerException.

- Прочитайте сообщение ошибки, если оно есть.

- Найдите первую строку, которая относится к вашему файлу.

- Кликните по строке в консоли IDEA, чтобы перейти к коду.

- Поставьте breakpoint перед этой строкой и запустите Debug.

4. NullPointerException: самый частый случай

String text = null;
int length = text.length();

NullPointerException означает: вы пытаетесь обратиться к полю или методу через ссылку, которая равна null.

🔍 Что проверять

- Где должна была быть создана ссылка через new?

- Было ли поле заполнено до вызова метода?

- Может ли метод вернуть null?

- Нужна ли проверка if (value == null)?

5. Отладка падения через Debug

✅ Порядок действий

- Запустите программу обычным Run и прочитайте stack trace.

- Перейдите к строке, где программа упала.

- Поставьте breakpoint на 1-2 строки выше.

- Запустите программу через Debug.

- Посмотрите значения переменных перед падением.

- Исправьте причину, а не только симптом.

6. Пример: ошибка в массиве

public class Main {
public static void main(String[] args) {
int[] numbers = {10, 20, 30};

for (int i = 0; i <= numbers.length; i++) {
System.out.println(numbers[i]);
}
}
}

Этот код упадёт с ArrayIndexOutOfBoundsException, потому что последний индекс массива равен numbers.length - 1. Условие цикла должно быть i < numbers.length.

🧠 Как увидеть это в Debug

Поставьте breakpoint внутри цикла и следите за i. На последней итерации i станет равен 3, а допустимые индексы массива: 0, 1, 2.

7. План самостоятельной отладки

- Сформулируйте, какой результат ожидался.

- Сравните с фактическим результатом или текстом ошибки.

- Найдите участок кода, который отвечает за этот результат.

- Поставьте breakpoint перед этим участком.

- Пройдите код по шагам и смотрите переменные.

- Найдите первую строку, после которой значение стало неправильным.

- Исправьте код и снова запустите проверку.
