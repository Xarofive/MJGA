Java Base. Модуль 7. Массивы

Цель подкаста:
Объяснить, зачем хранить несколько значений вместе и почему индекс - центральная идея массива.

Уровень:
Новичок, уверенно знакомый с циклами.

Общая идея модуля:
Массив хранит набор значений одного типа. У каждого элемента есть номер, а длина массива после создания не меняется.

Темы для дебатов:
- Массив - это просто много переменных или новая модель мышления?
- Фиксированная длина массива - ограничение или полезная определенность?
- Почему индекс начинается с нуля, и стоит ли это объяснять сразу?
- Что сложнее: создать массив или правильно пройти по нему циклом?
- Ошибка выхода за границы - наказание Java или подсказка о неверной логике?
- Когда массив понятнее списка, а когда уже хочется динамическую коллекцию?

Вопросы, которые стоит раскрыть простыми словами:
- Зачем нужен массив, если можно создать много переменных?
- Что значит "элемент массива"?
- Почему последний индекс меньше длины на единицу?
- Как цикл и массив работают вместе?
- Почему массивы удобно использовать для однотипных данных?

Хорошие аналогии:
- Ряд подписанных ячеек.
- Полка с местами, где каждое место имеет номер.
- Поезд с вагонами: длина известна, вагоны идут по порядку.

Чего избегать:
- Не обсуждать внутреннюю память массивов глубоко.
- Не уходить в сортировки и сложные алгоритмы.
- Не сравнивать подробно все коллекции Java.
