Java Base. Модуль 2. Переменные и типы данных

Цель подкаста:
Объяснить, зачем программе память для значений, почему у значений есть типы и как арифметика в Java связана с типами.

Уровень:
Новичок, который уже видел первую Java-программу, но еще не думает как разработчик.

Общая идея модуля:
Переменная - это именованное место для значения. Тип говорит Java, какое значение там хранится и какие действия с ним допустимы.

Темы для дебатов:
- Переменная - это "коробка", "ярлык" или "имя для значения"? Какая аналогия лучше для новичка?
- Строгая типизация помогает избежать ошибок или сначала кажется лишней сложностью?
- Почему нельзя хранить любое значение в любой переменной?
- Что важнее: правильно выбрать тип или правильно назвать переменную?
- Почему деление целых чисел иногда удивляет новичков?
- Арифметика в коде - это школьная математика или отдельный набор правил?

Вопросы, которые стоит раскрыть простыми словами:
- Зачем давать значениям имена?
- Почему имя переменной должно объяснять смысл, а не просто быть коротким?
- Чем отличается число от текста, даже если текст содержит цифры?
- Почему Java не всегда "угадывает", что имел в виду ученик?
- Как типы связаны с ошибками до запуска программы?

Хорошие аналогии:
- Переменная как подписанная ячейка.
- Тип как правило хранения: в одну ячейку кладем числа, в другую текст.
- Операции как действия, которые разрешены только для подходящих значений.

Чего избегать:
- Не уходить в память, байты и внутреннее устройство типов.
- Не обсуждать все примитивные типы подробно.
- Не превращать выпуск в таблицу синтаксиса.
