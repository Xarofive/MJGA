Java Base. Модуль 1. Введение в программирование

Цель подкаста:
Сделать спокойный разговор для новичка о том, что такое программирование, зачем нужна Java, почему нужно окружение разработки и как выглядит первая программа.

Уровень:
Абсолютный новичок. Не превращать обсуждение в разговор опытных разработчиков. Не уходить в Spring, микросервисы, JVM internals, производительность и архитектуру.

Общая идея модуля:
Программа - это набор точных инструкций для компьютера. Java - язык, на котором эти инструкции пишутся. Чтобы программа запустилась, нужен установленный инструмент разработки и понятная структура файла.

Темы для дебатов:
- Что важнее в самом начале: понять общую картину или сразу писать код?
- Почему компьютер требует точных инструкций, а не понимает "примерно"?
- Java кажется строгой: это мешает новичку или помогает учиться аккуратно?
- Нужно ли новичку понимать JDK, JVM и компиляцию, или достаточно нажать Run?
- Почему первая программа выглядит многословно, хотя делает простую вещь?
- Что лучше для первого занятия: теория про язык или маленький успешный запуск программы?

Вопросы, которые стоит раскрыть простыми словами:
- Чем программа отличается от обычной инструкции для человека?
- Почему Java-код сначала нужно подготовить к запуску?
- Зачем нужна IntelliJ IDEA, если код можно написать в любом текстовом редакторе?
- Что ученик должен почувствовать после первой запущенной программы?

Хорошие аналогии:
- Программа как рецепт, но очень точный.
- JDK как набор инструментов мастера.
- IntelliJ IDEA как рабочий стол, где видно файлы, ошибки и результат запуска.

Чего избегать:
- Не обсуждать сложные особенности JVM.
- Не спорить о том, лучше ли Java других языков.
- Не превращать выпуск в карьерный обзор backend-разработки.
