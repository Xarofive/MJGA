Поля и методы: состояние, операции и правила

Продолжаем тему классов: учимся не просто хранить данные, а управлять состоянием объекта через методы.

📌 TL;DR — Суть за 30 секунд

- Поля объекта хранят его состояние.

- Методы объекта описывают операции над этим состоянием.

- Метод может читать поля, менять поля, возвращать результат или делать всё сразу.

- Хороший метод не просто меняет данные, а соблюдает правила предметной области.

- Если операция может не выполниться, метод может вернуть boolean.

- В учебных задачах методы обычно возвращают результат, а не печатают его.

Что такое состояние объекта

Состояние объекта — это текущие значения его полей. У банковского счёта состояние может включать владельца и баланс. У товара — название, цену и количество. У лампы — модель и флаг включения.

class Lamp {
String model;
boolean isOn;
}

Если isOn == false, лампа выключена. Если isOn == true, лампа включена. Когда метод меняет isOn, он меняет состояние объекта.

Когда важно состояние:

- заказ может быть новым, оплаченным или отменённым;

- счёт может иметь разный баланс;

- товар может иметь разный остаток на складе;

- пользователь может быть активным или заблокированным;

- игровой персонаж может иметь разное здоровье.

Основные операции: чтение, изменение, расчёт

Операция 1: чтение состояния

Метод может вернуть информацию об объекте, не меняя его.

class Student {
String name;
int age;

String getCard() {
return "Student: " + name + ", age: " + age;
}
}

Операция 2: изменение состояния

Метод может изменить поле объекта.

class Lamp {
boolean isOn;

void turnOn() {
isOn = true;
}

void turnOff() {
isOn = false;
}
}

Операция 3: расчёт на основе состояния

Метод может вычислить значение по полям объекта.

class Product {
int price;
int quantity;

int totalPrice() {
return price * quantity;
}
}

Методы как правила безопасности

Главная сила методов не в том, что они “сокращают код”. Они позволяют спрятать правило рядом с данными, к которым это правило относится.

Например, банковский счёт не должен уходить в минус:

class BankAccount {
String owner;
int balance;

boolean withdraw(int amount) {
if (amount <= 0) {
return false;
}

if (amount > balance) {
return false;
}

balance = balance - amount;
return true;
}
}

📌 Почему это важно

Если менять balance напрямую, легко получить отрицательный баланс. Метод withdraw задаёт единое правило: деньги снимаются только при корректной сумме и достаточном балансе.

Метод возвращает результат или печатает?

Для новичков частая привычка — сразу писать System.out.println. Но в большинстве задач и реальных программ метод должен вернуть результат.

| Подход
| Когда подходит

| return
| Когда результат нужен тестам, другому методу, интерфейсу или расчёту

| System.out.println
| Когда задача прямо просит вывести результат в консоль

// Хорошо для объекта: метод возвращает данные
String getInfo() {
return owner + ": " + balance;
}

// Менее гибко: метод сам решает, куда выводить
void printInfo() {
System.out.println(owner + ": " + balance);
}

Несколько объектов одного класса

У каждого объекта своя копия полей.

Product phone = new Product();
phone.quantity = 2;

Product laptop = new Product();
laptop.quantity = 5;

phone.quantity++;

System.out.println(phone.quantity); // 3
System.out.println(laptop.quantity); // 5

Оба объекта созданы по классу Product, но это разные товары с независимым состоянием.

Реальный кейс: складской товар

Постановка задачи

На складе у товара есть остаток. Поставка увеличивает остаток, продажа уменьшает, но нельзя продать больше, чем есть.

Решение

class Product {
String name;
int price;
int quantity;

void add(int amount) {
if (amount > 0) {
quantity = quantity + amount;
}
}

boolean sell(int amount) {
if (amount <= 0 || amount > quantity) {
return false;
}

quantity = quantity - amount;
return true;
}

int totalPrice() {
return price * quantity;
}
}

Разбор решения

- add игнорирует некорректную поставку.

- sell возвращает false, если операцию выполнить нельзя.

- totalPrice ничего не меняет, а только считает.

- Все операции связаны с товаром, поэтому находятся внутри класса Product.

Частые ошибки и как их избежать

❌ Ошибка 1: Метод меняет не то поле

Проверяйте, что sell уменьшает quantity, а не price, и что deposit меняет balance, а не создаёт локальную переменную.

❌ Ошибка 2: Нет проверки граничных значений

Ноль и отрицательные значения почти всегда нужно обрабатывать отдельно: нельзя продать -3 товара или пополнить счёт на -100.

❌ Ошибка 3: Метод всегда возвращает true

Если операция может не выполниться, возвращаемое значение должно отражать реальный результат операции.

Мини-квиз для самопроверки

🧩 Вопрос 1: Что изменится?

Product product = new Product();
product.quantity = 3;
product.add(2);

Ответ: quantity станет 5.

🧩 Вопрос 2: Почему метод sell возвращает boolean?

Ответ: потому что продажа может быть успешной или неуспешной, и вызывающий код должен узнать результат.

📋 Шпаргалка: методы объекта

class Entity {
int value;

// читает состояние
int getValue() {
return value;
}

// меняет состояние
void add(int amount) {
if (amount > 0) {
value += amount;
}
}

// пытается выполнить операцию
boolean remove(int amount) {
if (amount <= 0 || amount > value) {
return false;
}

value -= amount;
return true;
}
}
