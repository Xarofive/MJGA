Классы и объекты: основы и операции

Учимся описывать сущности программы: данные, действия, создание объектов и работу с ними через точку.

📌 TL;DR — Суть за 30 секунд

- class создаёт новый тип данных, который описывает сущность программы.

- Класс не является конкретным объектом. Это шаблон: какие поля и методы будут у объектов.

- Объект создаётся через new и получает собственные значения полей.

- Поля хранят состояние объекта: имя студента, баланс счёта, количество товара.

- Методы описывают поведение объекта: включить лампу, пополнить счёт, посчитать стоимость.

- Через точку object.field и object.method() мы обращаемся к данным и действиям конкретного объекта.

- Классы нужны, когда несколько данных и действий логически относятся к одной сущности.

Что такое класс и зачем он нужен

Класс в Java — это способ собрать связанные данные и действия в одну понятную модель. До классов вы могли хранить данные отдельными переменными, массивами и коллекциями. Это работает для маленьких задач, но плохо масштабируется.

Представьте студента:

- имя;

- возраст;

- средний балл;

- действие: получить краткую карточку;

- действие: проверить, проходит ли студент по среднему баллу.

Если хранить всё отдельно, связь между данными держится только в вашей голове:

String studentName = "Alice";
int studentAge = 20;
double studentAverageGrade = 4.7;

Класс делает эту связь явной:

class Student {
String name;
int age;
double averageGrade;
}

Почему это важно: программа становится ближе к предметной области. Вы работаете не просто со строками и числами, а со студентами, товарами, заказами, счетами и пользователями.

Когда использовать классы:

- нужно хранить несколько связанных значений;

- у сущности есть действия над этими значениями;

- нужно создавать много похожих объектов;

- нужно передавать сущность целиком в методы;

- код с отдельными переменными стал трудно читать и поддерживать.

Класс против объекта

Класс — это описание. Объект — конкретный экземпляр этого описания.

| Понятие
| Что означает
| Пример

| Класс
| Шаблон, чертёж, новый тип данных
| class Student

| Объект
| Конкретная сущность, созданная по классу
| new Student()

| Переменная-ссылка
| Имя, через которое мы обращаемся к объекту
| Student alice

💡 Аналогия: класс — чертёж дома, объект — конкретный построенный дом. По одному чертежу можно построить много домов. По одному классу можно создать много объектов.

Анатомия простого класса

Простой класс обычно состоит из полей и методов. В будущем появятся конструкторы, модификаторы доступа, наследование и интерфейсы, но для первого знакомства достаточно этой модели.

class BankAccount {
String owner; // поле
int balance; // поле

void deposit(int amount) { // метод
balance = balance + amount;
}

String getInfo() { // метод
return owner + ": " + balance;
}
}

| Элемент
| Роль

| class BankAccount
| Объявление нового типа

| owner, balance
| Поля, которые хранят состояние объекта

| deposit, getInfo
| Методы, которые описывают поведение объекта

Создание объекта и работа через точку

Чтобы использовать класс, нужно создать объект:

BankAccount account = new BankAccount();

После этого через точку можно записывать поля, читать поля и вызывать методы:

account.owner = "Alice";
account.balance = 100;

account.deposit(50);

String info = account.getInfo(); // "Alice: 150"

📌 Суть объекта

Объект полезен тем, что данные и действия живут рядом. Баланс хранится внутри счёта, а метод пополнения меняет именно этот счёт.

Значения полей по умолчанию

Если создать объект и не заполнить поля, Java присвоит им значения по умолчанию.

| Тип поля
| Значение по умолчанию

| int | 0

| double | 0.0

| boolean | false

| String | null

class Student {
int id;
String name;
}

Student student = new Student();

System.out.println(student.id); // 0
System.out.println(student.name); // null

Реальный кейс: карточка товара

Постановка задачи

В интернет-магазине нужно хранить название товара, цену и количество на складе. Ещё нужно уметь посчитать общую стоимость остатков.

Решение через класс

class Product {
String name;
int price;
int quantity;

int totalPrice() {
return price * quantity;
}

String label() {
return name + " (" + quantity + " pcs)";
}
}

Использование

Product notebook = new Product();
notebook.name = "Notebook";
notebook.price = 100;
notebook.quantity = 3;

System.out.println(notebook.totalPrice()); // 300
System.out.println(notebook.label()); // Notebook (3 pcs)

Такой код читается как описание предметной области: есть товар, у товара есть цена и остаток, товар умеет считать свою общую стоимость.

Частые ошибки и как их избежать

❌ Ошибка 1: Класс есть, объекта нет

Student student;
student.name = "Alice"; // ошибка: объект не создан

Сначала создайте объект через new Student().

❌ Ошибка 2: Метод печатает вместо return

В учебных задачах метод чаще должен вернуть результат, а не вывести его.

String getInfo() {
return owner + ": " + balance;
}

❌ Ошибка 3: Делать всё static

Если поле относится к конкретному объекту, оно не должно быть static. У каждого студента своё имя, у каждого счёта свой баланс.

Мини-квиз для самопроверки

🧩 Вопрос 1: Что здесь класс, а что объект?

Student alice = new Student();

Ответ: Student — класс, new Student() — объект, alice — переменная-ссылка.

🧩 Вопрос 2: Что выведется?

class Lamp {
boolean isOn;
}

Lamp lamp = new Lamp();
System.out.println(lamp.isOn);

Ответ: false, потому что это значение по умолчанию для boolean.

// 1. Объявление класса
class Student {
String name;
int age;

String getInfo() {
return name + ", " + age;
}
}

// 2. Создание объекта
Student student = new Student();

student.name = "Alice";
student.age = 20;

// 4. Вызов метода
String info = student.getInfo();
