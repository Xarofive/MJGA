Java Base. Модуль 11. Коллекции: ArrayList и HashMap

Цель подкаста:
Объяснить, зачем нужны динамические структуры данных и чем список отличается от словаря ключ-значение.

Уровень:
Новичок, который уже понимает массивы и циклы.

Общая идея модуля:
ArrayList хранит упорядоченный список элементов. HashMap хранит пары "ключ - значение" и помогает быстро находить данные по ключу.

Темы для дебатов:
- Если есть массивы, зачем нужен ArrayList?
- Динамический размер - удобство или повод забыть о структуре данных?
- ArrayList отвечает на вопрос "что идет по порядку", а HashMap - "что связано с этим ключом". Насколько это полезное разделение?
- Почему ключ в HashMap должен быть понятным и уникальным?
- Порядок элементов важен всегда или иногда важнее быстрый поиск?
- Коллекции упрощают задачи или скрывают слишком много деталей?

Вопросы, которые стоит раскрыть простыми словами:
- Когда данные лучше хранить списком?
- Когда данные лучше хранить по ключу?
- Почему в ArrayList есть индекс, а в HashMap есть ключ?
- Что происходит, если положить новое значение по уже существующему ключу?
- Почему коллекции часто используются вместе с циклами?

Хорошие аналогии:
- ArrayList как список покупок.
- HashMap как телефонная книга: имя -> номер.
- Ключ как подпись, по которой быстро находится значение.

Чего избегать:
- Не обсуждать устройство хеш-таблицы подробно.
- Не уходить в интерфейсы List и Map глубоко.
- Не разбирать сложность алгоритмов на профессиональном уровне.
