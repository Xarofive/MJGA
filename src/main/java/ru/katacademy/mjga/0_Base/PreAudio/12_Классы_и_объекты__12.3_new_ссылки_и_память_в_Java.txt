new, ссылки и объекты в памяти

Разбираем самый частый источник ошибок у новичков: объектная переменная хранит ссылку, а не сам объект.

📌 TL;DR — Суть за 30 секунд

- new создаёт новый объект.

- Переменная объектного типа хранит ссылку на объект.

- Присваивание second = first копирует ссылку, а не объект.

- Две переменные могут указывать на один и тот же объект.

- null означает, что ссылки на объект нет.

- Вызов поля или метода через null приводит к NullPointerException.

- == для объектов проверяет, одна ли это ссылка.

Что делает new и зачем он нужен

Когда вы пишете new ClassName(), Java создаёт новый объект в памяти и возвращает ссылку на него.

Lamp lamp = new Lamp();

Эта строка состоит из нескольких действий:

- создать объект класса Lamp;

- выделить место под его поля;

- заполнить поля значениями по умолчанию;

- вернуть ссылку на объект;

- сохранить ссылку в переменную lamp.

💡 Аналогия: объект — квартира, ссылка — адрес квартиры. Переменная хранит адрес, по которому Java находит объект.

Ссылки против примитивов

Примитивы копируют значение:

int a = 10;
int b = a;
b = 20;

System.out.println(a); // 10
System.out.println(b); // 20

Объектные переменные копируют ссылку:

Counter first = new Counter();
Counter second = first;

second.value = 10;

System.out.println(first.value); // 10
System.out.println(second.value); // 10

first и second указывают на один объект. Поэтому изменение через одну переменную видно через другую.

Основные операции со ссылками

Операция 1: создать новый объект

Counter counter = new Counter();

Операция 2: скопировать ссылку

Counter alias = counter;

Новый объект не создаётся. Появляется ещё одна переменная для доступа к тому же объекту.

Операция 3: проверить, один ли это объект

boolean same = counter == alias;

== отвечает на вопрос: “эти переменные указывают на один и тот же объект?”

null: ссылки нет

null — специальное значение для объектных переменных. Оно означает: переменная сейчас ни на какой объект не указывает.

Student student = null;

Через null нельзя обращаться к полям и методам:

student.name = "Alice"; // NullPointerException

📌 Практическое правило

Если переменная может быть null, перед обращением к полям и методам нужно либо проверить её, либо организовать код так, чтобы null туда не попадал.

Объекты в методах

Когда объект передаётся в метод, копируется ссылка. Метод получает доступ к тому же объекту и может изменить его состояние.

class Counter {
int value;
}

static void increment(Counter counter) {
counter.value++;
}

Counter counter = new Counter();
increment(counter);

System.out.println(counter.value); // 1

Метод increment получил копию ссылки, но объект за этой ссылкой тот же самый.

Реальный кейс: один объект в разных частях программы

Постановка задачи

Есть счётчик событий. Его передают в несколько методов. Все методы должны увеличивать один общий счётчик, а не создавать свои отдельные копии.

Решение

class Counter {
int value;

void increment() {
value++;
}
}

static void processClick(Counter counter) {
counter.increment();
}

static void processSubmit(Counter counter) {
counter.increment();
}

Counter events = new Counter();

processClick(events);
processSubmit(events);

System.out.println(events.value); // 2

Разбор решения

- events хранит ссылку на один объект Counter.

- Оба метода получают ссылку на тот же объект.

- Каждый метод увеличивает поле value общего объекта.

- Итоговое значение равно 2.

Частые ошибки и как их избежать

❌ Ошибка 1: Ожидать копию объекта

Counter second = first;

Эта строка копирует ссылку. Новый объект не создаётся.

❌ Ошибка 2: Вызов метода через null

Counter counter = null;
counter.increment(); // NullPointerException

Создайте объект или проверьте переменную на null.

❌ Ошибка 3: Сравнивать содержимое через ==

== сравнивает ссылки. Два разных объекта с одинаковыми полями всё равно дадут false.

Мини-квиз для самопроверки

🧩 Вопрос 1: Что выведется?

Counter first = new Counter();
Counter second = first;
second.value = 7;

System.out.println(first.value);

Ответ: 7. Обе переменные указывают на один объект.

🧩 Вопрос 2: Что вернёт сравнение?

Counter first = new Counter();
Counter second = new Counter();

System.out.println(first == second);

Ответ: false. Это два разных объекта.

12.2.5 Счётчик и ссылки

Будущие задачи с коллекциями объектов

Когда вы будете хранить ArrayList<Student>, список будет хранить ссылки на объекты студентов.

📋 Шпаргалка: new, ссылки, null

// новый объект
Counter first = new Counter();

// копия ссылки, не копия объекта
Counter second = first;

// новый независимый объект
Counter third = new Counter();

// сравнение ссылок
boolean same = first == second; // true

// ссылки нет
Counter empty = null;
