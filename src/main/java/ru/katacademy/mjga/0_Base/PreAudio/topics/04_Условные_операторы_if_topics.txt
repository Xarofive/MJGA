Java Base. Модуль 4. Условные операторы if

Цель подкаста:
Обсудить, как программа принимает простые решения и почему условия нужно формулировать точно.

Уровень:
Новичок, знакомый с переменными и методами.

Общая идея модуля:
if позволяет программе выбрать действие в зависимости от условия. Условие должно давать ответ true или false.

Темы для дебатов:
- if делает программу умнее или просто добавляет развилки?
- Что сложнее для новичка: синтаксис if или формулировка правильного условия?
- Лучше писать один большой if или несколько маленьких проверок?
- Ранний return делает код понятнее или сначала выглядит непривычно?
- Почему порядок проверок в цепочке if/else if может менять результат?
- Логические операторы && и || помогают читать код или путают?

Вопросы, которые стоит раскрыть простыми словами:
- Что такое условие в программе?
- Почему условие не должно быть "примерным"?
- Когда нужен else, а когда можно обойтись без него?
- Как программа выбирает только одну ветку?
- Почему важно думать о крайних случаях?

Хорошие аналогии:
- Развилка на дороге.
- Правило допуска: если условие выполнено, действие разрешено.
- Фильтр, который пропускает только подходящие значения.

Чего избегать:
- Не обсуждать сложные boolean-алгебры.
- Не уходить в паттерны проектирования.
- Не превращать разговор в разбор длинных бизнес-правил.
