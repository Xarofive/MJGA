🐞 Debug: breakpoints, шаги и переменные

📝 TL;DR

Debug — это запуск программы с возможностью остановиться на выбранной строке и посмотреть, что происходит внутри кода.

Breakpoint ставится кликом слева от строки кода или клавишами Ctrl+F8 на Windows/Linux и ⌘F8 на macOS. После запуска через Debug программа остановится на этой строке, и IDEA покажет значения переменных.

Главные кнопки: Step Over выполняет текущую строку, Step Into заходит внутрь метода, Resume Program продолжает выполнение до следующей остановки. Их горячие клавиши: F8, F7 и F9.

🎯 Чему вы научитесь

- Ставить и убирать breakpoint

- Запускать программу в режиме Debug

- Смотреть значения локальных переменных

- Выполнять код по шагам

- Понимать, почему if выбрал конкретную ветку

- Отслеживать изменение переменной в цикле

- Проверять состояние объекта после вызова метода

📚 Предпосылки

- Вы умеете запускать класс с методом main

- Вы знаете переменные, условия, циклы и методы

- Вы работаете в IntelliJ IDEA Community Edition

1. Что такое breakpoint

📚 Точка остановки

Breakpoint — это место, где программа должна остановиться во время Debug. Вы ставите его кликом в серой области слева от строки кода или клавишами Ctrl+F8 на Windows/Linux и ⌘F8 на macOS.

Когда выполнение дойдёт до этой строки, IDEA остановит программу до выполнения строки. В этот момент можно смотреть переменные, стек вызовов и выполнять код по шагам.

public class Main {
public static void main(String[] args) {
int price = 100;
int discount = 15; // поставьте breakpoint здесь
int finalPrice = price - discount;

System.out.println(finalPrice);
}
}

2. Как запустить Debug

✅ Порядок действий

- Поставьте breakpoint на строку, которую хотите проверить: клик слева от строки, Ctrl+F8 на Windows/Linux или ⌘F8 на macOS.

- Нажмите зелёную кнопку рядом с main. Быстрый запуск обычного Run: Shift+F10 на Windows/Linux или ⌃R на macOS.

- Выберите Debug 'Main.main()', а не Run.

- Дождитесь, пока программа остановится на breakpoint.

- Посмотрите значения переменных в нижнем окне Debug.

💡 Run против Debug

Run просто выполняет программу от начала до конца.

Debug выполняет программу с остановками и позволяет смотреть состояние программы во время выполнения.

⌨️ Горячие клавиши запуска

- Run: Shift+F10 на Windows/Linux, ⌃R на macOS

- Debug: Shift+F9 на Windows/Linux, ⌃D на macOS

- Остановить программу: Ctrl+F2 на Windows/Linux, ⌘F2 на macOS

- Показать все breakpoints: Ctrl+Shift+F8 на Windows/Linux, ⌘⇧F8 на macOS

Если клавиши F7, F8 или F9 на ноутбуке управляют яркостью, громкостью или экраном, нажимайте их вместе с Fn.

3. Основные кнопки Debug

| Действие
| Горячие клавиши
| Что делает
| Когда использовать

| Toggle Breakpoint
| Ctrl+F8 на Windows/Linux, ⌘F8 на macOS
| Ставит или убирает breakpoint на текущей строке
| Когда нужно выбрать строку остановки без клика мышью

| Step Over
| F8
| Выполняет текущую строку и переходит к следующей
| Когда хотите идти по коду сверху вниз

| Step Into
| F7
| Заходит внутрь метода, который вызывается на строке
| Когда ошибка может быть внутри метода

| Step Out
| Shift+F8
| Выходит из текущего метода обратно к месту вызова
| Когда зашли в метод и уже поняли, что там происходит

| Resume Program
| F9
| Продолжает выполнение до следующего breakpoint или конца программы
| Когда текущий участок уже проверен

| Run to Cursor
| Alt+F9 на Windows/Linux, ⌥F9 на macOS
| Продолжает выполнение до строки, на которой стоит курсор
| Когда нужно быстро дойти до конкретного места без нового breakpoint

| Evaluate Expression
| Alt+F8 на Windows/Linux, ⌥F8 на macOS
| Позволяет проверить выражение прямо во время остановки
| Когда хотите быстро посчитать условие или значение переменной

| Stop
| Ctrl+F2 на Windows/Linux, ⌘F2 на macOS
| Останавливает запущенную программу
| Когда проверка закончена или программа зависла

4. Пример: почему if пошёл не туда

public class Main {
static String accessStatus(int age, boolean hasTicket) {
if (age > 18 && hasTicket) {
return "allowed";
}

return "denied";
}

public static void main(String[] args) {
String result = accessStatus(18, true);
System.out.println(result);
}
}

🔍 Что проверять в Debug

Поставьте breakpoint на строку с if через Ctrl+F8 / ⌘F8 и запустите Debug через Shift+F9 / ⌃D. В переменных будет видно: age = 18, hasTicket = true.

Условие age > 18 ложно для возраста 18. Если по правилу 18 уже разрешено, условие нужно заменить на age >= 18.

5. Пример: переменная в цикле

public class Main {
static int sumEven(int[] numbers) {
int sum = 0;

for (int number : numbers) {
if (number % 2 == 0) {
sum = number; // ошибка: старое значение суммы теряется
}
}

return sum;
}

public static void main(String[] args) {
int result = sumEven(new int[] {2, 4, 5, 6});
System.out.println(result);
}
}

🔍 Как увидеть ошибку

Поставьте breakpoint внутри if. Нажимайте Step Over (F8) и следите за sum.

После числа 2 сумма станет 2, после числа 4 станет 4, после числа 6 станет 6. Значит, код не накапливает сумму. Нужно написать sum = sum + number.

6. Пример: состояние объекта

class Counter {
int value;

void increment() {
value++;
}
}

public class Main {
public static void main(String[] args) {
Counter counter = new Counter();

counter.increment();
counter.increment();

System.out.println(counter.value);
}
}

Поставьте breakpoint на первый вызов counter.increment(). Нажмите Step Into (F7), чтобы зайти внутрь метода и увидеть, как меняется поле value. Чтобы выйти из метода обратно к вызову, используйте Step Out (Shift+F8).

🧠 Практическое правило

Если ошибка связана с результатом метода, используйте Step Into (F7). Если метод уже понятен и вы хотите пройти дальше, используйте Step Over (F8). Если нужно продолжить выполнение до следующей остановки, используйте Resume Program (F9).

7. Мини-чек-лист Debug

- Поставьте breakpoint до подозрительной строки: Ctrl+F8 / ⌘F8

- Запускайте через Debug, а не через Run: Shift+F9 / ⌃D

- Смотрите значения переменных до и после выполнения строки

- Для циклов проверяйте несколько итераций, а не только первую

- Для методов используйте Step Into: F7

- Для понятных строк используйте Step Over: F8

- Для продолжения до следующей остановки используйте Resume Program: F9

- Для объектов раскрывайте объект в панели Variables и смотрите поля

- После проверки убирайте ненужные breakpoints: Ctrl+F8 / ⌘F8

- Если нужно остановить программу, используйте Stop: Ctrl+F2 / ⌘F2
