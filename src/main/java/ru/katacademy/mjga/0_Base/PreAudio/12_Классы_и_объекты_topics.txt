Java Base. Модуль 12. Классы и объекты

Цель подкаста:
Подвести новичка к объектному мышлению: программа может описывать сущности, у которых есть данные и действия.

Уровень:
Новичок после переменных, методов, массивов и коллекций.

Общая идея модуля:
Класс описывает тип сущности. Объект - конкретный экземпляр. Поля хранят состояние, методы описывают поведение.

Темы для дебатов:
- Класс - это чертеж, шаблон или новый тип данных? Какая аналогия лучше?
- Объекты делают программу понятнее или сначала добавляют лишний слой?
- Поля и методы должны жить вместе или можно хранить данные отдельно?
- Почему объект через new - это не просто переменная?
- Ссылки на объекты помогают работать с данными или сначала путают?
- Когда стоит создавать класс, а когда достаточно нескольких переменных?

Вопросы, которые стоит раскрыть простыми словами:
- Чем класс отличается от объекта?
- Почему у двух объектов одного класса могут быть разные значения?
- Что значит "состояние объекта"?
- Почему метод объекта работает с его полями?
- Почему присваивание одной ссылки другой не создает копию объекта?

Хорошие аналогии:
- Класс как форма анкеты, объект как заполненная анкета.
- Класс как чертеж, объект как построенная вещь.
- Ссылка как адрес, по которому находится объект.

Чего избегать:
- Не обсуждать наследование, полиморфизм и интерфейсы.
- Не уходить в heap/stack подробно.
- Не превращать выпуск в большую лекцию по ООП.
