Java Base. Модуль 10. switch

Цель подкаста:
Показать switch как инструмент выбора из ограниченного набора вариантов, без усложнения синтаксисом.

Уровень:
Новичок, который уже знает if и строки.

Общая идея модуля:
switch удобен, когда программа выбирает действие по одному значению: команде, статусу, роли, номеру месяца или символу.

Темы для дебатов:
- switch заменяет if или решает другую задачу?
- Когда список вариантов читается лучше, чем цепочка if/else if?
- default - это необязательная ветка или защита от неожиданностей?
- Fall-through в классическом switch - полезная возможность или источник ошибок?
- Новый синтаксис switch делает код понятнее или сначала выглядит непривычно?
- Почему switch плохо подходит для диапазонов?

Вопросы, которые стоит раскрыть простыми словами:
- Что значит "выбор по одному значению"?
- Почему switch хорош для меню и команд?
- Чем case отличается от условия if?
- Зачем нужна ветка default?
- Как понять, что в задаче лучше использовать if, а не switch?

Хорошие аналогии:
- Меню в автомате: нажал номер - получил действие.
- Табличка соответствий: команда ведет к конкретной реакции.
- Развилка с заранее известными направлениями.

Чего избегать:
- Не обсуждать pattern matching.
- Не уходить в enum как отдельную большую тему.
- Не сравнивать switch в разных языках.
