<div style="background: linear-gradient(to right, #e3f2fd, #bbdefb); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #1565c0; margin-top: 0;">📝 TL;DR</h3>

<p style="margin: 10px 0;">Когда Java-программа падает, в консоли появляется stack trace &mdash; список вызовов, который помогает найти место ошибки.</p>

<p style="margin: 10px 0;">В первую очередь ищите название исключения, сообщение ошибки и первую строку из вашего кода. В IntelliJ IDEA по строкам stack trace часто можно кликнуть, чтобы перейти к нужной строке файла.</p>

<p style="margin: 10px 0;">Debugger полезен не только для неправильных результатов, но и для падений: поставьте breakpoint перед подозрительным местом и проверьте значения переменных до ошибки.</p>

<h2 style="color: #34495e; margin-top: 28px;">2. Что такое stack trace</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">class Student {
    String name;

    String upperName() {
        return name.toUpperCase();
    }
}

public class Main {
    public static void main(String[] args) {
        Student student = new Student();
        System.out.println(student.upperName());
    }
}</code></pre>
</div>

<p>Поле <code>name</code> не заполнено, поэтому оно равно <code>null</code>. При вызове <code>name.toUpperCase()</code> программа упадёт.</p>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">📌 Пример stack trace</h4>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 12px;"><code>Exception in thread "main" java.lang.NullPointerException
    at Student.upperName(Main.java:5)
    at Main.main(Main.java:12)</code></pre>

<p style="margin-bottom: 0;">Здесь важно: тип ошибки &mdash; <code>NullPointerException</code>, строка падения &mdash; <code>Main.java:5</code>, путь вызовов идёт через <code>Main.main</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">3. Как читать stack trace</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Алгоритм чтения</h4>

<ol style="margin: 10px 0;">
	<li>Найдите название исключения: например, <code>NullPointerException</code>.</li>
	<li>Прочитайте сообщение ошибки, если оно есть.</li>
	<li>Найдите первую строку, которая относится к вашему файлу.</li>
	<li>Кликните по строке в консоли IDEA, чтобы перейти к коду.</li>
	<li>Поставьте breakpoint перед этой строкой и запустите Debug.</li>
</ol>
</div>

<h2 style="color: #34495e; margin-top: 28px;">4. NullPointerException: самый частый случай</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">String text = null;
int length = text.length();</code></pre>
</div>

<p><code>NullPointerException</code> означает: вы пытаетесь обратиться к полю или методу через ссылку, которая равна <code>null</code>.</p>

<div style="background: #e8eaf6; border: 1px solid #3f51b5; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #283593; margin-top: 0;">🔍 Что проверять</h4>

<ul style="margin: 10px 0;">
	<li>Где должна была быть создана ссылка через <code>new</code>?</li>
	<li>Было ли поле заполнено до вызова метода?</li>
	<li>Может ли метод вернуть <code>null</code>?</li>
	<li>Нужна ли проверка <code>if (value == null)</code>?</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">5. Отладка падения через Debug</h2>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">✅ Порядок действий</h4>

<ol style="margin: 10px 0;">
	<li>Запустите программу обычным Run и прочитайте stack trace.</li>
	<li>Перейдите к строке, где программа упала.</li>
	<li>Поставьте breakpoint на 1-2 строки выше.</li>
	<li>Запустите программу через Debug.</li>
	<li>Посмотрите значения переменных перед падением.</li>
	<li>Исправьте причину, а не только симптом.</li>
</ol>
</div>

<h2 style="color: #34495e; margin-top: 28px;">6. Пример: ошибка в массиве</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-java">public class Main {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30};

        for (int i = 0; i &lt;= numbers.length; i++) {
            System.out.println(numbers[i]);
        }
    }
}</code></pre>
</div>

<p>Этот код упадёт с <code>ArrayIndexOutOfBoundsException</code>, потому что последний индекс массива равен <code>numbers.length - 1</code>. Условие цикла должно быть <code>i &lt; numbers.length</code>.</p>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🧠 Как увидеть это в Debug</h4>

<p style="margin-bottom: 0;">Поставьте breakpoint внутри цикла и следите за <code>i</code>. На последней итерации <code>i</code> станет равен <code>3</code>, а допустимые индексы массива: <code>0</code>, <code>1</code>, <code>2</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">7. План самостоятельной отладки</h2>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 15px; margin: 20px 0;">
<ol style="margin: 10px 0;">
	<li>Сформулируйте, какой результат ожидался.</li>
	<li>Сравните с фактическим результатом или текстом ошибки.</li>
	<li>Найдите участок кода, который отвечает за этот результат.</li>
	<li>Поставьте breakpoint перед этим участком.</li>
	<li>Пройдите код по шагам и смотрите переменные.</li>
	<li>Найдите первую строку, после которой значение стало неправильным.</li>
	<li>Исправьте код и снова запустите проверку.</li>
</ol>
</div>
