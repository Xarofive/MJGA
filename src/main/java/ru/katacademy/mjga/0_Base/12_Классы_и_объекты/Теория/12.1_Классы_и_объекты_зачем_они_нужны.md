<h1 style="text-align: center; color: #2c3e50;">Классы и объекты: основы и операции</h1>

<p style="text-align: center; color: #666; margin-bottom: 30px;">Учимся описывать сущности программы: данные, действия, создание объектов и работу с ними через точку.</p>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 TL;DR &mdash; Суть за 30 секунд</h3>
<ul>
	<li><code>class</code> создаёт новый тип данных, который описывает сущность программы.</li>
	<li>Класс не является конкретным объектом. Это шаблон: какие поля и методы будут у объектов.</li>
	<li>Объект создаётся через <code>new</code> и получает собственные значения полей.</li>
	<li>Поля хранят состояние объекта: имя студента, баланс счёта, количество товара.</li>
	<li>Методы описывают поведение объекта: включить лампу, пополнить счёт, посчитать стоимость.</li>
	<li>Через точку <code>object.field</code> и <code>object.method()</code> мы обращаемся к данным и действиям конкретного объекта.</li>
	<li>Классы нужны, когда несколько данных и действий логически относятся к одной сущности.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Что такое класс и зачем он нужен</h2>

<p>Класс в Java &mdash; это способ собрать связанные данные и действия в одну понятную модель. До классов вы могли хранить данные отдельными переменными, массивами и коллекциями. Это работает для маленьких задач, но плохо масштабируется.</p>

<p>Представьте студента:</p>
<ul>
	<li>имя;</li>
	<li>возраст;</li>
	<li>средний балл;</li>
	<li>действие: получить краткую карточку;</li>
	<li>действие: проверить, проходит ли студент по среднему баллу.</li>
</ul>

<p>Если хранить всё отдельно, связь между данными держится только в вашей голове:</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px;"><code class="language-java">String studentName = "Alice";
int studentAge = 20;
double studentAverageGrade = 4.7;</code></pre>

<p>Класс делает эту связь явной:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Student {
    String name;
    int age;
    double averageGrade;
}</code></pre>

<p><strong>Почему это важно:</strong> программа становится ближе к предметной области. Вы работаете не просто со строками и числами, а со студентами, товарами, заказами, счетами и пользователями.</p>

<p><strong>Когда использовать классы:</strong></p>
<ul>
	<li>нужно хранить несколько связанных значений;</li>
	<li>у сущности есть действия над этими значениями;</li>
	<li>нужно создавать много похожих объектов;</li>
	<li>нужно передавать сущность целиком в методы;</li>
	<li>код с отдельными переменными стал трудно читать и поддерживать.</li>
</ul>

<h2 style="color: #34495e; margin-top: 30px;">Анатомия простого класса</h2>

<p>Простой класс обычно состоит из полей и методов. В будущем появятся конструкторы, модификаторы доступа, наследование и интерфейсы, но для первого знакомства достаточно этой модели.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class BankAccount {
    String owner; // поле
    int balance;  // поле

    void deposit(int amount) { // метод
        balance = balance + amount;
    }

    String getInfo() { // метод
        return owner + ": " + balance;
    }
}</code></pre>

<h2 style="color: #34495e; margin-top: 30px;">Создание объекта и работа через точку</h2>

<p>Чтобы использовать класс, нужно создать объект:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">BankAccount account = new BankAccount();</code></pre>

<p>После этого через точку можно записывать поля, читать поля и вызывать методы:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">account.owner = "Alice";
account.balance = 100;

account.deposit(50);

String info = account.getInfo(); // "Alice: 150"</code></pre>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #856404; margin-top: 0;">📌 Суть объекта</h4>
<p style="margin-bottom: 0;">Объект полезен тем, что данные и действия живут рядом. Баланс хранится внутри счёта, а метод пополнения меняет именно этот счёт.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Реальный кейс: карточка товара</h2>

<h3 style="color: #34495e; margin-top: 20px;">Постановка задачи</h3>

<p>В интернет-магазине нужно хранить название товара, цену и количество на складе. Ещё нужно уметь посчитать общую стоимость остатков.</p>

<h3 style="color: #34495e; margin-top: 20px;">Решение через класс</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Product {
    String name;
    int price;
    int quantity;

    int totalPrice() {
        return price * quantity;
    }

    String label() {
        return name + " (" + quantity + " pcs)";
    }
}</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Использование</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Product notebook = new Product();
notebook.name = "Notebook";
notebook.price = 100;
notebook.quantity = 3;

System.out.println(notebook.totalPrice()); // 300
System.out.println(notebook.label());      // Notebook (3 pcs)</code></pre>

<p>Такой код читается как описание предметной области: есть товар, у товара есть цена и остаток, товар умеет считать свою общую стоимость.</p>

<h2 style="color: #34495e; margin-top: 30px;">Частые ошибки и как их избежать</h2>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 1: Класс есть, объекта нет</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">Student student;
student.name = "Alice"; // ошибка: объект не создан</code></pre>
<p style="margin-bottom: 0;">Сначала создайте объект через <code>new Student()</code>.</p>
</div>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 2: Метод печатает вместо return</h4>
<p>В учебных задачах метод чаще должен вернуть результат, а не вывести его.</p>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">String getInfo() {
    return owner + ": " + balance;
}</code></pre>
</div>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 3: Делать всё static</h4>
<p style="margin-bottom: 0;">Если поле относится к конкретному объекту, оно не должно быть <code>static</code>. У каждого студента своё имя, у каждого счёта свой баланс.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Мини-квиз для самопроверки</h2>

<div style="background: #f3e5f5; border: 1px solid #ce93d8; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #6a1b9a; margin-top: 0;">🧩 Вопрос 1: Что здесь класс, а что объект?</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">Student alice = new Student();</code></pre>
<p style="margin-bottom: 0;">Ответ: <code>Student</code> &mdash; класс, <code>new Student()</code> &mdash; объект, <code>alice</code> &mdash; переменная-ссылка.</p>
</div>

<div style="background: #f3e5f5; border: 1px solid #ce93d8; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #6a1b9a; margin-top: 0;">🧩 Вопрос 2: Что выведется?</h4>
<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px;"><code class="language-java">class Lamp {

<p style="margin-bottom: 0;">Ответ: <code>false</code>, потому что это значение по умолчанию для <code>boolean</code>.</p>
