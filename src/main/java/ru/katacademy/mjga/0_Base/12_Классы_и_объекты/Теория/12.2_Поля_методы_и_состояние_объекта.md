<h2 style="color: #34495e; margin-top: 30px;">Что такое состояние объекта</h2>

<p>Состояние объекта &mdash; это текущие значения его полей. У банковского счёта состояние может включать владельца и баланс. У товара &mdash; название, цену и количество. У лампы &mdash; модель и флаг включения.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Lamp {
    String model;
    boolean isOn;
}</code></pre>

<p>Если <code>isOn == false</code>, лампа выключена. Если <code>isOn == true</code>, лампа включена. Когда метод меняет <code>isOn</code>, он меняет состояние объекта.</p>

<p><strong>Когда важно состояние:</strong></p>
<ul>
	<li>заказ может быть новым, оплаченным или отменённым;</li>
	<li>счёт может иметь разный баланс;</li>
	<li>товар может иметь разный остаток на складе;</li>
	<li>пользователь может быть активным или заблокированным;</li>
	<li>игровой персонаж может иметь разное здоровье.</li>
</ul>

<h2 style="color: #34495e; margin-top: 30px;">Основные операции: чтение, изменение, расчёт</h2>

<h3 style="color: #34495e; margin-top: 20px;">Операция 1: чтение состояния</h3>

<p>Метод может вернуть информацию об объекте, не меняя его.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Student {
    String name;
    int age;

    String getCard() {
        return "Student: " + name + ", age: " + age;
    }
}</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Операция 2: изменение состояния</h3>

<p>Метод может изменить поле объекта.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Lamp {
    boolean isOn;

    void turnOn() {
        isOn = true;
    }

    void turnOff() {
        isOn = false;
    }
}</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Операция 3: расчёт на основе состояния</h3>

<p>Метод может вычислить значение по полям объекта.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Product {
    int price;
    int quantity;

    int totalPrice() {
        return price * quantity;
    }
}</code></pre>

<h2 style="color: #34495e; margin-top: 30px;">Методы как правила безопасности</h2>

<p>Главная сила методов не в том, что они “сокращают код”. Они позволяют спрятать правило рядом с данными, к которым это правило относится.</p>

<p>Например, банковский счёт не должен уходить в минус:</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class BankAccount {
    String owner;
    int balance;

    boolean withdraw(int amount) {
        if (amount &lt;= 0) {
            return false;
        }

        if (amount &gt; balance) {
            return false;
        }

        balance = balance - amount;
        return true;
    }
}</code></pre>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #856404; margin-top: 0;">📌 Почему это важно</h4>
<p style="margin-bottom: 0;">Если менять <code>balance</code> напрямую, легко получить отрицательный баланс. Метод <code>withdraw</code> задаёт единое правило: деньги снимаются только при корректной сумме и достаточном балансе.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Несколько объектов одного класса</h2>

<p>У каждого объекта своя копия полей.</p>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">Product phone = new Product();
phone.quantity = 2;

Product laptop = new Product();
laptop.quantity = 5;

phone.quantity++;

System.out.println(phone.quantity);  // 3
System.out.println(laptop.quantity); // 5</code></pre>

<p>Оба объекта созданы по классу <code>Product</code>, но это разные товары с независимым состоянием.</p>

<h2 style="color: #34495e; margin-top: 30px;">Реальный кейс: складской товар</h2>

<h3 style="color: #34495e; margin-top: 20px;">Постановка задачи</h3>
<p>На складе у товара есть остаток. Поставка увеличивает остаток, продажа уменьшает, но нельзя продать больше, чем есть.</p>

<h3 style="color: #34495e; margin-top: 20px;">Решение</h3>

<pre style="background: #263238; color: #aed581; padding: 15px; border-radius: 5px;"><code class="language-java">class Product {
    String name;
    int price;
    int quantity;

    void add(int amount) {
        if (amount &gt; 0) {
            quantity = quantity + amount;
        }
    }

    boolean sell(int amount) {
        if (amount &lt;= 0 || amount &gt; quantity) {
            return false;
        }

        quantity = quantity - amount;
        return true;
    }

    int totalPrice() {
        return price * quantity;
    }
}</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Разбор решения</h3>
<ul>
	<li><code>add</code> игнорирует некорректную поставку.</li>
	<li><code>sell</code> возвращает <code>false</code>, если операцию выполнить нельзя.</li>
	<li><code>totalPrice</code> ничего не меняет, а только считает.</li>
	<li>Все операции связаны с товаром, поэтому находятся внутри класса <code>Product</code>.</li>
</ul>

<h2 style="color: #34495e; margin-top: 30px;">Частые ошибки и как их избежать</h2>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 1: Метод меняет не то поле</h4>
<p style="margin-bottom: 0;">Проверяйте, что <code>sell</code> уменьшает <code>quantity</code>, а не <code>price</code>, и что <code>deposit</code> меняет <code>balance</code>, а не создаёт локальную переменную.</p>
</div>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 2: Нет проверки граничных значений</h4>
<p style="margin-bottom: 0;">Ноль и отрицательные значения почти всегда нужно обрабатывать отдельно: нельзя продать <code>-3</code> товара или пополнить счёт на <code>-100</code>.</p>
</div>

<div style="background: #fee; border-left: 4px solid #e74c3c; padding: 15px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 3: Метод всегда возвращает true</h4>
<p style="margin-bottom: 0;">Если операция может не выполниться, возвращаемое значение должно отражать реальный результат операции.</p>
</div>
