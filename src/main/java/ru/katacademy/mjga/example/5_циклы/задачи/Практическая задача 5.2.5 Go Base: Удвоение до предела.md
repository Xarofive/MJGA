<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔄 Удвоение до предела</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться создавать циклы в стиле while для итеративного увеличения значений с проверкой граничных условий и корректной обработкой случаев, когда цикл не должен выполняться.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В реальных системах часто встречаются задачи экспоненциального роста: расчёт процентов в банковских приложениях, увеличение размера буфера в алгоритмах, подбор оптимальной мощности в игровой механике. Умение контролировать геометрическую прогрессию с ограничением критично для оптимизации ресурсов и предотвращения переполнений.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию, которая удваивает начальное значение до тех пор, пока оно остаётся меньше заданного предела, и возвращает первое значение, достигшее или превысившее предел.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Цикл for в стиле while</h4>

<p>В Go нет отдельного ключевого слова while, но цикл for может работать только с условием:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Классический while из других языков
// while (условие) { ... }

// Эквивалент в Go
for условие {
    // тело цикла
}

// Пример: подсчёт степени двойки
value := 1
for value &lt; 100 {
    value *= 2
}
// value будет 128 (первое значение &gt;= 100)</code></pre>

<p style="margin-bottom: 0;">Цикл продолжается, пока условие истинно. Завершается, когда условие становится ложным.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Защитное программирование</h4>

<p>Проверяйте граничные условия ДО входа в цикл:</p>

<ul style="margin: 10px 0;">
	<li><strong>Предварительная проверка:</strong> если начальное значение уже &gt;= предела, цикл не нужен</li>
	<li><strong>Что верно на каждом шаге:</strong> значение всегда положительное и растёт &mdash; это гарантирует, что цикл не зависнет</li>
	<li><strong>Что верно в конце:</strong> после завершения цикла результат всегда &gt;= предела</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для удвоения числа до заданного предела:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция doubleUntilLimit</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func doubleUntilLimit(start, limit int) int</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает два параметра: <code>start</code> (начальное значение) и <code>limit</code> (предельное значение)</li>
	<li>Удваивает значение <code>start</code> пока оно меньше <code>limit</code></li>
	<li>Возвращает первое значение, которое больше или равно <code>limit</code></li>
	<li><strong>Важно:</strong> если <code>start &gt;= limit</code> изначально, вернуть <code>start</code> без выполнения цикла</li>
	<li>Использовать цикл <code>for</code> только с условием (в стиле while): <code>for value &lt; limit { ... }</code></li>
	<li>Внутри цикла значение должно удваиваться операцией <code>*= 2</code></li>
	<li>НЕ использовать <code>break</code> или <code>continue</code> &mdash; цикл завершается естественно по условию</li>
	<li><strong>Структура решения:</strong> только одна функция <code>doubleUntilLimit</code>, без вспомогательных функций &mdash; вся логика реализуется в теле <code>doubleUntilLimit</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📐 Математическая модель</h4>

<p>Функция удваивает <code>start</code> раз за разом, пока результат не станет &gt;= <code>limit</code>, и возвращает это первое достаточно большое значение.</p>

<p>Например: начали с 3, предел 50 &rarr; 3 &rarr; 6 &rarr; 12 &rarr; 24 &rarr; 48 &rarr; 96. Результат: 96.</p>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Отсутствие предварительной проверки</strong><br />
❌ Сразу входить в цикл без проверки <code>start &gt;= limit</code><br />
✅ Сначала проверить граничное условие, затем запускать цикл</p>

<p><strong>Ошибка 2: Использование обычного for-цикла</strong><br />
❌ <code>for i := 0; i &lt; n; i++ { ... }</code><br />
✅ <code>for value &lt; limit { ... }</code> &mdash; только условие, без инициализации и постинкремента</p>

<p><strong>Ошибка 3: Неправильная операция удвоения</strong><br />
❌ <code>value = value + 2</code> или <code>value++</code><br />
✅ <code>value *= 2</code> или <code>value = value * 2</code></p>

<p><strong>Ошибка 4: Использование break для выхода</strong><br />
❌ <code>if value &gt;= limit { break }</code><br />
✅ Условие цикла само контролирует завершение</p>

<p><strong>Ошибка 5: Попытка вернуть значение ДО превышения предела</strong><br />
❌ Сохранять предыдущее значение и возвращать его<br />
✅ Возвращать текущее значение после завершения цикла (которое уже &gt;= limit)</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Бесконечный цикл</strong><br />
❌ Забыть изменять значение внутри цикла<br />
✅ Обязательно удваивать значение на каждой итерации</p>
</div>

<h4 style="color: #6a1b9a; margin-top: 0;">🎨 Где это пригодится</h4>

<p><strong>Системное программирование:</strong> Динамическое увеличение размера буфера или кэша</p>

<p><strong>Игровая механика:</strong> Расчёт урона/опыта с мультипликаторами</p>

<p><strong>Финансовые расчёты:</strong> Моделирование сложных процентов с ограничением</p>

<p><strong>Сетевые протоколы:</strong> Экспоненциальная задержка при повторных попытках (exponential backoff)</p>

<p style="margin-bottom: 0;"><strong>Алгоритмы поиска:</strong> Подбор оптимального размера для хеш-таблиц</p>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Представьте, что вы подбираете размер контейнера для растущей колонии бактерий. Колония удваивается каждый час, начиная с <code>start</code> бактерий. Вам нужен минимальный контейнер, который вместит колонию, когда она достигнет или превысит <code>limit</code> бактерий. Цикл продолжается, пока колония помещается (меньше limit), и останавливается, как только она перерастёт этот размер.</p>
