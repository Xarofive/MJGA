<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">➕ Сумма чисел от 1 до N: арифметическая прогрессия</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Освоить итеративное накопление суммы и понять базовый паттерн обработки последовательностей чисел с помощью циклов.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст</h3>

<p style="margin-bottom: 10px;">Суммирование последовательностей &mdash; основа для расчёта средних значений, статистических метрик и агрегации данных. Этот навык применяется везде: от подсчёта общей стоимости товаров до вычисления контрольных сумм в сетевых протоколах.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию для вычисления суммы всех натуральных чисел от 1 до N включительно.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Арифметическая последовательность</h4>

<p>Числа от 1 до N образуют арифметическую прогрессию:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code>1, 2, 3, 4, ..., N

Сумма = 1 + 2 + 3 + ... + N

Примеры:
N=3: 1 + 2 + 3 = 6
N=5: 1 + 2 + 3 + 4 + 5 = 15</code></pre>

<p style="margin-bottom: 0;">Ключевая идея: начинаем с нуля и последовательно добавляем каждое число от 1 до N.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Паттерн накопления суммы</h4>

<p>Классический подход к суммированию последовательности:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code><span style="color: #546e7a;">// Общий паттерн</span>
sum := 0
<span style="color: #c792ea;">for</span> каждое_число_в_последовательности {
    sum = sum + текущее_число
}</code></pre>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию суммирования:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Сигнатура функции</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code><span style="color: #c792ea;">func</span> <span style="color: #82aaff;">sumToN</span>(n <span style="color: #82aaff;">int</span>) <span style="color: #82aaff;">int</span></code></pre>

<h4 style="color: #e65100; margin-top: 16px;">🎯 Требования</h4>

<ul style="margin: 10px 0;">
	<li><strong>Входные данные:</strong> целое число <code>n</code> (может быть положительным или нулём)</li>
	<li><strong>Выходные данные:</strong> сумма всех чисел от 1 до <code>n</code> включительно</li>
	<li><strong>Метод решения:</strong> использовать цикл <code>for</code> для итеративного суммирования</li>
	<li><strong>Ограничения:</strong> <code>n &ge; 0</code></li>
	<li><strong>Формат решения:</strong> отправляйте только функцию без <code>package</code> и <code>import</code></li>
	<li><strong>Запрещено:</strong> вывод в консоль через <code>print</code>, <code>println</code> и <code>fmt.*</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Особые случаи</h4>

<ul style="margin: 10px 0;">
	<li>При <code>n = 0</code> результат равен 0 (пустая сумма)</li>
	<li>При <code>n = 1</code> результат равен 1</li>
	<li>Для отрицательных <code>n</code> поведение не определено (в тестах не встречается)</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки</h4>

<p><strong>Ошибка 1: Начало с единицы вместо нуля</strong><br />
Если накопитель суммы начинается с 1, результат будет на единицу больше.</p>

<p><strong>Ошибка 2: Пропуск последнего числа</strong><br />
Убедитесь, что цикл включает число N. Условие должно быть <code>i &lt;= n</code>, а не <code>i &lt; n</code>.</p>

<p><strong>Ошибка 3: Начало цикла с нуля</strong><br />
Если цикл начинается с 0, в сумму попадёт лишний ноль (хотя это не изменит результат, это неэффективно).</p>

<p style="margin-bottom: 0;"><strong>Ошибка 4: Неверная обработка n=0</strong><br />
При n=0 цикл не должен выполняться вообще, результат должен быть 0.</p>
</div>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">✅ Проверьте себя</h4>

<ul style="margin-bottom: 0;">
	<li>Функция возвращает 0 при n=0?</li>
	<li>Функция возвращает 1 при n=1?</li>
	<li>Используется цикл <code>for</code> для итерации?</li>
	<li>Накопитель суммы инициализирован нулём?</li>
	<li>Цикл проходит от 1 до n включительно?</li>
	<li>Каждое число от 1 до n добавляется к сумме ровно один раз?</li>
	<li>В решении нет <code>package</code>/<code>import</code> и вызовов <code>print</code>, <code>println</code>, <code>fmt.*</code>?</li>
</ul>
</div>

<p>Представьте, что вы складываете монеты в копилку по одной. С какого номера монеты вы начнёте? До какого номера дойдёте? Что будет в копилке, если монет вообще нет (n=0)?</p>

<p style="margin-bottom: 0;">Подумайте о границах цикла: если n=5, какие числа должны быть просуммированы? Проверьте мысленно: 1, 2, 3, 4, 5 &mdash; все ли они попадут в вашу сумму?</p>
