<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔐 Валидатор паролей</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать короткую форму условного оператора if с инициализацией переменной для валидации данных.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Валидация паролей &mdash; критически важная функция безопасности любого приложения. Правильная проверка длины и состава пароля предотвращает взломы методом перебора и защищает пользовательские данные. Современные системы проверяют множество критериев: длину, наличие специальных символов, пробелов и других потенциально опасных элементов.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для проверки статуса пароля по базовым критериям безопасности.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Короткая форма if с инициализацией</h4>

<p>Go позволяет инициализировать переменную прямо в условии if:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Обычная форма
length := len(text)
if length &gt; 10 {
    // действие
}
// Короткая форма - переменная видна только внутри if/else
if length := len(text); length &gt; 10 {
// переменная length доступна здесь
} else {
// и здесь тоже
}
// но НЕ доступна здесь</code></pre>

<p style="margin-bottom: 0;">Эта форма делает код компактнее и ограничивает область видимости переменной.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Полезная функция: len()</h4>

<p>Встроенная функция <code>len(s)</code> возвращает длину строки. Это удобно использовать в короткой форме <code>if</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">if n := len(password); n &lt; 8 {
    return "too short"
} else if n &gt; 16 {
    return "too long"
}</code></pre>

<p style="margin-bottom: 0;">Переменная <code>n</code> вычисляется один раз и видна во всей цепочке <code>if/else if/else</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для проверки статуса пароля:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция passwordStatus</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func passwordStatus(password string) string</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>password</code> (string) &mdash; проверяемый пароль</li>
</ul>

<p><strong>Правила валидации (в порядке проверки):</strong></p>

<ol style="margin: 10px 0;">
	<li><strong>&quot;too short&quot;</strong> &mdash; пароль короче 8 символов</li>
	<li><strong>&quot;too long&quot;</strong> &mdash; пароль длиннее 16 символов</li>
	<li><strong>&quot;ok&quot;</strong> &mdash; пароль прошёл все проверки</li>
</ol>

<p><strong>Требования к реализации:</strong></p>

<ul style="margin: 10px 0;">
	<li>Обязательно использовать короткую форму <code>if</code> с инициализацией для вычисления длины</li>
	<li>Проверки выполнять в указанном порядке</li>
	<li>Запрещено использовать циклы (<code>for</code>) и оператор <code>switch</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📝 Формат решения</h4>

<ul style="margin: 10px 0;">
	<li>Включите в ответ полную сигнатуру функции, например <code>func passwordStatus(password string) string { ... }</code></li>
	<li>НЕ указывайте <code>package main</code></li>
	<li>НЕ добавляйте <code>import</code> директивы</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Обычная форма if вместо короткой</strong><br />
❌ <code>len := len(password); if len &lt; 8 {...}</code><br />
✅ <code>if len := len(password); len &lt; 8 {...}</code></p>

<p><strong>Ошибка 2: Неправильный порядок проверок</strong><br />
❌ Сначала проверять максимум, потом минимум<br />
✅ Порядок: короткий &rarr; длинный &rarr; ok</p>

<p><strong>Ошибка 3: Неверные границы длины</strong><br />
❌ Минимум 7, максимум 17 символов<br />
✅ Минимум 8, максимум 16 символов включительно</p>

<p><strong>Ошибка 4: Опечатки в возвращаемых строках</strong><br />
❌ <code>&quot;Too short&quot;</code>, <code>&quot;OK&quot;</code><br />
✅ Точные значения: <code>&quot;too short&quot;</code>, <code>&quot;too long&quot;</code>, <code>&quot;ok&quot;</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 5: Множественные return без else</strong><br />
❌ Цепочка if с return без else if<br />
✅ Использовать if - else if для ясности логики</p>
</div>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Короткая форма if особенно полезна, когда переменная нужна для нескольких проверок. В нашем случае длина пароля используется дважды: для проверки минимума и максимума. Благодаря области видимости внутри if-else, переменная <code>n</code> доступна во всей цепочке условий, но не засоряет внешнюю область видимости.</p>
