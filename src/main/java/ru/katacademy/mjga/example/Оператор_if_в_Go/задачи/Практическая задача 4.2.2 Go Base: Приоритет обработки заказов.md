<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📦 Приоритет обработки заказов</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать составные условия для принятия решений на основе нескольких параметров и корректно обрабатывать недопустимые входные данные.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В логистике и e-commerce системы автоматически определяют приоритет обработки заказов на основе суммы покупки и типа доставки. Это критически важно для оптимизации складских операций, управления очередями и обеспечения качественного сервиса. Правильная приоритизация напрямую влияет на удовлетворённость клиентов и эффективность бизнеса.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для определения приоритета обработки заказа на складе.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Составные условия</h4>

<p>В Go можно комбинировать несколько условий с помощью логических операторов:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Логическое И (AND) - оба условия должны быть истинны
if условие1 &amp;&amp; условие2 {
    // выполняется только если ОБА условия истинны
}
// Логическое ИЛИ (OR) - хотя бы одно условие истинно
if условие1 || условие2 {
// выполняется если ХОТЯ БЫ ОДНО условие истинно
}
// Комбинирование условий
if (x &gt; 10 &amp;&amp; y &lt; 5) || z == 0 {
// сложная логика с приоритетами
}</code></pre>

<p style="margin-bottom: 0;">Скобки помогают явно указать приоритет операций.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Работа с булевым типом</h4>

<p>Тип <code>bool</code> представляет логические значения:</p>

<ul style="margin: 10px 0;">
	<li><strong>Значения:</strong> только <code>true</code> или <code>false</code></li>
	<li><strong>В условиях:</strong> можно использовать напрямую без сравнения</li>
	<li><strong>Пример:</strong> <code>if express { ... }</code> эквивалентно <code>if express == true { ... }</code></li>
	<li><strong>Отрицание:</strong> <code>if !express { ... }</code> проверяет на <code>false</code></li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для определения приоритета обработки заказа:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция orderPriority</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func orderPriority(total int, express bool) string</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>total</code> (int) &mdash; общая сумма заказа в условных единицах</li>
	<li><code>express</code> (bool) &mdash; признак экспресс-доставки (<code>true</code> &mdash; экспресс, <code>false</code> &mdash; обычная)</li>
</ul>

<p><strong>Бизнес-логика приоритизации:</strong></p>

<ul style="margin: 10px 0;">
	<li><strong>&quot;invalid&quot;</strong> &mdash; для недопустимых заказов (сумма &le; 0)</li>
	<li><strong>&quot;rush&quot;</strong> &mdash; экспресс-доставка при любой допустимой сумме</li>
	<li><strong>&quot;priority&quot;</strong> &mdash; обычная доставка при сумме &gt; 100</li>
	<li><strong>&quot;standard&quot;</strong> &mdash; обычная доставка при сумме от 1 до 100 включительно</li>
</ul>

<p><strong>Требования к реализации:</strong></p>

<ul style="margin: 10px 0;">
	<li>Обязательно использовать условные операторы <code>if</code></li>
	<li>Запрещено использовать циклы (<code>for</code>) и оператор <code>switch</code></li>
	<li>Запрещено использовать функции вывода (<code>fmt.Print*</code>)</li>
	<li>Функция должна возвращать строку с приоритетом</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📝 Формат решения</h4>

<ul style="margin: 10px 0;">
	<li>Включите в ответ полную сигнатуру функции, например <code>func orderPriority(total int, express bool) string { ... }</code></li>
	<li>НЕ указывайте <code>package main</code></li>
	<li>НЕ добавляйте <code>import</code> директивы</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неправильный порядок проверок</strong><br />
❌ Сначала проверять размер заказа (<code>total &gt; 100</code>), потом тип доставки (<code>express</code>) &mdash; заказ с большой суммой и экспрессом получит <code>&quot;priority&quot;</code> вместо <code>&quot;rush&quot;</code><br />
✅ Сначала отсеять недопустимые (<code>total &le; 0</code> &rarr; <code>&quot;invalid&quot;</code>), затем проверить <code>express</code> &rarr; <code>&quot;rush&quot;</code>, и только для обычной доставки различать <code>&quot;priority&quot;</code>/<code>&quot;standard&quot;</code> по сумме</p>

<p><strong>Ошибка 2: Некорректная обработка нуля</strong><br />
❌ Ноль считается допустимой суммой заказа<br />
✅ При total &le; 0 возвращать <code>&quot;invalid&quot;</code></p>

<p><strong>Ошибка 3: Неверные границы для priority</strong><br />
❌ <code>&quot;priority&quot;</code> при total &ge; 100<br />
✅ <code>&quot;priority&quot;</code> строго при total &gt; 100</p>

<p><strong>Ошибка 4: Опечатки в возвращаемых строках</strong><br />
❌ <code>&quot;Invalid&quot;</code>, <code>&quot;RUSH&quot;</code>, <code>&quot;standart&quot;</code><br />
✅ Точные значения: <code>&quot;invalid&quot;</code>, <code>&quot;rush&quot;</code>, <code>&quot;priority&quot;</code>, <code>&quot;standard&quot;</code></p>

<p><strong>Ошибка 5: Использование запрещённых конструкций</strong><br />
❌ <code>switch</code>, <code>for</code>, <code>fmt.Println</code><br />
✅ Только <code>if</code>, <code>else if</code>, <code>else</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Неправильная сигнатура</strong><br />
❌ <code>func orderPriority(sum int, fast bool)</code> &mdash; забыт тип возврата<br />
✅ <code>func orderPriority(total int, express bool) string</code> &mdash; имена параметров могут быть любыми, главное &mdash; типы</p>
</div>

<h4 style="color: #2e7d32; margin-top: 0;">✅ Чек-лист самопроверки</h4>

<ul style="margin-bottom: 0;">
	<li>Функция называется <code>orderPriority</code>?</li>
	<li>Принимает параметры типов <code>int</code> и <code>bool</code>?</li>
	<li>Возвращает тип <code>string</code>?</li>
	<li>Проверяет недопустимые суммы (&le; 0) первыми?</li>
	<li>Экспресс-доставка всегда возвращает <code>&quot;rush&quot;</code> для допустимых сумм?</li>
	<li>Обычная доставка правильно различает <code>&quot;standard&quot;</code> и <code>&quot;priority&quot;</code>?</li>
	<li>Граница между standard и priority проходит строго по 100?</li>
	<li>Использует условные операторы <code>if</code>?</li>
	<li>Нет циклов, switch и функций вывода?</li>
	<li>Код содержит полное определение <code>func orderPriority(total int, express bool) string { ... }</code> (с сигнатурой и телом) без <code>package</code> и <code>import</code>?</li>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Представьте дерево решений: сначала отсеиваем недопустимые заказы (это защита от некорректных данных). Затем проверяем самый важный фактор &mdash; тип доставки. Экспресс-заказы имеют абсолютный приоритет независимо от суммы. Только для обычной доставки сумма определяет уровень приоритета. Подумайте, в каком порядке нужно проверять условия, чтобы каждый заказ попал в правильную категорию.</p>
