<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📊 Система оценивания результатов</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать множественные операторы return для раннего выхода из функции и создания чёткой логики проверок.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Системы оценивания используются в образовании, сертификации и тестировании. Правильная категоризация результатов помогает студентам понимать свой прогресс, а преподавателям &mdash; принимать решения. Техника раннего выхода (early return) делает код чище, убирая лишнюю вложенность и делая логику более линейной и понятной.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для классификации экзаменационных баллов.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Ранний выход из функции</h4>

<p>Оператор return может использоваться в любом месте функции для немедленного выхода:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func checkValue(x int) string {
    // Сначала проверяем некорректные данные
    if x &lt; 0 {
        return "invalid"  // выход здесь
    }
// Затем особые случаи
if x == 0 {
    return "zero"     // или здесь
}

// Основная логика
if x &lt; 100 {
    return "normal"   // или здесь
}

return "high"         // финальный выход
}</code></pre>

<p style="margin-bottom: 0;">Такой подход называется &quot;guard clauses&quot; &mdash; защитные условия в начале функции.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Преимущества множественных return</h4>

<p>Использование нескольких return улучшает читаемость кода:</p>

<ul style="margin: 10px 0;">
	<li><strong>Линейная логика:</strong> каждая проверка на своём уровне</li>
	<li><strong>Меньше вложенности:</strong> избегаем глубоких if-else цепочек</li>
	<li><strong>Явные условия:</strong> каждый случай обрабатывается отдельно</li>
	<li><strong>Защитный стиль:</strong> сначала отсеиваем невалидные данные</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для оценивания экзаменационных баллов:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция scoreLabel</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func scoreLabel(score int) string</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>score</code> (int) &mdash; количество баллов (от 0 до 100 для валидных значений)</li>
</ul>

<p><strong>Система оценивания:</strong></p>

<ul style="margin: 10px 0;">
	<li><strong>&quot;invalid&quot;</strong> &mdash; баллы &lt; 0 или &gt; 100</li>
	<li><strong>&quot;failed&quot;</strong> &mdash; от 0 до 69 включительно</li>
	<li><strong>&quot;passed&quot;</strong> &mdash; от 70 до 89 включительно</li>
	<li><strong>&quot;excellent&quot;</strong> &mdash; от 90 до 100 включительно</li>
</ul>

<p><strong>Требования к реализации:</strong></p>

<ul style="margin: 10px 0;">
	<li>Использовать минимум 2 оператора <code>return</code> для раннего выхода</li>
	<li>Сначала проверять невалидные значения</li>
	<li>Избегать глубокой вложенности if-else</li>
	<li>Запрещено использовать циклы (<code>for</code>) и оператор <code>switch</code></li>
	<li>Запрещено использовать функции вывода (<code>fmt.Print*</code>)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📝 Формат решения</h4>

<ul style="margin: 10px 0;">
	<li>Включите в ответ полную сигнатуру функции, например <code>func scoreLabel(score int) string { ... }</code></li>
	<li>НЕ указывайте <code>package main</code></li>
	<li>НЕ добавляйте <code>import</code> директивы</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Только один return в конце</strong><br />
❌ Вложенные if-else с одним return<br />
✅ Несколько return для раннего выхода</p>

<p><strong>Ошибка 2: Неправильные границы</strong><br />
❌ 89 баллов считается excellent<br />
✅ 70-89 &mdash; passed, 90-100 &mdash; excellent</p>

<p><strong>Ошибка 3: Забыта проверка на &gt; 100</strong><br />
❌ Принимать любые положительные числа<br />
✅ Валидный диапазон строго 0-100</p>

<p><strong>Ошибка 4: Неверный порядок проверок</strong><br />
❌ Сначала проверять на passed/failed<br />
✅ Сначала invalid, затем категории оценок</p>

<p><strong>Ошибка 5: Опечатки в возвращаемых строках</strong><br />
❌ <code>&quot;Invalid&quot;</code>, <code>&quot;pass&quot;</code>, <code>&quot;fail&quot;</code><br />
✅ Точные значения: <code>&quot;invalid&quot;</code>, <code>&quot;failed&quot;</code>, <code>&quot;passed&quot;</code>, <code>&quot;excellent&quot;</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Глубокая вложенность</strong><br />
❌ if { if { if { ... } } }<br />
✅ Плоская структура с ранними return</p>
</div>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Техника раннего выхода делает код читаемым как книгу: сверху вниз, без прыжков. Представьте охранника на входе: сначала он отсеивает тех, кто точно не пройдёт (невалидные данные), затем пропускает по категориям. Каждая проверка &mdash; это фильтр, и если условие выполнено, дальше проверять не нужно. Это избавляет от необходимости помнить, в какой ветке else мы находимся.</p>
