<div style="background: #fff3cd; border: 2px solid #ffc107; border-radius: 8px; padding: 10px 14px; margin: 20px 0;">
<p style="margin: 0; color: #856404;"><strong>⭐ Задача повышенной сложности.</strong> Здесь нужно учесть два набора пороговых значений в зависимости от состояния сети. Если пока сложно &mdash; переходите к задаче 07 и вернитесь позже.</p>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать условные операторы для адаптивного выбора качества видео на основе скорости соединения и состояния сети.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Стриминговые платформы (Netflix, YouTube, Twitch) автоматически подстраивают качество видео под скорость интернета пользователя. Это обеспечивает плавное воспроизведение без буферизации, экономит трафик и улучшает пользовательский опыт. Адаптивная потоковая передача &mdash; ключевая технология современного интернет-видео.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для выбора оптимального качества видеопотока.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Адаптивная логика</h4>

<p>При выборе качества учитываются несколько факторов:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Проверка валидности данных
if скорость &lt;= 0 {
    return "недопустимо"
}
// Учёт перегруженности сети
if перегружена {
// снижаем качество для стабильности
}
// Выбор качества по скорости
if высокая_скорость {
// максимальное качество
}</code></pre>

<p style="margin-bottom: 0;">Перегруженная сеть требует снижения качества независимо от скорости.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Уровни качества видео</h4>

<p>Стандартные уровни качества и требования к скорости:</p>

<ul style="margin: 10px 0;">
	<li><strong>Low (480p):</strong> минимальное качество, для медленных соединений</li>
	<li><strong>Medium (720p):</strong> стандартное HD, баланс качества и скорости</li>
	<li><strong>High (1080p):</strong> Full HD, требует стабильного соединения</li>
	<li><strong>Ultra (4K):</strong> максимальное качество, только для быстрых соединений</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для определения качества видеопотока:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция streamQuality</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func streamQuality(speed int, congested bool) string</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>speed</code> (int) &mdash; скорость интернет-соединения в Мбит/с</li>
	<li><code>congested</code> (bool) &mdash; признак перегруженности сети (<code>true</code> &mdash; перегружена)</li>
</ul>

<p><strong>Правила выбора качества:</strong></p>

<ul style="margin: 10px 0;">
	<li><strong>&quot;invalid&quot;</strong> &mdash; недопустимая скорость (&le; 0)</li>
	<li><strong>При перегруженной сети (congested = true):</strong>
	<ul>
		<li>&quot;low&quot; &mdash; скорость &lt; 15 Мбит/с</li>
		<li>&quot;medium&quot; &mdash; скорость &ge; 15 Мбит/с</li>
	</ul>
	</li>
	<li><strong>При нормальной сети (congested = false):</strong>
	<ul>
		<li>&quot;low&quot; &mdash; скорость &lt; 8 Мбит/с</li>
		<li>&quot;medium&quot; &mdash; скорость от 8 до 14 Мбит/с включительно</li>
		<li>&quot;high&quot; &mdash; скорость от 15 до 24 Мбит/с включительно</li>
		<li>&quot;ultra&quot; &mdash; скорость &ge; 25 Мбит/с</li>
	</ul>
	</li>
</ul>

<p><strong>Требования к реализации:</strong></p>

<ul style="margin: 10px 0;">
	<li>Обязательно использовать условные операторы <code>if</code></li>
	<li>Учитывать оба параметра: <code>speed</code> и <code>congested</code></li>
	<li>Запрещено использовать циклы (<code>for</code>) и оператор <code>switch</code></li>
	<li>Запрещено использовать функции вывода (<code>fmt.Print*</code>)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📝 Формат решения</h4>

<ul style="margin: 10px 0;">
	<li>Включите в ответ полную сигнатуру функции, например <code>func streamQuality(speed int, congested bool) string { ... }</code></li>
	<li>НЕ указывайте <code>package main</code></li>
	<li>НЕ добавляйте <code>import</code> директивы</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Игнорирование перегруженности</strong><br />
❌ Выбирать качество только по скорости<br />
✅ При перегруженной сети максимум &mdash; &quot;medium&quot;</p>

<p><strong>Ошибка 2: Неправильные границы диапазонов</strong><br />
❌ Путать границы между уровнями качества<br />
✅ Внимательно проверить: 8-14 (medium), 15-24 (high), 25+ (ultra)</p>

<p><strong>Ошибка 3: Некорректная валидация</strong><br />
❌ Скорость 0 считается допустимой<br />
✅ Скорость должна быть строго больше 0</p>

<p><strong>Ошибка 4: Неправильный порядок проверок</strong><br />
❌ Сначала проверять качество, потом валидность<br />
✅ Сначала валидация, затем перегруженность, затем скорость</p>

<p><strong>Ошибка 5: Опечатки в строках</strong><br />
❌ <code>&quot;Low&quot;</code>, <code>&quot;MEDIUM&quot;</code>, <code>&quot;4K&quot;</code><br />
✅ Точные значения: <code>&quot;invalid&quot;</code>, <code>&quot;low&quot;</code>, <code>&quot;medium&quot;</code>, <code>&quot;high&quot;</code>, <code>&quot;ultra&quot;</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Неиспользование параметра congested</strong><br />
❌ Не проверять состояние сети<br />
✅ Обязательно учитывать параметр <code>congested</code></p>
</div>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Представьте двухуровневую систему принятия решений. На первом уровне проверяем критические условия: валидность данных и состояние сети. Если сеть перегружена, мы ограничены только двумя качествами независимо от скорости. На втором уровне, при нормальной сети, у нас полный спектр качеств, и выбор зависит только от скорости. Помните: перегруженная сеть &mdash; это как узкое горлышко, через которое не пройдёт высокое качество.</p>
