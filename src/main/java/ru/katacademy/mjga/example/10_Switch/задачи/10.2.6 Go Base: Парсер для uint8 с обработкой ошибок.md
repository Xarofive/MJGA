<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔢 Парсер uint8 с классификацией результата через switch</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать оператор <code>switch</code> для классификации результата операции: успешный парсинг или одна из нескольких подсказок об ошибке.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Парсинг пользовательского ввода &mdash; критическая задача в веб-формах, конфигурационных файлах, API. Полезно сообщить пользователю, что именно пошло не так: число вышло за допустимые пределы или ввод вообще не выглядит как число.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для парсинга строки в uint8, использующую <code>switch</code> для возврата подходящей подсказки.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Тип uint8</h4>

<p>Тип <code>uint8</code> &mdash; беззнаковое 8-битное целое число:</p>

<ul style="margin: 10px 0;">
	<li>Диапазон значений: 0&ndash;255</li>
	<li>Занимает 1 байт памяти</li>
	<li>Синоним типа <code>byte</code></li>
	<li>Часто используется для работы с байтами и цветовыми компонентами RGB</li>
</ul>

<p style="margin-bottom: 0;">Числа вне диапазона 0&ndash;255 не могут быть представлены в uint8.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🔧 Парсинг строки в число</h4>

<p>Для разбора строки в беззнаковое число используется <code>strconv.ParseUint</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ParseUint(строка, основание, размер в битах)
n, err := strconv.ParseUint("42", 10, 8)
value := uint8(n)</code></pre>

<p style="margin-bottom: 0;">При ошибке функция возвращает значение типа <code>error</code>. На текущем уровне достаточно проверить <code>if err != nil</code> и различать причины по тексту сообщения.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔀 Switch для классификации причины ошибки</h4>

<p><code>strconv.ParseUint</code> при ошибке возвращает текст, в котором обычно встречается одна из двух фраз: <code>&quot;out of range&quot;</code> (число вышло за пределы 0&ndash;255) или <code>&quot;invalid syntax&quot;</code> (строка вообще не похожа на число). Это можно различить с помощью <code>strings.Contains</code> и <code>switch</code> без выражения:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">msg := err.Error()
switch {
case strings.Contains(msg, "out of range"):
    return 0, "out of range"
default:
    return 0, "bad syntax"
}</code></pre>

<p style="margin-bottom: 0;">Эта форма <code>switch</code> группирует разные ветви проверки и явно показывает их взаимоисключающий характер.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте парсер строки в uint8 с классификацией ошибок через switch:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура пакета</h4>

<ul style="margin: 10px 0;">
	<li>Пакет называется <code>u8</code></li>
	<li>Импортирует пакеты: <code>strconv</code>, <code>strings</code> (без псевдонимов)</li>
	<li>Содержит функцию <code>ParseUint8</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Функция ParseUint8</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func ParseUint8(s string) (uint8, string)</code></div>

<p><strong>Поведение:</strong></p>

<ul style="margin: 10px 0;">
	<li>При успешном парсинге возвращает (число, <code>&quot;&quot;</code>) &mdash; пустая строка означает &laquo;ошибки нет&raquo;</li>
	<li>При ошибке возвращает (0, причина) &mdash; где причина это либо <code>&quot;out of range&quot;</code>, либо <code>&quot;bad syntax&quot;</code></li>
	<li>Используйте <code>strconv.ParseUint(s, 10, 8)</code> для парсинга</li>
	<li>Внутри функции <strong>обязательно</strong> используйте оператор <code>switch</code> &mdash; это тема модуля M10</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📊 Правила классификации</h4>

<p>Если <code>strconv.ParseUint</code> вернул ошибку, разберите её текст:</p>

<ul style="margin: 10px 0;">
	<li>Если <code>strings.Contains(msg, &quot;out of range&quot;)</code> &rarr; причина <code>&quot;out of range&quot;</code></li>
	<li>Иначе &rarr; причина <code>&quot;bad syntax&quot;</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🎯 Важные детали</h4>

<ul style="margin: 10px 0;">
	<li>Основание 10: парсим десятичные числа</li>
	<li>Размер 8 бит: результат помещается в uint8</li>
	<li>При любой ошибке всегда возвращайте 0 как первое значение</li>
	<li>Пустая строка причины (<code>&quot;&quot;</code>) означает &laquo;всё хорошо&raquo;</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неправильные параметры ParseUint</strong><br />
❌ <code>strconv.ParseUint(s, 0, 8)</code> или <code>strconv.ParseUint(s, 10, 64)</code><br />
✅ <code>strconv.ParseUint(s, 10, 8)</code> &mdash; основание 10, размер 8</p>

<p><strong>Ошибка 2: Возврат не-нуля при ошибке</strong><br />
❌ Возвращать исходное значение при ошибке<br />
✅ Всегда возвращать 0 при любой ошибке</p>

<p><strong>Ошибка 3: Неправильный текст причины</strong><br />
❌ <code>&quot;invalid syntax&quot;</code> или <code>&quot;value out of range&quot;</code><br />
✅ Точно <code>&quot;bad syntax&quot;</code> и <code>&quot;out of range&quot;</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 4: Решение без switch</strong><br />
❌ Цепочка <code>if/else if</code><br />
✅ Используйте <code>switch</code> &mdash; это тема модуля M10</p>
