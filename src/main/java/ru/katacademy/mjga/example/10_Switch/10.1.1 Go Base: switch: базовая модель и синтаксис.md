<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🎛️ switch: базовая модель и синтаксис</h1>

<div style="background: linear-gradient(to right, #e3f2fd, #bbdefb); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #1565c0; margin-top: 0;">📝 TL;DR</h3>

<p style="margin: 10px 0;"><code>switch</code> в Go &mdash; это конструкция ветвления для выбора одного из множества вариантов. В Go срабатывает <strong>только первая подходящая ветка</strong>, затем выполнение автоматически выходит из switch (нет неявного &laquo;проваливания&raquo;).</p>

<p style="margin: 10px 0;">Switch может проверять значение выражения или работать как цепочка if-else с условиями в каждом case. Ветка <code>default</code> срабатывает, когда ни один case не подошёл &mdash; это повышает устойчивость кода к неожиданным значениям.</p>

<p style="margin: 10px 0;">Go позволяет объединять несколько значений в одном case через запятую и использовать &laquo;короткое объявление&raquo; переменной прямо в заголовке switch для локализации области видимости.</p>
</div>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">

<ul style="margin: 10px 0;">
	<li>Писать switch с выражением и без выражения</li>
	<li>Объединять несколько значений в одном case через запятую</li>
	<li>Правильно использовать ветку default для обработки неожиданных случаев</li>
	<li>Применять &laquo;короткое объявление&raquo; переменной в заголовке switch</li>
	<li>Понимать область видимости переменных в switch</li>
	<li>Выбирать между switch и if-else для лучшей читаемости</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">1. Формы switch</h2>

<h3 style="color: #455a64; margin-top: 24px;">Switch по значению</h3>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Классический switch</h4>

<p style="margin-bottom: 10px;">Самая распространённая форма &mdash; проверка значения выражения. Go сравнивает результат выражения с каждым case по порядку и выполняет код первого совпадения.</p>

<p style="margin-bottom: 10px;"><strong>Важно:</strong> выражение вычисляется один раз перед проверкой всех case. В форме <code>switch v { case ... }</code> сопоставление идёт через оператор <code>==</code>, поэтому значения в case должны быть сопоставимы с типом <code>v</code>.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

func main() {
day := 3

    switch day {
    case 1:
        fmt.Println("Понедельник")
    case 2:
        fmt.Println("Вторник")
    case 3:
        fmt.Println("Среда")
    case 4:
        fmt.Println("Четверг")
    case 5:
        fmt.Println("Пятница")
    case 6, 7: // Несколько значений в одном case
        fmt.Println("Выходной!")
    default:
        fmt.Println("Неверный день недели")
    }
    // Вывод: Среда
}</code></pre>
</div>

<h3 style="color: #455a64; margin-top: 24px;">Switch без выражения</h3>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">💡 Альтернатива цепочке if-else</h4>

<p style="margin-bottom: 10px;">Switch без выражения эквивалентен <code>switch true</code> &mdash; каждый case содержит булево условие. Это часто читабельнее длинной цепочки if-else if.</p>

<p style="margin: 0;">Выполняется первый case, условие которого истинно. Условия проверяются по порядку, поэтому важна их последовательность. В этой форме не требуется сравнимость типов, так как используются булевы условия.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

func classifyNumber(n int) string {
switch {
case n &lt; 0:
return "отрицательное"
case n == 0:
return "ноль"
case n &gt; 0 &amp;&amp; n &lt; 10:
return "однозначное положительное"
case n &gt;= 10 &amp;&amp; n &lt; 100:
return "двузначное"
default:
return "большое число"
}
}

func main() {
fmt.Println(classifyNumber(-5))   // отрицательное
fmt.Println(classifyNumber(0))    // ноль
fmt.Println(classifyNumber(7))    // однозначное положительное
fmt.Println(classifyNumber(42))   // двузначное
fmt.Println(classifyNumber(1000)) // большое число
}</code></pre>
</div>

<h3 style="color: #455a64; margin-top: 24px;">Несколько значений в case</h3>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

func getMonthSeason(month int) string {
switch month {
case 12, 1, 2:
return "Зима"
case 3, 4, 5:
return "Весна"
case 6, 7, 8:
return "Лето"
case 9, 10, 11:
return "Осень"
default:
return "Неверный месяц"
}
}

func main() {
fmt.Println(getMonthSeason(1))  // Зима
fmt.Println(getMonthSeason(7))  // Лето
fmt.Println(getMonthSeason(10)) // Осень
fmt.Println(getMonthSeason(13)) // Неверный месяц
}</code></pre>
</div>

<div style="background: #e8eaf6; border: 1px solid #3f51b5; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #283593; margin-top: 0;">🔍 Порядок выполнения</h4>

<p style="margin-bottom: 10px;">Go проверяет case сверху вниз и останавливается на первом совпадении. После выполнения кода выбранного case управление автоматически выходит из switch &mdash; нет &laquo;проваливания&raquo; в следующий case.</p>

<p style="margin: 0;">Если нужно явное проваливание (редкий случай), используется ключевое слово <code>fallthrough</code>. На практике он встречается редко; подробно &mdash; в третьей лекции этого модуля.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">2. Короткое объявление в заголовке switch</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Паттерн: Инициализация перед проверкой</h4>

<p style="margin-bottom: 10px;">Как и в if, можно объявить переменную прямо в заголовке switch. Эта переменная видна только внутри switch, что помогает изолировать временные вычисления.</p>

<p style="margin: 0;">Синтаксис: <code>switch переменная := выражение; переменная { ... }</code></p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"fmt"
"strings"
)

func processCommand(input string) {
// Объявляем cmd прямо в switch, она видна только внутри
switch cmd := strings.TrimSpace(strings.ToLower(input)); cmd {
case "start", "run", "go":
fmt.Println("Запускаем процесс...")
case "stop", "halt", "end":
fmt.Println("Останавливаем процесс...")
case "pause":
fmt.Println("Приостанавливаем процесс...")
case "":
fmt.Println("Пустая команда")
default:
fmt.Printf("Неизвестная команда: %q\n", cmd)
}
// cmd здесь уже не существует
}

func main() {
processCommand("  START  ")  // Запускаем процесс...
processCommand("halt")       // Останавливаем процесс...
processCommand("restart")    // Неизвестная команда: "restart"
processCommand("   ")        // Пустая команда
}</code></pre>
</div>

<h2 style="color: #34495e; margin-top: 28px;">3. Область видимости переменных в switch</h2>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">✅ Каждый case &mdash; отдельный блок</h4>

<p style="margin-bottom: 10px;">В Go каждый case создаёт собственный неявный блок области видимости (согласно спецификации языка). Это означает, что переменные, объявленные в одном case, <strong>не видны</strong> в других case и не конфликтуют с ними.</p>

<p style="margin: 0;">Это отличие от C/C++, где все case делят общую область видимости. В Go дополнительные фигурные скобки внутри case не требуются.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

func demonstrateScope(value int) {
switch value {
case 1:
// msg видна только в этом case
msg := "Значение равно единице"
fmt.Println(msg)
case 2:
// Другая msg — в другом блоке, конфликта нет
msg := "Значение равно двум"
fmt.Println(msg)
case 3, 4, 5:
count := value * 2
fmt.Printf("Удвоенное значение: %d\n", count)
default:
// count здесь не виден — он в блоке case 3,4,5
result := "Другое значение"
fmt.Println(result)
}
}

func main() {
demonstrateScope(1)  // Значение равно единице
demonstrateScope(2)  // Значение равно двум
demonstrateScope(4)  // Удвоенное значение: 8
demonstrateScope(10) // Другое значение
}</code></pre>
</div>

<h2 style="color: #34495e; margin-top: 28px;">4. Сравнение с if-else</h2>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🤔 Когда switch читабельнее?</h4>

<p style="margin-bottom: 10px;"><strong>Используйте switch когда:</strong></p>

<ul style="margin: 10px 0;">
	<li>Проверяете одну переменную на множество конкретных значений</li>
	<li>Есть чёткие категории или состояния</li>
	<li>Нужна ветка default для неожиданных случаев</li>
</ul>

<p style="margin-bottom: 10px;"><strong>Используйте if-else когда:</strong></p>

<ul style="margin: 10px 0;">
	<li>Проверяете сложные условия с разными переменными</li>
	<li>Всего 2-3 варианта</li>
	<li>Условия не взаимоисключающие</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">5. Практические примеры</h2>

<h3 style="color: #455a64; margin-top: 24px;">Пример 1: Обработка оценок</h3>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

func getGradeLetter(score int) string {
switch {
case score &gt;= 90:
return "A"
case score &gt;= 80:
return "B"
case score &gt;= 70:
return "C"
case score &gt;= 60:
return "D"
default:
return "F"
}
}

func main() {
scores := []int{95, 85, 72, 55, 100}

    for _, score := range scores {
        grade := getGradeLetter(score)
        fmt.Printf("Балл %d = оценка %s\n", score, grade)
    }
}</code></pre>

<pre style="margin: 5px 0; color: #2e7d32;">
Балл 95 = оценка A
Балл 85 = оценка B
Балл 72 = оценка C
Балл 55 = оценка F
Балл 100 = оценка A</pre>

<h3 style="color: #455a64; margin-top: 24px;">Пример 2: Парсинг опций командной строки</h3>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"fmt"
"strings"
)

func parseOption(opt string) {
// Нормализуем: обрезаем пробелы, приводим к нижнему регистру
o := strings.ToLower(strings.TrimSpace(opt))
// Удаляем все ведущие дефисы: "-", "--", "---" → ""
o = strings.TrimLeft(o, "-")

    switch o {
    case "help", "h":
        fmt.Println("Показать справку")
    case "version", "v":
        fmt.Println("Версия 1.0.0")
    case "verbose":
        fmt.Println("Включён подробный вывод")
    case "quiet", "q":
        fmt.Println("Тихий режим")
    default:
        if strings.HasPrefix(o, "output=") {
            file := strings.TrimPrefix(o, "output=")
            fmt.Printf("Вывод в файл: %s\n", file)
        } else {
            fmt.Printf("Неизвестная опция: %s\n", o)
        }
    }
}

func main() {
options := []string{"--help", "--Version", "-q", "--output=result.txt", "--debug"}

    for _, opt := range options {
        parseOption(opt)
    }
}</code></pre>

<div style="background: #f1f8e9; border: 1px solid #7cb342; border-radius: 8px; padding: 12px; margin: 12px 0;">
<h4 style="color: #558b2f; margin-top: 0;">💡 Подсказка: Визуальная аналогия</h4>

<p style="margin: 10px 0;">Представьте switch как сортировочную машину на почте: посылка (значение) попадает в первый подходящий лоток (case) и дальше не проверяется. Если ни один лоток не подошёл, посылка попадает в корзину &laquo;разное&raquo; (default).</p>

<p style="margin: 0;">В Go посылка останавливается в первом подходящем лотке: после сработавшего case проверка остальных веток не продолжается.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">6. Типичные ошибки</h2>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 1: Ожидание неявного проваливания</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// В Go выведет только "one"
switch 1 {
case 1:
    fmt.Println("one")
    // Нет автоматического перехода к case 2
case 2:
    fmt.Println("two")
}</code></pre>

<p style="margin: 0;"><strong>Решение:</strong> В Go каждый case изолирован. Если нужно проваливание, используйте явное <code>fallthrough</code> (редкий случай).</p>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 2: Отсутствие default</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

// ПЛОХО: неожиданное значение игнорируется молча
func processStatus(status int) {
    switch status {
    case 200:
        fmt.Println("OK")
    case 404:
        fmt.Println("Not Found")
    case 500:
        fmt.Println("Server Error")
    }
}

// ХОРОШО: явная обработка неожиданных значений
func processStatusSafe(status int) {
    switch status {
    case 200:
        fmt.Println("OK")
    case 404:
        fmt.Println("Not Found")
    case 500:
        fmt.Println("Server Error")
    default:
        fmt.Printf("Неизвестный статус: %d\n", status)
    }
}</code></pre>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 3: Попытка использовать переменную из другого case</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ПЛОХО: переменная result не видна за пределами case
switch mode {
case "calc":
    result := compute()
    fmt.Println(result)
case "print":
    fmt.Println(result) // Ошибка: result не определена
}</code></pre>

<p style="margin: 0;"><strong>Решение:</strong> Каждый case &mdash; отдельный блок. Если переменная нужна нескольким веткам, объявите её до switch.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📌 Итоги</h2>

<div style="background: #e8f5e9; border: 1px solid #66bb6a; border-radius: 8px; padding: 14px; margin: 20px 0;">
<ul style="margin: 10px 0;">
	<li>Switch в Go выполняет только первый подходящий case и автоматически выходит</li>
	<li>Можно использовать switch с выражением или без (как улучшенный if-else)</li>
	<li>Несколько значений в case объединяются через запятую</li>
	<li>Default обрабатывает все неучтённые случаи &mdash; используйте для устойчивости</li>
	<li>Короткое объявление в заголовке switch изолирует временные переменные</li>
	<li>Каждый case имеет свой блок видимости &mdash; переменные из одного case не видны в других</li>
</ul>
</div>
