<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">⚙️ switch: идиомы, fallthrough, ошибки и поток</h1>

<div style="background: linear-gradient(to right, #e3f2fd, #bbdefb); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #1565c0; margin-top: 0;">📝 TL;DR</h3>

<p style="margin: 10px 0;">Switch без выражения (<code>switch { ... }</code>) &mdash; мощная идиома Go для проверки диапазонов и сложных условий с приоритетами. В отличие от цепочки if-else, switch визуально группирует связанные проверки и явно показывает их взаимоисключающий характер.</p>

<p style="margin: 10px 0;">Fallthrough в Go требует явного указания и выполняет следующую ветку БЕЗ проверки её условия. Используется редко &mdash; обычно для накопления действий при иерархии прав или состояний.</p>

<p style="margin: 10px 0;">При использовании switch внутри цикла важно понимать: <code>break</code> выходит только из switch (не из цикла), <code>continue</code> переходит к следующей итерации внешнего цикла, а <code>return</code> завершает всю функцию. Switch без выражения отлично подходит для ветвления по форме результата (успех или ошибка, разные виды ошибок).</p>

<h2 style="color: #34495e; margin-top: 28px;">1. Диапазоны и приоритеты</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Switch для диапазонов</h4>

<p style="margin-bottom: 10px;">Switch без выражения идеален для проверки диапазонов значений. Порядок case критически важен: выполняется первая подходящая ветка, поэтому более специфичные условия должны идти раньше общих.</p>

<p style="margin: 0;"><strong>Преимущества перед if-else:</strong> визуальная группировка связанных проверок, явная взаимоисключаемость веток, лучшая читаемость при множестве условий.</p>

<h2 style="color: #34495e; margin-top: 28px;">2. Когда уместен fallthrough</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Правила fallthrough</h4>

<p style="margin-bottom: 10px;"><strong>Ключевые правила fallthrough:</strong></p>

<ul style="margin: 10px 0;">
	<li>Должен быть последней инструкцией в case (не может быть в середине)</li>
	<li>Нельзя использовать в последней ветке switch (включая default, если она последняя)</li>
	<li>Следующая ветка выполняется БЕЗ проверки условия</li>
	<li>Редко используется &mdash; обычно есть лучшие альтернативы</li>
</ul>

<p style="margin: 0;"><strong>Уместное применение:</strong> иерархия прав доступа, накопление действий по уровням, состояния с инкрементальным поведением.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"fmt"
"strings"
)

// permissions демонстрирует корректное использование fallthrough
// для накопления прав доступа
func permissions(role string) []string {
var perms []string

    switch strings.ToLower(role) {
    case "admin":
        perms = append(perms, "delete")

        fallthrough
    case "viewer":

    return perms

<code class="language-go">// ПЛОХО: fallthrough перезаписывает результат
func badFallthrough(x int) string {
    var result string
    switch x {
    case 1:
        result = "one"
        fallthrough // следующий case выполнится без проверки!
    case 2:
        result = "two" // Перезапишет "one" даже для x=1
    }
    return result
}

// ХОРОШО: множественные значения в case
func goodMultiCase(x int) string {
switch x {
case 1, 2:
return "one or two"
case 3:
return "three"
default:
return "other"
}

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Предостережение: Риски fallthrough</h4>

<p style="margin-bottom: 10px;">Fallthrough может привести к неожиданному поведению, так как следующая ветка выполняется БЕЗ проверки условия. Используйте его только когда переход в следующий case действительно нужен.</p>

<p style="margin: 0;"><strong>Альтернативы fallthrough:</strong> множественные значения в case, вызов общей функции, явное дублирование кода (если он короткий).</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">3. Ошибки и switch</h2>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🔧 Новый инструмент: strconv.ParseUint</h4>

<p style="margin-bottom: 10px;">Вы уже знаете <code>strconv.Atoi</code> (строка &rarr; int). Для беззнаковых чисел с заданным размером используется <code>strconv.ParseUint</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ParseUint(строка, основание, размер в битах)
n, err := strconv.ParseUint("42", 10, 8)  // десятичное число, до 8 бит (0-255)
value := uint8(n)                          // безопасное приведение после ParseUint</code></pre>

<p style="margin: 0;">При ошибке <code>ParseUint</code> возвращает значение типа <code>error</code>. На текущем уровне достаточно проверить <code>if err != nil</code> и вернуть свою ошибку или сообщение пользователю. Различение конкретных видов ошибок изучается в следующем курсе.</p>
</div>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Switch для анализа результата</h4>

<p style="margin-bottom: 10px;">Switch без выражения удобен для ветвления по результату операции: успех &mdash; одна ветка, разные виды ошибок &mdash; другие ветки. Различить причину ошибки можно по тексту сообщения.</p>

<p style="margin: 0;"><strong>Паттерн:</strong> сначала проверяйте успех (<code>err == nil</code>), затем известные подсказки, в конце &mdash; общий случай.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">

<code class="language-go">package main

import (
"fmt"
"strconv"
"strings"
)

// parseUint8 парсит строку в uint8 и возвращает текстовое описание проблемы.
// Различаем два класса ошибок по тексту сообщения strconv.
func parseUint8(s string) (uint8, string) {
n, err := strconv.ParseUint(s, 10, 8)
if err != nil {
msg := err.Error()
switch {
case strings.Contains(msg, "out of range"):
return 0, "вне диапазона 0..255"
default:
return 0, "не является числом"
}
}
return uint8(n), ""
}

// processValue показывает switch по форме результата.
func processValue(input string) {
value, problem := parseUint8(input)

    switch {
    case problem == "":
        fmt.Printf("✓ Успешно: %d\n", value)
    case problem == "вне диапазона 0..255":
        fmt.Printf("⚠️ %s\n", problem)
    default:
        fmt.Printf("❌ %s\n", problem)
    }
}

// validateAndParse показывает многоуровневую обработку без обёртывания ошибок.
func validateAndParse(s string) (int, string) {
// Уровень 1: базовая валидация
if s == "" {
return 0, "пустой ввод"
}

    // Уровень 2: парсинг
    n, err := strconv.Atoi(s)
    if err != nil {
        return 0, "не удалось разобрать число"
    }

    // Уровень 3: бизнес-валидация — switch без выражения
    switch {
    case n &lt; 0:
        return 0, fmt.Sprintf("отрицательные числа недопустимы: %d", n)
    case n &gt; 1000:
        return 0, fmt.Sprintf("число больше 1000: %d", n)
    default:
        return n, ""
    }
}

func main() {
// Тестируем parseUint8
fmt.Println("=== Парсинг uint8 ===")
testInputs := []string{"42", "255", "256", "-1", "abc", ""}

    for _, input := range testInputs {
        fmt.Printf("Вход: %-6q → ", input)
        processValue(input)
    }

    // Тестируем validateAndParse
    fmt.Println("\n=== Валидация и парсинг ===")
    validateInputs := []string{"", "100", "-50", "2000", "xyz"}

    for _, input := range validateInputs {
        result, problem := validateAndParse(input)
        if problem != "" {
            fmt.Printf("%-10q → Ошибка: %s\n", input, problem)
        } else {
            fmt.Printf("%-10q → Успех: %d\n", input, result)
        }
    }
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">4. Управление потоком</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: break, continue, return в switch</h4>

<p style="margin-bottom: 10px;"><strong>Важные правила:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>break</code> &mdash; выходит из switch, НЕ из внешнего цикла</li>
	<li><code>continue</code> &mdash; работает только в циклах, переходит к следующей итерации</li>
	<li><code>return</code> &mdash; завершает всю функцию</li>
	<li>Для выхода из цикла изнутри switch используйте метки или флаги</li>
</ul>

<p style="margin: 0;">Switch сам по себе не требует break в конце case &mdash; выход происходит автоматически.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"fmt"
"strings"
)

// processCommands демонстрирует управление потоком
func processCommands(commands []string) {
for i, cmd := range commands {
fmt.Printf("Команда %d: %q → ", i+1, cmd)

        switch strings.ToLower(cmd) {
        case "skip":
            fmt.Println("Пропускаем")
            continue // Переход к следующей итерации цикла

        case "stop":
            fmt.Println("Остановка")
            return // Выход из функции

        case "pause":
            fmt.Println("Пауза (выход из switch)")
            break // Выход только из switch, цикл продолжается

        case "process":
            fmt.Println("Обработка...")
            // Дополнительная логика
            if i &gt; 2 {
                fmt.Println("  (поздняя обработка)")
            }

        default:
            fmt.Printf("Неизвестная команда\n")
        }

        // Этот код выполняется после switch (кроме continue и return)
        fmt.Printf("  [После switch для команды %d]\n", i+1)
    }

    fmt.Println("Функция завершена нормально")
}

func main() {
commands := []string{"process", "skip", "pause", "process", "stop", "ignored"}
processCommands(commands)
}</code></pre>

<p style="margin: 12px 0;"><strong>Разница break и continue внутри switch:</strong></p>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">for i := 1; i &lt;= 5; i++ {

    default:
        fmt.Println("обычная обработка")
    }
    fmt.Println("  [этот код после switch]") // НЕ выполнится для case 2
}</code></pre>
</div>

<p style="margin: 12px 0;"><strong>break с меткой</strong> &mdash; выход из вложенного цикла. Метки для циклов мы изучили в модуле 6 (break/continue). Здесь тот же приём, но внутри switch:</p>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func findFirstWithDigit(items []string) string {
    found := ""
outer: // Метка для внешнего цикла
    for _, item := range items {
        for _, ch := range item {
            switch {
            case ch &gt;= '0' &amp;&amp; ch &lt;= '9':
                found = item
                break outer       // Нашли — выходим из обоих циклов
            case ch == '!':
                continue outer    // Пропускаем элемент целиком
            }
        }
    }
    return found
}</code></pre>

<div style="background: #e8f5e9; border: 1px solid #66bb6a; border-radius: 4px; padding: 10px; margin-top: 10px;"><strong>📋 Результат:</strong> <code>findFirstWithDigit([]string{&quot;abc&quot;, &quot;def!&quot;, &quot;x1y&quot;})</code> вернёт <code>&quot;x1y&quot;</code>.</div>
</div>

<h2 style="color: #34495e; margin-top: 28px;">5. Практические примеры</h2>

<h3 style="color: #455a64; margin-top: 24px;">Пример: Калькулятор с обработкой ошибок через switch</h3>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"fmt"
"strconv"
"strings"
)

func calculate(expr string) (float64, error) {
parts := strings.Fields(expr)
if len(parts) != 3 {
return 0, fmt.Errorf("ожидаемый формат: число операция число")
}

    a, err := strconv.ParseFloat(parts[0], 64)
    if err != nil {
        return 0, fmt.Errorf("первое число %q не распознано", parts[0])
    }

    b, err := strconv.ParseFloat(parts[2], 64)
    if err != nil {
        return 0, fmt.Errorf("второе число %q не распознано", parts[2])
    }

    switch parts[1] {
    case "+":
        return a + b, nil
    case "-":
        return a - b, nil
    case "*":
        return a * b, nil
    case "/":
        if b == 0 {
            return 0, fmt.Errorf("деление на ноль")
        }
        return a / b, nil
    default:
        return 0, fmt.Errorf("неизвестная операция %q", parts[1])
    }
}</code></pre>

<p style="margin: 8px 0;">Использование с разделением успеха и ошибки через простой <code>if</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func main() {
    expressions := []string{"10 + 5", "15 / 3", "10 / 0", "5 ^ 2"}

    for _, expr := range expressions {
        result, err := calculate(expr)
        fmt.Printf("%-15s → ", expr)
        if err != nil {
            fmt.Printf("❌ Ошибка: %v\n", err)
            continue
        }
        fmt.Printf("%.2f\n", result)
    }
}</code></pre>
</div>

<div style="background: #f1f8e9; border: 1px solid #7cb342; border-radius: 8px; padding: 12px; margin: 12px 0;">
<h4 style="color: #558b2f; margin-top: 0;">💡 Подсказка: Ментальная модель</h4>

<p style="margin: 10px 0;">Представьте switch как железнодорожную стрелку: поезд (выполнение) идёт по первому подходящему пути и выходит на главную магистраль после станции. Fallthrough &mdash; это специальный мост на следующий путь, игнорирующий семафоры (условия).</p>

<p style="margin: 0;">При работе с ошибками switch работает как сортировочный центр: каждый тип ошибки направляется в свой обработчик. Порядок важен: от специфичных к общим.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">6. Типичные ошибки</h2>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 1: Использование fallthrough вместо множественных значений</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ПЛОХО: fallthrough для группировки значений
switch x {
case 1:
    fallthrough
case 2:
    fallthrough
case 3:
    return "small"
}

// ХОРОШО: множественные значения в одном case
switch x {
case 1, 2, 3:
    return "small"
}</code></pre>

<p style="margin: 0;"><strong>Решение:</strong> Используйте запятую для перечисления значений в одном case.</p>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 2: Использование переменной из другого case</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ОШИБКА: переменная data не видна за пределами своего case
switch mode {
case "read":
    data := readData()
    process(data)
case "write":
    process(data) // Ошибка: data не определена — она в блоке case "read"
}

// РЕШЕНИЕ: объявите переменную до switch
var data []byte
switch mode {
case "read":
    data = readData() // = вместо :=, т.к. переменная объявлена выше
case "write":
    data = writeData()
}
process(data)</code></pre>

<p style="margin: 0;"><strong>Помните:</strong> каждый case в Go создаёт свой блок видимости. Переменные из одного case не видны в других.</p>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 3: Путаница с break в цикле</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// НЕОЖИДАННОЕ ПОВЕДЕНИЕ: break выходит только из switch
for _, item := range items {
    switch item {
    case "exit":
        break // Выходит из switch, НЕ из цикла!
    }
    // Код продолжает выполняться
}

// РЕШЕНИЕ 1: используйте return или метку
outer:
for _, item := range items {
    switch item {
    case "exit":
        break outer // Выходит из цикла
    }
}

// РЕШЕНИЕ 2: используйте флаг
shouldBreak := false
for _, item := range items {
    switch item {
    case "exit":
        shouldBreak = true
    }
    if shouldBreak {
        break
    }
}</code></pre>

<p style="margin: 0;"><strong>Решение:</strong> Используйте метки или флаги для управления внешними циклами.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📌 Итоги</h2>

<div style="background: #e8f5e9; border: 1px solid #66bb6a; border-radius: 8px; padding: 14px; margin: 20px 0;">
<ul style="margin: 10px 0;">
	<li>Switch без выражения идеален для диапазонов и приоритетных проверок</li>
	<li>Порядок case критичен &mdash; выполняется первая подходящая ветка</li>
	<li>Fallthrough выполняет следующую ветку БЕЗ проверки условия</li>
	<li>Используйте fallthrough только для накопительной логики</li>
	<li>Switch без выражения помогает структурировать ветвление по состоянию (успех или разные виды ошибок) с явной взаимоисключаемостью</li>
	<li>Break выходит только из switch, не из внешнего цикла</li>
	<li>Continue в switch работает с внешним циклом</li>
	<li>Return завершает всю функцию</li>
	<li>Каждый case имеет свой блок видимости &mdash; переменные из одного case не видны в других</li>
	<li>Для множественных значений используйте запятую, не fallthrough</li>
</ul>
</div>
