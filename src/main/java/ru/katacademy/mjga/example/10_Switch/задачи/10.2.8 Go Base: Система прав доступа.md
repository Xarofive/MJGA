<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔐 Система прав доступа</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться создавать системы авторизации с иерархическими правами, использовать оператор fallthrough для накопления прав и работать с нормализацией входных данных.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Системы управления правами доступа &mdash; основа безопасности современных приложений. От файловых систем до веб-сервисов, везде используется принцип иерархических ролей: каждая следующая роль включает права предыдущей плюс дополнительные. Это упрощает администрирование и делает систему предсказуемой.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для определения прав доступа по роли пользователя.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Иерархия прав доступа</h4>

<p>В системах авторизации права часто организованы иерархически:</p>

<ul style="margin: 10px 0;">
	<li><strong>Viewer</strong> &mdash; только чтение</li>
	<li><strong>Editor</strong> &mdash; чтение + запись</li>
	<li><strong>Admin</strong> &mdash; чтение + запись + удаление</li>
</ul>

<p style="margin-bottom: 0;">Каждая роль включает все права предыдущих ролей.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Оператор fallthrough</h4>

<p>В Go оператор <code>fallthrough</code> позволяет продолжить выполнение следующего case:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">switch level {
case 3:
    perms = append(perms, "delete")
    fallthrough
case 2:
    perms = append(perms, "write")
    fallthrough
case 1:
    perms = append(perms, "read")
}</code></pre>

<p style="margin-bottom: 0;">Это удобно для накопления прав от высших ролей к низшим.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔀 Нормализация регистра</h4>

<p>Для устойчивости к опечаткам роли часто приводят к единому регистру:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">role = strings.ToLower(role) // "ADMIN" → "admin"</code></pre>

<p style="margin-bottom: 0;">Это делает систему более дружелюбной к пользователю.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для определения прав доступа по роли:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура пакета</h4>

<ul style="margin: 10px 0;">
	<li>Пакет называется <code>authz</code></li>
	<li>Импортируйте только пакет <code>strings</code> (без псевдонимов)</li>
	<li>Создайте только одну функцию <code>Permissions</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция Permissions</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func Permissions(role string) []string</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает название роли (регистронезависимо)</li>
	<li>Возвращает срез строк с правами доступа</li>
	<li>Используйте <code>strings.ToLower</code> для нормализации роли</li>
	<li>Используйте <code>switch</code> с <code>fallthrough</code> для накопления прав</li>
	<li>НЕ используйте map для хранения прав</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📊 Роли и их права</h4>

<ul style="margin: 10px 0;">
	<li><code>&quot;viewer&quot;</code> &rarr; <code>[&quot;read&quot;]</code></li>
	<li><code>&quot;editor&quot;</code> &rarr; <code>[&quot;write&quot;, &quot;read&quot;]</code></li>
	<li><code>&quot;admin&quot;</code> &rarr; <code>[&quot;delete&quot;, &quot;write&quot;, &quot;read&quot;]</code></li>
	<li>Любая другая роль &rarr; <code>[]</code> (пустой срез)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Важные детали</h4>

<ul style="margin: 10px 0;">
	<li>Порядок прав в срезе важен (соответствует иерархии)</li>
	<li>Роли &quot;VIEWER&quot;, &quot;Viewer&quot;, &quot;viewer&quot; эквивалентны</li>
	<li>Каждый вызов должен возвращать новый независимый срез</li>
	<li>Используйте <code>fallthrough</code> для реализации иерархии</li>
	<li>Сначала приведите роль к нижнему регистру, потом используйте switch</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Использование map</strong><br />
❌ <code>perms := map[string][]string{&quot;admin&quot;: ...}</code><br />
✅ Используйте switch с fallthrough</p>

<p><strong>Ошибка 2: Забыть про нормализацию</strong><br />
❌ Сразу использовать switch по role<br />
✅ Сначала <code>role = strings.ToLower(role)</code>, потом switch</p>

<p><strong>Ошибка 3: Неправильный порядок прав</strong><br />
❌ <code>[&quot;read&quot;, &quot;write&quot;, &quot;delete&quot;]</code> для admin (не соответствует естественному порядку fallthrough)<br />
✅ <code>[&quot;delete&quot;, &quot;write&quot;, &quot;read&quot;]</code> &mdash; от уникальных прав роли к базовым через fallthrough</p>

<p><strong>Ошибка 4: Возврат общего среза</strong><br />
❌ Возвращать один и тот же срез для одинаковых ролей<br />
✅ Каждый вызов создаёт новый срез</p>

<p><strong>Ошибка 5: Забыть fallthrough</strong><br />
❌ В каждом case дублировать все права<br />
✅ Использовать fallthrough для накопления</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Возврат nil вместо пустого среза</strong><br />
❌ <code>return nil</code> для неизвестной роли<br />
✅ <code>return []string{}</code> &mdash; пустой срез</p>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка по архитектуре</h4>

<p>Представьте офисное здание с системой пропусков. У уборщика есть ключ только от подсобки (read). У сотрудника есть ключи от подсобки и офиса (read, write). У директора есть все ключи: от подсобки, офиса и сейфа (read, write, delete). Вместо выдачи отдельных ключей каждой роли, можно использовать принцип наследования: директор получает все ключи сотрудника, плюс ключ от сейфа. Это и реализует fallthrough &mdash; &quot;проваливание&quot; через уровни доступа.</p>
