<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📊 Система оценивания</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться работать с диапазонами значений, использовать конструкцию switch без условия для проверки составных условий и создавать системы классификации числовых данных.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Системы оценивания используются повсеместно: в образовании для конвертации баллов в оценки, в играх для определения рангов, в бизнесе для категоризации клиентов. Умение корректно работать с диапазонами и граничными значениями &mdash; важный навык для создания справедливых и предсказуемых систем классификации.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для конвертации числового балла в буквенную оценку по американской системе.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Switch без условия для диапазонов</h4>

<p>Конструкция <code>switch</code> без указания переменной позволяет проверять сложные условия, включая диапазоны:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">switch {
case x &gt;= 10 &amp;&amp; x &lt;= 20:
    return "диапазон 10-20"
case x &gt;= 21 &amp;&amp; x &lt;= 30:
    return "диапазон 21-30"
default:
    return "вне диапазонов"
}</code></pre>

<p style="margin-bottom: 0;">Проверки выполняются последовательно сверху вниз до первого совпадения.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Американская система оценок</h4>

<p>В американской образовательной системе используется буквенная система оценок:</p>

<ul style="margin: 10px 0;">
	<li><strong>A</strong> &mdash; отлично (excellent)</li>
	<li><strong>B</strong> &mdash; хорошо (good)</li>
	<li><strong>C</strong> &mdash; удовлетворительно (average)</li>
	<li><strong>D</strong> &mdash; ниже среднего (below average)</li>
	<li><strong>F</strong> &mdash; неудовлетворительно (fail)</li>
</ul>

<p style="margin-bottom: 0;">Оценка E традиционно не используется, чтобы избежать путаницы с &quot;Excellent&quot;.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔢 Проверка диапазонов</h4>

<p>При работе с диапазонами важно учитывать граничные значения:</p>

<ul style="margin: 10px 0;">
	<li>Используйте <code>&gt;=</code> и <code>&lt;=</code> для включения границ в диапазон</li>
	<li>Объединяйте условия через <code>&amp;&amp;</code> (логическое И)</li>
	<li>Проверяйте диапазоны в правильном порядке</li>
	<li>Не забывайте про значения вне всех диапазонов</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для конвертации баллов в оценки:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура пакета</h4>

<ul style="margin: 10px 0;">
	<li>Пакет называется <code>grade</code></li>
	<li>НЕ требуется импортировать какие-либо пакеты</li>
	<li>Содержит только одну функцию <code>Grade</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция Grade</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func Grade(score int) string</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает параметр типа <code>int</code></li>
	<li>Возвращает строку с буквенной оценкой</li>
	<li>Используйте ровно один оператор <code>switch</code> без условия: <code>switch { ... }</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📊 Шкала оценивания</h4>

<p>Конвертируйте баллы в оценки по следующей шкале (включая границы):</p>

<ul style="margin: 10px 0;">
	<li><strong>90&ndash;100 баллов</strong> &rarr; возвращать <code>&quot;A&quot;</code></li>
	<li><strong>80&ndash;89 баллов</strong> &rarr; возвращать <code>&quot;B&quot;</code></li>
	<li><strong>70&ndash;79 баллов</strong> &rarr; возвращать <code>&quot;C&quot;</code></li>
	<li><strong>60&ndash;69 баллов</strong> &rarr; возвращать <code>&quot;D&quot;</code></li>
	<li><strong>0&ndash;59 баллов</strong> &rarr; возвращать <code>&quot;F&quot;</code></li>
	<li><strong>Все остальные значения</strong> &rarr; возвращать <code>&quot;invalid&quot;</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Важные детали</h4>

<ul style="margin: 10px 0;">
	<li>Используйте <code>switch</code> без условия: <code>switch { ... }</code></li>
	<li>Каждый <code>case</code> должен корректно определять свой диапазон &mdash; можно использовать двусторонние условия (<code>score &gt;= 90 &amp;&amp; score &lt;= 100</code>) или каскадные (<code>score &gt;= 90</code>)</li>
	<li>Рекомендуется располагать проверки от высших оценок к низшим: A &rarr; B &rarr; C &rarr; D &rarr; F</li>
	<li>Значения вне диапазона 0&ndash;100 должны возвращать <code>&quot;invalid&quot;</code></li>
	<li>Границы включаются в диапазон (90 и 100 дают &quot;A&quot;, 80 и 89 дают &quot;B&quot;)</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Switch с условием</strong><br />
❌ <code>switch score { ... }</code><br />
✅ <code>switch { ... }</code> &mdash; без переменной после switch</p>

<p><strong>Ошибка 2: Пропущенные значения</strong><br />
❌ Не все значения от 0 до 100 попадают в оценку<br />
✅ Используйте полные диапазоны (<code>score &gt;= 90 &amp;&amp; score &lt;= 100</code>) или каскадные условия</p>

<p><strong>Ошибка 3: Неправильный порядок при каскадном подходе</strong><br />
❌ При каскадном подходе (<code>score &gt;= X</code>) порядок критичен &mdash; более специфичные условия должны идти первыми<br />
✅ Рекомендуется: A &rarr; B &rarr; C &rarr; D &rarr; F &rarr; default</p>

<p><strong>Ошибка 4: Пропущенные границы</strong><br />
❌ Диапазоны не покрывают все значения от 0 до 100<br />
✅ Каждое значение от 0 до 100 должно попасть в какую-то оценку</p>

<p><strong>Ошибка 5: Лишний код</strong><br />
❌ Дополнительные переменные, if-else, несколько операторов<br />
✅ Только один switch с прямыми return в каждой ветви</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Забытый default</strong><br />
❌ Только 5 case без default<br />
✅ Обязательно добавьте default для &quot;invalid&quot;</p>
</div>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка по логике</h4>

<p>Представьте, что вы проверяете попадание числа в коридоры. Каждый коридор имеет две стены (нижнюю и верхнюю границу). Число должно быть не меньше нижней стены И не больше верхней стены, чтобы попасть в коридор. Порядок проверки коридоров важен &mdash; идите от самого престижного (A) к наименее престижному (F), а всё за пределами коридоров &mdash; это &quot;invalid&quot;.</p>
