<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📅 Календарь: месяцы по-русски с обработкой ошибок</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться создавать надёжные функции с корректной обработкой ошибок, используя предопределённые переменные ошибок и конструкцию switch для выбора значений.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В реальных приложениях часто требуется конвертировать числовые коды в текстовые представления: коды стран в названия, номера месяцев в их имена, статус-коды в описания. Грамотная обработка некорректных входных данных через систему ошибок &mdash; ключевой навык для создания надёжного ПО.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для получения русского названия месяца по его номеру с корректной обработкой неверных значений.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Обработка ошибок в Go</h4>

<p>В Go для сигнализации об ошибках функции возвращают значение типа <code>error</code> последним параметром:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func doSomething() (string, error) {
    if somethingWrong {
        return "", errors.New("что-то пошло не так")
    }
    return "результат", nil
}</code></pre>

<p style="margin-bottom: 0;">При успешном выполнении возвращается <code>nil</code> вместо ошибки.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Предопределённые ошибки</h4>

<p>Для часто используемых ошибок создают глобальные переменные:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">var ErrNotFound = errors.New("элемент не найден")
var ErrInvalidInput = errors.New("некорректный ввод")</code></pre>

<p style="margin-bottom: 0;">Это позволяет переиспользовать одну и ту же ошибку: вызывающий код может сравнить полученное значение с переменной через <code>==</code>, а не создавать новую ошибку каждый раз.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔀 Конструкция switch</h4>

<p>Оператор <code>switch</code> позволяет выбирать действие на основе значения:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">switch value {
case 1:
    return "один"
case 2:
    return "два"
default:
    return "неизвестно"
}</code></pre>

<p style="margin-bottom: 0;">Это более читаемая альтернатива множественным if-else.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте пакет для работы с месяцами на русском языке:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура пакета</h4>

<ul style="margin: 10px 0;">
	<li>Пакет называется <code>month</code></li>
	<li>Импортирует только пакет <code>errors</code> (без псевдонимов)</li>
	<li>Содержит ровно одну функцию (<code>MonthRU</code>) и одну переменную ошибки (<code>ErrMonthOutOfRange</code>) &mdash; без вспомогательных приватных функций или дополнительных переменных</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Переменная ошибки</h4>

<p><strong>Объявление:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>var ErrMonthOutOfRange = errors.New(&quot;month out of range&quot;)</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Имя переменной точно <code>ErrMonthOutOfRange</code></li>
	<li>Текст ошибки точно <code>&quot;month out of range&quot;</code> (на английском)</li>
	<li>Создаётся через <code>errors.New()</code></li>
	<li>Объявляется на уровне пакета (вне функций)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Функция MonthRU</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func MonthRU(n int) (string, error)</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает номер месяца (1-12)</li>
	<li>Возвращает название месяца на русском языке в нижнем регистре и <code>nil</code> при корректном вводе</li>
	<li>Возвращает пустую строку и <code>ErrMonthOutOfRange</code> при некорректном номере</li>
	<li>Использует конструкцию <code>switch</code> по параметру для выбора месяца</li>
	<li>НЕ использует массивы, срезы или map для хранения названий</li>
	<li>Каждое число от 1 до 12 обрабатывается отдельной веткой case</li>
</ul>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Использование массива или среза</strong><br />
❌ <code>months := []string{&quot;январь&quot;, &quot;февраль&quot;, ...}</code><br />
✅ Используйте switch с отдельными case для каждого месяца</p>

<p><strong>Ошибка 2: Создание новой ошибки каждый раз</strong><br />
❌ <code>return &quot;&quot;, errors.New(&quot;month out of range&quot;)</code><br />
✅ Возвращайте предопределённую переменную <code>ErrMonthOutOfRange</code></p>

<p><strong>Ошибка 3: Неправильный регистр</strong><br />
❌ <code>return &quot;Январь&quot;, nil</code> или <code>return &quot;ЯНВАРЬ&quot;, nil</code><br />
✅ Все названия в нижнем регистре: <code>return &quot;январь&quot;, nil</code></p>

<p><strong>Ошибка 4: Забытые граничные случаи</strong><br />
❌ Обработка только положительных чисел<br />
✅ Проверьте: 0, отрицательные числа, числа больше 12</p>

<p><strong>Ошибка 5: Объединение нескольких case</strong><br />
❌ <code>case 1, 2, 3:</code> с условной логикой внутри<br />
✅ Каждый месяц в отдельном case</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Лишние функции или методы</strong><br />
❌ Дополнительные вспомогательные функции<br />
✅ Только одна функция <code>MonthRU</code></p>
