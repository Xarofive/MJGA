<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🌐 Классификация HTTP-кодов</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться работать с целочисленным делением, определять принадлежность числа к диапазону и классифицировать трёхзначные коды по их старшей цифре.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">HTTP-коды состояния &mdash; основа веб-разработки. Они сообщают браузеру и разработчику о результате запроса: успешно ли он выполнен, нужно ли перенаправление, произошла ли ошибка. Группировка кодов по классам помогает быстро понять общий характер ответа сервера и упрощает обработку ошибок в приложениях.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для определения класса HTTP-кода по его числовому значению.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Классы HTTP-кодов</h4>

<p>HTTP-коды состояния &mdash; это трёхзначные числа, где первая цифра определяет класс ответа:</p>

<ul style="margin: 10px 0;">
	<li><strong>1xx</strong> &mdash; информационные (100-199)</li>
	<li><strong>2xx</strong> &mdash; успешные (200-299)</li>
	<li><strong>3xx</strong> &mdash; перенаправления (300-399)</li>
	<li><strong>4xx</strong> &mdash; ошибки клиента (400-499)</li>
	<li><strong>5xx</strong> &mdash; ошибки сервера (500-599)</li>
</ul>

<p style="margin-bottom: 0;">Числа вне диапазона 100-599 не являются валидными HTTP-кодами.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Целочисленное деление</h4>

<p>В Go оператор <code>/</code> для целых чисел выполняет целочисленное деление:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">404 / 100 = 4    // результат — старшая цифра
250 / 100 = 2
199 / 100 = 1
99 / 100 = 0     // числа меньше 100 дают 0</code></pre>

<p style="margin-bottom: 0;">Это позволяет легко извлечь первую цифру трёхзначного числа.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔀 Примеры известных HTTP-кодов</h4>

<ul style="margin: 10px 0;">
	<li><strong>200 OK</strong> &mdash; запрос успешно выполнен</li>
	<li><strong>301 Moved Permanently</strong> &mdash; ресурс перемещён навсегда</li>
	<li><strong>404 Not Found</strong> &mdash; ресурс не найден</li>
	<li><strong>500 Internal Server Error</strong> &mdash; внутренняя ошибка сервера</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для определения класса HTTP-кода:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура пакета</h4>

<ul style="margin: 10px 0;">
	<li>Пакет называется <code>httpclass</code></li>
	<li>НЕ требуется импортировать какие-либо пакеты</li>
	<li>Содержит только одну функцию <code>HTTPClass</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция HTTPClass</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func HTTPClass(code int) string</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает параметр <code>code</code> типа <code>int</code> &mdash; числовой HTTP-код</li>
	<li>Возвращает строку с классом кода или <code>&quot;unknown&quot;</code></li>
	<li>Используйте оператор <code>switch</code> для классификации кода (тема модуля M10)</li>
	<li>НЕ используйте дополнительные функции или глобальные переменные</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📊 Правила классификации</h4>

<p>Функция должна возвращать:</p>

<ul style="margin: 10px 0;">
	<li><code>&quot;1xx&quot;</code> для кодов от 100 до 199 включительно</li>
	<li><code>&quot;2xx&quot;</code> для кодов от 200 до 299 включительно</li>
	<li><code>&quot;3xx&quot;</code> для кодов от 300 до 399 включительно</li>
	<li><code>&quot;4xx&quot;</code> для кодов от 400 до 499 включительно</li>
	<li><code>&quot;5xx&quot;</code> для кодов от 500 до 599 включительно</li>
	<li><code>&quot;unknown&quot;</code> для всех остальных значений</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Важные детали</h4>

<ul style="margin: 10px 0;">
	<li>Коды меньше 100 или больше 599 &mdash; возвращать <code>&quot;unknown&quot;</code></li>
	<li>Отрицательные числа &mdash; возвращать <code>&quot;unknown&quot;</code></li>
	<li>Ноль &mdash; возвращать <code>&quot;unknown&quot;</code></li>
	<li>Формат ответа: цифра + &quot;xx&quot; в нижнем регистре (не &quot;XX&quot;)</li>
	<li>Границы диапазонов включаются (100 даёт &quot;1xx&quot;, 199 даёт &quot;1xx&quot;)</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неправильные границы</strong><br />
❌ Забыть проверку на диапазон 100-599<br />
✅ Сначала проверить, что код в диапазоне 100-599</p>

<p><strong>Ошибка 2: Неправильный формат</strong><br />
❌ Возвращать &quot;1XX&quot; или &quot;1хх&quot;<br />
✅ Строго &quot;1xx&quot; с маленькими буквами x</p>

<p><strong>Ошибка 3: Усложнение логики</strong><br />
❌ Проверять каждый код индивидуально<br />
✅ Использовать целочисленное деление для определения класса</p>

<p><strong>Ошибка 4: Забытые граничные случаи</strong><br />
❌ Не обработать 100, 199, 200, 599<br />
✅ Проверить, что границы диапазонов включаются</p>

<p><strong>Ошибка 5: Неучтённые отрицательные числа</strong><br />
❌ Обрабатывать -404 как 404<br />
✅ Отрицательные числа всегда дают &quot;unknown&quot;</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Лишние импорты</strong><br />
❌ Импортировать fmt, strconv или другие пакеты<br />
✅ Решение не требует никаких импортов</p>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка по реализации</h4>

<p>Подумайте о HTTP-коде как о трёхзначном номере квартиры в доме. Первая цифра &mdash; это этаж. Чтобы узнать этаж, достаточно разделить номер квартиры на 100. Квартира 404 находится на 4-м этаже, квартира 200 &mdash; на 2-м этаже. Но помните: в этом &quot;доме&quot; только этажи с 1 по 5, и квартиры начинаются со 100.</p>
