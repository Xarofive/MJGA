<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔤 Классификация символов Unicode</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться работать с символами Unicode, использовать стандартный пакет unicode для классификации символов и применять конструкцию switch без условия для множественных проверок.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В современных приложениях обработка текста часто требует классификации символов: валидация паролей, парсинг токенов, очистка данных, анализ текста. Стандартная библиотека Go предоставляет мощные инструменты для работы с Unicode, позволяющие корректно обрабатывать символы всех языков мира.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для определения категории символа: буква, цифра, пробельный символ или другое.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Тип rune и Unicode</h4>

<p>В Go тип <code>rune</code> представляет Unicode-символ (кодовую точку). Это псевдоним для <code>int32</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">var r rune = 'А'  // кириллическая А
var e rune = '😀'  // эмодзи тоже rune
var n rune = '7'   // цифра как символ</code></pre>

<p style="margin-bottom: 0;">Тип rune позволяет работать с любыми символами Unicode, а не только ASCII.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Пакет unicode</h4>

<p>Стандартный пакет <code>unicode</code> предоставляет функции для классификации символов:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">unicode.IsLetter('A')  // true - латинская буква
unicode.IsLetter('Я')  // true - кириллическая буква
unicode.IsDigit('5')   // true - цифра
unicode.IsSpace(' ')   // true - пробел
unicode.IsSpace('\t')  // true - табуляция тоже пробельный символ</code></pre>

<p style="margin-bottom: 0;">Эти функции корректно работают со всеми символами Unicode.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔀 Switch без условия</h4>

<p>Конструкция <code>switch</code> без указания переменной позволяет проверять различные условия последовательно:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">switch {
case x &gt; 0:
    return "положительное"
case x &lt; 0:
    return "отрицательное"
default:
    return "ноль"
}</code></pre>

<p style="margin-bottom: 0;">Выполняется первая ветвь, условие которой истинно. Если ни одно условие не подходит, выполняется <code>default</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте пакет для классификации Unicode-символов:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура пакета</h4>

<ul style="margin: 10px 0;">
	<li>Пакет называется <code>classify</code></li>
	<li>Импортирует только пакет <code>unicode</code> (без псевдонимов)</li>
	<li>Содержит ровно одну функцию <code>ClassifyRune</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция ClassifyRune</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func ClassifyRune(r rune) string</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает один символ типа <code>rune</code> с именем параметра <code>r</code></li>
	<li>Возвращает строку с категорией символа</li>
	<li>Тело функции содержит только один оператор <code>switch</code> без условия</li>
	<li>Использует функции пакета <code>unicode</code> для проверки категории</li>
	<li>НЕ используйте дополнительные переменные или инструкции</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📊 Категории и порядок проверок</h4>

<p>Функция должна проверять категории строго в следующем порядке:</p>

<ol style="margin: 10px 0;">
	<li><strong>Буква</strong>: проверка <code>unicode.IsLetter(r)</code> &rarr; возврат <code>&quot;letter&quot;</code></li>
	<li><strong>Цифра</strong>: проверка <code>unicode.IsDigit(r)</code> &rarr; возврат <code>&quot;digit&quot;</code></li>
	<li><strong>Пробельный символ</strong>: проверка <code>unicode.IsSpace(r)</code> &rarr; возврат <code>&quot;space&quot;</code></li>
	<li><strong>Всё остальное</strong>: ветвь <code>default</code> &rarr; возврат <code>&quot;other&quot;</code></li>
</ol>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Структура switch</h4>

<ul style="margin: 10px 0;">
	<li>Используйте конструкцию <code>switch { ... }</code> без указания переменной после switch</li>
	<li>Каждый <code>case</code> содержит одну проверку через функцию <code>unicode.*</code></li>
	<li>Каждый <code>case</code> и <code>default</code> должны сразу возвращать результат через <code>return</code></li>
	<li>Все проверки выполняются для параметра <code>r</code></li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Switch с условием</strong><br />
❌ <code>switch r { ... }</code><br />
✅ <code>switch { ... }</code> &mdash; без указания переменной</p>

<p><strong>Ошибка 2: Неправильный порядок проверок</strong><br />
❌ Сначала digit, потом letter<br />
✅ Строгий порядок: letter &rarr; digit &rarr; space &rarr; other</p>

<p><strong>Ошибка 3: Лишние переменные или код</strong><br />
❌ <code>result := &quot;&quot;; switch { ... }; return result</code><br />
✅ Только switch с прямыми return в каждой ветви</p>

<p><strong>Ошибка 4: Проверка не того параметра</strong><br />
❌ <code>case unicode.IsLetter(&#39;A&#39;):</code><br />
✅ <code>case unicode.IsLetter(r):</code> &mdash; проверяем параметр r</p>

<p><strong>Ошибка 5: Множественные условия в case</strong><br />
❌ <code>case unicode.IsLetter(r) || unicode.IsDigit(r):</code><br />
✅ Каждый case содержит ровно одну проверку</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Забытый default</strong><br />
❌ Только три case без default<br />
✅ Обязательно добавьте default для категории &quot;other&quot;</p>
</div>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">✅ Чек-лист самопроверки</h4>

<ul style="margin-bottom: 0;">
	<li>Пакет называется <code>classify</code>?</li>
	<li>Импортирован только пакет <code>unicode</code>?</li>
	<li>Есть только одна функция <code>ClassifyRune</code>?</li>
	<li>Сигнатура точно <code>func ClassifyRune(r rune) string</code>?</li>
	<li>В теле функции только один оператор <code>switch</code>?</li>
	<li>Switch без условия: <code>switch { ... }</code>?</li>
	<li>Ровно 4 ветви: 3 case и 1 default?</li>
	<li>Порядок проверок: letter &rarr; digit &rarr; space &rarr; other?</li>
	<li>Каждый case использует функцию <code>unicode.*</code>?</li>
	<li>Каждая ветвь сразу возвращает строковый литерал?</li>
</ul>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка по архитектуре</h4>

<p>Представьте, что вы пишете лексический анализатор для языка программирования. Первый шаг &mdash; классификация каждого символа входного потока.
