<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🌍 Извлечение подстроки с учётом Unicode</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться корректно работать с многобайтовыми символами Unicode при извлечении подстрок, понимая разницу между байтами и рунами в Go.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В современных приложениях текст содержит символы из разных языков: эмодзи 😊, китайские иероглифы 你好, арабскую вязь العربية. Каждый такой символ может занимать от 1 до 4 байт в UTF-8. Неправильная работа с индексами приведёт к &quot;битым&quot; символам, крашам или некорректному отображению. Навык работы с рунами критичен для интернационализации, обработки пользовательского ввода и парсинга данных.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию извлечения подстроки, корректно обрабатывающую символы любых языков.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Байты vs Руны в Go</h4>

<p>В Go строки хранятся в кодировке UTF-8:</p>

<ul style="margin: 10px 0;">
	<li><strong>Байт (byte):</strong> единица хранения, 8 бит. ASCII-символ = 1 байт</li>
	<li><strong>Руна (rune):</strong> Unicode code point, логический символ. Тип <code>rune</code> = <code>int32</code></li>
	<li><strong>Проблема:</strong> <code>s[i]</code> возвращает i-й байт, не i-й символ!</li>
</ul>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">s := "Hello, 世界"
fmt.Println(len(s))        // 13 байт (не 9 символов!)
fmt.Println(s[7])          // 228 (первый байт иероглифа 世)
fmt.Println(string(s[7]))  // � (битый символ!)

// Правильная работа с рунами:
runes := []rune(s)
fmt.Println(len(runes))    // 9 символов
fmt.Println(string(runes[7])) // 世 (корректный символ)</code></pre>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Способы работы с рунами</h4>

<p>Три основных подхода для посимвольной обработки:</p>

<ol style="margin: 10px 0;">
	<li><strong>Конверсия в слайс рун:</strong> <code>[]rune(строка)</code> &mdash; простейший способ</li>
	<li><strong>Итерация через range:</strong> <code>for i, r := range строка</code> &mdash; память-эффективно</li>
	<li><strong>Пакет utf8:</strong> <code>utf8.DecodeRuneInString()</code> &mdash; для точечной работы</li>
</ol>

<p style="margin: 0;">Выбор зависит от задачи: для извлечения подстроки обычно удобнее слайс рун.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию извлечения подстроки с учётом Unicode:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция SubstrRunes</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func SubstrRunes(s string, start, length int) string</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>s</code> &mdash; исходная строка (может содержать любые Unicode-символы)</li>
	<li><code>start</code> &mdash; позиция начала подстроки (в символах, не байтах!)</li>
	<li><code>length</code> &mdash; количество символов для извлечения</li>
</ul>

<p><strong>Предусловия (гарантии):</strong></p>

<ul style="margin: 10px 0;">
	<li><code>start &ge; 0</code> и <code>start &le; длина_строки_в_рунах</code></li>
	<li><code>length &ge; 0</code></li>
	<li>Параметры всегда валидны &mdash; не нужна защитная проверка границ</li>
</ul>

<p><strong>Постусловие:</strong></p>

<ul style="margin: 10px 0;">
	<li>Возвращает подстроку из <code>length</code> символов, начиная с позиции <code>start</code></li>
	<li>Если <code>start + length &gt; длина_строки</code>, вернуть символы до конца строки</li>
	<li>Все многобайтовые символы должны остаться целыми</li>
</ul>

<p><strong>Ограничения:</strong></p>

<ul style="margin: 10px 0;">
	<li>Использовать только стандартную библиотеку Go</li>
	<li>Решение целиком в одной функции <code>SubstrRunes</code> &mdash; без вспомогательных функций и глобальных объявлений</li>
	<li>Параметры гарантированно корректны &mdash; функция должна возвращать результат через <code>return</code> без аварийной остановки программы</li>
	<li>Обязательна работа через руны, не через байтовые индексы</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Прямая индексация строки</strong><br />
❌ <code>return s[start:start+length]</code> &mdash; работает с байтами, разрушит Unicode!<br />
✅ Конвертируйте в руны или используйте range-итерацию</p>

<p><strong>Ошибка 2: Работа через []byte</strong><br />
❌ <code>bytes := []byte(s)</code> &mdash; всё ещё байты, не символы<br />
✅ Используйте <code>[]rune(s)</code> для работы с символами</p>

<p><strong>Ошибка 3: Игнорирование параметров</strong><br />
❌ Возврат фиксированной подстроки без учёта <code>start</code> или <code>length</code><br />
✅ Оба параметра должны влиять на результат</p>

<p><strong>Ошибка 4: Аварийная остановка программы при некорректных параметрах</strong><br />
❌ Прерывать выполнение программы для невалидных значений <code>start</code> или <code>length</code><br />
✅ Параметры гарантированно валидны, защита не нужна &mdash; возвращайте результат через <code>return</code></p>

<p><strong>Ошибка 5: Сложность выше O(n)</strong><br />
❌ Множественные проходы по строке, вложенные циклы<br />
✅ Достаточно одного прохода или конверсии в руны</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Модификация исходной строки</strong><br />
❌ Присваивание <code>s = s[start:]</code> внутри функции<br />
✅ Работайте с копиями или слайсами, не меняйте параметр</p>
</div>
