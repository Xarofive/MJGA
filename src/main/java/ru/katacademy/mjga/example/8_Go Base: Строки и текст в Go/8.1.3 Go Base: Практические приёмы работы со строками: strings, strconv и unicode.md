<h2 style="color: #34495e; margin-top: 30px;">1. Нормализация пробелов: Fields и Join</h2>

<p><strong>Теоретическая основа:</strong> В реальных приложениях текст часто приходит из внешних источников &mdash; пользовательский ввод, файлы, API. Этот текст может содержать непредсказуемое количество пробелов, табуляций, переносов строк. Нормализация &mdash; это процесс приведения текста к единообразному виду.</p>

<p><strong>Почему это важно:</strong> Представьте, что вы сравниваете два email-адреса, введённых пользователями. Один напечатал &quot;john@example.com&quot;, а другой случайно добавил пробелы: &quot; john@example.com &quot;. Без нормализации система посчитает их разными адресами, хотя по сути они идентичны.</p>

<p><strong>Как работает механизм:</strong></p>

<ul>
	<li><code>strings.Fields(s)</code> &mdash; разбивает строку на слова, используя любые пробельные символы как разделители (пробелы, табы, переносы строк). Важно: пустые элементы автоматически пропускаются!</li>
	<li><code>strings.Join(slice, &quot; &quot;)</code> &mdash; собирает слова обратно, вставляя между ними ровно один пробел</li>
	<li>Комбинация этих функций за один проход решает задачу нормализации</li>
</ul>

<p><strong>Альтернативные подходы и их проблемы:</strong></p>

<ul>
	<li>Продвинутые инструменты (поиск по шаблону) &mdash; мощно, но избыточно для простой задачи</li>
	<li>Ручной проход по символам &mdash; много кода, легко ошибиться с граничными случаями</li>
	<li>Множественные Replace &mdash; не покрывает все случаи (табы, переносы, множественные пробелы)</li>
</ul>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 15px 0;">
<p style="margin: 0;"><strong>⚠️ Важный нюанс:</strong> <code>strings.Fields</code> склеивает ВСЕ пробельные символы (включая переводы строк) между словами в один пробел. Многострочный текст превратится в одну строку! Если нужно сохранить переносы, см. упражнение в конце раздела.</p>
</div>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0; overflow-x: auto;">
<code class="language-go">package main

import (
    "fmt"
    "strings"
)

func main() {
    // Пример: пользователь ввёл адрес с лишними пробелами
    messy := "  ул. Пушкина   \t  дом  10  "

    // Fields разбивает по пробельным символам
    // Join объединяет обратно с единичными пробелами
    clean := strings.Join(strings.Fields(messy), " ")

    fmt.Printf("До:    %q\n", messy)
    fmt.Printf("После: %q\n\n", clean)

    // TrimSpace убирает только с краёв
    trimmed := strings.TrimSpace(messy)
    fmt.Printf("TrimSpace:  %q (внутри остались)\n", trimmed)
    fmt.Printf("Fields+Join: %q (всё очищено)\n", clean)
}</code></pre>

<h2 style="color: #34495e; margin-top: 30px;">2. Сравнение без учёта регистра</h2>

<p><strong>Теоретическая основа:</strong> Сравнение строк без учёта регистра &mdash; частая задача в программировании. На первый взгляд кажется простой: приведи обе строки к нижнему регистру и сравни. Но в Unicode всё сложнее из-за особенностей разных языков.</p>

<p><strong>Проблема с наивным подходом:</strong> В некоторых языках преобразование регистра работает не так, как мы ожидаем:</p>

<ul>
	<li><strong>Турецкий язык:</strong> заглавная İ (U+0130, 2 байта) при <code>ToLower</code> в Go становится обычной i (1 байт) &mdash; длина строки меняется</li>
	<li><strong>Немецкий язык:</strong> &szlig; (эсцет) не имеет простого uppercase-аналога в Go &mdash; <code>strings.ToUpper(&quot;&szlig;&quot;)</code> возвращает &quot;&szlig;&quot; без изменений. Полное преобразование &szlig; &rarr; SS реализовано только в специализированных библиотеках (golang.org/x/text)</li>
	<li><strong>Греческий язык:</strong> имеет особые правила для сигмы в конце слова</li>
</ul>

<p><strong>Решение &mdash; Case Folding:</strong> <code>strings.EqualFold</code> использует алгоритм &quot;case folding&quot; из стандарта Unicode. Это специальная форма нормализации, разработанная именно для сравнения без учёта регистра. Она учитывает все особенности языков и работает корректно для любого текста.</p>

<p><strong>Когда что использовать:</strong></p>

<ul>
	<li><code>EqualFold</code> &mdash; для сравнения двух строк целиком (email, логины, теги)</li>
	<li><code>ToLower</code> + <code>Contains</code> &mdash; для поиска подстроки (но помните об изменении размера!)</li>
	<li><code>ToLower</code> для индексации &mdash; когда нужен нормализованный ключ для поиска</li>
</ul>

<p><strong>Производительность:</strong> <code>EqualFold</code> работает быстрее, чем две операции <code>ToLower</code> + сравнение, так как не создаёт промежуточные строки, а сравнивает &quot;на лету&quot;.</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0; overflow-x: auto;">
<code class="language-go">package main

import (
    "fmt"
    "strings"
)

func main() {
    // EqualFold vs ToLower
    s1, s2 := "Go", "go"

    fmt.Printf("%q == %q?\n", s1, s2)
    fmt.Printf("EqualFold: %v\n", strings.EqualFold(s1, s2))
    fmt.Printf("ToLower:   %v\n\n",
        strings.ToLower(s1) == strings.ToLower(s2))

    // Поиск подстроки без учёта регистра через ToLower+Contains
    text := "Learn Go programming"
    search := "GO"
    found := strings.Contains(strings.ToLower(text), strings.ToLower(search))
    fmt.Printf("Ищем %q в %q: %v\n", search, text, found)

    // Практический пример: email
    email1 := "John.Doe@Example.COM"
    email2 := "john.doe@example.com"

    if strings.EqualFold(email1, email2) {
        fmt.Printf("\nEmails совпадают (без учёта регистра)")
    }

    // Примечание: по RFC локальная часть email (до @)
    // технически чувствительна к регистру, но на практике
    // большинство провайдеров игнорирует регистр
}</code></pre>

<code>"Go" == "go"?

<h2 style="color: #34495e; margin-top: 30px;">3. Эффективная конкатенация: strings.Builder</h2>

<p><strong>Теоретическая основа &mdash; почему конкатенация медленная:</strong> Строки в Go иммутабельны (неизменяемы). Когда вы пишете <code>s = s + &quot;новая часть&quot;</code>, Go не может изменить существующую строку. Вместо этого происходит следующее:</p>

<ol>
	<li>Выделяется новый участок памяти размером len(s) + len(&quot;новая часть&quot;)</li>
	<li>Копируется содержимое старой строки в новую память</li>
	<li>Копируется новая часть в конец</li>
	<li>Старая строка остаётся в памяти пока Go автоматически не освободит её</li>
</ol>

<p><strong>Визуальная аналогия:</strong> Представьте, что вы пишете книгу, но каждый раз, добавляя новую страницу, вы должны:</p>

<ul>
	<li>Взять новую тетрадь большего размера</li>
	<li>Переписать туда все предыдущие страницы</li>
	<li>Добавить новую страницу</li>
	<li>Выбросить старую тетрадь</li>
</ul>

<p>Если в книге 100 страниц, вы перепишете: 1 + 2 + 3 + ... + 100 = 5050 страниц! Это и есть квадратичная сложность O(n&sup2;).</p>

<p><strong>Как работает strings.Builder:</strong> Builder &mdash; это как черновик на отдельных листах. Вы просто добавляете новые листы, а в конце собираете их в книгу за один проход. Внутри Builder использует изменяемый байтовый срез ([]byte), который может расти без копирования всего содержимого каждый раз.</p>

<p><strong>Математика производительности:</strong></p>

<ul>
	<li><strong>Конкатенация через +:</strong> для n частей выполняется n(n+1)/2 копирований = O(n&sup2;)</li>
	<li><strong>Builder:</strong> для n частей выполняется n записей + 1 финальное создание = O(n)</li>
	<li><strong>Разница:</strong> для 1000 частей это ~500,000 операций против 1,001 операции!</li>
</ul>

<p><strong>Когда использовать Builder:</strong></p>

<ul>
	<li>Сборка строки в цикле (генерация HTML, CSV, логов)</li>
	<li>Построение структурированных текстовых сообщений</li>
	<li>Конкатенация более 5-10 частей</li>
	<li>Когда заранее известен примерный размер результата (используйте Grow)</li>
</ul>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 15px 0;">
<p style="margin: 0;"><strong>⚠️ Важно:</strong> Нельзя копировать непустой strings.Builder! После первой записи Builder привязывается к своему внутреннему буферу. Всегда создавайте новый Builder, а не копируйте существующий.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 15px; margin: 15px 0;">
<p style="margin: 0;"><strong>📝 Новый синтаксис:</strong> В этом примере вы впервые видите вызовы вида <code>builder.WriteString(&quot;...&quot;)</code> &mdash; это <strong>методы</strong>, то есть функции, привязанные к переменной. Паттерн прост: <code>переменная.Функция(аргументы)</code>. Подробно методы разбираются в модуле 12, а пока достаточно запомнить эту запись.</p>
</div>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0; overflow-x: auto;">
<code class="language-go">package main

import (
    "fmt"
    "strconv"
    "strings"
)

func main() {
    // ПЛОХО: квадратичная сложность O(n²)
    bad := ""
    for i := 0; i &lt; 5; i++ {
        bad = bad + fmt.Sprintf("Line %d\n", i)
    }

    // ЛУЧШЕ: линейная сложность O(n), но с промежуточными строками
    var builder1 strings.Builder
    for i := 0; i &lt; 5; i++ {
        builder1.WriteString(fmt.Sprintf("Line %d\n", i))
    }

    // ОПТИМАЛЬНО: без fmt вообще + предварительное выделение памяти
    var builder3 strings.Builder
    builder3.Grow(50)  // Резервируем ~50 байт, снижаем реаллокации
    for i := 0; i &lt; 5; i++ {
        builder3.WriteString("Line ")
        builder3.WriteString(strconv.Itoa(i))
        builder3.WriteByte('\n')
    }

    fmt.Println("Результат оптимального Builder:")
    fmt.Print(builder3.String())

    // Различные методы записи
    var b strings.Builder
    b.WriteString("Hello")      // Строка
    b.WriteByte(' ')           // Байт
    b.WriteRune('🌍')          // Руна

    fmt.Printf("\nСобрано: %q\n", b.String())
    fmt.Printf("Длина: %d байт\n", b.Len())
}</code></pre>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 0;">
<code>Результат оптимального Builder:
Line 0
Line 1
Line 2
Line 3
Line 4

Собрано: "Hello 🌍"
Длина: 10 байт</code></pre>

<p style="margin: 0;"><strong>⚠️ Важно:</strong> Никогда не используйте <code>s = s + part</code> в больших циклах! Это создаёт новую строку на каждой итерации, что приводит к квадратичной сложности.</p>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">🔀 Когда Join, а когда Builder?</h4>

<ul style="margin-bottom: 0;">
	<li><code>strings.Join(slice, sep)</code> &mdash; когда у вас уже есть готовый <code>[]string</code> и нужно просто объединить. Идеально для нормализации пробелов: <code>strings.Join(strings.Fields(s), &quot; &quot;)</code></li>
	<li><code>strings.Builder</code> &mdash; когда вы собираете строку по частям в цикле, особенно если части &mdash; не строки (числа, байты, руны). Builder позволяет писать <code>WriteString</code>, <code>WriteByte</code>, <code>WriteRune</code> без промежуточных аллокаций.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 30px;">4. Быстрое преобразование чисел: strconv</h2>

<p><strong>Контекст проблемы:</strong> Преобразование чисел в строки &mdash; одна из самых частых операций в программировании. Вы делаете это при:</p>

<ul>
	<li>Формировании текстовых ответов с числовыми значениями</li>
	<li>Создании CSV файлов</li>
	<li>Генерации SQL запросов</li>
	<li>Логировании с числовыми значениями</li>
	<li>Построении URL с параметрами</li>
</ul>

<p><strong>Почему fmt.Sprintf медленный:</strong> <code>fmt.Sprintf</code> &mdash; это универсальный швейцарский нож форматирования. Он может работать с любыми типами, любыми форматами, поддерживает выравнивание, отступы, точность. Но за универсальность приходится платить:</p>

<ol>
	<li><strong>Парсинг формата:</strong> fmt должен разобрать строку формата (&quot;%d&quot;, &quot;%v&quot; и т.д.)</li>
	<li><strong>Определение типа:</strong> fmt должен во время работы программы выяснить, какой тип у каждого аргумента</li>
	<li><strong>Универсальная логика:</strong> один код для всех типов данных</li>
	<li><strong>Аллокации:</strong> создание промежуточных буферов</li>
</ol>

<p><strong>Преимущество strconv:</strong> Функции в strconv специализированы для конкретных типов. <code>strconv.Itoa</code> знает, что работает только с int, поэтому:</p>

<ul>
	<li>Нет парсинга формата</li>
	<li>Тип известен заранее &mdash; не нужно определять его во время работы</li>
	<li>Оптимизированный алгоритм именно для целых чисел</li>
	<li>Минимальные аллокации памяти</li>
</ul>

<p><strong>Реальные цифры производительности:</strong> На преобразовании большого количества чисел strconv.Itoa работает в разы быстрее fmt.Sprintf. При работе с большим количеством данных разница становится заметной.</p>

<p><strong>Правило выбора:</strong></p>

<ul>
	<li><code>strconv</code> &mdash; когда вы точно знаете тип и нужна скорость</li>

<h2 style="color: #34495e; margin-top: 30px;">5. Классификация символов с unicode</h2>

<p><strong>Зачем нужна классификация символов:</strong> При обработке текста часто нужно понимать, с каким типом символа мы работаем. Реальные задачи:</p>

<ul>
	<li><strong>Валидация паролей:</strong> минимум одна буква, одна цифра, один специальный символ</li>
	<li><strong>Очистка телефонных номеров:</strong> оставить только цифры</li>
	<li><strong>Проверка имён пользователей:</strong> только буквы, цифры и подчёркивания</li>
	<li><strong>Парсинг данных:</strong> отделить числовые значения от текстовых</li>
	<li><strong>Безопасность:</strong> обнаружение потенциально опасных символов в вводе</li>
</ul>

<p><strong>Проблема с простыми проверками:</strong> Вы не можете просто проверить <code>r &gt;= &#39;a&#39; &amp;&amp; r &lt;= &#39;z&#39;</code> для букв, потому что:</p>

<ul>
	<li>Это покрывает только латиницу в нижнем регистре</li>
	<li>Игнорирует кириллицу, китайский, арабский и сотни других алфавитов</li>
	<li>Не учитывает Unicode, где буквы разбросаны по разным диапазонам</li>
</ul>

<p><strong>Как работает пакет unicode:</strong> Он содержит таблицы всех символов Unicode, разбитых по категориям согласно стандарту. Функции типа <code>IsLetter</code> проверяют, входит ли символ в соответствующую категорию. Это работает для ВСЕХ языков мира.</p>

<p><strong>Основные категории Unicode:</strong></p>

<ul>
	<li><strong>Letter (L):</strong> все буквы всех алфавитов мира</li>
	<li><strong>Digit (Nd):</strong> цифры 0-9 во всех системах счисления</li>
	<li><strong>Space (Z):</strong> пробелы, табуляции, переносы строк</li>
	<li><strong>Punct (P):</strong> знаки препинания</li>
	<li><strong>Symbol (S):</strong> математические, валютные и другие символы</li>
	<li><strong>Mark (M):</strong> диакритические знаки (ударения, умлауты)</li>
</ul>

<p><strong>Практическая польза:</strong> Используя unicode функции, ваш код автоматически становится международным. Валидация имени будет работать для русских, китайских, арабских имён без изменения кода.</p>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 15px; margin: 15px 0;">
<p style="margin: 0;"><strong>💡 Оптимизация:</strong> Если вам нужны только ASCII-цифры (0-9), проверка <code>r &gt;= &#39;0&#39; &amp;&amp; r &lt;= &#39;9&#39;</code> работает быстрее, чем <code>unicode.IsDigit</code>. Но для поддержки всех цифр Unicode (арабские, тайские и др.) используйте <code>unicode.IsDigit</code>.</p>
</div>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0; overflow-x: auto;">
<code class="language-go">package main

import (
    "fmt"
    "strings"
    "unicode"
)

// ExtractDigits извлекает только цифры
func ExtractDigits(s string) string {
    var builder strings.Builder

    for _, r := range s {
        if unicode.IsDigit(r) {
            builder.WriteRune(r)
        }
    }

    return builder.String()
}

func main() {
    // Анализ строки
    text := "Go 1.22"

    for i, r := range text {
        fmt.Printf("'%c': ", r)

        if unicode.IsLetter(r) {
            fmt.Print("буква ")
        }
        if unicode.IsDigit(r) {
            fmt.Print("цифра ")
        }
        if unicode.IsSpace(r) {
            fmt.Print("пробел ")
        }
        if unicode.IsPunct(r) {
            fmt.Print("пунктуация ")
        }

        fmt.Printf("(позиция %d)\n", i)
    }

    // Извлечение цифр
    phone := "+7 (495) 123-45-67"
    digits := ExtractDigits(phone)
    fmt.Printf("\nТелефон: %q\n", phone)
    fmt.Printf("Цифры:   %q\n", digits)
}</code></pre>

    local, domain, found := strings.Cut(email, "@")
