<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔗 Эффективная конкатенация чисел в строку</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться эффективно собирать строки из множества элементов в Go, избегая проблем с производительностью при работе с большими объёмами данных.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">При обработке больших массивов данных &mdash; логов, CSV-файлов, JSON-генерации &mdash; критически важна скорость сборки строк. Наивная конкатенация через <code>+</code> в цикле создаёт O(n&sup2;) сложность из-за иммутабельности строк в Go, что делает программу непригодной для production.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию, которая эффективно преобразует срез чисел в строку с разделителями-запятыми.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Проблема конкатенации строк</h4>

<p>В Go строки иммутабельны &mdash; каждая операция <code>+</code> создаёт новую строку:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ПЛОХО: O(n²) сложность
result := ""
for i := 0; i &lt; 10000; i++ {
    result += strconv.Itoa(i) + ","  // создаёт 20000 промежуточных строк!
}</code></pre>

<p style="margin-bottom: 0;">При 10,000 элементах это создаст ~20,000 промежуточных строк в памяти!</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Эффективные подходы</h4>

<p><strong>strings.Builder</strong> &mdash; специализированный тип для сборки строк:</p>

<ul style="margin: 10px 0;">
	<li>Внутренний буфер растёт динамически</li>
	<li>Методы: <code>WriteString()</code>, <code>WriteByte()</code>, <code>String()</code></li>
	<li>Минимизирует аллокации памяти</li>
</ul>

<p><strong>Альтернатива: байтовый буфер</strong> для продвинутой оптимизации:</p>

<ul style="margin: 10px 0;">
	<li><code>make([]byte, 0, capacity)</code> &mdash; предаллокация</li>
	<li><code>append()</code> для добавления байтов</li>
	<li><code>string(buffer)</code> для финального преобразования</li>
</ul>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">📊 Преобразование чисел</h4>

<p>Пакет <code>strconv</code> предоставляет оптимизированные функции:</p>

<ul style="margin: 10px 0;">
	<li><code>strconv.Itoa(int)</code> &mdash; int в string</li>
	<li><code>strconv.FormatInt(int64, base)</code> &mdash; с указанием системы счисления</li>
	<li><code>strconv.AppendInt([]byte, int64, base)</code> &mdash; прямое добавление в буфер</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для эффективного преобразования среза чисел в строку с запятыми:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция JoinNumbers</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func JoinNumbers(nums []int) string</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает срез целых чисел</li>
	<li>Возвращает строку, где числа разделены запятыми БЕЗ пробелов</li>
	<li>Для пустого среза возвращает пустую строку</li>
	<li>После последнего числа запятая НЕ ставится</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Используйте цикл для перебора элементов среза</li>
	<li>НЕ используйте конкатенацию строк (<code>+</code> или <code>+=</code>) внутри цикла</li>
	<li>НЕ используйте <code>fmt.Sprintf</code> внутри цикла</li>
	<li>Используйте <code>strings.Builder</code> или байтовый буфер для сборки результата</li>
	<li>Для преобразования чисел используйте функции из пакета <code>strconv</code></li>
	<li>Разрешены только импорты: <code>&quot;strings&quot;</code> и <code>&quot;strconv&quot;</code></li>
	<li>НЕ используйте <code>fmt.Print*</code> функции</li>
	<li>Решение содержит ровно одну функцию <code>JoinNumbers</code> &mdash; без вспомогательных функций, глобальных переменных и констант: все вычисления внутри тела <code>JoinNumbers</code></li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Антипаттерн 1: Конкатенация в цикле</strong><br />
❌ <code>result += strconv.Itoa(num) + &quot;,&quot;</code> &mdash; квадратичная сложность!<br />
✅ Используйте <code>strings.Builder</code> или байтовый буфер</p>

<p><strong>Антипаттерн 2: fmt.Sprintf в цикле</strong><br />
❌ <code>result += fmt.Sprintf(&quot;%d,&quot;, num)</code> &mdash; избыточные аллокации<br />
✅ Применяйте <code>strconv.Itoa</code> + <code>builder.WriteString</code></p>

<p><strong>Антипаттерн 3: Лишняя запятая в конце</strong><br />
❌ <code>&quot;1,2,3,&quot;</code> &mdash; запятая после последнего элемента<br />
✅ Проверяйте позицию элемента или обрезайте результат</p>

<p><strong>Антипаттерн 4: Пробелы после запятых</strong><br />
❌ <code>&quot;1, 2, 3&quot;</code> &mdash; есть пробелы<br />
✅ Только запятые: <code>&quot;1,2,3&quot;</code></p>

<p><strong>Антипаттерн 5: Игнорирование пустого среза</strong><br />
❌ Паника или некорректный результат<br />
✅ Верните пустую строку для <code>[]int{}</code></p>

<p style="margin-bottom: 0;"><strong>Антипаттерн 6: Использование готовых функций</strong><br />
❌ <code>strings.Join</code> с предварительным преобразованием<br />
✅ Реализуйте алгоритм самостоятельно через цикл</p>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка по алгоритму</h4>

<p>Представьте, что вы пишете в блокнот: сначала подготавливаете место (создаёте Builder), затем последовательно записываете каждое число и, если это не последнее, добавляете запятую. В конце получаете готовый текст вызовом <code>String()</code>.</p>
