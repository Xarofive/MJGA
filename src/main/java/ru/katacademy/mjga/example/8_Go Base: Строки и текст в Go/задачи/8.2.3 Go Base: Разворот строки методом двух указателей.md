<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔄 Разворот строки методом двух указателей</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Освоить классический алгоритм разворота последовательности &quot;на месте&quot; через встречное движение индексов и обмен элементов, корректно работая с многобайтовыми символами Unicode.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Разворот строки &mdash; базовая операция в обработке текста: от палиндромов до криптографии. Алгоритм двух указателей &mdash; фундаментальная техника, применимая к спискам, массивам, строкам. Она минимизирует количество операций (O(n/2) обменов) и является основой для задач на перестановки, партиционирование, поиск пар. Понимание этого паттерна откроет дверь к решению сложных задач на LeetCode и в реальной разработке.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать in-place разворот строки через симметричный обмен символов с учётом Unicode.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Алгоритм двух указателей</h4>

<p>Классический паттерн для работы с последовательностями:</p>

<ul style="margin: 10px 0;">
	<li><strong>Идея:</strong> два индекса движутся навстречу с противоположных концов</li>
	<li><strong>Операция:</strong> на каждом шаге &mdash; обмен элементов по индексам</li>
	<li><strong>Завершение:</strong> когда индексы встречаются или пересекаются</li>
	<li><strong>Эффективность:</strong> O(n) по времени, O(1) дополнительной памяти (не считая входных данных)</li>
</ul>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Концептуальная схема (не готовое решение!):
// left начинается с 0
// right начинается с len-1
// пока left &lt; right:
//     обменять элементы[left] и элементы[right]
//     left++, right--</code></pre>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Обмен значений в Go</h4>

<p>Go поддерживает элегантный синтаксис параллельного присваивания:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Обмен без временной переменной:
a, b = b, a

// Для элементов слайса:
slice[i], slice[j] = slice[j], slice[i]

// Это атомарная операция — значения меняются одновременно</code></pre>

<p style="margin: 0;">Помните: строки в Go неизменяемы, но слайсы рун &mdash; изменяемы!</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию разворота строки алгоритмом двух указателей:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция ReverseRunes</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func ReverseRunes(s string) string</code></div>

<p><strong>Алгоритмические требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Преобразовать строку в <code>[]rune</code> для корректной работы с Unicode</li>
	<li>Использовать цикл <code>for</code> с двумя индексными переменными</li>
	<li>Левый индекс должен начинаться с <code>0</code></li>
	<li>Правый индекс должен начинаться с <code>len(runes)-1</code></li>
	<li>На каждой итерации: обменивать элементы и сдвигать индексы навстречу</li>
	<li>Условие продолжения: пока левый индекс меньше правого</li>
	<li>После разворота конвертировать обратно через <code>string(...)</code></li>
</ul>

<p><strong>Запреты:</strong></p>

<ul style="margin: 10px 0;">
	<li>Функция должна возвращать результат через <code>return</code> без аварийной остановки программы</li>
	<li>Не работать с <code>[]byte</code> &mdash; только <code>[]rune</code></li>
	<li>Не использовать встроенные функции разворота или библиотеки</li>
	<li>Не создавать новый слайс для результата &mdash; модифицировать существующий</li>
	<li>Не добавлять package и import &mdash; только тело функции</li>
</ul>

<p><strong>Формат решения:</strong></p>

<ul style="margin: 10px 0;">
	<li>Предоставить только функцию <code>ReverseRunes</code></li>
	<li>Не включать <code>package</code>, <code>import</code> или другие функции</li>
	<li>Код должен быть самодостаточным</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Работа с байтами вместо рун</strong><br />
❌ <code>bytes := []byte(s)</code> &mdash; разрушит многобайтовые символы<br />
✅ Используйте <code>runes := []rune(s)</code></p>

<p><strong>Ошибка 2: Неправильная инициализация индексов</strong><br />
❌ <code>right := len(runes)</code> &mdash; выход за границы массива<br />
✅ <code>right := len(runes) - 1</code> &mdash; последний валидный индекс</p>

<p><strong>Ошибка 3: Неоптимальное условие цикла</strong><br />
⚠️ <code>for left &lt;= right</code> &mdash; работает корректно, но делает лишний обмен центрального элемента с самим собой<br />
✅ <code>for left &lt; right</code> &mdash; оптимально: остановка при встрече индексов</p>

<p><strong>Ошибка 4: Забыли обновить индексы</strong><br />
❌ Только обмен без <code>left++</code> и <code>right--</code> &mdash; бесконечный цикл<br />
✅ Обязательно сдвигайте оба индекса на каждой итерации</p>

<p><strong>Ошибка 5: Создание нового слайса</strong><br />
❌ <code>result := make([]rune, len(runes))</code> и копирование в обратном порядке<br />
✅ Модифицируйте исходный слайс рун через обмены</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Использование встроенных функций</strong><br />
❌ <code>sort.Reverse()</code> или сторонние библиотеки<br />
✅ Реализуйте алгоритм самостоятельно для понимания</p>
