<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔍 Поиск подстроки без учёта регистра</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться работать с текстовыми данными с учётом регистрозависимости, реализовав алгоритм поиска подстроки, игнорирующий различия между заглавными и строчными буквами.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В реальных приложениях &mdash; поисковых системах, текстовых редакторах, системах автодополнения &mdash; пользователи ожидают, что поиск будет работать независимо от регистра. Написание &quot;Hello&quot;, &quot;HELLO&quot; или &quot;hello&quot; должно давать одинаковые результаты при поиске текста.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию поиска подстроки, которая игнорирует различия в регистре символов.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Работа с регистром в Go</h4>

<p>Пакет <code>strings</code> предоставляет функции для работы с регистром:</p>

<ul style="margin: 10px 0;">
	<li><code>strings.ToLower(s)</code> &mdash; преобразует все символы в нижний регистр</li>
	<li><code>strings.ToUpper(s)</code> &mdash; преобразует все символы в верхний регистр</li>
	<li><code>strings.Contains(s, substr)</code> &mdash; проверяет наличие подстроки (регистрозависимо)</li>
</ul>

<p style="margin-bottom: 0;">В Go строки иммутабельны, поэтому эти функции возвращают новую строку.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Подходы к регистронезависимому поиску</h4>

<p><strong>Подход 1: Нормализация регистра</strong></p>

<ul style="margin: 10px 0;">
	<li>Привести обе строки к одному регистру</li>
	<li>Использовать стандартный поиск подстроки</li>
	<li>Простая реализация, но создаёт копии строк</li>
</ul>

<p><strong>Подход 2: Посимвольное сравнение</strong></p>

<ul style="margin: 10px 0;">
	<li>Функция <code>strings.EqualFold(s1, s2)</code> &mdash; сравнивает строки без учёта регистра</li>
	<li>Можно использовать для сравнения фрагментов в цикле</li>
	<li>Более эффективно по памяти для больших строк</li>
</ul>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">📊 Особый случай: пустая подстрока</h4>

<p>По соглашению в Go (и многих других языках), пустая подстрока считается содержащейся в любой строке:</p>

<ul style="margin: 10px 0;">
	<li><code>strings.Contains(&quot;hello&quot;, &quot;&quot;)</code> возвращает <code>true</code></li>
	<li>Это логично: пустая строка &quot;находится&quot; в начале, конце и между любыми символами</li>
	<li>Ваша функция должна следовать этому соглашению</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию для поиска подстроки без учёта регистра:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция ContainsFold</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func ContainsFold(s, sub string) bool</code></div>

<p><strong>Требования:</strong></p>

<ul style="margin: 10px 0;">
	<li>Принимает две строки: исходную строку <code>s</code> и искомую подстроку <code>sub</code></li>
	<li>Возвращает <code>true</code>, если <code>sub</code> содержится в <code>s</code> (без учёта регистра)</li>
	<li>Возвращает <code>false</code>, если подстрока не найдена</li>
	<li>Если <code>sub</code> &mdash; пустая строка, всегда возвращает <code>true</code></li>
	<li>Параметры должны быть именованы (не анонимны)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Импортировать только пакет <code>strings</code></li>
	<li>Создать только одну функцию <code>ContainsFold</code></li>
	<li>Функция должна возвращать значение через <code>return</code> без аварийной остановки программы</li>
	<li>НЕ использовать функции вывода (<code>fmt.Print*</code>)</li>
	<li>Обязательно использовать либо <code>strings.ToLower</code>/<code>strings.ToUpper</code>, либо <code>strings.EqualFold</code></li>
	<li>При использовании приведения к регистру &mdash; применить <code>strings.Contains</code></li>
	<li>При использовании <code>strings.EqualFold</code> &mdash; реализовать перебор позиций в цикле</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Антипаттерн 1: Забытый пустой случай</strong><br />
❌ Не проверяется <code>sub == &quot;&quot;</code><br />
✅ Обязательно вернуть <code>true</code> для пустой подстроки</p>

<p><strong>Антипаттерн 2: Использование регистрозависимого поиска</strong><br />
❌ <code>return strings.Contains(s, sub)</code> &mdash; учитывает регистр!<br />
✅ Сначала нормализовать регистр или использовать <code>EqualFold</code></p>

<p><strong>Антипаттерн 3: Неэффективное использование EqualFold</strong><br />
❌ Сравнение всей строки целиком<br />
✅ При использовании <code>EqualFold</code> нужен цикл для проверки всех позиций</p>

<p><strong>Антипаттерн 4: Неправильная обработка границ</strong><br />
❌ Выход за границы строки при переборе<br />
✅ Проверять, что остаток строки достаточной длины</p>

<p><strong>Антипаттерн 5: Аварийная остановка программы или вывод на экран вместо возврата</strong><br />
❌ Прерывать программу при отсутствии совпадения или печатать результат через <code>fmt.Println</code><br />
✅ Только возврат <code>bool</code> результата через <code>return</code></p>

<p style="margin-bottom: 0;"><strong>Антипаттерн 6: Лишние функции или импорты</strong><br />
❌ Дополнительные вспомогательные функции или импорт <code>fmt</code><br />
✅ Только одна функция и импорт <code>strings</code></p>
