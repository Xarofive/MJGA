<h2 style="color: #34495e; margin-top: 28px;">🌊 Философия ошибок в Go</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Ошибки &mdash; это значения</h4>

<p style="margin-bottom: 10px;">Ошибки в Go &mdash; это обычные значения. Это даёт полный контроль над потоком выполнения и делает обработку ошибок явной и предсказуемой.</p>

<p style="margin-bottom: 10px;">В Go <code>error</code> &mdash; это встроенный тип. Любое значение, у которого есть метод <code>Error() string</code>, может быть ошибкой. На практике вы будете создавать ошибки через <code>errors.New(&quot;текст&quot;)</code> и <code>fmt.Errorf(&quot;формат&quot;, ...)</code>.</p>

<p style="margin-bottom: 10px;"><strong>Важно:</strong> для этого модуля не нужны знания <code>try/catch</code> из других языков. Базовое правило простое: после вызова функции проверяйте <code>err != nil</code> и при ошибке сразу возвращайте её.</p>

<p><strong>Преимущества подхода Go:</strong></p>

<ul style="margin: 10px 0;">
	<li>Явность &mdash; всегда видно, где может произойти ошибка</li>
	<li>Простота &mdash; нет скрытых путей выполнения</li>
	<li>Гибкость &mdash; ошибку можно сохранить, сравнить и обработать как любое другое значение</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">1. Ошибка &mdash; обычное значение</h2>

<h3 style="color: #455a64; margin-top: 24px;">Паттерн (result, error)</h3>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Guard Clauses и ранний возврат</h4>

<p style="margin-bottom: 10px;">Guard clause &mdash; это проверка условия в начале функции с ранним возвратом при ошибке. Это делает основной путь выполнения линейным и читаемым.</p>

<p style="margin-bottom: 10px;"><strong>Принципы:</strong></p>

<ul style="margin: 10px 0;">
	<li>Проверяйте ошибки сразу после вызова</li>
	<li>Возвращайтесь рано при ошибке</li>
	<li>Основная логика остаётся не вложенной</li>
	<li>Избегайте глубокой вложенности if-else</li>
</ul>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"errors"
"fmt"
"strconv"
"strings"
)

// Хороший стиль: guard clauses и ранний возврат
func ParseAndDouble(s string) (int, error) {
// Guard: пустая строка
if s == "" {
return 0, errors.New("empty string")
}

    // Guard: проверка префикса
    if !strings.HasPrefix(s, "num:") {
        return 0, errors.New("missing 'num:' prefix")
    }

    // Извлекаем число
    numStr := strings.TrimPrefix(s, "num:")

    // Guard: парсинг
    n, err := strconv.Atoi(numStr)
    if err != nil {
        return 0, fmt.Errorf("invalid number %q: %v", numStr, err)
    }

    // Guard: диапазон
    if n &lt; 0 || n &gt; 1000 {
        return 0, fmt.Errorf("number %d out of range [0, 1000]", n)
    }

    // Основная логика — не вложена
    return n * 2, nil
}</code></pre>

<p style="margin: 8px 0;">Пример использования этой функции:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func main() {
    tests := []string{"", "hello", "num:abc", "num:42", "num:2000"}

    for _, test := range tests {
        result, err := ParseAndDouble(test)
        if err != nil {
            fmt.Printf("❌ %q: %v\n", test, err)
        } else {
            fmt.Printf("✓ %q: %d\n", test, result)
        }
    }
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">2. Виды ошибок</h2>

<h3 style="color: #455a64; margin-top: 24px;">Sentinel-ошибки</h3>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Когда использовать sentinel-ошибки</h4>

<p style="margin-bottom: 10px;">Sentinel-ошибки &mdash; это предопределённые значения ошибок для известных состояний. Они экспортируются как переменные пакета и позволяют вызывающему коду проверять конкретные условия прямым сравнением <code>==</code>.</p>

<p style="margin-bottom: 10px;"><strong>Используйте sentinel-ошибки когда:</strong></p>

<ul style="margin: 10px 0;">
	<li>Ошибка представляет конкретное, хорошо определённое состояние</li>
	<li>Вызывающему коду нужно различать типы ошибок</li>
	<li>Ошибка не требует дополнительного контекста</li>
</ul>

<p style="margin: 0;"><strong>Пример из стандартной библиотеки:</strong> <code>io.EOF</code> &mdash; сигнал о конце данных, который вы уже видели в лекции про ввод.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"errors"
"fmt"
"strings"
)

// Sentinel-ошибки для email валидации
var (
ErrEmailEmpty    = errors.New("email is empty")
ErrEmailNoAt     = errors.New("email missing @ symbol")
ErrEmailMultiAt  = errors.New("email has multiple @ symbols")
ErrEmailNoDomain = errors.New("email missing domain")
)

func ValidateEmail(email string) error {
// Используем sentinel-ошибки для конкретных состояний
if email == "" {
return ErrEmailEmpty
}

    atCount := strings.Count(email, "@")
    if atCount == 0 {
        return ErrEmailNoAt
    }
    if atCount &gt; 1 {
        return ErrEmailMultiAt
    }

    parts := strings.Split(email, "@")
    if len(parts[1]) == 0 {
        return ErrEmailNoDomain
    }

    if !strings.Contains(parts[1], ".") {
        return ErrEmailNoDomain
    }

    return nil
}</code></pre>

<p style="margin: 8px 0;">А теперь протестируем наши sentinel-ошибки через прямое сравнение <code>==</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func main() {
    emails := []string{
        "",
        "user",
        "user@@example.com",
        "user@",
        "user@domain",
        "user@example.com",
    }

    for _, email := range emails {
        err := ValidateEmail(email)
        if err != nil {
            // Проверяем конкретные ошибки прямым сравнением
            if err == ErrEmailEmpty {
                fmt.Printf("❌ %q: Пустой email\n", email)
            } else if err == ErrEmailNoAt {
                fmt.Printf("❌ %q: Нужен символ @\n", email)
            } else if err == ErrEmailMultiAt {
                fmt.Printf("❌ %q: Слишком много @\n", email)
            } else if err == ErrEmailNoDomain {
                fmt.Printf("❌ %q: Неверный домен\n", email)
            } else {
                fmt.Printf("❌ %q: %v\n", email, err)
            }
        } else {
            fmt.Printf("✓ %q: Валидный email\n", email)
        }
    }
}</code></pre>

<div style="background: #e8f5e9; border: 1px solid #66bb6a; border-radius: 4px; padding: 10px; margin-top: 10px;"><strong>📋 Ожидаемый вывод:</strong>

<pre style="margin: 5px 0; color: #2e7d32;">
❌ &quot;&quot;: Пустой email
❌ &quot;user&quot;: Нужен символ @
❌ &quot;user@@example.com&quot;: Слишком много @
❌ &quot;user@&quot;: Неверный домен
❌ &quot;user@domain&quot;: Неверный домен
✓ &quot;user@example.com&quot;: Валидный email</pre>

<div style="background: #e3f2fd; border: 1px dashed #64b5f6; border-radius: 6px; padding: 10px; margin: 16px 0;">
<p style="margin: 0;">💡 <strong>Ключевая идея:</strong> если функция возвращает sentinel-ошибку напрямую, вызывающий код сравнивает результат с этой переменной через <code>==</code>. Без дополнительного контекста этого достаточно, чтобы различать известные состояния.</p>

<h2 style="color: #34495e; margin-top: 28px;">3. Сообщения об ошибках</h2>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Принципы хороших сообщений об ошибках</h4>

<p style="margin-bottom: 10px;">Хорошее сообщение об ошибке должно отвечать на три вопроса:</p>

<ol style="margin: 10px 0;">
	<li><strong>Что произошло?</strong> &mdash; описание проблемы</li>
	<li><strong>Где произошло?</strong> &mdash; контекст (файл, строка, поле)</li>
	<li><strong>Почему произошло?</strong> &mdash; причина или ожидаемое значение</li>
</ol>

<p style="margin: 0;"><strong>Правила:</strong> начинайте с маленькой буквы, не заканчивайте точкой, добавляйте контекст к сообщению об ошибке.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

"fmt"
"strconv"
"strings"
)

// ❌ Плохие сообщения об ошибках
func badErrors() {
fmt.Println("=== Плохие примеры ===")

    // Слишком общее
    err1 := fmt.Errorf("Error")
    fmt.Printf("❌ %v\n", err1)

    // Нет контекста
    err2 := fmt.Errorf("invalid value")
    fmt.Printf("❌ %v\n", err2)

    // Потеря исходной ошибки
    original := fmt.Errorf("connection timeout")
    err3 := fmt.Errorf("database error") // original потеряна!
    fmt.Printf("❌ %v (исходная: %v)\n", err3, original)

    // Избыточные детали
    err4 := fmt.Errorf("ERROR!!! FATAL!!! The value you provided is completely wrong!!!")
    fmt.Printf("❌ %v\n", err4)
}</code></pre>

<p style="margin: 8px 0;">А теперь более качественные формулировки:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ✓ Хорошие сообщения об ошибках

<p style="margin: 8px 0;">И отдельный пример функции с контекстом ошибок:</p>

<code class="language-go">// Пример функции с информативными ошибками
func ParseCSVInts(s string) ([]int, error) {
    if s == "" {
        return nil, fmt.Errorf("empty input")
    }

    parts := strings.Split(s, ",")
    result := make([]int, 0, len(parts))

    for i, part := range parts {
        part = strings.TrimSpace(part)
        if part == "" {
            return nil, fmt.Errorf("empty value at index %d", i)

        n, err := strconv.Atoi(part)
        if err != nil {
            // Хорошее сообщение: что, где, почему
            return nil, fmt.Errorf("invalid integer %q at index %d: %v",
                part, i, err)
        }

        result = append(result, n)
    }

    return result, nil
}</code></pre>

<h4 style="color: #2e7d32; margin-top: 0;">Упражнение 2: ValidateEmail с sentinel-ошибками</h4>

<p style="margin-bottom: 10px;">Реализуйте функцию <code>ValidateEmail(s string) error</code> с sentinel-ошибками <code>ErrEmpty</code> и <code>ErrFormat</code>. Покажите пример использования с прямым сравнением <code>==</code>.</p>

    ErrEmpty  = errors.New("email is empty")
    ErrFormat = errors.New("invalid email format")
)

func ValidateEmail(s string) error {
    if s == "" {
        return ErrEmpty
    }

    if !strings.Contains(s, "@") {
        return ErrFormat
    }

    parts := strings.Split(s, "@")
    if len(parts) != 2 {
        return ErrFormat
    }

    if len(parts[0]) == 0 || len(parts[1]) == 0 {
        return ErrFormat
    }

    if !strings.Contains(parts[1], ".") {
        return ErrFormat
    }

    return nil
}</code></pre>
