<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔢 Суммирование чисел из потока</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться извлекать числовые данные из текстового потока с надёжной обработкой ошибок парсинга и граничных случаев.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Обработка числовых данных из текстовых источников &mdash; базовая задача в логах, конфигурациях, CSV-файлах и API-ответах. Корректная валидация и информативные сообщения об ошибках критичны для отладки и мониторинга систем обработки данных.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для суммирования всех целых чисел из текстового потока с детальной диагностикой ошибок.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Парсинг чисел и обработка ошибок</h4>

<p><strong>strconv.Atoi</strong> &mdash; конвертация строки в целое число:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">num, err := strconv.Atoi("123")
if err != nil {
    // err имеет тип *strconv.NumError
    // содержит информацию о проблемной строке
}</code></pre>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Работа с потоками и Scanner</h4>

<ul style="margin: 10px 0;">
	<li><strong>bufio.Scanner:</strong> построчное чтение из io.Reader</li>
	<li><strong>scanner.Err():</strong> после цикла <code>for scanner.Scan()</code> нужно вызвать <code>scanner.Err()</code> и вернуть результат, если он не nil &mdash; сам по себе <code>Scan()</code> не возвращает ошибку чтения</li>
	<li><strong>strings.TrimSpace:</strong> удаление пробелов по краям строки</li>
	<li><strong>Пустые строки:</strong> после обрезки пробелов могут стать пустыми &mdash; их нужно пропускать</li>
	<li><strong>errors.New:</strong> создание простых ошибок с текстовым сообщением</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию для суммирования целых чисел из текстового потока:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция SumInts</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func SumInts(r io.Reader) (int, error)</code></div>

<p><strong>Требования к поведению:</strong></p>

<ul style="margin: 10px 0;">
	<li>Читает строки из io.Reader построчно</li>
	<li>Пропускает пустые строки и строки только из пробелов</li>
	<li>Парсит каждую непустую строку как целое число через strconv.Atoi</li>
	<li>Суммирует все успешно распарсенные числа</li>
	<li>При успехе возвращает сумму и nil</li>
	<li>При ошибке парсинга: возвращает 0 и ошибку с указанием проблемной строки (например <code>fmt.Errorf(&quot;парсинг %q: %v&quot;, line, err)</code>)</li>
	<li>При ошибке чтения из io.Reader (её возвращает <code>scanner.Err()</code> после цикла): возвращает 0 и эту ошибку</li>
	<li>Если нет ни одного числа: возвращает 0 и ошибку с текстом &quot;no numbers&quot;</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📝 Ограничения реализации</h4>

<ul style="margin: 10px 0;">
	<li>Используйте только пакеты: bufio, errors, fmt, io, strconv, strings</li>
	<li>fmt можно использовать только для fmt.Errorf</li>
	<li>Возвращайте ошибки через возвращаемое значение <code>error</code> &mdash; никакой аварийной остановки программы, <code>print</code>/<code>println</code> или вывода на консоль</li>
	<li>Обязательно проверяйте <code>scanner.Err()</code> после цикла чтения</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неинформативная ошибка парсинга</strong><br />
❌ <code>return 0, errors.New(&quot;parse error&quot;)</code> &mdash; без проблемной строки<br />
✅ <code>return 0, fmt.Errorf(&quot;парсинг %q: %v&quot;, line, err)</code> &mdash; включает проблемную строку и текст ошибки strconv</p>

<p><strong>Ошибка 2: Суммирование не всех чисел</strong><br />
❌ Остановка после первой ошибки парсинга<br />
✅ При ошибке сразу возвращать 0 и ошибку</p>

<p><strong>Ошибка 3: Неправильная обработка пустого ввода</strong><br />
❌ Возврат успеха для пустого потока<br />
✅ Возврат ошибки &quot;no numbers&quot; если не найдено ни одного числа</p>

<p><strong>Ошибка 4: Включение пробельных строк</strong><br />
❌ Попытка парсить строки без TrimSpace<br />
✅ Обрезать пробелы и пропускать пустые строки</p>

<p><strong>Ошибка 5: Вывод на консоль</strong><br />
❌ fmt.Println для отладки<br />
✅ Никакого вывода, только возврат результата</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Неинформативные ошибки</strong><br />
❌ <code>errors.New(&quot;error&quot;)</code> без контекста<br />
✅ Включать проблемную строку в сообщение об ошибке</p>
