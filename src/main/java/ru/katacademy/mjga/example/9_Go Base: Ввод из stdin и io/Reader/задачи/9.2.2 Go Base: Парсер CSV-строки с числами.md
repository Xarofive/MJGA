<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📊 Парсер CSV-строки с числами</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться парсить структурированные текстовые данные, корректно обрабатывать ошибки с указанием позиции проблемного элемента и работать со срезами в Go.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">CSV (Comma-Separated Values) &mdash; один из самых популярных форматов для обмена табличными данными между системами. При импорте данных из Excel, обработке логов или парсинге конфигураций часто требуется преобразовать строки с числами, разделёнными запятыми. Умение создавать надёжные парсеры с информативными сообщениями об ошибках критично для построения систем интеграции данных.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию для парсинга строки с целыми числами, разделёнными запятыми, с детальной диагностикой ошибок.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Работа со строками и срезами</h4>

<p>Пакет <code>strings</code> предоставляет функции для работы со строками:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Разбиение строки по разделителю
parts := strings.Split("1,2,3", ",")  // []string{"1", "2", "3"}

// Удаление пробелов по краям
clean := strings.TrimSpace("  42  ")  // "42"

// Создание среза целых чисел
numbers := make([]int, 0)  // или []int{} или var numbers []int
numbers = append(numbers, 42)</code></pre>

<p style="margin-bottom: 0;">Срезы в Go &mdash; динамические массивы, которые могут расти по мере добавления элементов.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Создание информативных ошибок</h4>

<p>При обработке последовательных данных важно указывать позицию проблемного элемента:</p>

<ul style="margin: 10px 0;">
	<li><strong>Индексация:</strong> элементы в срезах нумеруются с 0</li>
	<li><strong>Контекст ошибки:</strong> включайте проблемное значение и его позицию</li>
	<li><strong>fmt.Errorf:</strong> позволяет создавать форматированные сообщения об ошибках</li>
</ul>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">📝 Обработка ошибок парсинга</h4>

<p>При невозможности выполнить операцию функция должна:</p>

<ul style="margin: 10px 0;">
	<li>Возвращать <code>nil</code> вместо частично заполненного результата</li>
	<li>Предоставлять ошибку с детальной информацией о проблеме</li>
	<li>Указывать индекс и значение проблемного элемента</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию ParseCSVInts для парсинга CSV-строки с целыми числами:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура функции</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func ParseCSVInts(s string) ([]int, error)</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li><strong>Разделитель:</strong> числа разделены запятыми</li>
	<li><strong>Пробелы:</strong> удаляйте начальные и конечные пробелы у каждого элемента</li>
	<li><strong>Успешный парсинг:</strong> возвращайте срез чисел и <code>nil</code>-ошибку</li>
	<li><strong>Пустая строка:</strong> возвращайте <code>(nil, error)</code> с текстом <code>&quot;empty input&quot;</code></li>
	<li><strong>Только пробельные символы:</strong> если входная строка состоит исключительно из пробельных символов (пробелы, табы, переводы строк) &mdash; обрабатывайте её так же, как пустую: возвращайте <code>(nil, errors.New(&quot;empty input&quot;))</code>. Удобный guard в начале функции: <code style="white-space: pre;">strings.TrimSpace(s) == &quot;&quot;</code>.</li>
	<li><strong>Пустой элемент:</strong> после удаления пробелов элемент стал пустым &mdash; возвращайте ошибку с указанием индекса</li>
	<li><strong>Некорректное число:</strong> возвращайте <code>(nil, error)</code> с проблемным значением и индексом</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Разрешённые импорты</h4>

<ul style="margin: 10px 0;">
	<li><code>strconv</code> &mdash; для преобразования строк в числа</li>
	<li><code>strings</code> &mdash; для работы со строками</li>
	<li><code>fmt</code> &mdash; для форматирования ошибок</li>
	<li><code>errors</code> &mdash; для создания ошибок (опционально)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Возвращайте ошибки через возвращаемое значение <code>error</code> &mdash; не пытайтесь сигнализировать об ошибке аварийной остановкой программы</li>
	<li>Не используйте функции вывода (<code>fmt.Print*</code>)</li>
	<li>Не создавайте дополнительные функции или методы</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
	<li>Не добавляйте глобальные переменные</li>
</ul>
</div>

<div style="background: #e1f5fe; border: 2px solid #0277bd; border-radius: 8px; padding: 16px; margin: 20px 0;">
<h4 style="color: #01579b; margin-top: 0;">📋 Формат сообщений об ошибках</h4>

<p style="margin-top: 10px;"><strong>⚠️ ВАЖНО:</strong> Сообщения об ошибках должны содержать позицию проблемного элемента. Рекомендуется использовать слово <strong>&quot;индекс&quot;</strong> (на русском). Допускается также: &quot;index&quot;, &quot;position&quot;, &quot;позиция&quot; или формат <code>[N]</code>.</p>

<h5 style="color: #0277bd; margin-top: 16px;">Для пустого элемента:</h5>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0; font-family: monospace;">
return nil, fmt.Errorf(&quot;индекс %d: %q: empty&quot;, i, trimmedValue)</pre>

<p style="margin: 10px 0;"><strong>Пример:</strong> <code>&quot;индекс 1: &quot;&quot;: empty&quot;</code></p>

<h5 style="color: #0277bd; margin-top: 16px;">Для некорректного числа:</h5>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0; font-family: monospace;">
return nil, fmt.Errorf(&quot;индекс %d: %q: %v&quot;, i, trimmedValue, err)</pre>

<p style="margin: 0;"><strong>Пример:</strong> <code>&quot;индекс 1: &quot;abc&quot;: invalid syntax&quot;</code></p>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Возврат частичного результата при ошибке</strong><br />
❌ При ошибке парсинга возвращать уже распарсенные числа<br />
✅ Всегда возвращайте <code>nil</code> в качестве среза при любой ошибке</p>

<p><strong>Ошибка 2: Неправильная обработка пустых элементов</strong><br />
❌ Игнорировать пустые элементы или считать их нулями<br />
✅ Пустой элемент после обрезки пробелов &mdash; это ошибка с сообщением вида: <code>&quot;индекс %d: %q: empty&quot;</code></p>

<p><strong>Ошибка 3: Отсутствие индекса в сообщении об ошибке</strong><br />
❌ <code>return nil, fmt.Errorf(&quot;invalid value&quot;)</code><br />
✅ Включайте индекс проблемного элемента: <code>&quot;индекс %d: ...&quot;</code></p>

<p><strong>Ошибка 4: Отсутствие позиции элемента в сообщении</strong><br />
❌ Ошибка без указания позиции проблемного элемента<br />
✅ Указывайте позицию: <strong>&quot;индекс&quot;</strong>, &quot;index&quot;, &quot;position&quot; или <code>[N]</code></p>

<p><strong>Ошибка 5: Неправильная проверка пустой строки</strong><br />
❌ Проверка после <code>strings.Split</code> (пустая строка даст <code>[&quot;&quot;]</code>)<br />
✅ Проверяйте пустоту входной строки до разбиения</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Аварийная остановка программы при ошибках</strong><br />
❌ Прерывать выполнение программы при некорректных данных<br />
✅ Всегда возвращайте ошибку через <code>return nil, fmt.Errorf(...)</code></p>
