<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🧮 Калькулятор арифметических операций</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться создавать функции с множественными возвращаемыми значениями, использовать sentinel-ошибки для различения конкретных ошибочных ситуаций и реализовывать диспетчеризацию операций.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Калькуляторы &mdash; основа многих приложений: от простых утилит до сложных финансовых систем и научных программ. Корректная обработка ошибок (деление на ноль, неизвестные операции) с использованием sentinel-паттерна позволяет клиентскому коду различать конкретные ошибочные ситуации (деление на ноль, неизвестная операция) и реагировать соответствующим образом.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию-калькулятор для базовых арифметических операций с корректной обработкой ошибок.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Множественные возвращаемые значения</h4>

<p>В Go функции могут возвращать несколько значений одновременно:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func divide(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// Использование
result, err := divide(10, 2)
if err != nil {
    // обработка ошибки
}
// result == 5</code></pre>

<p style="margin-bottom: 0;">Паттерн возврата (значение, ошибка) &mdash; стандартный подход в Go для функций, которые могут завершиться неудачей.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Sentinel-ошибки для различения ошибочных ситуаций</h4>

<p>Предопределённые ошибки позволяют точно определить, какая именно ситуация произошла:</p>

<ul style="margin: 10px 0;">
	<li><strong>Объявление:</strong> <code>var ErrDivideByZero = errors.New(&quot;divide by zero&quot;)</code></li>
	<li><strong>Возврат:</strong> <code>return 0, ErrDivideByZero</code></li>
	<li><strong>Проверка:</strong> <code>if err == ErrDivideByZero { ... }</code> &mdash; простое сравнение значения на равенство, поскольку sentinel-ошибки сохраняют идентичность</li>
	<li><strong>Преимущество:</strong> клиентский код может различать конкретные ситуации (деление на ноль vs неизвестная операция) по конкретному значению ошибки</li>
</ul>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">📝 Целочисленное деление в Go</h4>

<p>Особенности операций с целыми числами:</p>

<ul style="margin: 10px 0;">
	<li><strong>Деление:</strong> результат округляется к нулю (7 / 5 = 1, -7 / 5 = -1)</li>
	<li><strong>Деление на ноль:</strong> вызывает ошибку времени выполнения, поэтому требует предварительной проверки</li>
	<li><strong>Умножение:</strong> может привести к переполнению (проверка не требуется для этой задачи)</li>
	<li><strong>Вычитание отрицательных:</strong> a - (-b) эквивалентно a + b</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте калькулятор базовых операций с sentinel-ошибками:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Sentinel-ошибки (обязательные)</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">var ErrUnknownOp = errors.New("unknown op")
var ErrDivideByZero = errors.New("divide by zero")</code></pre>

<h4 style="color: #e65100; margin-top: 16px;">📝 Сигнатура функции</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func CalcOp(a, b int, op string) (int, error)</code></div>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li><strong>Поддерживаемые операции:</strong>

	<ul>
		<li><code>&quot;+&quot;</code> &mdash; сложение (a + b)</li>
		<li><code>&quot;-&quot;</code> &mdash; вычитание (a - b)</li>
		<li><code>&quot;*&quot;</code> &mdash; умножение (a * b)</li>
		<li><code>&quot;/&quot;</code> &mdash; целочисленное деление (a / b)</li>
	</ul>
	</li>
	<li><strong>Успешное выполнение:</strong> возвращайте (результат, nil)</li>
	<li><strong>Неизвестная операция:</strong> возвращайте (0, ErrUnknownOp)</li>
	<li><strong>Деление на ноль:</strong> при b == 0 и op == &quot;/&quot; возвращайте (0, ErrDivideByZero)</li>
	<li><strong>Порядок проверок:</strong> сначала определите операцию, затем проверяйте специальные случаи</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">💡 Разрешённые импорты</h4>

<ul style="margin: 10px 0;">
	<li><code>errors</code> &mdash; для создания sentinel-ошибок</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Возвращайте ошибки через возвращаемое значение <code>error</code> &mdash; не пытайтесь сигнализировать об ошибке аварийной остановкой программы</li>
	<li>Не используйте функции вывода (<code>fmt.*</code>, <code>print</code>, <code>println</code>)</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
	<li>Sentinel-ошибки должны быть созданы через <code>errors.New</code> с точными сообщениями</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Создание новых ошибок вместо sentinel</strong><br />
❌ <code>return 0, errors.New(&quot;divide by zero&quot;)</code> &mdash; создаёт новую ошибку<br />
✅ <code>return 0, ErrDivideByZero</code> &mdash; возвращает sentinel-ошибку</p>

<p><strong>Ошибка 2: Неправильный порядок проверок</strong><br />
❌ Сначала проверять деление на ноль, потом операцию<br />
✅ Сначала определить операцию, затем проверять специфичные условия</p>

<p><strong>Ошибка 3: Возврат ненулевого значения при ошибке</strong><br />
❌ <code>return -1, ErrUnknownOp</code> или <code>return a+b, ErrDivideByZero</code><br />
✅ При любой ошибке всегда возвращайте <code>(0, error)</code></p>

<p><strong>Ошибка 4: Забыли проверку на ноль перед делением</strong><br />
❌ <code>case &quot;/&quot;: return a / b, nil</code> &mdash; ошибка времени выполнения при b == 0<br />
✅ Проверяйте b на равенство нулю перед делением</p>

<p><strong>Ошибка 5: Неправильные тексты sentinel-ошибок</strong><br />
❌ <code>errors.New(&quot;деление на ноль&quot;)</code> или <code>errors.New(&quot;unknown operation&quot;)</code><br />
✅ Точные тексты: &quot;divide by zero&quot; и &quot;unknown op&quot;</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Аварийная остановка программы вместо возврата ошибки</strong><br />
❌ Прерывать выполнение программы при делении на ноль<br />
✅ Всегда возвращайте ошибку через <code>return 0, ErrDivideByZero</code></p>
