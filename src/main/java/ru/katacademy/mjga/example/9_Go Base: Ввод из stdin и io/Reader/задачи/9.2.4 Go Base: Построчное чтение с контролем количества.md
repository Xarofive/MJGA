<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📖 Построчное чтение с контролем количества</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться реализовывать надёжное чтение текстовых данных с контролем количества строк и корректной обработкой ошибок при работе с потоками ввода.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В реальных системах часто требуется читать данные порциями: заголовки протоколов, батчи для обработки, конфигурационные файлы с фиксированной структурой. Умение контролировать количество прочитанных строк и корректно обрабатывать ситуации недостатка данных критично для парсеров, валидаторов и систем потоковой обработки.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для чтения ровно N непустых строк из потока с гарантиями целостности результата.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Источник данных (io.Reader) и bufio.Scanner</h4>

<p><strong>io.Reader</strong> &mdash; воспринимайте как &laquo;любой источник данных, из которого можно читать байты&raquo;: stdin, файл, строка в памяти, сеть. Для этой задачи <em>не нужно знать теорию интерфейсов</em> &mdash; просто передавайте в функцию источник, а внутри используйте Scanner.</p>

<p style="margin: 0;">У <code>io.Reader</code> есть метод <code>Read</code>, через который Scanner получает данные. Подробнее об этом &mdash; в следующем курсе.</p>

<p><strong>bufio.Scanner</strong> &mdash; удобная обёртка для построчного чтения:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">scanner := bufio.NewScanner(r) // r: io.Reader — источник данных
for scanner.Scan() {
    line := scanner.Text()
}
if err := scanner.Err(); err != nil {
    // обработка ошибки чтения
}</code></pre>

<div style="background: #e3f2fd; border: 1px dashed #64b5f6; border-radius: 6px; padding: 10px; margin: 10px 0;">💡 <strong>Важно:</strong> в этой задаче достаточно запомнить: <code>io.Reader</code> &mdash; это &laquo;источник данных&raquo;. Теория интерфейсов будет позже.</div>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Обработка строк и ошибок</h4>

<ul style="margin: 10px 0;">
	<li><strong>strings.TrimSpace:</strong> удаляет пробельные символы в начале и конце строки</li>
	<li><strong>Пустые строки:</strong> строка после TrimSpace может стать пустой &mdash; такие нужно пропускать</li>
	<li><strong>scanner.Err():</strong> всегда проверяйте после цикла чтения &mdash; может содержать ошибку ввода-вывода</li>
	<li><strong>errors.New:</strong> создание собственных ошибок с информативными сообщениями</li>
</ul>
</div>

<ul style="margin: 10px 0;">
	<li>Используйте только пакеты: bufio, errors, fmt, io, strings</li>
	<li>Обязательно используйте bufio.NewScanner для чтения</li>
	<li>Обязательно проверяйте scanner.Err() после чтения</li>
	<li>Возвращайте ошибки через возвращаемое значение <code>error</code> &mdash; никакой аварийной остановки программы, <code>fmt.Print*</code> или другого вывода на экран</li>
	<li>Читайте только построчно (не меняйте режим разбиения Scanner)</li>
</ul>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Включение пустых строк в результат</strong><br />
❌ Добавление строк без проверки после TrimSpace<br />
✅ Проверяйте: если после TrimSpace строка пустая &mdash; пропускайте её</p>

<p><strong>Ошибка 2: Игнорирование scanner.Err()</strong><br />
❌ Возврат результата без проверки ошибок чтения<br />
✅ После цикла Scan() всегда проверяйте scanner.Err()</p>

<p><strong>Ошибка 3: Неинформативные ошибки</strong><br />
❌ return nil, errors.New(&quot;error&quot;)<br />
✅ Включайте в сообщение числа: сколько требовалось и сколько получено</p>

<p><strong>Ошибка 4: Частичный возврат при ошибке</strong><br />
❌ Возврат частично заполненного среза при недостатке данных<br />
✅ При любой ошибке возвращайте nil в качестве среза</p>

<p><strong>Ошибка 5: Модификация разбиения Scanner</strong><br />
❌ scanner.Split(bufio.ScanWords) или другие режимы<br />
✅ Используйте построчное чтение по умолчанию</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Вывод на экран</strong><br />
❌ fmt.Println для отладки внутри функции<br />
✅ Никакого вывода &mdash; только возврат результата</p>
