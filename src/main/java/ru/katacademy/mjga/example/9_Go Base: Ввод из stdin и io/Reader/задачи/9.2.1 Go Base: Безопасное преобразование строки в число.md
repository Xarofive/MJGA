<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔢 Безопасное преобразование строки в число</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться создавать надёжные функции преобразования данных с корректной обработкой ошибок и использованием стандартных библиотек Go.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">При обработке пользовательского ввода, чтении конфигураций или парсинге API-ответов часто требуется преобразовать строковые представления чисел в целочисленные значения. Прямое использование <code>strconv.Atoi</code> на некорректных данных возвращает ошибку, которую нужно явно обработать &mdash; иначе программа может аварийно завершиться. Создание безопасной обёртки с информативными ошибками &mdash; критически важный навык для построения устойчивых к сбоям систем.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию безопасного преобразования строки в целое число с детальной обработкой ошибок.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Преобразование строк в числа в Go</h4>

<p>Пакет <code>strconv</code> предоставляет функцию <code>Atoi</code> для преобразования строки в <code>int</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">value, err := strconv.Atoi("42")
if err != nil {
    // обработка ошибки
}
// value == 42</code></pre>

<p style="margin-bottom: 0;">Функция возвращает два значения: результат и ошибку. При невозможности преобразования ошибка будет типа <code>*strconv.NumError</code>.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">📊 Ошибки strconv.Atoi</h4>

<p style="margin-bottom: 0;">При невалидном вводе <code>strconv.Atoi</code> возвращает ненулевое значение типа <code>error</code>. Результат в этом случае использовать не следует &mdash; при ошибке возвращайте <code>0</code>. Текст ошибки от <code>strconv.Atoi</code> уже понятный и самодостаточный.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию SafeAtoi для безопасного преобразования строки в число:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура функции</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func SafeAtoi(s string) (int, error)</code></div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li><strong>Предобработка:</strong> удалите начальные и конечные пробелы из входной строки</li>
	<li><strong>Преобразование:</strong> используйте <code>strconv.Atoi</code> для конвертации</li>
	<li><strong>Успешный случай:</strong> при корректном вводе возвращайте (число, nil)</li>
	<li><strong>Обработка ошибок:</strong> при невозможности преобразования возвращайте <code>(0, err)</code> &mdash; текст ошибки от <code>strconv.Atoi</code> уже самодостаточен; при желании можно добавить контекст через <code>fmt.Errorf(&quot;преобразование %q: %v&quot;, s, err)</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Разрешённые импорты</h4>

<ul style="margin: 10px 0;">
	<li><code>strconv</code> &mdash; для преобразования строки в число</li>
	<li><code>fmt</code> &mdash; для форматирования ошибок</li>
	<li><code>strings</code> &mdash; для обработки строк (опционально)</li>
	<li><code>errors</code> &mdash; для работы с ошибками (опционально)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🚫 Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Возвращайте ошибки через возвращаемое значение <code>error</code> &mdash; не пытайтесь сигнализировать об ошибке аварийной остановкой программы</li>
	<li>Не используйте функции вывода (<code>fmt.Print*</code>, кроме <code>fmt.Errorf</code>)</li>
	<li>Не создавайте дополнительные функции</li>
	<li>Не указывайте <code>package</code> внутри решения</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Забыли обработать пробелы</strong><br />
❌ Прямая передача строки в <code>strconv.Atoi</code> без <code>TrimSpace</code><br />
✅ Сначала удалите пробелы: <code>strings.TrimSpace(s)</code></p>

<p><strong>Ошибка 2: Возврат ненулевого значения при ошибке</strong><br />
❌ <code>return -1, err</code> &mdash; возвращается -1 вместо 0<br />
✅ При ошибке всегда возвращайте <code>(0, error)</code></p>

<p><strong>Ошибка 3: Собственная логика преобразования</strong><br />
❌ Ручной парсинг цифр циклом по символам<br />
✅ Используйте стандартную функцию <code>strconv.Atoi</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 4: Аварийная остановка программы при ошибках</strong><br />
❌ Прерывать выполнение программы при некорректном вводе<br />
✅ Всегда возвращайте ошибку через <code>return 0, err</code></p>
