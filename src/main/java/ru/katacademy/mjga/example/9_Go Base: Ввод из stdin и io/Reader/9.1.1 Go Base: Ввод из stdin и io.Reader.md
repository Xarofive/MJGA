<h2 style="color: #34495e; margin-top: 28px;">🌊 Понимание потоков данных</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Что такое поток ввода?</h4>

<p style="margin-bottom: 10px;">Поток (stream) &mdash; это последовательность данных, которая поступает постепенно, а не вся сразу. Представьте воду из крана: вы не знаете заранее, сколько воды польётся, но можете обрабатывать её по мере поступления.</p>

<p style="margin-bottom: 10px;">В Go основной поток ввода &mdash; это <code>os.Stdin</code>, который представляет стандартный ввод программы. Когда пользователь печатает на клавиатуре или программа получает данные через pipe, они приходят именно через <code>os.Stdin</code>.</p>

<p><strong>Ключевые особенности потоков:</strong></p>

<ul style="margin: 10px 0;">
	<li><strong>Последовательность:</strong> данные читаются по порядку, нельзя &laquo;перемотать&raquo; назад</li>
	<li><strong>Буферизация:</strong> для эффективности данные часто читаются блоками, а не по байту</li>
	<li><strong>Блокировка:</strong> если данных нет, операция чтения ждёт их появления</li>
	<li><strong>EOF (End of File):</strong> специальный сигнал о том, что данные закончились</li>
</ul>
</div>

<div style="background: #f1f8e9; border: 1px solid #7cb342; border-radius: 8px; padding: 12px; margin: 12px 0;">
<h4 style="color: #558b2f; margin-top: 0;">🧩 Мини-глоссарий</h4>

<ul style="margin: 8px 0;">
	<li><strong>Источник данных</strong> &mdash; откуда читаем (например, <code>os.Stdin</code> или строка в памяти).</li>
	<li><strong>Буфер</strong> &mdash; временное хранилище прочитанных байтов.</li>
	<li><strong>Токен</strong> &mdash; кусок данных, например строка до <code>\n</code>.</li>
	<li><strong>EOF</strong> &mdash; &laquo;данные закончились&raquo;, это не ошибка.</li>
	<li><strong>io.Reader</strong> &mdash; общий &laquo;тип-способность&raquo; для чтения байтов из чего угодно.</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">1. Обзор способов ввода</h2>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🔍 Три уровня абстракции</h4>

<p style="margin-bottom: 10px;">Go предоставляет три уровня работы с вводом, каждый со своими компромиссами:</p>

<ol style="margin: 10px 0;">
	<li><strong>Высокий уровень (fmt.Scan*):</strong> автоматический парсинг типов, минимум кода, но ограниченный контроль</li>
	<li><strong>Средний уровень (bufio.Scanner):</strong> эффективное построчное чтение, настраиваемые разделители</li>
	<li><strong>Низкий уровень (bufio.Reader):</strong> полный контроль над процессом чтения, работа с байтами и рунами</li>
</ol>

<p style="margin: 0;">Выбор зависит от задачи: для простого ввода чисел подойдёт <code>fmt.Scan</code>, для обработки файлов &mdash; <code>Scanner</code>, для бинарных протоколов &mdash; <code>Reader</code>.</p>

<h3 style="color: #455a64; margin-top: 24px;">fmt.Scan* &mdash; простой старт</h3>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Как работает fmt.Scan*</h4>

<p style="margin-bottom: 10px;">Семейство функций <code>fmt.Scan*</code> предназначено для чтения форматированного ввода. Они автоматически пропускают пробельные символы (пробелы, табуляции, переводы строк) и пытаются интерпретировать следующий токен согласно типу переменной.</p>

<p style="margin-bottom: 10px;"><strong>Основные функции:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>fmt.Scan(&amp;var1, &amp;var2)</code> &mdash; читает значения, разделённые пробелами</li>
	<li><code>fmt.Scanln(&amp;var1)</code> &mdash; останавливается на переводе строки, но всё равно разбивает по пробелам</li>
	<li><code>fmt.Scanf(&quot;format&quot;, &amp;var1)</code> &mdash; использует форматную строку</li>
</ul>

<p style="margin-bottom: 10px;"><strong>Зачем <code>&amp;</code> перед переменной?</strong> Символ <code>&amp;</code> означает &laquo;адрес переменной&raquo; &mdash; он нужен, чтобы <code>fmt.Scan</code> могла записать прочитанное значение прямо в вашу переменную. Пока просто запомните этот паттерн: <code>fmt.Scan(&amp;x)</code>. Подробнее про адреса и указатели &mdash; в модуле 12.</p>

<p style="margin-bottom: 10px;"><strong>Важный нюанс:</strong> эти функции оставляют непрочитанные символы (включая \n) в буфере, что может привести к неожиданному поведению при последующем чтении.</p>

<p style="margin-bottom: 10px;"><strong>Семейство <code>fmt.Fscan*</code></strong> работает аналогично <code>fmt.Scan*</code>, но читает из произвольного <code>io.Reader</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 8px; border-radius: 4px; margin: 8px 0;">
<code class="language-go">fmt.Fscan(r, &amp;a, &amp;b)    // аналог fmt.Scan(&amp;a, &amp;b), но из r
fmt.Fscanf(r, "%d", &amp;n) // аналог fmt.Scanf("%d", &amp;n), но из r
fmt.Fscanln(r, &amp;a)      // аналог fmt.Scanln(&amp;a), но из r</code></pre>

<p style="margin: 0;">Это удобно, когда нужно прочитать несколько значений из <code>io.Reader</code> без создания <code>bufio.Scanner</code>.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import "fmt"

func main() {
var name string
var age int

    fmt.Print("Введите имя и возраст: ")
    fmt.Scan(&amp;name, &amp;age)  // Читает два значения через пробел
    fmt.Printf("Имя: %s, Возраст: %d\n", name, age)

    // Scanln останавливается на переводе строки, но читает пробело-разделённые токены
    // Для чтения всей строки с пробелами используйте Scanner или bufio.Reader
    var city string
    fmt.Print("Город (одно слово): ")
    fmt.Scanln(&amp;city)
    fmt.Printf("Город: %s\n", city)
}</code></pre>

<div style="background: #e8f5e9; border: 1px solid #66bb6a; border-radius: 4px; padding: 10px; margin-top: 10px;"><strong>📋 Ожидаемый вывод:</strong>

<pre style="margin: 5px 0; color: #2e7d32;">
Введите имя и возраст: Alice 25
Имя: Alice, Возраст: 25
Город (одно слово): Moscow
Город: Moscow</pre>
</div>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Предостережения при использовании fmt.Scan*</h4>

<ul style="margin: 10px 0;">
	<li>Оставляет символ новой строки в буфере после чтения</li>
	<li>Плохо работает с пустыми строками</li>
	<li>Не используйте разные механизмы над одним источником без единого буфера</li>
	<li>Может &quot;зависнуть&quot; при неожиданном формате ввода</li>
	<li>Scanln не читает &quot;всю строку&quot; &mdash; только токены до \n</li>
</ul>
</div>

<h3 style="color: #455a64; margin-top: 24px;">bufio.Scanner &mdash; рабочая лошадка</h3>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Архитектура Scanner</h4>

<p style="margin-bottom: 10px;"><code>bufio.Scanner</code> &mdash; это обёртка над <code>io.Reader</code>, которая эффективно читает данные блоками и разбивает их на токены. По умолчанию токены &mdash; это строки, но можно настроить другие разделители.</p>

<p style="margin-bottom: 10px;"><strong>Как работает Scanner:</strong></p>

<ol style="margin: 10px 0;">
	<li>Читает данные из источника большими блоками (буферизация)</li>
	<li>Ищет разделители токенов (по умолчанию &mdash; \n)</li>
	<li>Возвращает токен без разделителя</li>
	<li>Запоминает позицию для следующего чтения</li>
</ol>

<p style="margin-bottom: 10px;"><strong>Преимущества буферизации:</strong></p>

<ul style="margin: 10px 0;">
	<li>Меньше системных вызовов (быстрее)</li>
	<li>Эффективная работа с памятью</li>
	<li>Возможность &quot;заглянуть вперёд&quot; без потери данных</li>
</ul>

<p style="margin: 0;"><strong>Ограничение:</strong> по умолчанию максимальный размер токена &mdash; 64KB. Это защита от исчерпания памяти при чтении очень длинных строк.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"bufio"
"fmt"
"os"
)

func main() {
scanner := bufio.NewScanner(os.Stdin)

    // Чтение строк
    fmt.Println("Введите несколько строк (Ctrl+D для завершения):")

    lineCount := 0
    for scanner.Scan() {
        line := scanner.Text()
        lineCount++
        fmt.Printf("Строка %d: %s\n", lineCount, line)
    }

    // ВАЖНО: всегда проверяем ошибки после цикла
    if err := scanner.Err(); err != nil {
        fmt.Printf("Ошибка чтения: %v\n", err)
    }

}</code></pre>

<div style="background: #e3f2fd; border: 1px dashed #64b5f6; border-radius: 6px; padding: 10px; margin: 10px 0;">
<p style="margin: 0;">⌨️ <strong>Завершение ввода:</strong> macOS/Linux &mdash; <code>Ctrl+D</code>; Windows &mdash; <code>Ctrl+Z</code> затем <code>Enter</code>.</p>
</div>

<div style="background: #e8f5e9; border: 1px solid #66bb6a; border-radius: 4px; padding: 10px; margin-top: 10px;"><strong>📋 Ожидаемый вывод:</strong>

<pre style="margin: 5px 0; color: #2e7d32;">
Введите несколько строк (Ctrl+D для завершения):
Hello
Строка 1: Hello
World
Строка 2: World
^D</pre>
</div>

<p style="margin-top: 14px;">Scanner также может разбивать вход по словам вместо строк &mdash; для этого используется <code>Split(bufio.ScanWords)</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">r := strings.NewReader("Go is awesome language")
wordScanner := bufio.NewScanner(r)
wordScanner.Split(bufio.ScanWords)

for wordScanner.Scan() {
    fmt.Println(wordScanner.Text())
}
// Go
// is
// awesome
// language</code></pre>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">💡 Важно: Лимит токена Scanner</h4>

<p style="margin-bottom: 10px;">По умолчанию <code>Scanner</code> может прочитать токен размером до 64KB. Для больших строк используйте <code>Scanner.Buffer()</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">scanner := bufio.NewScanner(os.Stdin)
scanner.Buffer(nil, 1024*1024) // Увеличиваем лимит до 1MB на токен</code></pre>

<p style="margin: 0;">Для очень длинных строк или бинарных протоколов лучше использовать <code>bufio.Reader</code> с методами <code>ReadSlice</code>, <code>ReadBytes</code> или <code>ReadString</code> &mdash; они не имеют фиксированных ограничений.</p>
</div>

<h3 style="color: #455a64; margin-top: 24px;">bufio.Reader &mdash; низкоуровневый контроль</h3>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Возможности bufio.Reader</h4>

<p style="margin-bottom: 10px;"><code>bufio.Reader</code> предоставляет максимальную гибкость при чтении данных. В отличие от Scanner, который работает с токенами, Reader позволяет читать данные различными способами:</p>

<ul style="margin: 10px 0;">
	<li><strong>По байтам:</strong> <code>ReadByte()</code> &mdash; для протоколов с побайтовой структурой</li>
	<li><strong>По рунам:</strong> <code>ReadRune()</code> &mdash; корректная работа с Unicode</li>
	<li><strong>До разделителя:</strong> <code>ReadString(&#39;\n&#39;)</code>, <code>ReadBytes(&#39;\n&#39;)</code></li>
	<li><strong>Заглядывание вперёд:</strong> <code>Peek(n)</code> &mdash; посмотреть байты без их чтения</li>
	<li><strong>Пропуск данных:</strong> <code>Discard(n)</code> &mdash; эффективный пропуск байтов</li>
</ul>

<p style="margin: 0;"><strong>Когда выбрать Reader:</strong> бинарные протоколы, смешанный формат данных (текст + бинарные), необходимость точного контроля над процессом чтения.</p>
</div>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">package main

import (
"bufio"
"fmt"
"io"
"os"
"strings"
)

func main() {
reader := bufio.NewReader(os.Stdin)

    // Чтение строки целиком
    fmt.Print("Введите строку: ")
    line, err := reader.ReadString('\n')
    if err != nil &amp;&amp; err != io.EOF {
        fmt.Printf("Ошибка: %v\n", err)
        return
    }

    // Убираем \r\n или \n в конце (кроссплатформенно)
    line = strings.TrimRight(line, "\r\n")
    fmt.Printf("Вы ввели: %q\n", line)

    // Чтение по одному байту

    // Очистка буфера от остатка строки
    _, _ = reader.ReadString('\n')
}</code></pre>

<h3 style="color: #455a64; margin-top: 24px;">io.Reader &mdash; универсальная абстракция</h3>

<p><code>io.Reader</code> &mdash; это универсальный источник данных. Функции, принимающие <code>io.Reader</code>, работают одинаково с <code>os.Stdin</code> и <code>strings.NewReader(...)</code>.</p>

<div style="background: #fff8e1; border: 1px solid #ffc107; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">📚 Теория: Философия io.Reader</h4>

<p style="margin-bottom: 10px;"><code>io.Reader</code> &mdash; это название встроенного типа Go, который принимают функции вроде <code>bufio.NewScanner(...)</code>. Главное практическое следствие: одна функция, объявленная с параметром этого типа, работает и с реальным <code>os.Stdin</code>, и с <code>strings.NewReader(...)</code> в тестах. Точное внутреннее устройство <code>io.Reader</code> рассматривается в course-1 &mdash; на текущем этапе достаточно знать, что и <code>os.Stdin</code>, и <code>strings.NewReader(...)</code> оба годятся как такой источник.</p>

<p style="margin-bottom: 10px;"><strong>Почему это важно:</strong></p>

<ul style="margin: 10px 0;">
	<li><strong>Гибкость:</strong> одна функция работает и с <code>os.Stdin</code>, и с <code>strings.NewReader(...)</code> без изменений</li>
	<li><strong>Тестируемость:</strong> легко подменить реальный ввод на <code>strings.NewReader</code> в тестах</li>
	<li><strong>Переиспользование:</strong> одна функция работает с любым источником данных</li>
</ul>

<p style="margin: 0;"><strong>Золотое правило:</strong> если ваша функция читает данные, она должна принимать <code>io.Reader</code>, а не жёстко работать с <code>os.Stdin</code>.</p>
</div>

<div style="background: #fffde7; border: 1px solid #fbc02d; border-radius: 8px; padding: 12px; margin: 16px 0;">
<h4 style="color: #f57f17; margin-top: 0;">🔁 Один и тот же код &mdash; сначала просто, потом универсально</h4>

<p><strong>Шаг 1. Просто (жёстко читаем из stdin):</strong></p>

<pre style="background: #263238; color: #aed581; padding: 10px; border-radius: 4px; margin: 8px 0;">
<code class="language-go">func ReadIntsFromStdin() ([]int, error) {

<code class="language-go">package main

    "bufio"
    "fmt"
    "io"
    "os"
    "strconv"
    "strings"

func AvgFromReader(r io.Reader) (float64, error) {
    scanner := bufio.NewScanner(r)
    sum := 0.0
    count := 0

    for scanner.Scan() {
        line := strings.TrimSpace(scanner.Text())
        if line == "" { continue }

        num, err := strconv.ParseFloat(line, 64)
        if err != nil {
            return 0, fmt.Errorf("неверное число %q: %v", line, err)
        }
        sum += num
        count++
    }

    if err := scanner.Err(); err != nil {
        return 0, err
    }

    if count == 0 {
        return 0, fmt.Errorf("нет данных для вычисления среднего")
    }

    return sum / float64(count), nil
}</code></pre>

<code class="language-go">func run(r io.Reader) error {

    if err != nil { return err }
    fmt.Printf("Среднее: %.2f\n", avg)
    return nil
}

func main() {

<h2 style="color: #34495e; margin-top: 28px;">3. Типичные ошибки и как их избежать</h2>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 1: Смешивание fmt.Scan* и bufio.*</h4>

<p style="margin-bottom: 10px;">Проблема возникает из-за независимой буферизации. Решения:</p>

<ul style="margin: 10px 0;">
	<li>Используйте один подход последовательно</li>
	<li>Или создайте единый bufio.Reader и передавайте его всем функциям</li>
</ul>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// НЕПРАВИЛЬНО
var n int
fmt.Scan(&amp;n) // Оставляет \n в буфере

scanner := bufio.NewScanner(os.Stdin)
scanner.Scan() // Прочитает пустую строку!

// ПРАВИЛЬНО: используйте один метод
scanner := bufio.NewScanner(os.Stdin)
scanner.Scan()
n, _ := strconv.Atoi(scanner.Text())

// ИЛИ единый буфер для всех
br := bufio.NewReader(os.Stdin)
fmt.Fscan(br, &amp;n)     // Читаем через br
line, _ := br.ReadString('\n') // Тоже через br</code></pre>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 2: Игнорирование Scanner.Err()</h4>

<p style="margin-bottom: 10px;">Scanner не прерывает программу при ошибках &mdash; он тихо прекращает сканирование, и вы должны явно проверить результат через <code>Err()</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// НЕПРАВИЛЬНО
for scanner.Scan() {
    // обработка
}
// Ошибки потеряны!

// ПРАВИЛЬНО
for scanner.Scan() {
    // обработка
}
if err := scanner.Err(); err != nil {
    return err
}</code></pre>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 3: Неверная интерпретация EOF</h4>

<p style="margin-bottom: 10px;">EOF (End of File) &mdash; это не ошибка, а нормальный сигнал о конце данных. Обрабатывайте его отдельно:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// НЕПРАВИЛЬНО: EOF это не ошибка
line, err := reader.ReadString('\n')
if err != nil {
    return err // EOF будет считаться ошибкой
}

// ПРАВИЛЬНО
line, err := reader.ReadString('\n')
if err != nil &amp;&amp; err != io.EOF {
    return err
}
// Обрабатываем line даже при EOF
// (последняя строка может не иметь \n)</code></pre>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">❌ Ошибка 4: Непредусмотренные длинные строки</h4>

<p style="margin-bottom: 10px;">Scanner по умолчанию ограничен 64KB на токен. Для больших данных:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// НЕПРАВИЛЬНО: упадёт на строке &gt; 64KB
scanner := bufio.NewScanner(r)
for scanner.Scan() {
    // bufio.Scanner: token too long
}

// ПРАВИЛЬНО
scanner := bufio.NewScanner(r)
scanner.Buffer(nil, 1024*1024) // 1MB лимит
for scanner.Scan() {
    // Теперь обработает длинные строки
}

// АЛЬТЕРНАТИВА для очень длинных строк
reader := bufio.NewReader(r)
line, err := reader.ReadString('\n')
// ReadString не имеет лимита</code></pre>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📚 Шпаргалка</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Быстрые рецепты</h4>

<p><strong>Ментальная модель на сейчас:</strong> <code>io.Reader</code> &mdash; это любой источник, из которого можно одинаково читать байты. Благодаря этому одна функция подходит и для <code>os.Stdin</code>, и для <code>strings.NewReader(...)</code>.</p>

<h4 style="color: #2e7d32; margin-top: 0;">Упражнение 1: Чтение N строк</h4>

<p style="margin-bottom: 10px;">Напишите функцию <code>ReadNLines(r io.Reader, n int) ([]string, error)</code>, которая читает ровно N непустых строк из потока. Если строк меньше N, верните ошибку.</p>

<details style="margin-top: 10px;"><summary style="cursor: pointer; color: #388e3c; font-weight: bold;">📝 Решение</summary>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func ReadNLines(r io.Reader, n int) ([]string, error) {
    scanner := bufio.NewScanner(r)
    var lines []string

    for scanner.Scan() {
        line := strings.TrimSpace(scanner.Text())
        if line == "" {
            continue // Игнорируем пустые строки
        }

        lines = append(lines, line)
        if len(lines) == n {
            return lines, nil
        }
    }

    if err := scanner.Err(); err != nil {
        return nil, err
    }

    return nil, fmt.Errorf("недостаточно строк: нужно %d, найдено %d", n, len(lines))
}</code></pre>
</details>

<h4 style="color: #2e7d32; margin-top: 0;">Упражнение 2: Сумма чисел из потока</h4>

<p style="margin-bottom: 10px;">Напишите функцию <code>SumInts(r io.Reader) (int, error)</code>, которая суммирует все целые числа из потока, игнорируя пустые и пробельные строки.</p>

<details style="margin-top: 10px;"><summary style="cursor: pointer; color: #388e3c; font-weight: bold;">📝 Решение</summary>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func SumInts(r io.Reader) (int, error) {
    scanner := bufio.NewScanner(r)
    sum := 0
    hasNumbers := false

    for scanner.Scan() {
        line := strings.TrimSpace(scanner.Text())
        if line == "" {
            continue
        }

        num, err := strconv.Atoi(line)
        if err != nil {
            return 0, fmt.Errorf("неверное число %q: %v", line, err)
        }

        sum += num
        hasNumbers = true
    }

    if err := scanner.Err(); err != nil {
        return 0, err
    }

    if !hasNumbers {
        return 0, fmt.Errorf("нет чисел в потоке")
    }

    return sum, nil
}</code></pre>

<h2 style="color: #34495e; margin-top: 28px;">🎲 Квиз: Что выведет код?</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 20px 0;">
<h4 style="color: #e65100; margin-top: 0;">Вопрос 1: Лимит Scanner</h4>

<pre style="background: #263238; color: #aed581; padding: 8px; border-radius: 4px;">
<code class="language-go">scanner := bufio.NewScanner(strings.NewReader(strings.Repeat("A", 70000)))
if scanner.Scan() {
    fmt.Println("OK")
} else {
    fmt.Println("FAIL")
}</code></pre>

<details style="margin-top: 10px;"><summary style="cursor: pointer; color: #ef6c00; font-weight: bold;">🔍 Ответ</summary>

<p style="margin: 10px 0;"><strong>FAIL</strong> &mdash; по умолчанию Scanner имеет лимит 64KB (65536 байт), а мы пытаемся прочитать 70000 байт.</p>
</details>
</div>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 20px 0;">
<h4 style="color: #e65100; margin-top: 0;">Вопрос 2: Scanner.Err()</h4>

<pre style="background: #263238; color: #aed581; padding: 8px; border-radius: 4px;">
<code class="language-go">input := "line1\nline2"
scanner := bufio.NewScanner(strings.NewReader(input))
count := 0
for scanner.Scan() {
    count++
}
fmt.Println(count, scanner.Err() == nil)</code></pre>

<details style="margin-top: 10px;"><summary style="cursor: pointer; color: #ef6c00; font-weight: bold;">🔍 Ответ</summary>

<p style="margin: 10px 0;"><strong>2 true</strong> &mdash; прочитано 2 строки, ошибок нет (EOF не считается ошибкой в Scanner).</p>
</details>
</div>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 20px 0;">
<h4 style="color: #e65100; margin-top: 0;">Вопрос 3: fmt.Scan vs Scanner</h4>

<pre style="background: #263238; color: #aed581; padding: 8px; border-radius: 4px;">
<code class="language-go">input := "10 20\n30"
r := strings.NewReader(input)
var a, b int
fmt.Fscan(r, &amp;a, &amp;b)
scanner := bufio.NewScanner(r)
scanner.Scan()
fmt.Println(a, b, scanner.Text())</code></pre>

<details style="margin-top: 10px;"><summary style="cursor: pointer; color: #ef6c00; font-weight: bold;">🔍 Ответ</summary>

<p style="margin: 10px 0;"><strong>10 20 </strong> &mdash; <code>fmt.Fscan</code> прочитал два запрошенных значения (10 и 20) и остановился. В Reader остался <code>&quot;\n30&quot;</code>. Затем <code>bufio.Scanner</code> вызывает <code>Scan()</code>, который читает до первого <code>\n</code> &mdash; а <code>\n</code> стоит первым символом, поэтому <code>scanner.Text()</code> возвращает пустую строку <code>&quot;&quot;</code>. Итого: <code>fmt.Println(10, 20, &quot;&quot;)</code> выводит <code>10 20 </code> (с пробелом перед пустой строкой). Это наглядный пример проблемы смешивания <code>fmt.Fscan</code> и <code>bufio.Scanner</code>: непрочитанный <code>\n</code> после <code>Fscan</code> &laquo;съедает&raquo; первый вызов <code>Scan()</code>.</p>
</details>
</div>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 20px 0;">
<h4 style="color: #e65100; margin-top: 0;">Вопрос 4: Зачем функциям принимать <code>io.Reader</code>?</h4>

<p>Выберите все верные варианты.</p>

<ul>
	<li>A. Чтобы одна и та же функция работала со stdin, файлами, строками и сетью.</li>
	<li>B. Чтобы легко писать тесты через <code>strings.NewReader</code>.</li>
	<li>C. Чтобы функция была быстрее, чем при чтении напрямую из <code>os.Stdin</code>.</li>
</ul>

<details style="margin-top: 10px;"><summary style="cursor: pointer; color: #ef6c00; font-weight: bold;">🔍 Ответ</summary>

<p style="margin: 10px 0;"><strong>Правильно: A и B.</strong> Пункт C не гарантируется: скорость зависит от буферизации и задачи, а не от самого факта использования <code>io.Reader</code>.</p>
</details>
</div>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 20px 0;">
