# 7.7 Финальное задание: Создание калькулятора

## Практическая задача

Теперь, когда мы изучили все необходимые компоненты для создания калькулятора, пришло время собрать их вместе и реализовать полноценное приложение. В этом задании вам предстоит применить все знания, полученные в предыдущих лекциях.

## Шаблон для реализации

```go
package main

import (
	"bufio"
	"errors"
	"fmt"
	"os"
	"strconv"
	"strings"
)

func main() {
	// Получение ввода от пользователя
	input := getInput()

	// Валидация и парсинг входных данных
	left, operator, right, err := validateInput(input)
	if err != nil {
		panic(err)
	}

	// Валидация и парсинг операндов
	a, b, isRoman, err := validateOperands(left, right)
	if err != nil {
		panic(err)
	}

	// Выполнение операции
	result, err := calculate(a, operator, b)
	if err != nil {
		panic(err)
	}

	// Вывод результата согласно введенным данным
	err = outputResult(result, isRoman)
	if err != nil {
		panic(err)
	}
}

// Реализуйте следующие функции:

// getInput - получает ввод от пользователя

// validateInput - проверяет формат ввода и возвращает операнды и оператор

// validateOperands - проверяет операнды и возвращает их числовые значения и тип

// calculate - выполняет арифметическую операцию

// outputResult - выводит результат в соответствующей системе счисления

// romanToArabic - преобразует римское число в арабское

// arabicToRoman - преобразует арабское число в римское
```

## Требования к реализации

1. Функция `getInput` должна считывать строку ввода от пользователя.
2. Функция `validateInput` должна проверять формат ввода и возвращать левый операнд, оператор и правый операнд.
3. Функция `validateOperands` должна определять тип операндов (римские или арабские), проверять их корректность и возвращать их числовые значения.
4. Функция `calculate` должна выполнять арифметическую операцию и возвращать результат.
5. Функция `outputResult` должна выводить результат в соответствующей системе счисления.
6. Функции `romanToArabic` и `arabicToRoman` должны корректно преобразовывать числа между системами счисления.

## Критерии проверки

- Программа должна корректно обрабатывать все возможные входные данные.
- Программа должна выдавать ошибки при некорректном вводе.
- Программа должна корректно работать с римскими и арабскими числами.
- Код должен быть хорошо структурирован и читаем.
