Обработка ошибок в калькуляторе на Go

## Введение
Обработка ошибок — важный аспект разработки надежных программ. В этом разделе мы научимся корректно обрабатывать ошибки в калькуляторе, чтобы он мог информировать пользователя о некорректном вводе и прекращать работу в случае критических ошибок.

## Что такое ошибки в Go

В языке Go ошибки представлены типом `error`. Встроенный интерфейс выглядит следующим образом:

```go
type error interface {
    Error() string
}
```

Это позволяет нам получать удобные текстовые сообщения об ошибках.

## Пример функции с обработкой ошибок

Рассмотрим пример функции, проверяющей корректность ввода данных:

```go
func validateInput(parts []string) (string, string, string, error) {
    if len(parts) != 3 {
        return "", "", "", errors.New("неправильный формат ввода: ожидалось 'число оператор число'")
    }

    leftOperand := parts[0]
    operator := parts[1]
    rightOperand := parts[2]

    if operator != "+" && operator != "-" && operator != "*" && operator != "/" {
        return "", "", "", errors.New("неподдерживаемый оператор")
    }

    return leftOperand, operator, rightOperand, nil
}
```

## Как использовать обработку ошибок в программе

Рекомендуемый подход:

1. В функциях, где могут возникать ошибки, возвращайте значение типа `error` последним аргументом.
2. Проверяйте ошибки сразу после вызова функции и, при необходимости, завершайте выполнение программы с помощью `panic`.

Пример:

```go
left, operator, right, err := validateInput(parts)
if err != nil {
    panic(err)
}
```

## Использование panic

Функция `panic()` завершает работу программы и выводит сообщение об ошибке:

```go
panic("сообщение об ошибке")
```

Используйте `panic` только при критических ошибках, после которых невозможно дальнейшее выполнение программы.

## Итоговая реализация обработки ошибок в калькуляторе

Стандартный шаблон обработки ошибок:

```go
func main() {
    input := getInput()
    parts := strings.Fields(input)

    left, operator, right, err := validateInput(parts)
    if err != nil {
        panic(err)
    }

    a, b, isRoman, err := validateOperands(left, right)
    if err != nil {
        panic(err)
    }

    result, err := calculate(a, operator, b)
    if err != nil {
        panic(err)
    }

    outputResult(result, isRoman)
}
```
