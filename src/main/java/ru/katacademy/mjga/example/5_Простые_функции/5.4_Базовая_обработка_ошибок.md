# Базовая обработка ошибок в Go

В программировании ошибки неизбежны, и важно правильно их обрабатывать. В Go обработка ошибок реализована просто и понятно, что помогает делать программы надёжными и понятными.

## Как обрабатываются ошибки в Go

В Go ошибки представлены встроенным типом данных `error`. Обычно функции возвращают значение ошибки вместе с основным результатом:

```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("деление на ноль")
    }
    return a / b, nil
}
```

## Что происходит в примере:

- Мы создаём функцию `divide`, которая принимает два числа.
- Если второе число равно нулю, функция возвращает ошибку с описанием проблемы.
- Если ошибки нет, возвращаем результат и значение ошибки как `nil`.

## Обработка ошибки при вызове функции

Вот пример вызова и обработки ошибок:

```go
package main

import (
    "errors"
    "fmt"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("деление на ноль")
    }
    return a / b, nil
}

func main() {
    result, err := divide(10, 0)

    if err != nil {
        fmt.Println("Произошла ошибка:", err)
        return
    }

    fmt.Println("Результат:", result)
}
```

### Что происходит при выполнении:

- Вызвали функцию `divide(10, 0)`.
- Проверили наличие ошибки через переменную `err`.
- Если ошибка есть, выводим сообщение и завершаем выполнение функции с помощью `return`.
- Если ошибки нет, выводим результат.

Вывод программы будет:
```
Произошла ошибка: деление на ноль
```

## Важные моменты при обработке ошибок:

- Всегда проверяй ошибки сразу после вызова функции.
- Используй чёткие и понятные сообщения об ошибках.
- Никогда не игнорируй ошибки без веских причин.
