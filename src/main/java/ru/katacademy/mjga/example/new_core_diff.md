| --- | --- | --- |
| 1. Переход от Base к Core | Повторение базы и качество кода | Декомпозиция, небольшие чистые методы, понятные имена, Code Style, чтение и рефакторинг существующего кода |
|  | Полная система типов Java | `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`, диапазоны типов, литералы, суффиксы `L` и `f` |
|  | Преобразования и вычисления | Неявные и явные преобразования, numeric promotion, переполнение, потеря точности, целочисленное деление |
|  | Пакеты и структура кода | `package`, `import`, полное имя класса, структура исходников, разделение классов по пакетам |
| 2. Инкапсуляция и объектная модель | Классы и состояние | Поля и методы экземпляра, состояние объекта, поведение объекта, ответственность класса |
|  | Инкапсуляция | `private`, `public`, package-private, getters и setters, валидация, объектные инварианты |
|  | Конструкторы | Объявление конструкторов, `this`, перегрузка конструкторов, цепочки конструкторов, создание валидного объекта |
|  | Члены класса и композиция | `static`, `final`, константы, композиция объектов, передача зависимостей через конструктор |
| 3. Объектные контракты | Класс `Object` | Базовая роль `Object`, ссылочное и логическое равенство, методы общего контракта объектов |
|  | Представление объекта | Переопределение `toString`, читаемый вывод состояния, использование при диагностике |
|  | Равенство объектов | `equals`, `hashCode`, требования контракта, согласованность методов, объекты в `HashSet` и ключи в `HashMap` |
|  | Перечисления | `enum`, поля и методы перечисления, конструктор `enum`, `switch` по перечислению, статусы и ограниченные наборы значений |
| 4. Наследование, полиморфизм и интерфейсы | Наследование | `extends`, отношение is-a, наследование состояния и поведения, `protected`, `super`, конструкторы наследников |
|  | Переопределение и полиморфизм | `@Override`, динамический полиморфизм, тип ссылки и тип объекта, upcasting, `instanceof` и pattern matching базово |
|  | Абстрактные классы | `abstract`, абстрактные методы, общее состояние и шаблон поведения |
|  | Интерфейсы | Контракт поведения, реализация нескольких интерфейсов, default и static методы базово, зависимость от абстракции |
|  | Выбор связи между классами | Композиция против наследования, слабая связанность, простое разделение `model`, `service`, `repository` |
| 5. Collections Framework и Generics | Обобщения | Generic-классы и методы, параметр типа `<T>`, type safety, ограничения типов базово, инвариантность generics |
|  | Классы-обёртки | Wrapper types, autoboxing, unboxing, сравнение и преобразование обёрток |
|  | Интерфейсы коллекций | `Collection`, `List`, `Set`, `Queue`, `Map`, выбор коллекции под задачу |
|  | Реализации коллекций | `ArrayList`, `LinkedList` обзорно, `HashSet`, `LinkedHashSet`, `TreeSet`, `HashMap`, `LinkedHashMap`, `TreeMap`, `ArrayDeque` |
|  | Обход и изменение коллекций | Enhanced `for`, `Iterator`, безопасное удаление, неизменяемые коллекции базово |
|  | Сравнение и сортировка | `Comparable`, `Comparator`, цепочки компараторов, `Collections` и базовые алгоритмы |
| 6. Исключения и Files API | Модель исключений | `Throwable`, `Exception`, `RuntimeException`, `Error`, checked и unchecked исключения |
|  | Обработка ошибок | `try-catch-finally`, multi-catch, `throw`, `throws`, распространение исключений, сохранение причины ошибки |
|  | Проектирование исключений | Выбор стандартного исключения, пользовательские исключения, валидация аргументов, исключения на границах слоёв |
|  | Управление ресурсами | `AutoCloseable`, try-with-resources, корректное закрытие ресурсов |
|  | Файлы и пути | `Path`, `Files`, файлы и директории, проверка существования, создание, копирование, перемещение и удаление |
|  | Текстовые данные | Чтение и запись текста, `BufferedReader`, `BufferedWriter`, кодировки, CSV и простые текстовые форматы |
| 7. Lambda и Stream API | Функциональные интерфейсы | SAM-интерфейсы, `Predicate`, `Function`, `Consumer`, `Supplier`, собственные функциональные интерфейсы |
|  | Лямбда-выражения | Синтаксис lambda, захват переменных, effectively final, method reference |
|  | Основы Stream API | Создание stream, pipeline, ленивые вычисления, промежуточные и терминальные операции |
|  | Обработка данных | `filter`, `map`, `flatMap` базово, `sorted`, `distinct`, `limit`, `skip`, `reduce` |
|  | Сбор результата | `toList`, `Collectors.toSet`, `toMap`, `joining`, `groupingBy`, `partitioningBy`, статистические collectors |
|  | `Optional` | Создание и получение значения, `map`, `filter`, `orElse`, `orElseGet`, использование как результата поиска |
| 8. SQL и JDBC | Реляционная модель | Таблицы, строки, столбцы, типы данных, первичный и внешний ключ, связи между таблицами |
|  | Основы SQL | `SELECT`, `WHERE`, `ORDER BY`, агрегатные функции, `GROUP BY`, `INSERT`, `UPDATE`, `DELETE` |
|  | Связи и выборки | `JOIN`, алиасы, выборка связанных данных, базовое проектирование таблиц |
|  | Подключение через JDBC | JDBC driver, `Connection`, URL подключения, открытие и закрытие соединения |
|  | Выполнение запросов | `PreparedStatement`, параметры запроса, защита от SQL injection, `ResultSet`, преобразование строки БД в объект |
|  | Хранение данных приложения | DAO и Repository, CRUD-операции, отделение SQL от бизнес-логики |
|  | Транзакции | `commit`, `rollback`, auto-commit, атомарность нескольких операций базово |
| 9. Базовая многопоточность | Модель выполнения | Процесс и поток, последовательное и параллельное выполнение, жизненный цикл потока |
|  | Создание задач | `Thread`, `Runnable`, `Callable`, запуск и ожидание завершения |
|  | Пулы потоков | `ExecutorService`, фиксированный пул, отправка задач, корректное завершение executor |
|  | Асинхронный результат | `Future`, получение результата, обработка исключений, отмена и timeout базово |
|  | Общие изменяемые данные | Race condition, visibility базово, критическая секция, `synchronized` |
|  | Потокобезопасность | Immutability, минимизация общего состояния, atomic-классы и concurrent collections обзорно |
