<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Удаление дубликатов с сохранением порядка</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать map как множество (set) для эффективной проверки уникальности элементов за O(1), комбинируя map с срезом для сохранения порядка первого появления.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Удаление дубликатов с сохранением порядка &mdash; базовая операция в обработке данных. От анализа логов до очистки пользовательского ввода, везде встречается задача: оставить уникальные элементы в том порядке, как они появились впервые. Наивное решение через вложенные циклы работает за O(n&sup2;), что неприемлемо для больших данных. Map позволяет решить эту задачу за O(n) &mdash; проверка существования элемента занимает константное время.</p>

<p style="margin: 0;"><strong>Задача:</strong> получить список уникальных чисел в порядке их первого появления в исходной последовательности.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Map как множество (set)</h4>

<p>В Go нет встроенного типа set (множество), но map можно использовать для этой цели:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Map с bool значениями работает как set
seen := make(map[int]bool)

// Проверка наличия элемента
if seen[42] {
    fmt.Println("Элемент 42 уже встречался")
}

// Добавление элемента в set
seen[42] = true</code></pre>

<p style="margin-bottom: 0;"><strong>Преимущество:</strong> Проверка <code>seen[x]</code> выполняется мгновенно, в отличие от поиска в срезе, где нужно перебирать элементы один за другим.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Паттерн: Map + Slice для сохранения порядка</h4>

<p>Классический приём для удаления дубликатов с сохранением порядка:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">seen := make(map[int]bool)    // Для проверки "видели ли раньше"
unique := []int{}             // Для сохранения порядка

for each element {
    if !seen[element] {       // Элемент встретился впервые
        seen[element] = true  // Помечаем как виденный
        unique = append(unique, element) // Добавляем в результат
    }
}</code></pre>

<p style="margin-bottom: 0;"><strong>Инвариант:</strong> Срез unique содержит ровно те элементы, для которых seen[element] == true, в порядке их добавления.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">📊 Сложность алгоритма</h4>

<p>Сравнение двух подходов:</p>

	<tbody>
		<tr style="background: #f5f5f5;">

</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте программу для удаления дубликатов с сохранением порядка:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Импортирует только пакет <code>&quot;fmt&quot;</code></li>
	<li>Содержит функцию <code>main()</code></li>
	<li>Использует <code>map[int]bool</code> для отслеживания уникальности</li>
	<li>Использует срез <code>[]int</code> для сохранения порядка</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📥 Формат входных данных</h4>

<p><strong>Первая строка:</strong> целое число N (0 &le; N &le; 100000) &mdash; количество элементов</p>

<p><strong>Вторая строка:</strong> N целых чисел через пробел (каждое в диапазоне int: -2&sup3;&sup1; до 2&sup3;&sup1;-1)</p>

<p><strong>Особые случаи:</strong></p>

<ul style="margin: 10px 0;">
	<li>При N=0 вторая строка может отсутствовать или быть пустой</li>
	<li>Числа могут быть отрицательными, нулевыми, очень большими</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📤 Формат выходных данных</h4>

<p>Одна строка: уникальные числа в порядке первого появления, разделённые одним пробелом.</p>

<p><strong>Требования к формату:</strong></p>

<ul style="margin: 10px 0;">
	<li>Между числами ровно один пробел</li>
	<li>Нет лишнего пробела в конце строки</li>
	<li>Строка завершается переносом строки</li>
	<li>При N=0 допустим пустой вывод или одна пустая строка</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Важные детали</h4>

<ul style="margin: 10px 0;">
	<li>Сохраняйте порядок первого появления (не сортируйте!)</li>
	<li>Каждое уникальное число встречается в выводе ровно один раз</li>
	<li>Если число повторяется, берётся его первое вхождение</li>
	<li>Программа должна работать за O(n) времени и O(n) памяти</li>
	<li>Используйте map для проверки, срез для порядка</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Использовать только срез без map (O(n&sup2;))</strong><br />
❌ Проверять наличие элемента циклом по срезу<br />
✅ Используйте map для O(1) проверки существования</p>

<p><strong>Ошибка 2: Использовать только map без среза</strong><br />
❌ <code>for num := range seen { ... }</code> &mdash; порядок случайный!<br />
✅ Map для проверки, срез для порядка</p>

<p><strong>Ошибка 3: Сортировать результат</strong><br />
❌ <code>sort.Ints(result)</code> &mdash; теряется порядок появления<br />
✅ Сохраняйте порядок добавления в срез, не сортируйте</p>

<p><strong>Ошибка 4: Добавлять лишний пробел в конце</strong><br />
❌ <code>fmt.Print(num, &quot; &quot;)</code> для каждого элемента<br />
✅ Проверяйте <code>if i &gt; 0</code> перед выводом пробела</p>

<p><strong>Ошибка 5: Не инициализировать map</strong><br />
❌ <code>var seen map[int]bool</code> &mdash; запись в nil map приведёт к ошибке времени выполнения<br />
✅ <code>seen := make(map[int]bool)</code> или <code>map[int]bool{}</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Забыть добавить в срез result</strong><br />
❌ Только <code>seen[num] = true</code> без <code>append</code><br />
✅ После проверки !seen[num] выполните оба действия</p>
</div>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказки по реализации</h4>

<p><strong>Зачем нужны и map, и срез одновременно?</strong><br />
Map даёт быстрый ответ &quot;видели ли мы этот элемент раньше?&quot;, но не сохраняет порядок. Срез сохраняет порядок добавления, но поиск в нём медленный. Комбинация даёт лучшее из обоих миров: O(1) проверка + сохранение порядка.</p>

<p><strong>Как избежать лишнего пробела в конце?</strong><br />
Классический приём: выводите пробел перед числом, начиная со второго элемента. Условие <code>if i &gt; 0</code> перед <code>fmt.Print(&quot; &quot;)</code> решает проблему.</p>

<p><strong>Что делать при N=0?</strong><br />
Если элементов нет, срез result будет пустым. Цикл вывода не выполнится ни разу, и программа завершится без вывода или с одной пустой строкой &mdash; оба варианта допустимы.</p>

<p style="margin-bottom: 0;"><strong>Какой тип использовать для map?</strong><br />
Используйте <code>map[int]bool</code> &mdash; это простой и читаемый способ проверять, встречался ли элемент: <code>if seen[x] { ... }</code>.</p>
