<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📊 Частотный анализ категорий</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Освоить работу с отображениями (maps) для подсчёта частоты встречаемости элементов, научиться выводить результаты в отсортированном виде.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">Подсчёт частоты элементов &mdash; базовая операция в анализе данных, статистике и обработке текстов. От анализа логов и метрик до обработки естественного языка &mdash; везде требуется быстро определить, сколько раз встречается каждый уникальный элемент в наборе данных.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать систему подсчёта количества вхождений каждой категории с выводом результатов в лексикографическом порядке.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Maps для подсчёта частоты</h4>

<p>Maps в Go идеально подходят для подсчёта количества вхождений:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Создание map для счётчиков
counts := make(map[string]int)

// Увеличение счётчика при встрече элемента
category := "apple"
counts[category]++  // Работает даже если ключа нет (0 + 1 = 1)

// Проверка существования
if count, exists := counts[category]; exists {
    fmt.Printf("%s встретился %d раз\n", category, count)
}</code></pre>

<p style="margin-bottom: 0;">Инкремент <code>++</code> автоматически работает с нулевым значением для новых ключей.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Чтение последовательности элементов</h4>

<p>Для подсчёта частоты нужно прочитать все элементы и обработать каждый:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">var n int
fmt.Scan(&amp;n)  // Количество элементов

counts := make(map[string]int)

for i := 0; i &lt; n; i++ {
    var category string
    fmt.Scan(&amp;category)

    // Увеличиваем счётчик для этой категории
    counts[category]++
}</code></pre>

<p style="margin-bottom: 0;">Каждый элемент увеличивает соответствующий счётчик на единицу.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ffb74d; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔒 Сортировка и форматированный вывод</h4>

<p>Для вывода в алфавитном порядке нужно отсортировать ключи:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">import "sort"

// Собираем ключи
var categories []string
for category := range counts {
    categories = append(categories, category)
}

// Сортируем лексикографически
sort.Strings(categories)

// Выводим в формате "категория количество"
for _, category := range categories {
    fmt.Printf("%s %d\n", category, counts[category])
}</code></pre>

<p style="margin-bottom: 0;">Важно помнить про регистр: заглавные буквы идут перед строчными при лексикографической сортировке.</p>
</div>

<div style="background: #fce4ec; border: 1px solid #f06292; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c2185b; margin-top: 0;">⚡ Обработка граничных случаев</h4>

<p>При работе с пустыми данными важен специальный вывод:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Проверка на пустоту
if n == 0 {
    fmt.Println("EMPTY")
    return
}

// Или после обработки
if len(counts) == 0 {
    fmt.Println("EMPTY")
    return
}</code></pre>

<p style="margin-bottom: 0;">Всегда обрабатывайте случай отсутствия данных явно.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте программу для частотного анализа категорий:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Содержит функцию <code>main</code></li>
	<li>Использует <code>map[string]int</code> для подсчёта частоты</li>
	<li>Может импортировать: <code>fmt</code>, <code>bufio</code>, <code>os</code>, <code>strings</code>, <code>strconv</code>, <code>sort</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Формат ввода</h4>

<p><strong>Первая строка:</strong> целое число n (количество элементов, 0 &le; n &le; 200000)</p>

<p><strong>Следующие n строк:</strong> по одной категории в каждой строке (строка без пробелов)</p>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Формат вывода</h4>

<ul style="margin: 10px 0;">
	<li><strong>Если n = 0:</strong> вывести <code>EMPTY</code></li>
	<li><strong>Иначе:</strong> вывести каждую уникальную категорию и количество её вхождений</li>
	<li>Формат строки: <code>&lt;категория&gt; &lt;количество&gt;</code></li>
	<li>Категории выводятся в лексикографическом порядке</li>
	<li>Регистр букв имеет значение (Food &ne; food)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📊 Поведение программы</h4>

<ul style="margin: 10px 0;">
	<li>Подсчитать, сколько раз встречается каждая категория</li>
	<li>Учитывать регистр (заглавные и строчные буквы &mdash; разные категории)</li>
	<li>Сортировать категории лексикографически перед выводом</li>
	<li>Обработать случай пустого ввода</li>
	<li>Программа должна обрабатывать до 200 000 элементов за разумное время (&le; 10 секунд)</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неправильный порядок вывода</strong><br />
❌ Вывод в порядке встречаемости или случайном порядке<br />
✅ Обязательная лексикографическая сортировка через <code>sort.Strings()</code></p>

<p><strong>Ошибка 2: Игнорирование регистра</strong><br />
❌ <code>strings.ToLower(category)</code> перед подсчётом<br />
✅ Сохраняйте оригинальный регистр: <code>Food</code> и <code>food</code> &mdash; разные ключи</p>

<p><strong>Ошибка 3: Перезапись вместо инкремента</strong><br />
❌ <code>counts[category] = 1</code> &mdash; теряется предыдущее значение<br />
✅ <code>counts[category]++</code> &mdash; увеличивает счётчик</p>

<p><strong>Ошибка 4: Неверная обработка пустого ввода</strong><br />
❌ Вывод пустой строки или ничего<br />
✅ При <code>n == 0</code> обязательно выводите <code>EMPTY</code></p>

<p><strong>Ошибка 5: Лишние переводы строк или отсутствие перевода</strong><br />
❌ <code>fmt.Print(category, &quot; &quot;, counts[category])</code> &mdash; нет перевода строки в конце<br />
✅ <code>fmt.Println(category, counts[category])</code> или <code>fmt.Printf(&quot;%s %d\n&quot;, ...)</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Забытая инициализация map</strong><br />
❌ <code>var counts map[string]int</code> без <code>make</code> &mdash; запись в nil map приведёт к ошибке времени выполнения<br />
✅ <code>counts := make(map[string]int)</code> &mdash; правильная инициализация</p>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказки для размышления</h4>

<p><strong>Вопрос 1:</strong> Почему <code>counts[category]++</code> работает для новых ключей?<br />
<em>Go инициализирует нулевое значение (0) для несуществующих ключей типа int.</em></p>

<p><strong>Вопрос 2:</strong> Как определить самую частую категорию?<br />
<em>После подсчёта можно пройти по map и найти максимальное значение счётчика.</em></p>

<p><strong>Вопрос 3:</strong> Что эффективнее: подсчёт через map или сортировка и группировка?<br />
<em>Map даёт O(1) для каждой операции, сортировка требует O(n log n) &mdash; map эффективнее.</em></p>

<p style="margin-bottom: 0;"><strong>Вопрос 4:</strong> Как работает лексикографический порядок с разными регистрами?<br />
<em>В UTF-8 коды заглавных букв (65-90) меньше кодов строчных (97-122), поэтому &quot;A&quot; &lt; &quot;a&quot;.</em></p>
