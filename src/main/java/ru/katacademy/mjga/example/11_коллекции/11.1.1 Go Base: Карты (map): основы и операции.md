<h1 style="text-align: center; color: #2c3e50;">Карты (map): основы и операции</h1>

<p style="text-align: center; color: #666; margin-bottom: 30px;">Изучаем ассоциативные массивы Go &mdash; эффективное хранилище пар ключ-значение для быстрого поиска и подсчёта.</p>

<h3 style="color: #2e7d32; margin-top: 0;">📌 TL;DR &mdash; Суть за 30 секунд</h3>

<ul>
	<li>map[K]V &mdash; ассоциативный массив: ключи типа K, значения типа V</li>
	<li>Ключи должны быть сравнимыми (==): string, int, bool &mdash; да; slice, map &mdash; нет</li>
	<li>Нулевое значение map &mdash; nil; чтение из nil безопасно (даёт zero value), запись приведёт к ошибке времени выполнения</li>
	<li>Создание: var m map[K]V (nil), make(map[K]V), литерал map[K]V{...}</li>
	<li>Операции: m[k]=v (запись), v:=m[k] (чтение), v,ok:=m[k] (проверка), delete(m,k)</li>
	<li>range проходит в произвольном порядке &mdash; никаких гарантий последовательности</li>
	<li>Чтение, запись и удаление выполняются за приблизительно константное время &mdash; практически не зависят от размера map</li>
	<li>len(m) возвращает количество пар ключ-значение; len(nil map) == 0</li>
</ul>

<h2 style="color: #34495e; margin-top: 30px;">Что такое map и зачем она нужна</h2>

<p>Представьте телефонный справочник: вы ищете человека по имени и мгновенно находите его номер. Или словарь: по слову находите определение. Map в Go работает именно так &mdash; это структура данных, которая связывает ключи со значениями и позволяет быстро находить нужное значение по ключу.</p>

<p><strong>Почему не срез?</strong> В срезе для поиска элемента нужно пройти все элементы по одному. В map поиск происходит за приблизительно константное время &mdash; практически не зависит от размера коллекции (в среднем). Если у вас миллион записей, срез будет проверять в среднем 500 000 элементов, а map найдёт практически мгновенно.</p>

<p><strong>Когда использовать map:</strong></p>

<ul>
	<li>Подсчёт частоты элементов (сколько раз встречается слово)</li>
	<li>Кеширование результатов вычислений</li>
	<li>Группировка данных по категориям</li>
	<li>Быстрый поиск по уникальному идентификатору</li>
	<li>Проверка наличия элемента в множестве</li>
</ul>

<h2 style="color: #34495e; margin-top: 30px;">Требования к ключам: что может быть ключом</h2>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 15px; margin: 20px 0;">
<p style="margin: 0;"><strong>💡 Почему срезы не могут быть ключами?</strong> В Go существует понятие &laquo;сравнимый тип&raquo; &mdash; тип, два значения которого можно сравнить оператором <code>==</code>. Срезы (а также map и функции) не сравнимы в этом смысле: компилятор не разрешает писать <code>s1 == s2</code> для двух срезов (кроме сравнения с <code>nil</code>). Поскольку map&#39;у нужно уметь сравнивать ключи, чтобы найти нужную пару, она требует от ключа поддержки <code>==</code>. Срезы такой поддержки не имеют &mdash; поэтому компилятор отклонит <code>make(map[[]int]string)</code> ещё до запуска программы.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Три способа создания карт</h2>

<p>В Go есть три основных способа создать map, и каждый подходит для своей ситуации. Понимание различий поможет вам выбрать правильный подход.</p>

<div style="background: #f8f9fa; border-left: 4px solid #2196F3; padding: 15px; margin: 20px 0;">
<h4 style="color: #1565c0; margin-top: 0;">Способ 1: Объявление переменной (nil map)</h4>

<p>Когда вы объявляете map без инициализации, она равна nil. Это особое состояние: читать можно (получите нулевое значение), а записывать нельзя.</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">var m map[string]int
fmt.Println(m == nil)  // true
fmt.Println(m["key"])   // 0 - чтение безопасно, вернёт нулевое значение
// m["key"] = 1         // ошибка времени выполнения: запись в nil map запрещена</code></pre>

<p><strong>Когда использовать:</strong> Когда map может остаться неинициализированной (например, в структуре, где она заполняется только при определённых условиях). Также полезно для отложенной инициализации &mdash; сначала проверяете условие, потом создаёте map.</p>

<p><strong>Частая ошибка:</strong> Забыть инициализировать map перед записью &mdash; программа аварийно завершится с ошибкой времени выполнения.</p>
</div>

<div style="background: #f8f9fa; border-left: 4px solid #4CAF50; padding: 15px; margin: 20px 0;">
<h4 style="color: #2e7d32; margin-top: 0;">Способ 2: make (пустая инициализированная map)</h4>

<p>Функция make создаёт инициализированную пустую map, готовую к использованию. Это самый частый способ создания map.</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">m := make(map[string]int)        // без начальной ёмкости
m2 := make(map[string]int, 1000) // с подсказкой о размере

m["USD"] = 1  // теперь можно записывать
m["EUR"] = 2
fmt.Println(len(m))  // 2</code></pre>

<p><strong>Про ёмкость:</strong> Второй параметр (1000) &mdash; это подсказка рантайму (среде выполнения) о предполагаемом размере. Если вы знаете, что будет около 1000 элементов, указание этого числа помогает Go подготовить структуру под ожидаемое количество элементов. Это подсказка, а не жёсткое ограничение &mdash; map может расти дальше.</p>

<p><strong>Когда использовать:</strong> Практически всегда, когда нужна пустая map для последующего заполнения. Если знаете примерный размер &mdash; указывайте его как подсказку.</p>
</div>

<div style="background: #f8f9fa; border-left: 4px solid #FF9800; padding: 15px; margin: 20px 0;">
<h4 style="color: #e65100; margin-top: 0;">Способ 3: Литерал (с начальными значениями)</h4>

<p>Литерал позволяет создать map сразу с данными. Это как создать и заполнить map в одном выражении.</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">rates := map[string]int{
    "USD": 1,
    "EUR": 2,
    "GBP": 3,
}

// Пустой литерал эквивалентен make
empty := map[string]int{}
fmt.Println(len(rates), len(empty))  // 3 0</code></pre>

<p><strong>Когда использовать:</strong> Для статических данных, известных на момент написания кода &mdash; конфигурации, константные таблицы соответствий, начальные значения счётчиков. Удобно для тестов, где нужны предзаполненные данные.</p>

<p><strong>Синтаксическая тонкость:</strong> Последняя запятая после &quot;GBP&quot;: 3 обязательна, если закрывающая скобка на новой строке. Это упрощает добавление новых элементов и улучшает читаемость diff в системах контроля версий.</p>
</div>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #856404; margin-top: 0;">📌 Динамические ключи &mdash; суть map</h4>

<p>В практических задачах модуля 11 важно использовать map с переменными-ключами, а не только со строковыми литералами. Если вы пишете <code>phonebook[&quot;Alice&quot;] = phone</code> исключительно с фиксированными именами, вы не используете главное преимущество map &mdash; возможность быстро находить значение по любому ключу, известному только в рантайме.</p>

<p><strong>Мини-пример (динамический ключ):</strong></p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">for _, name := range names {
    counts[name]++  // m[var]: ключ — переменная, не литерал
}</code></pre>

<p><strong>Почему так:</strong> если все ключи известны на этапе написания кода как литералы, обычно достаточно нескольких отдельных переменных, и map не даёт реального преимущества. M11 учит именно динамической работе с ключами &mdash; поэтому в каждой практической задаче должно быть хотя бы одно использование map&#39;а с переменной-ключом в логике решения.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Основные операции: чтение, запись, проверка</h2>

<p>Map предоставляет простой синтаксис для базовых операций, но есть важные нюансы, которые нужно понимать для корректной работы.</p>

<h3 style="color: #34495e; margin-top: 20px;">Запись и чтение значений</h3>

<p>Запись в map выглядит как присваивание элементу массива, но работает иначе &mdash; создаёт или перезаписывает пару ключ-значение:</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">scores := make(map[string]int)

// Запись новых значений
scores["Alice"] = 95
scores["Bob"] = 87

// Перезапись существующего значения
scores["Alice"] = 98  // Было 95, стало 98

// Простое чтение
alice := scores["Alice"]     // 98
charlie := scores["Charlie"] // 0 - ключа нет, вернулось нулевое значение</code></pre>

<p><strong>Критически важно:</strong> При чтении несуществующего ключа Go не выдаёт ошибку, а возвращает нулевое значение типа (0 для int, &quot;&quot; для string, false для bool). Это удобно, но может привести к логическим ошибкам.</p>

<h3 style="color: #34495e; margin-top: 20px;">Ok-идиома: как отличить &quot;нет ключа&quot; от &quot;нулевое значение&quot;</h3>

<p>Представьте систему голосования: если за вариант &quot;Yes&quot; проголосовало 0 человек, это отличается от ситуации, когда варианта &quot;Yes&quot; вообще нет в бюллетене. Ok-идиома решает эту проблему:</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">votes := map[string]int{"Yes": 0, "No": 5}

// Проблема: не можем отличить отсутствие от нуля
if votes["Yes"] == 0 {
    // Это может означать:
    // 1. За "Yes" никто не голосовал (0 голосов)
    // 2. "Yes" вообще нет в списке вариантов
}

// Решение: ok-идиома
value, exists := votes["Yes"]
if exists {
    fmt.Printf("За Yes проголосовало: %d\n", value)  // 0
} else {
    fmt.Println("Варианта Yes нет в голосовании")
}

// Проверка отсутствующего ключа
value, exists = votes["Maybe"]
if !exists {
    fmt.Println("Maybe отсутствует")  // Выведется это
}</code></pre>

<p><strong>Синтаксический сахар:</strong> Часто проверку объединяют с условием для компактности:</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">if score, ok := scores["Bob"]; ok {
    fmt.Printf("Bob набрал %d баллов\n", score)
}</code></pre>

<div style="background: #fff3cd; border: 1px solid #ffeaa7; border-radius: 8px; padding: 15px; margin: 20px 0;">
<h4 style="color: #856404; margin-top: 0;">⚠️ Распространённая ошибка новичков</h4>

<p>Многие начинающие полагаются на нулевое значение вместо ok-идиомы. Это работает только если нулевое значение никогда не является валидным в вашей логике. Например, для множества (set) на основе map[string]bool это безопасно, но для счётчиков или флагов &mdash; опасно.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Удаление элементов и подсчёт размера</h2>

<p>Операции delete и len работают предсказуемо и безопасно, что делает работу с map удобной.</p>

<h3 style="color: #34495e; margin-top: 20px;">Функция delete: безопасное удаление</h3>

<p>delete удаляет пару ключ-значение из map. Важная особенность &mdash; она безопасна: не приводит к ошибке времени выполнения при удалении несуществующего ключа или работе с nil map.</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">users := map[int]string{
    1: "Alice",
    2: "Bob",
    3: "Charlie",
}

// Удаление существующего ключа
delete(users, 2)  // Bob удалён

// Безопасное удаление несуществующего ключа
delete(users, 999)  // Ничего не происходит, без ошибки

// delete на nil map тоже безопасен
var nilMap map[string]int
delete(nilMap, "key")  // Безопасно, без ошибки</code></pre>

<p><strong>Почему это важно:</strong> Вам не нужно проверять наличие ключа перед удалением. Это упрощает код и делает его более устойчивым к ошибкам.</p>

<h3 style="color: #34495e; margin-top: 20px;">Функция len: количество элементов</h3>

<p>len возвращает количество пар ключ-значение в map. Для nil map возвращает 0, что логично и удобно:</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">m := map[string]int{"a": 1, "b": 2}
fmt.Println(len(m))  // 2

var nilMap map[string]int
fmt.Println(len(nilMap))  // 0 - nil map имеет длину 0

empty := make(map[string]int)
fmt.Println(len(empty))  // 0 - пустая map тоже</code></pre>

<h2 style="color: #34495e; margin-top: 30px;">Итерация с range: порядок не гарантирован</h2>

<p>Одна из самых важных особенностей map в Go &mdash; отсутствие гарантированного порядка при итерации. Это сделано намеренно, чтобы разработчики не полагались на порядок, который может измениться.</p>

<h3 style="color: #34495e; margin-top: 20px;">Почему порядок случайный?</h3>

<p>Map оптимизирована для быстрого поиска по ключу, а не для сохранения порядка. Порядок элементов при итерации не определён языком и может различаться при разных запусках программы.</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">capitals := map[string]string{
    "Russia": "Moscow",
    "France": "Paris",
    "Japan":  "Tokyo",
    "Brazil": "Brasilia",
}

// Первый обход
fmt.Println("Первый обход:")
for country, capital := range capitals {
    fmt.Printf("  %s: %s\n", country, capital)
}

// Второй обход - порядок может отличаться!
fmt.Println("\nВторой обход:")
for country, capital := range capitals {
    fmt.Printf("  %s: %s\n", country, capital)
}</code></pre>

<p><strong>Важное следствие:</strong> Никогда не пишите тесты, зависящие от порядка элементов в map. Никогда не используйте порядок итерации для бизнес-логики.</p>

<h3 style="color: #34495e; margin-top: 20px;">Как получить детерминированный порядок</h3>

<p>Если нужен определённый порядок вывода (например, алфавитный), соберите ключи в срез и отсортируйте его. Для сортировки используется пакет <code>sort</code> &mdash; стандартный инструмент Go. Функция <code>sort.Strings(срез)</code> сортирует срез строк по алфавиту (на месте, без создания копии):</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">import "sort"

// Собираем ключи в срез
countries := make([]string, 0, len(capitals))
for country := range capitals {
    countries = append(countries, country)
}

// Сортируем ключи
sort.Strings(countries)

// Выводим в отсортированном порядке
fmt.Println("Алфавитный порядок:")
for _, country := range countries {
    fmt.Printf("  %s: %s\n", country, capitals[country])
}</code></pre>

<p><strong>Паттерн для запоминания:</strong> Собрать ключи &rarr; Отсортировать &rarr; Итерировать по отсортированным ключам.</p>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 15px; margin: 20px 0;">
<p style="margin: 0;"><strong>💡 Совет:</strong> Указание ёмкости при создании среза ключей (make([]string, 0, len(capitals))) &mdash; хорошая практика. Это избегает перевыделения памяти при append.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Вложенные map</h2>

<p>Иногда нужно группировать данные по двум уровням. Для этого значением map может быть другая map:</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">// Категория -&gt; (товар -&gt; количество)
inventory := make(map[string]map[string]int)

// Важно: перед записью во внутреннюю map
// нужно убедиться, что она создана
if inventory["фрукты"] == nil {
    inventory["фрукты"] = make(map[string]int)
}
inventory["фрукты"]["яблоки"] = 5</code></pre>

<div style="background: #fff3e0; border: 1px solid #ffb74d; border-radius: 8px; padding: 12px; margin: 16px 0;">
<p style="margin: 0;"><strong>⚠️ Не забывайте инициализировать внутреннюю map!</strong> Если написать <code>inventory[&quot;фрукты&quot;][&quot;яблоки&quot;] = 5</code> без проверки, программа аварийно завершится с ошибкой времени выполнения, потому что внутренняя map равна nil.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Реальный кейс: Анализ логов веб-сервера</h2>

<p>Рассмотрим практическую задачу, с которой сталкивается каждый веб-разработчик &mdash; анализ логов для понимания, какие HTTP статус-коды возвращает сервер и как часто.</p>

<h3 style="color: #34495e; margin-top: 20px;">Постановка задачи</h3>

<p>У нас есть логи веб-сервера, где каждая строка содержит информацию о запросе. Нужно подсчитать, сколько раз встречается каждый статус-код (200, 404, 500 и т.д.). Это поможет понять:</p>

<ul>
	<li>Процент успешных запросов (коды 2xx)</li>
	<li>Количество ошибок клиента (4xx) и сервера (5xx)</li>
	<li>Самые частые типы ответов</li>
</ul>

<h3 style="color: #34495e; margin-top: 20px;">Решение с использованием map</h3>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">package main

import (
    "fmt"
    "strconv"
    "strings"
)

// CountStatuses анализирует логи и подсчитывает статус-коды
func CountStatuses(lines []string) map[int]int {
    counts := make(map[int]int)

    for _, line := range lines {
        // strings.Fields разбивает строку по пробелам в срез слов
        fields := strings.Fields(line)

        // Проверка корректности формата
        if len(fields) &lt; 4 {
            continue  // Пропускаем некорректные строки
        }

        // strconv.Atoi переводит строку в int (знаком из модуля 9)
        code, err := strconv.Atoi(fields[3])
        if err != nil {
            continue  // Игнорируем строки с ошибкой парсинга
        }

        // Магия map: инкремент счётчика
        counts[code]++
        // Если ключа не было, сначала вернётся 0,
        // затем увеличится до 1
    }

    return counts
}</code></pre>

<h3 style="color: #34495e; margin-top: 20px;">Разбор решения по шагам</h3>

<p><strong>1. Инициализация map:</strong> Создаём пустую map[int]int, где ключ &mdash; статус-код, значение &mdash; количество.</p>

<p><strong>2. Обработка строк:</strong> Используем strings.Fields для разбиения строки на слова. Это устойчиво к множественным пробелам.</p>

<p><strong>3. Защита от ошибок:</strong> Проверяем длину среза полей и корректность парсинга числа. Реальные данные всегда содержат &quot;мусор&quot;.</p>

<p><strong>4. Инкремент счётчика:</strong> counts[code]++ &mdash; элегантная особенность Go. Если ключа нет, сначала вернётся 0 (нулевое значение int), затем произойдёт инкремент.</p>

<pre style="background: #f8f9fa; border: 1px solid #e9ecef; border-radius: 5px; padding: 15px; margin: 10px 0;">
<code class="language-go">func main() {
    logs := []string{
        "2024-01-15 GET /api/users 200 145ms",
        "2024-01-15 POST /api/login 401 23ms",
        "2024-01-15 GET /api/users 200 89ms",
        "2024-01-15 GET /api/posts 404 12ms",
        "malformed log line",  // Будет проигнорирована
    }

    stats := CountStatuses(logs)

    // Анализ результатов
    fmt.Println("Статистика:")
    total := 0
    successful := 0

    for code, count := range stats {
        fmt.Printf("  %d: %d раз\n", code, count)
        total += count
        if code &gt;= 200 &amp;&amp; code &lt; 300 {
            successful += count
        }
    }

    if total &gt; 0 {
        percent := float64(successful) * 100 / float64(total)
        fmt.Printf("\nУспешных: %.1f%%\n", percent)
    }
}</code></pre>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 15px; margin: 20px 0;">
<p style="margin: 0;"><strong>🛡️ Принцип устойчивости:</strong> Функция не падает на некорректных данных, а пропускает их. Это критически важно для production-кода, работающего с реальными данными.</p>
</div>

<h2 style="color: #34495e; margin-top: 30px;">Частые ошибки и как их избежать</h2>

<p>Рассмотрим типичные ошибки, которые делают разработчики при работе с map, и научимся их избегать.</p>

<div style="background: #fee2e2; border: 1px solid #ef4444; border-radius: 8px; padding: 20px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 1: Запись в nil map</h4>

<p><strong>Что происходит:</strong> При попытке записи в неинициализированную map программа аварийно завершается с ошибкой времени выполнения.</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">var m map[string]int
m["key"] = 42  // ошибка времени выполнения: запись в nil map запрещена</code></pre>

<p><strong>Как избежать:</strong> Всегда инициализируйте map перед использованием. Используйте make или литерал:</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">m := make(map[string]int)
m["key"] = 42  // OK</code></pre>

<p><strong>Защитная проверка:</strong> Если map может быть nil (например, поле структуры), проверяйте перед использованием:</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">if m == nil {
    m = make(map[string]int)
}
m["key"] = 42</code></pre>
</div>

<div style="background: #fee2e2; border: 1px solid #ef4444; border-radius: 8px; padding: 20px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 2: Использование NaN как ключа</h4>

<p><strong>Проблема:</strong> NaN (Not a Number) имеет особое свойство: NaN != NaN. Это делает его непригодным для использования в качестве ключа.</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">import "math"

m := make(map[float64]string)
nan := math.NaN()
m[nan] = "value"

// Попытка получить значение не работает!
v, ok := m[nan]
fmt.Println(v, ok)  // "" false</code></pre>

<p><strong>Решение:</strong> Избегайте float как ключей, если возможны NaN значения. Используйте целые числа или строки.</p>
</div>

<div style="background: #fee2e2; border: 1px solid #ef4444; border-radius: 8px; padding: 20px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 3: Полагаться на порядок итерации</h4>

<p><strong>Проблема:</strong> Код, зависящий от порядка элементов в map, будет работать непредсказуемо.</p>

<p><strong>Решение:</strong> Если нужен определённый порядок, всегда сортируйте ключи явно (как показано выше).</p>
</div>

<div style="background: #fee2e2; border: 1px solid #ef4444; border-radius: 8px; padding: 20px; margin: 20px 0;">
<h4 style="color: #991b1b; margin-top: 0;">❌ Ошибка 4: Попытка сравнить две map</h4>

<p><strong>Проблема:</strong> Go не позволяет сравнивать map между собой оператором ==.</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">a := map[string]int{"x": 1}
b := map[string]int{"x": 1}
// fmt.Println(a == b)  // Ошибка компиляции!</code></pre>

<p><strong>Решение:</strong> Для сравнения map напишите собственную функцию, сравнивающую длины и все пары ключ-значение.</p>

<p><strong>Исключение:</strong> Можно сравнивать map только с nil:</p>

<pre style="background: white; border: 1px solid #e9ecef; border-radius: 5px; padding: 10px; margin: 10px 0;">
<code class="language-go">var m map[string]int
fmt.Println(m == nil)  // true</code></pre>
</div>

<p><strong>Эта лекция подготавливает вас к решению следующих задач модуля:</strong></p>

<h4 style="color: #1565c0;">11.2.1 Телефонная книга</h4>

<p>Вы будете создавать map[string]string для хранения имён и телефонов, использовать ok-идиому для проверки наличия контакта и delete для удаления записей. Это прямое применение базовых операций из лекции.</p>

<h4 style="color: #1565c0;">11.2.2 Удаление дубликатов</h4>

<p>Научитесь использовать map[T]bool как множество (set) для эффективной проверки уникальности элементов. Паттерн &quot;если ключ существует &mdash; элемент уже встречался&quot; &mdash; основа для дедупликации.</p>

<h4 style="color: #1565c0;">11.2.3 Частотный анализ слов</h4>

<p>Примените map[string]int для подсчёта частоты категорий (как в примере с логами). Научитесь работать с регистрозависимыми ключами и обеспечивать детерминированный вывод через сортировку ключей.</p>

<h4 style="color: #1565c0;">11.2.4 Суммы по категориям</h4>

<p>Освоите агрегацию данных через оператор += для накопления сумм по ключам. Научитесь обрабатывать отрицательные значения и выводить результаты в отсортированном порядке.</p>

<h4 style="color: #1565c0;">11.2.5 Учёт товаров по категориям</h4>

<p>Работа с вложенными map (map[string]map[string]int) &mdash; двухуровневая группировка данных. Научитесь инициализировать внутренние map, удалять элементы с очисткой пустых категорий и обрабатывать команды через switch.</p>
