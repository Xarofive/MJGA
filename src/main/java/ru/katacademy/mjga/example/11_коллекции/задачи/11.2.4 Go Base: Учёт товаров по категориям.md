<p style="margin: 0;">Освоить работу с отображениями (maps) для группировки данных, научиться управлять вложенными структурами и обеспечивать корректное удаление элементов из коллекций.</p>

<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В реальных приложениях часто возникает задача учёта товаров по категориям &mdash; от простого домашнего бюджета до складских систем. Maps в Go идеально подходят для организации данных с быстрым доступом по ключу, позволяя эффективно группировать товары и подсчитывать их суммарную стоимость.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать систему учёта товаров с возможностью добавления, удаления и подсчёта суммарной стоимости по категориям.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Maps во вложенных структурах</h4>

<p>В Go можно создавать maps с maps в качестве значений для группировки данных по двум измерениям:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Категория -&gt; (Товар -&gt; Суммарная стоимость)
inventory := make(map[string]map[string]int)

// Инициализация вложенного map при первом использовании
if inventory["groceries"] == nil {
    inventory["groceries"] = make(map[string]int)
}
inventory["groceries"]["milk"] = 100</code></pre>

<p style="margin-bottom: 0;">Такая структура позволяет эффективно организовать двухуровневую группировку данных.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Накопление значений в map</h4>

<p>При работе с учётом товаров часто нужно накапливать значения:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Добавление товара (цена * количество)
category := "tech"
item := "laptop"
totalCost := price * quantity

// Накопление стоимости
inventory[category][item] += totalCost  // Работает даже если item не существует

// Подсчёт общей суммы по категории
total := 0
for _, cost := range inventory[category] {
    total += cost
}</code></pre>

<p style="margin-bottom: 0;"><strong>Важно:</strong> оператор += корректно работает с нулевым значением int.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ffb74d; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔒 Удаление элементов и очистка пустых категорий</h4>

<p>При удалении товаров важно следить за пустыми категориями:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Удаление товара из категории
delete(inventory[category], item)

// Проверка и удаление пустой категории
if len(inventory[category]) == 0 {
    delete(inventory, category)
}

// Проверка существования товара перед удалением
if _, exists := inventory[category][item]; exists {
    delete(inventory[category], item)
    fmt.Println("OK")
} else {
    fmt.Println("NOT FOUND")
}</code></pre>

<p style="margin-bottom: 0;">Удаление несуществующего ключа из map не вызывает ошибку, но лучше проверять существование для корректного вывода.</p>
</div>

<div style="background: #fce4ec; border: 1px solid #f06292; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #c2185b; margin-top: 0;">⚡ Сортировка при выводе</h4>

<p>Maps в Go не сохраняют порядок элементов. Для сортированного вывода используйте срез ключей:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">import "sort"

// Собираем ключи
var categories []string
for category := range inventory {
    categories = append(categories, category)
}

// Сортируем
sort.Strings(categories)

// Выводим в отсортированном порядке
for _, category := range categories {
    fmt.Printf("%s %d\n", category, calculateTotal(inventory[category]))
}</code></pre>

<p style="margin-bottom: 0;">Лексикографическая сортировка обеспечивает предсказуемый порядок вывода.</p>
</div>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Содержит функцию <code>main</code></li>
	<li>Использует <code>map</code> для хранения данных о товарах</li>
	<li>Может импортировать пакеты: <code>fmt</code>, <code>bufio</code>, <code>os</code>, <code>strings</code>, <code>strconv</code>, <code>sort</code>, <code>io</code></li>
	<li>Обрабатывает команды в цикле</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Формат ввода</h4>

<p><strong>Первая строка:</strong> количество команд N (1 &le; N &le; 100000)</p>

<p><strong>Следующие N строк:</strong> команды в одном из форматов:</p>

<ul style="margin: 10px 0;">
	<li><code>ADD &lt;категория&gt; &lt;товар&gt; &lt;цена&gt; &lt;количество&gt;</code> &mdash; добавить товар</li>
	<li><code>DELETE &lt;категория&gt; &lt;товар&gt;</code> &mdash; удалить товар</li>
	<li><code>TOTAL &lt;категория&gt;</code> &mdash; вывести общую стоимость категории</li>
	<li><code>SUMMARY</code> &mdash; вывести все категории с их суммами</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Поведение команд</h4>

<p><strong>ADD категория товар цена количество:</strong></p>

<ul style="margin: 10px 0;">
	<li>Добавляет к стоимости товара произведение цена &times; количество</li>
	<li>Если товар уже существует, увеличивает его суммарную стоимость</li>
	<li>Если категория не существует, создаёт её</li>
	<li>Не выводит ничего</li>
</ul>

<p><strong>DELETE категория товар:</strong></p>

<ul style="margin: 10px 0;">
	<li>Если товар существует: удаляет его и выводит <code>OK</code></li>
	<li>Если товар не существует: выводит <code>NOT FOUND</code></li>
	<li>Если после удаления категория становится пустой, удаляет категорию</li>
</ul>

<p><strong>TOTAL категория:</strong></p>

<ul style="margin: 10px 0;">
	<li>Если категория существует: выводит суммарную стоимость всех товаров</li>
	<li>Если категория не существует или пуста: выводит <code>NOT FOUND</code></li>
</ul>

<p><strong>SUMMARY:</strong></p>

<ul style="margin: 10px 0;">
	<li>Если есть категории: выводит их в лексикографическом порядке, каждую с новой строки в формате <code>&lt;категория&gt; &lt;сумма&gt;</code></li>
	<li>Если категорий нет: выводит <code>EMPTY</code></li>
</ul>

<p><strong>Бюджет производительности:</strong></p>

<ul style="margin: 10px 0;">
	<li>Программа должна обрабатывать до 100 000 команд за разумное время (&le; 5 секунд)</li>
</ul>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Не инициализация вложенного map</strong><br />
❌ <code>inventory[category][item] = value</code> &mdash; программа аварийно завершится с ошибкой времени выполнения, если <code>inventory[category]</code> не инициализирован<br />
✅ Всегда проверяйте: <code>if inventory[category] == nil { inventory[category] = make(map[string]int) }</code></p>

<p><strong>Ошибка 2: Забытая очистка пустых категорий</strong><br />
❌ Удаляете товар, но оставляете пустую категорию в map<br />
✅ После удаления: <code>if len(inventory[category]) == 0 { delete(inventory, category) }</code></p>

<p><strong>Ошибка 3: Неправильная проверка существования</strong><br />
❌ <code>if inventory[category] == nil</code> для проверки пустоты (map может существовать, но быть пустым)<br />
✅ Используйте <code>len(inventory[category]) == 0</code> или проверку при чтении: <code>_, exists := inventory[category]</code></p>

<p><strong>Ошибка 4: Потеря порядка при выводе</strong><br />
❌ <code>for category := range inventory</code> &mdash; непредсказуемый порядок<br />
✅ Соберите ключи в срез, отсортируйте, затем выводите</p>

<p><strong>Ошибка 5: Перезапись вместо накопления</strong><br />
❌ <code>inventory[category][item] = price * quantity</code> &mdash; теряется предыдущее значение<br />
✅ <code>inventory[category][item] += price * quantity</code> &mdash; накапливает стоимость</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Неверная обработка несуществующих элементов</strong><br />
❌ Программа аварийно завершается с ошибкой времени выполнения или молчаливо пропускает несуществующую категорию (вместо корректного вывода <code>NOT FOUND</code>)<br />
✅ Проверка существования и вывод <code>NOT FOUND</code> когда требуется</p>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказки для размышления</h4>

<p><strong>Вопрос 1:</strong> Почему важно удалять пустые категории?<br />
<em>Накопление пустых категорий приводит к утечке памяти и некорректному SUMMARY.</em></p>

<p><strong>Вопрос 2:</strong> Как эффективно проверить, что категория стала пустой?<br />
<em>Используйте <code>len(inventory[category]) == 0</code> сразу после удаления товара.</em></p>

<p><strong>Вопрос 3:</strong> Что произойдёт при попытке удалить несуществующий ключ из map?<br />
<em>Ничего &mdash; операция delete безопасна, но для корректного вывода нужна проверка существования.</em></p>

<p style="margin-bottom: 0;"><strong>Вопрос 4:</strong> Можно ли обойтись одноуровневым map?<br />
<em>Можно использовать составной ключ типа &quot;category:item&quot;, но это усложнит итерацию по категориям.</em></p>
