Эффективная работа с такими структурами данных критична для создания баз данных, кэшей, индексов и любых систем, где нужен быстрый доступ по ключу.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать телефонную книгу с поддержкой четырёх операций: добавление/обновление, получение, удаление и подсчёт контактов.</p>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Коллекция map в Go</h4>

<p>Map (ассоциативный массив, словарь) хранит пары ключ-значение с O(1) доступом:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Создание map
phonebook := make(map[string]string)
// или
phonebook := map[string]string{}

// Добавление/обновление
phonebook["Alice"] = "123-456"

// Чтение с проверкой существования
if phone, exists := phonebook["Alice"]; exists {
    fmt.Println(phone)
}

// Удаление
delete(phonebook, "Alice")

// Размер
count := len(phonebook)</code></pre>

<p style="margin-bottom: 0;">Ключевое отличие map от массивов: доступ по произвольному ключу (не только по индексу), автоматическое расширение, отсутствие гарантий порядка элементов.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Проверка существования ключа</h4>

<p>При чтении из map важно различать отсутствие ключа и нулевое значение:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ❌ Неправильно — не различает отсутствие и пустую строку
phone := phonebook["Bob"]  // "" если ключа нет

// ✅ Правильно — явная проверка существования
phone, exists := phonebook["Bob"]
if exists {
    fmt.Println(phone)
} else {
    fmt.Println("NOT FOUND")
}</code></pre>

<p style="margin-bottom: 0;">Второе значение (exists) &mdash; булевый флаг, который true только если ключ присутствует в map.</p>
</div>

<div style="background: #fff3e0; border: 1px solid #ff9800; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #e65100; margin-top: 0;">🔄 Обработка команд</h4>

<p>Для обработки различных команд удобно использовать switch:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">var cmd string
fmt.Scan(&amp;cmd)

switch cmd {
case "PUT":
    var name, phone string
    fmt.Scan(&amp;name, &amp;phone)
    phonebook[name] = phone
case "GET":
    var name string
    fmt.Scan(&amp;name)
    // ... проверка и вывод
}</code></pre>

<p style="margin-bottom: 0;">Можно также использовать if-else, но switch делает код более читаемым при множественном выборе.</p>
</div>

<div style="background: #e1f5fe; border-left: 4px solid #03a9f4; padding: 16px; margin: 16px 0;">
<h4 style="color: #01579b; margin-top: 0;">📌 Напоминание про символ &amp;</h4>

<p>Во всех функциях <code>fmt.Scan*</code> перед переменной ставится символ <code>&amp;</code>:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">var q int
fmt.Scan(&amp;q)  // &amp; обязателен!

var name, phone string
fmt.Scan(&amp;name, &amp;phone)  // Перед каждой переменной</code></pre>

<p><strong>Почему нужен &amp;?</strong> Символ <code>&amp;</code> означает &quot;взять адрес переменной в памяти&quot;. Функция <code>fmt.Scan</code> изменяет значение переменной, поэтому ей нужен адрес, а не копия.</p>

<p><strong>Аналогия:</strong> Представьте, что вы заказали доставку. Курьеру нужен адрес вашего дома (переменной), чтобы доставить посылку (прочитанное значение). Если вы не дадите адрес (<code>&amp;</code>), посылка никогда не попадёт к вам.</p>

<p style="margin-bottom: 0;"><strong>Правило:</strong> Перед каждой переменной в <code>fmt.Scan</code>, <code>fmt.Scanln</code>, <code>fmt.Scanf</code> всегда ставьте <code>&amp;</code>. Это часть синтаксиса функции.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте программу для управления телефонной книгой:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Импортирует только пакет <code>&quot;fmt&quot;</code></li>
	<li>Содержит функцию <code>main()</code></li>
	<li>Использует <code>map[string]string</code> для хранения пар имя-телефон</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📥 Формат входных данных</h4>

<p><strong>Первая строка:</strong> целое число Q (1 &le; Q &le; 200000) &mdash; количество операций</p>

<p><strong>Следующие Q строк:</strong> одна из четырёх команд:</p>

<ul style="margin: 10px 0;">
	<li><code>PUT name phone</code> &mdash; сохранить/обновить номер для контакта</li>
	<li><code>GET name</code> &mdash; получить номер контакта</li>
	<li><code>DEL name</code> &mdash; удалить контакт</li>
	<li><code>COUNT</code> &mdash; вывести количество контактов</li>
</ul>

<p><strong>Ограничения:</strong></p>

<ul style="margin: 10px 0;">
	<li>Имена и номера &mdash; строки без пробелов, длина &le; 100 символов</li>
	<li>Имена регистрозависимы: <code>Ivan</code> и <code>IVAN</code> &mdash; разные контакты</li>
	<li>Гарантируется корректность формата команд</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📤 Формат выходных данных</h4>

<p>Для каждой команды, требующей вывода, выведите одну строку:</p>

<ul style="margin: 10px 0;">
	<li><strong>GET:</strong> номер телефона или <code>NOT FOUND</code> (если контакта нет)</li>
	<li><strong>DEL:</strong> <code>OK</code> (если контакт был удалён) или <code>NOT FOUND</code> (если контакта не было)</li>
	<li><strong>COUNT:</strong> целое число &mdash; количество контактов в книге</li>
	<li><strong>PUT:</strong> ничего не выводит</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Важные детали</h4>

<ul style="margin: 10px 0;">
	<li>PUT перезаписывает существующий номер без дополнительного вывода</li>
	<li>DEL несуществующего контакта выводит <code>NOT FOUND</code>, не <code>OK</code></li>
	<li>Порядок вывода строго соответствует порядку команд GET/DEL/COUNT</li>
	<li>Программа должна обрабатывать до 200,000 операций за разумное время (&lt; 10 секунд)</li>
	<li>Используйте <code>fmt.Scan</code> для чтения входных данных</li>
</ul>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Не проверять существование при GET</strong><br />
❌ <code>fmt.Println(phonebook[name])</code> &mdash; выведет пустую строку вместо NOT FOUND<br />
✅ Используйте <code>phone, exists := phonebook[name]</code> и проверяйте exists</p>

<p><strong>Ошибка 2: Не проверять существование при DEL</strong><br />
❌ <code>delete(phonebook, name); fmt.Println(&quot;OK&quot;)</code> &mdash; всегда выводит OK<br />
✅ Сначала проверьте существование, затем удаляйте и выводите OK или NOT FOUND</p>

<p><strong>Ошибка 3: Не инициализировать map</strong><br />
❌ <code>var phonebook map[string]string</code> &mdash; запись в nil map приведёт к ошибке времени выполнения<br />
✅ Используйте <code>make(map[string]string)</code> или <code>map[string]string{}</code></p>

<p><strong>Ошибка 4: Выводить сообщения для PUT</strong><br />
❌ Выводить &quot;ADDED&quot; или &quot;UPDATED&quot; после PUT<br />
✅ PUT ничего не выводит, даже при перезаписи</p>

<p><strong>Ошибка 5: Неправильный формат вывода</strong><br />
❌ <code>fmt.Println(&quot;Not found&quot;)</code> или <code>fmt.Println(&quot;not found&quot;)</code><br />
✅ Точный формат: <code>NOT FOUND</code> (заглавные буквы, с пробелом)</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Игнорировать регистр имён</strong><br />
❌ Преобразовывать имена в нижний регистр через <code>strings.ToLower</code><br />
✅ Использовать имена как есть &mdash; регистр важен!</p>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказки по реализации</h4>

<p><strong>Как обрабатывать большое количество операций?</strong><br />
Map в Go имеет O(1) сложность для добавления, поиска и удаления. Это означает, что даже 200,000 операций выполнятся за несколько секунд &mdash; каждая операция практически не зависит от размера коллекции.</p>

<p><strong>Какую структуру цикла использовать?</strong><br />
Считайте количество операций Q, затем запустите цикл от 0 до Q. На каждой итерации считывайте команду и обрабатывайте её через switch или if-else.</p>

<p><strong>Как различить отсутствие ключа и пустое значение?</strong><br />
При чтении из map всегда используйте форму с двумя переменными: <code>value, exists := map[key]</code>. Переменная exists будет true только если ключ присутствует, даже если значение &mdash; пустая строка.</p>

<p style="margin-bottom: 0;"><strong>Что делать с операцией PUT при существующем ключе?</strong><br />
Просто присваивайте новое значение &mdash; map автоматически перезапишет старое. Не нужно проверять существование или выводить что-либо.</p>
