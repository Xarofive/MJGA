# Установка GO и настройка окружения

## Почему важно выбрать подходящую операционную систему?

При разработке программного обеспечения чаще всего используют Unix-подобные системы (например, Linux или macOS). Это связано с несколькими причинами:

- **Удобство разработки:** Большинство инструментов и библиотек для программирования оптимизированы именно под Unix-подобные системы.
- **Стабильность и производительность:** Эти системы обеспечивают более стабильную работу и эффективное управление ресурсами.
- **Распространённость в индустрии:** Многие компании, особенно крупные, предпочитают разработку и размещение своих проектов на Linux.

Однако большинство студентов обычно используют Windows, и переход на Linux может быть сложным. Поэтому оптимальным решением является использование подсистемы Linux (WSL) в Windows и бесплатного редактора кода Visual Studio Code.

## Что такое WSL и почему именно он?

**WSL (Windows Subsystem for Linux)** — это специальная подсистема, позволяющая запускать Linux прямо из Windows без необходимости устанавливать отдельную систему или виртуальную машину.

**Преимущества WSL:**
- Легко устанавливается и настраивается.
- Позволяет пользоваться всеми преимуществами Linux без потери комфорта использования Windows.
- Совместим с популярными инструментами для разработки, включая VS Code.

## Почему Visual Studio Code?

- **Бесплатный и доступный в России**, в отличие от платной IDE GoLand.
- Множество плагинов и расширений, облегчающих работу с Go.
- Прост в использовании даже для новичков.

Студенты, которые уже используют Linux или macOS, могут продолжать использовать свои системы. Инструкции будут подходить и для них, так как VS Code и инструменты Go полностью кроссплатформенные.

## 🖥️ 2. Установка и настройка WSL2 и Ubuntu

### Шаг 1. Установка WSL

Открой **PowerShell от имени администратора** и выполни:

```powershell
wsl --install
```

После завершения перезагрузи компьютер.

После перезагрузки автоматически запустится установка Ubuntu, задай имя пользователя и пароль.

- Активируй интеграцию с WSL2.

## 🚀 5. Установка Go версии 1.19 через goenv (внутри WSL)

Открой терминал WSL и выполни команды:

```bash
sudo apt update
sudo apt install -y git build-essential curl

# установка goenv
git clone https://github.com/syndbg/goenv.git ~/.goenv
echo 'export GOENV_ROOT="$HOME/.goenv"' >> ~/.bashrc
echo 'export PATH="$GOENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(goenv init -)"' >> ~/.bashrc
source ~/.bashrc

# установка Go 1.19
goenv install 1.19.0
goenv global 1.19.0

go version
```

## 🟢 6. Настройка расширений VS Code для Go

Открой раздел **Extensions** в VS Code:
- Найди расширение **Go**.
- Выбери версию **v0.39.1** (Install Another Version).

После установки расширения VS Code предложит установить инструменты — соглашаемся:

- Нажми «Install All».

Если появляется ошибка из-за старой версии Go, перейди в терминал VS Code и установи вручную:
```bash
go install golang.org/x/tools/gopls@v0.11.0
go install github.com/go-delve/delve/cmd/dlv@latest
```

## 🚩 9. Решение возможных проблем (FAQ)

| Проблема | Решение |
| -------- | ------------------- |
| Docker не запускается | Перезапустить WSL (`wsl --shutdown`) |
| Нет автокомплита Go | Убедись, что используешь правильную версию Go и расширения (Go 1.19 и gopls v0.11.0) |
| Проблемы доступа к серверу из Docker | Проверь, что порт корректно проброшен: `docker run -p 8080:8080` |
| VS Code подключается не к тому дистрибутиву WSL | В PowerShell: `wsl --set-default Ubuntu` |

# 🍏 Установка Go на macOS

## 1. Установка Go 1.19 на macOS

### Метод 1: Через Homebrew (рекомендуется)

Homebrew - это популярный менеджер пакетов для macOS, который значительно упрощает установку программного обеспечения.

```bash
# Установка Homebrew, если еще не установлен
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Установка конкретной версии Go 1.19
brew install go@1.19

# Добавление Go в PATH
echo 'export PATH="/usr/local/opt/go@1.19/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Проверка установки
go version  # Должен отобразиться Go 1.19.x
```

### Метод 2: Через официальный установщик

1. Перейди на [официальный сайт Go](https://golang.org/dl/)
2. Найди и скачай **именно** версию Go 1.19.x для macOS
3. Открой скачанный файл .pkg и следуй инструкциям установщика
4. После установки проверь:

```bash
go version  # Должен отобразиться Go 1.19.x
```

## 2. Настройка рабочего окружения

Создай директорию для проектов Go и настрой переменные окружения:

```bash
# Создание рабочей директории
mkdir -p ~/go/{bin,src,pkg}

# Добавление в ~/.zshrc или ~/.bash_profile
echo 'export GOPATH=$HOME/go' >> ~/.zshrc
echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.zshrc
source ~/.zshrc
```

## 3. Установка инструментов Go

Установи необходимые инструменты Go (те же версии, что и для Windows):

```bash
# Установка gopls версии 0.11.0
go install golang.org/x/tools/gopls@v0.11.0

# Установка delve для отладки
go install github.com/go-delve/delve/cmd/dlv@latest
```

## 4. Установка Docker на macOS (опционально)

1. Скачай [Docker Desktop для Mac](https://www.docker.com/products/docker-desktop/)
2. Установи приложение, перетащив его в папку Applications
3. Запусти Docker Desktop и дождись его готовности (значок в статус-баре станет зелёным)
4. Проверь установку:

```bash
docker run hello-world
```

## 5. Создание первого приложения

Откройте терминал и выполните:

```bash
# Создание директории проекта
mkdir -p ~/go/src/hello
cd ~/go/src/hello

# Инициализация модуля Go
go mod init hello

# Создание файла main.go
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Привет, я учусь программировать на Go!")
}
EOF

# Запуск программы
go run main.go
```

# 🐧 Установка Go на Linux (Ubuntu)

## 1. Установка Go 1.19 на Ubuntu

```bash
# Обновление репозиториев
sudo apt update

# Установка необходимых пакетов
sudo apt install -y wget git curl build-essential

# Скачивание именно версии Go 1.19
wget https://golang.org/dl/go1.19.linux-amd64.tar.gz

# Установка Go
sudo tar -C /usr/local -xzf go1.19.linux-amd64.tar.gz
rm go1.19.linux-amd64.tar.gz

# Настройка переменных окружения
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPATH=$HOME/go' >> ~/.bashrc
echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.bashrc
source ~/.bashrc

# Проверка установки
go version  # Должен отобразиться Go 1.19.x
```

## 2. Настройка рабочего окружения

```bash
# Создание рабочей директории
mkdir -p ~/go/{bin,src,pkg}
```

## 3. Установка инструментов Go

Установи те же версии инструментов, что указаны для Windows:

```bash
# Установка gopls версии 0.11.0
go install golang.org/x/tools/gopls@v0.11.0

# Установка delve для отладки
go install github.com/go-delve/delve/cmd/dlv@latest
```

## 4. Установка Docker (опционально)

```bash
# Установка необходимых пакетов
sudo apt update
sudo apt install -y apt-transport-https ca-certificates curl software-properties-common

# Добавление Docker GPG ключа
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

# Добавление репозитория Docker
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

# Установка Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io

# Добавление текущего пользователя в группу docker
sudo usermod -aG docker $USER

# Перелогинься или используй следующую команду для применения изменений
newgrp docker

# Проверка установки Docker
docker run hello-world
```

## 5. Создание первого приложения

```bash
# Создание директории проекта
mkdir -p ~/go/src/hello
cd ~/go/src/hello

# Инициализация модуля Go
go mod init hello

# Создание файла main.go
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Привет, я учусь программировать на Go!")
}
EOF

# Запуск программы
go run main.go
```

## 6. Решение возможных проблем

| Проблема | Решение |
| -------- | ------------------- |
| Ошибка прав доступа Docker | `sudo chmod 666 /var/run/docker.sock` |
| Go не найден в PATH | Перепроверь настройки в ~/.bashrc или ~/.zshrc |
| Проблемы с зависимостями | `go mod tidy` для очистки и обновления зависимостей |

### Рекомендуемые среды разработки

Для комфортной работы с Go можно использовать разные среды разработки:

1. **Visual Studio Code (VS Code)** — бесплатный, легкий редактор с отличной поддержкой Go через расширения. Работает на всех платформах и, что особенно важно, полностью доступен в России. VS Code подходит как новичкам, так и опытным разработчикам благодаря гибкой системе расширений.

2. **GoLand от JetBrains** — профессиональная IDE, специально созданная для Go-разработки. Предлагает продвинутые инструменты рефакторинга, отладки и анализа кода. Существенный недостаток — это платное решение, и в настоящее время приобрести лицензию из России затруднительно из-за санкций. Для студентов доступна бесплатная лицензия, но её также непросто получить из России.

3. **Vim/Neovim или Emacs** с плагинами для Go — классические текстовые редакторы с полной поддержкой Go через плагины. Бесплатны и доступны на всех платформах.

**Наша рекомендация:** 
- VS Code — оптимальный выбор для большинства разработчиков в России из-за его доступности, функциональности и активного сообщества.
- GoLand — отличный вариант, если у вас уже есть лицензия или вы можете её получить.

Выбирай наиболее подходящий вариант установки в зависимости от твоей операционной системы, настраивай удобную для тебя среду разработки и смело начинай изучать Go!
