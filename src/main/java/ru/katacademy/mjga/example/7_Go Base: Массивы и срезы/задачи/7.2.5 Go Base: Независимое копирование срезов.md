<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">Независимая копия среза</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">Цель задачи</h3>

<p style="margin: 0;">Написать функцию <code>cloneSlice</code>, которая создаёт и возвращает полностью независимую копию среза целых чисел. Освоить связку <code>make</code> + <code>copy</code> для безопасного копирования данных.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">При работе с данными критически важно понимать разницу между ссылкой и копией. Если два среза указывают на одну область памяти, изменение через один срез неожиданно повлияет на другой. Создание независимых копий данных предотвращает такие побочные эффекты. Вынесение логики копирования в отдельную функцию &mdash; стандартная практика в Go-проектах.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию <code>cloneSlice(src []int) []int</code>, возвращающую независимую копию исходного среза.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Поверхностное vs глубокое копирование</h4>

<p><strong>Поверхностное копирование (разделяемая память):</strong></p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">clone := original  // Оба среза указывают на один массив
clone := original[:]  // То же самое - общий массив</code></pre>

<p><strong>Глубокое копирование (независимая память):</strong></p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">clone := make([]int, len(original))  // Выделяем новую память
copy(clone, original)  // Копируем данные</code></pre>

<p style="margin-bottom: 0;">При глубоком копировании изменения в одном срезе не затрагивают другой.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">Функция copy</h4>

<p>Встроенная функция <code>copy</code> копирует элементы из одного среза в другой:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">count := copy(destination, source)</code></pre>

<ul style="margin: 10px 0;">
	<li>Копирует минимум из длин обоих срезов</li>
	<li>Возвращает количество скопированных элементов</li>
	<li>Срезы должны быть одного типа</li>
	<li>Память destination должна быть выделена заранее</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Напишите функцию <code>cloneSlice</code>:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">Сигнатура</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">func cloneSlice(src []int) []int</code></pre>

<h4 style="color: #e65100; margin-top: 16px;">Требования к реализации</h4>

<ul style="margin: 10px 0;">
	<li>Функция называется <code>cloneSlice</code></li>
	<li>Параметр: <code>src []int</code></li>
	<li>Возвращаемый тип: <code>[]int</code></li>
	<li>Используйте <code>make</code> для создания нового среза нужной длины</li>
	<li>Используйте <code>copy</code> для копирования данных из <code>src</code> в новый срез</li>
	<li>Возвращённый срез должен быть независим от исходного &mdash; изменения в одном не влияют на другой</li>
	<li>Для пустого среза (<code>len == 0</code>) или <code>nil</code> возвращайте пустой срез (допустимы и <code>nil</code>, и <code>[]int{}</code>)</li>
	<li>Решение должно содержать ТОЛЬКО функцию <code>cloneSlice</code> &mdash; без дополнительных функций (включая <code>init()</code>) и без top-level переменных, констант или типов</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">Рекомендации</h4>

<ul style="margin: 10px 0;">
	<li>Решайте через связку <code>make</code> + <code>copy</code> (не через <code>append</code>)</li>
	<li>Присваивание <code>clone := src</code> или <code>clone := src[:]</code> создаёт поверхностную копию &mdash; избегайте этого</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Поверхностное копирование</strong><br />
<code>clone := src</code> &mdash; срезы делят память<br />
Правильно: <code>clone := make([]int, len(src))</code> + <code>copy(clone, src)</code></p>

<p><strong>Ошибка 2: Забыли выделить память</strong><br />
<code>var clone []int; copy(clone, src)</code> &mdash; clone пустой, copy ничего не скопирует<br />
Правильно: сначала <code>make</code>, потом <code>copy</code></p>

<p><strong>Ошибка 3: Неправильный порядок аргументов copy</strong><br />
<code>copy(src, clone)</code> &mdash; копирует из clone в src<br />
Правильно: <code>copy(clone, src)</code> &mdash; копирует из src в clone</p>

<p><strong>Ошибка 4: Ошибка при обращении к nil-срезу</strong><br />
Вызов <code>make([]int, len(src))</code> безопасен и при nil-аргументе: для nil-среза <code>len</code> возвращает 0, поэтому <code>make</code> создаст пустой срез без ошибки выхода за границы. Главное не обращаться к элементам пустого среза по индексу.</p>

<p style="margin-bottom: 0;"><strong>Ошибка 5: Использование append вместо make+copy</strong><br />
Задача направлена на освоение связки <code>make</code> + <code>copy</code></p>
</div>

	<li>Функция называется <code>cloneSlice</code>?</li>
	<li>Параметр называется <code>src</code> и имеет тип <code>[]int</code>?</li>
	<li>Возвращаемый тип &mdash; <code>[]int</code>?</li>
	<li>Используется <code>make</code> для создания нового среза?</li>
	<li>Используется <code>copy</code> для копирования данных?</li>
	<li>Для пустого/nil среза возвращается пустой срез?</li>
	<li>Изменение копии не влияет на оригинал?</li>
	<li>Функция содержит <code>return</code>?</li>
	<li>Решение содержит ТОЛЬКО функцию <code>cloneSlice</code> (без helper&#39;ов, <code>init()</code>, top-level переменных, констант, типов)?</li>
</ul>

<div style="background: #f3e5f5; border: 1px solid #ce93d8; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #6a1b9a; margin-top: 0;">Где это пригодится</h4>

<p><strong>Безопасное изменение:</strong> Можно менять копию, не боясь испортить оригинальные данные</p>

<p><strong>Откат изменений:</strong> Сохранение снимков состояния перед модификацией</p>

<p><strong>Тестирование:</strong> Изоляция тестовых данных от оригинальных</p>

<p style="margin-bottom: 0;"><strong>Передача данных между функциями:</strong> Безопасный возврат коллекции из функции, чтобы caller мог её менять без риска изменить внутреннее состояние модуля</p>
</div>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">Подсказка</h4>

<p>Представьте срезы как листы бумаги с данными. При присваивании <code>clone := src</code> вы просто создаёте второе имя для того же листа &mdash; любые изменения видны обоим. А комбинация <code>make</code> + <code>copy</code> &mdash; это как взять чистый лист и переписать на него все данные. Теперь у вас два независимых листа, и изменения на одном не затронут другой. Не забудьте обработать крайний случай: если исходный лист пуст или его нет &mdash; верните пустой лист.</p>
</div>
