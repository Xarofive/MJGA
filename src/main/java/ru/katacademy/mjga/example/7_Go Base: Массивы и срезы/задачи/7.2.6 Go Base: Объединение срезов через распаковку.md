<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🔗 Объединение двух срезов</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Написать функцию <code>mergeSlices(a, b []int) []int</code>, которая объединяет два среза в один с помощью <code>append</code> и оператора распаковки <code>...</code>.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст</h3>

<p style="margin-bottom: 10px;">При обработке данных часто требуется объединять коллекции: слияние результатов из разных источников, агрегация данных от микросервисов, построение отчётов. Оператор распаковки позволяет элегантно объединять срезы без написания циклов, что делает код чище и производительнее.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию, которая принимает два среза и возвращает новый срез, содержащий все элементы обоих.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Оператор распаковки (...)</h4>

<p>Троеточие после среза распаковывает его элементы как отдельные аргументы:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Без распаковки - для []int не скомпилируется
append(slice1, slice2)  // Ошибка компиляции: нельзя добавить []int как int

// С распаковкой - добавляются все элементы
append(slice1, slice2...)  // Правильно!</code></pre>

<p style="margin-bottom: 0;">Троеточие превращает срез в последовательность его элементов.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Паттерн объединения срезов</h4>

<p>Стандартный способ объединения двух срезов:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">result := append(first, second...)</code></pre>

<ul style="margin: 10px 0;">
	<li>Создаётся новый срез на основе первого</li>
	<li>Все элементы второго добавляются к результату</li>
	<li>Исходные срезы остаются неизменными</li>
	<li>Порядок элементов сохраняется</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте функцию объединения срезов:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Сигнатура функции</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code><span style="color: #c792ea;">func</span> <span style="color: #82aaff;">mergeSlices</span>(a, b <span style="color: #82aaff;">[]int</span>) <span style="color: #82aaff;">[]int</span></code></pre>

<h4 style="color: #e65100; margin-top: 16px;">🎯 Требования</h4>

<ul style="margin: 10px 0;">
	<li><strong>Параметры:</strong> два среза <code>a</code> и <code>b</code> типа <code>[]int</code></li>
	<li><strong>Возвращает:</strong> <strong>новый независимый</strong> срез <code>[]int</code>, содержащий сначала все элементы <code>a</code>, затем все элементы <code>b</code></li>
	<li><strong>Метод решения:</strong> используйте <code>append</code> с оператором распаковки <code>...</code></li>
	<li><strong>Результат должен быть независим от памяти <code>a</code> и <code>b</code>.</strong> Простой <code>return append(a, b...)</code> <strong>не подходит</strong>: если у <code>a</code> есть лишняя ёмкость, <code>append</code> запишет элементы <code>b</code> прямо в память <code>a</code>, и данные вызывающего кода будут испорчены. Создайте новый срез через <code>make([]int, 0, len(a)+len(b))</code>, а затем добавьте в него элементы обоих срезов</li>
	<li><strong>Для nil/пустых аргументов</strong> корректно возвращайте результат</li>
	<li><strong>Формат решения:</strong> отправляйте только функцию без <code>package</code> и <code>import</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚠️ Ограничения</h4>

<ul style="margin: 10px 0;">
	<li>Не используйте циклы для объединения</li>
	<li>Обязательно используйте <code>append</code> с оператором <code>...</code></li>
	<li>Не добавляйте <code>package</code>, <code>import</code> и другие функции</li>
	<li>Запрещён вывод в консоль (<code>print</code>, <code>println</code>, <code>fmt.*</code>)</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки</h4>

<p><strong>Ошибка 1: Забыли троеточие</strong><br />
❌ <code>append(a, b)</code> &mdash; попытка добавить срез как один элемент (ошибка компиляции)<br />
✅ <code>append(a, b...)</code> &mdash; распаковка элементов среза</p>

<p><strong>Ошибка 2: Использование цикла вместо распаковки</strong><br />
❌ <code>for _, v := range b { a = append(a, v) }</code><br />
✅ <code>append(a, b...)</code> &mdash; одна операция</p>

<p><strong>Ошибка 3: Мутация исходного среза a</strong><br />
❌ <code>return append(a, b...)</code> &mdash; если у <code>a</code> есть лишняя ёмкость, append запишет <code>b</code> в память <code>a</code> и испортит данные вызывающего кода<br />
✅ Создайте новый срез: <code>result := make([]int, 0, len(a)+len(b))</code>, затем <code>result = append(result, a...)</code> и <code>result = append(result, b...)</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 4: Добавление package/import</strong><br />
❌ Указание <code>package main</code> или <code>import &quot;fmt&quot;</code><br />
✅ Только функция <code>mergeSlices</code></p>
</div>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p style="margin-bottom: 0;">Представьте троеточие как команду &quot;распаковать коробку&quot;. Когда вы пишете <code>b...</code>, это как если бы вы вытащили все предметы из коробки <code>b</code> и разложили их по одному. Функция <code>append</code> получает не коробку целиком, а каждый элемент отдельно: вместо <code>append(a, [3,4,5])</code> происходит <code>append(a, 3, 4, 5)</code>.</p>
