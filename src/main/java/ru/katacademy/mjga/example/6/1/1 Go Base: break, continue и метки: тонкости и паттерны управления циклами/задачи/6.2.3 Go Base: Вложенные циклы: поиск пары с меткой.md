<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🎯 Вложенные циклы: поиск пары с меткой</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться прерывать несколько вложенных циклов одним действием и использовать метку, когда найден результат.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст</h3>

<p style="margin-bottom: 10px;">Представьте, что вы перебираете таблицу вариантов &mdash; этажи и места на парковке, строки и столбцы матрицы, комбинации параметров. Как только подходящая пара найдена, нет смысла продолжать обход. Метка позволяет мгновенно завершить оба цикла без вспомогательных флагов и лишних проверок.</p>

<p style="margin: 0;"><strong>Задача:</strong> найти первую пару чисел с произведением 45 и остановить весь поиск.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Метки и break с меткой</h4>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">search:
for i := 1; i &lt;= 10; i++ {
    for j := 1; j &lt;= 10; j++ {
        if подходящаяПара(i, j) {
            break search // выходит из обоих циклов
        }
    }
}
// продолжение программы</code></pre>

<p style="margin-bottom: 0;">Метка записывается перед внешним циклом, а <code>break search</code> завершает сразу оба уровня вложенности.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте программу, которая ищет первую пару чисел с произведением 45.</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Импортирует только пакет <code>&quot;fmt&quot;</code> без псевдонима (используйте <code>import &quot;fmt&quot;</code>, форма <code>import f &quot;fmt&quot;</code> не допускается)</li>
	<li>Содержит единственную функцию <code>main()</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Алгоритм</h4>

<ul style="margin: 10px 0;">
	<li>Переберите <code>i</code> от 1 до 9 во внешнем цикле <code>for</code></li>
	<li>Для каждого <code>i</code> переберите <code>j</code> от 1 до 9 во внутреннем цикле</li>
	<li>Используйте <code>continue</code>, чтобы сразу переходить к следующей итерации, если пара не подходит</li>
	<li>Когда найдено произведение 45, используйте <code>fmt.Println(i, j)</code> с двумя числовыми аргументами и выполните <code>break</code> с меткой для выхода из обоих циклов</li>
	<li>После успешного <code>break</code> программа завершается без дополнительных действий</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🧠 Почему это важно</h2>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<p style="margin: 0;">Без метки пришлось бы заводить логический флаг или дублировать проверку после внутреннего цикла. С меткой код остаётся линейным: условие &mdash; вывод &mdash; <code>break search</code>. Это тот же подход, который используют при поиске элемента в матрице или в задачах backtracking с досрочным завершением.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">🧪 Пример работы программы</h2>

<div style="background: #ede7f6; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin: 0 0 8px 0;">При запуске выводится первая подходящая пара.</p>

<details><summary style="cursor: pointer; color: #6a1b9a; font-weight: bold;">Показать ожидаемый вывод</summary>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0 0 0;">
5 9
</pre>

<p style="margin: 8px 0 0 0;">Пара <code>5 &times; 9 = 45</code> &mdash; первая комбинация в диапазоне <code>1..9</code>, поэтому цикл завершается сразу после её нахождения.</p>
</details>
</div>

<h2 style="color: #34495e; margin-top: 28px;">⚠️ Частые ошибки</h2>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 16px 0;">
<ul style="margin: 0;">
	<li><strong>Нет метки:</strong> обычный <code>break</code> остановит только внутренний цикл</li>
	<li><strong>Нет <code>continue</code>:</strong> проверку лучше оформлять как &laquo;если не подходит &mdash; <code>continue</code>&raquo;</li>
	<li><strong>Лишний вывод:</strong> должно быть ровно одно сообщение с найденной парой</li>
	<li><strong>Неверное произведение:</strong> проверяйте, что выводится именно пара для 45</li>
</ul>
</div>

<div style="background: #e8f5e9; border: 1px solid #4caf50; border-radius: 8px; padding: 14px; margin: 16px 0;">
<ul style="margin-bottom: 0;">

	<li>Циклы перебирают числа от 1 до 9?</li>
</ul>
