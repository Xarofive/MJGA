<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">🎯 FizzBuzz с паттерном Guard Clauses</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться применять паттерн early-continue (guard clauses) для создания плоского, читаемого кода без глубокой вложенности условий.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В промышленной разработке читаемость кода критична для командной работы и поддержки. Паттерн guard clauses устраняет &quot;пирамиду смерти&quot; из вложенных if-else, делая логику линейной и понятной. Этот подход используется в системах валидации, обработке бизнес-правил и фильтрации данных.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать классическую задачу FizzBuzz используя серию проверок с ранним выходом через <code>continue</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Паттерн Guard Clauses (Early Continue)</h4>

<p>Вместо вложенных условий используется серия независимых проверок с ранним выходом:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Вложенный подход (избегаем)
if условие1 {
    if условие2 {
        действие
    } else {
        другое_действие
    }
}

// Guard clauses (предпочтительно)
if условие1 {
    действие
    continue
}
if условие2 {
    другое_действие
    continue
}
// основной случай</code></pre>

<p style="margin-bottom: 0;">Каждая проверка обрабатывает свой случай и выходит, оставляя код плоским.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Правила FizzBuzz</h4>

<p>Для чисел от 1 до 30 вывести:</p>

<ul style="margin: 10px 0;">
	<li><strong>&quot;FizzBuzz&quot;</strong> &mdash; если число делится на 15 (или на 3 И на 5)</li>
	<li><strong>&quot;Fizz&quot;</strong> &mdash; если делится только на 3</li>
	<li><strong>&quot;Buzz&quot;</strong> &mdash; если делится только на 5</li>
	<li><strong>Само число</strong> &mdash; во всех остальных случаях</li>
</ul>

<p style="margin-bottom: 0;">Порядок проверок важен: сначала проверяем самое специфичное условие.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Реализуйте FizzBuzz для чисел от 1 до 30 используя паттерн guard clauses:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Импортирует только пакет <code>&quot;fmt&quot;</code> без псевдонима (используйте <code>import &quot;fmt&quot;</code>, форма <code>import f &quot;fmt&quot;</code> не допускается)</li>
	<li>Содержит только функцию <code>main()</code></li>
	<li>НЕ использует оператор <code>switch</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Требования к циклу</h4>

<ul style="margin: 10px 0;">
	<li>Используйте классическую форму: <code>for переменная := 1; переменная &lt;= 30; переменная++</code></li>
	<li>Начинайте с 1, заканчивайте на 30 (включительно)</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Требования к логике</h4>

<ul style="margin: 10px 0;">
	<li>Используйте серию отдельных <code>if</code> блоков с <code>continue</code></li>
	<li>Сначала проверьте деление на 15 (FizzBuzz)</li>
	<li>Затем на 3 (Fizz), затем на 5 (Buzz)</li>
	<li>После всех guard-проверок выведите само число</li>
	<li>НЕ используйте <code>else</code> или вложенные <code>if</code></li>
	<li>Каждый guard-блок должен содержать вывод и <code>continue</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📤 Формат вывода</h4>

<ul style="margin: 10px 0;">
	<li>Используйте <code>fmt.Println</code> или <code>fmt.Printf</code></li>
	<li>30 строк вывода &mdash; по одной на число</li>
	<li>Точное написание: &quot;FizzBuzz&quot;, &quot;Fizz&quot;, &quot;Buzz&quot;</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неправильный порядок проверок</strong><br />
❌ Сначала проверять на 3, потом на 15 (15 никогда не сработает)<br />
✅ Всегда начинайте с самого специфичного условия (15)</p>

<p><strong>Ошибка 2: Использование else</strong><br />
❌ <code>if ... else if ... else</code> &mdash; это не guard clauses<br />
✅ Серия независимых <code>if</code> с <code>continue</code></p>

<p><strong>Ошибка 3: Забыть continue</strong><br />
❌ Без <code>continue</code> выполнятся все последующие проверки<br />
✅ Каждый guard-блок должен заканчиваться <code>continue</code></p>

<p><strong>Ошибка 4: Вложенные условия</strong><br />
❌ <code>if i%3 == 0 { if i%5 == 0 {...} }</code><br />
✅ Плоская структура: проверка на 15 в одном условии</p>

<p><strong>Ошибка 5: Неправильные границы</strong><br />
❌ <code>for i := 0; i &lt; 30; i++</code> (начинается с 0, не включает 30)<br />
✅ <code>for i := 1; i &lt;= 30; i++</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Использование switch</strong><br />
❌ <code>switch { case i%15 == 0: ... }</code><br />
✅ Только серия <code>if</code> с <code>continue</code></p>
</div>

	<li>Есть только функция <code>main()</code>?</li>

	<li>Каждый блок содержит вывод (<code>fmt.Println</code> или <code>fmt.Printf</code>) и <code>continue</code>?</li>

<p><strong>Валидация данных:</strong> Последовательная проверка условий с ранним выходом</p>

<p><strong>Бизнес-логика:</strong> Применение правил в порядке приоритета</p>

<p><strong>Парсеры и фильтры:</strong> Обработка специальных случаев перед общими</p>

<p style="margin-bottom: 0;"><strong>Code review:</strong> Стандарт читаемого кода во многих компаниях</p>

<div style="background: #fff8e1; border-left: 4px solid #ffc107; padding: 16px; margin: 16px 0;">
<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Представьте сортировщика на конвейере: он проверяет каждый предмет по списку критериев сверху вниз. Как только критерий подходит &mdash; предмет убирается с ленты в соответствующий контейнер (continue). Самые специфичные критерии проверяются первыми (число делится на 15), затем менее специфичные (на 3 или 5). Если ни один критерий не подошёл &mdash; предмет проходит дальше как есть.</p>
