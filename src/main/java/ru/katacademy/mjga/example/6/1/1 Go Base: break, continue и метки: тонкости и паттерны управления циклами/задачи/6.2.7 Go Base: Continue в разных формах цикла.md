<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">⚠️ Continue в разных формах цикла</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться безопасно использовать оператор <code>continue</code> в классическом и while-стиле циклов, понимая критическое различие в обработке инкремента.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">При обработке потоков данных часто нужно пропускать определённые элементы: невалидные записи в логах, тестовые аккаунты в статистике, системные файлы при обходе директорий. Неправильное использование <code>continue</code> в while-циклах &mdash; частая причина зависания программ и потери данных в production.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать два цикла с пропуском элемента, демонстрируя безопасное использование <code>continue</code>.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Continue в классической форме</h4>

<p>В классическом <code>for i := 0; i &lt; n; i++</code> пост-выражение выполняется ВСЕГДА:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">for i := 0; i &lt; 10; i++ {
    if i == 5 {
        continue  // i++ всё равно выполнится!
    }
    fmt.Println(i)
}
// Безопасно: i увеличивается автоматически</code></pre>

<p style="margin-bottom: 0;">Пост-выражение &mdash; часть механизма цикла, а не тела.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Continue в while-стиле</h4>

<p>В while-стиле нет автоматического инкремента &mdash; опасность зацикливания:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// ОПАСНО - бесконечный цикл!
i := 0
for i &lt; 10 {
    if i == 5 {
        continue  // i остаётся 5 навсегда!
    }
    fmt.Println(i)
    i++
}

// БЕЗОПАСНО - инкремент перед continue
i := 0
for i &lt; 10 {
    if i == 5 {
        i++  // Обязательно увеличить!
        continue
    }
    fmt.Println(i)
    i++
}</code></pre>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте программу с двумя циклами, демонстрирующими безопасное использование continue:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Структура программы</h4>

<ul style="margin: 10px 0;">
	<li>Начинается с <code>package main</code></li>
	<li>Импортирует только пакет <code>&quot;fmt&quot;</code> без псевдонима (используйте <code>import &quot;fmt&quot;</code>, форма <code>import f &quot;fmt&quot;</code> не допускается)</li>
	<li>Содержит только функцию <code>main()</code></li>
	<li>НЕ использует <code>switch</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Первый цикл (классический)</h4>

<ul style="margin: 10px 0;">
	<li>Используйте форму: <code>for i := 0; i &lt; 10; i++</code></li>
	<li>Переменная цикла &mdash; например, <code>i</code> (имя на ваше усмотрение)</li>
	<li>Внутри цикла: если <code>i == 5</code>, выполните <code>continue</code></li>
	<li>Выведите значение <code>i</code> через <code>fmt.Println</code></li>
	<li>Результат: числа 0-9 без 5</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Второй цикл (while-стиль)</h4>

<ul style="margin: 10px 0;">
	<li>Объявите переменную перед циклом и установите в 0</li>
	<li>Используйте форму: <code>for условие {}</code> без init и post</li>
	<li>Условие: переменная меньше 10</li>
	<li>Если значение равно 5: сначала увеличьте счётчик, потом <code>continue</code></li>
	<li>После вывода увеличивайте счётчик</li>
	<li>Результат: те же числа 0-9 без 5</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📤 Формат вывода</h4>

<ul style="margin: 10px 0;">
	<li>Используйте только <code>fmt.Println</code></li>
	<li>18 строк вывода: два раза последовательность 0,1,2,3,4,6,7,8,9</li>
	<li>Каждое число на отдельной строке</li>
</ul>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Зацикливание в while-стиле</strong><br />
❌ Забыть увеличить счётчик перед <code>continue</code><br />
✅ Всегда инкрементируйте переменную ДО <code>continue</code></p>

<p><strong>Ошибка 2: Лишний инкремент в классическом</strong><br />
❌ Добавить <code>i++</code> перед <code>continue</code> (будет двойной инкремент)<br />
✅ В классической форме пост-выражение выполнится автоматически</p>

<p><strong>Ошибка 3: Неправильное условие</strong><br />
❌ Форма с пропуском последнего шага: <code>for i := 0; i &lt; 9; i++</code><br />
❌ Форма с лишней итерацией: <code>for i := 0; i &lt;= 10; i++</code><br />
✅ Корректная форма (любая из двух эквивалентов): <code>for i := 0; i &lt; 10; i++</code> или <code>for i := 0; i &lt;= 9; i++</code><br />
Обе ✅-формы дают итерацию i = 0..9 (десять шагов).</p>

<p><strong>Ошибка 4: Использование разных переменных</strong><br />
❌ Первый цикл с <code>i</code>, второй с <code>j</code><br />
✅ Можно использовать одну переменную или разные, но логика одинаковая</p>

<p><strong>Ошибка 5: Забыть основной инкремент</strong><br />
❌ В while-стиле забыть <code>i++</code> после вывода<br />
✅ Инкремент нужен И в блоке с continue, И в основном пути</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Использовать другие операторы вывода</strong><br />
❌ <code>fmt.Print</code> или <code>fmt.Printf</code><br />
✅ Только <code>fmt.Println</code></p>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Представьте конвейер с автоматическим и ручным управлением. В автоматическом режиме (классический for) лента движется сама &mdash; даже если вы пропустите предмет, лента продвинется. В ручном режиме (while-стиль) вы сами двигаете ленту &mdash; если забудете продвинуть перед пропуском предмета, будете обрабатывать один и тот же предмет бесконечно.</p>
