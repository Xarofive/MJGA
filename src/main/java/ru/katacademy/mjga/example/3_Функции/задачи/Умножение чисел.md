<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">✖️ Умножение чисел</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться сохранять промежуточные результаты вычислений в переменных для улучшения читаемости кода и облегчения отладки.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В промышленном программировании важно писать код, который легко понимать и отлаживать. Сохранение промежуточных результатов в именованных переменных делает код самодокументируемым и упрощает поиск ошибок. Это особенно критично в финансовых расчётах, научных вычислениях и системах обработки данных.</p>

<p style="margin: 0;"><strong>Задача:</strong> реализовать функцию умножения с явным сохранением результата в переменной.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Объявление переменных через var</h4>

<p>В Go переменные можно объявлять явно с указанием типа:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Объявление с типом
var result int
var price float64
// Объявление с инициализацией
var result int = 42
var name string = "Go"
// Краткая форма (тоже корректна, но здесь тренируем var)
result := 42  // автоматическое определение типа</code></pre>

<p style="margin-bottom: 0;">Явное объявление через <code>var</code> делает намерения программиста очевидными.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Промежуточные переменные</h4>

<p>Преимущества использования промежуточных переменных:</p>

<ul style="margin: 10px 0;">
	<li><strong>Читаемость:</strong> имя переменной объясняет смысл значения</li>
	<li><strong>Отладка:</strong> можно проверить промежуточное значение в отладчике</li>
	<li><strong>Переиспользование:</strong> результат можно использовать многократно</li>
	<li><strong>Оптимизация:</strong> избегаем повторных вычислений</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для умножения двух чисел:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">📝 Требования к функции</h4>

<ul style="margin: 10px 0;">
	<li>Имя функции: <code>Multiply</code></li>
	<li>Не указывайте <code>package</code> и <code>import</code> &mdash; только код функции</li>
	<li>Функция должна быть единственной в решении</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Сигнатура функции</h4>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func Multiply(a, b int) int</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>a</code> &mdash; первое число типа <code>int</code></li>
	<li><code>b</code> &mdash; второе число типа <code>int</code></li>
	<li>Возвращает произведение типа <code>int</code></li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">⚡ Рекомендуемая переменная (для тренировки <code>var</code>)</h4>

<p>Для тренировки рекомендуем объявить переменную через <code>var</code>. Можно и сразу вернуть выражение через <code>return a * b</code> &mdash; оба варианта корректны:</p>

<table style="width: 100%; margin: 10px 0; border-collapse: collapse;">
	<tbody>
		<tr style="background: #f5f5f5;">
			<th style="padding: 8px; text-align: left; border: 1px solid #ddd;">Переменная</th>
			<th style="padding: 8px; text-align: left; border: 1px solid #ddd;">Тип</th>
			<th style="padding: 8px; text-align: left; border: 1px solid #ddd;">Назначение</th>
		</tr>
		<tr>
			<td style="padding: 8px; border: 1px solid #ddd;"><code>result</code></td>
			<td style="padding: 8px; border: 1px solid #ddd;"><code>int</code></td>
			<td style="padding: 8px; border: 1px solid #ddd;">Хранит произведение <code>a * b</code></td>
		</tr>
	</tbody>
</table>

<h4 style="color: #e65100; margin-top: 16px;">📌 Алгоритм работы</h4>

<p style="margin: 10px 0; color: #555;"><em>Здесь мы тренируем <code>var</code> с явным типом. В реальном Go обычно используют <code>:=</code>, но сейчас важно закрепить полную форму объявления.</em></p>

<ol style="margin: 10px 0;">
	<li>Объявите переменную <code>result</code> типа <code>int</code> (через <code>var</code> или <code>:=</code>)</li>
	<li>Вычислите произведение <code>a * b</code> и сохраните в <code>result</code></li>
	<li>Верните <code>result</code> через оператор <code>return</code></li>
</ol>

<ul style="margin: 10px 0;">
	<li>Не выводите данные через <code>fmt.Print*</code></li>
	<li>Не создавайте дополнительные функции</li>
</ul>
</div>

<div style="background: #ffebee; border: 1px solid #ef5350; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Путаница со стилем объявления</strong><br />
<code>result := a * b</code> и <code>var result int; result = a * b</code> &mdash; оба варианта корректны.<br />
Для тренировки можно использовать форму с <code>var</code>.</p>

<p><strong>Ошибка 2: Неправильное выражение в return</strong><br />
Проверяйте, что возвращается именно произведение <code>a * b</code>.<br />
И <code>return a * b</code>, и <code>return result</code> корректны.</p>

<p><strong>Ошибка 3: Неправильная формула</strong><br />
Проверяйте, что вычисляется именно <code>a * b</code>, а не сумма или другое выражение.</p>

<p><strong>Ошибка 4: Неправильный тип</strong><br />
❌ <code>var result float64</code><br />
✅ <code>var result int</code></p>

<p><strong>Ошибка 5: Вывод внутри функции</strong><br />
❌ <code>fmt.Println(result)</code><br />
✅ Возвращайте результат через <code>return</code> (например <code>return result</code>)</p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Возврат не того значения</strong><br />
Убедитесь, что <code>return</code> отдаёт именно результат умножения.</p>
</div>
