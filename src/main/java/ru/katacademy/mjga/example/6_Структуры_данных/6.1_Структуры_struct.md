# Структуры (Structs) в Go

## Что такое структуры?

**Структура (struct)** — это пользовательский тип данных, который позволяет группировать различные типы данных под одним именем. Структуры помогают организовывать связанные данные вместе и создавать более сложные и выразительные абстракции.

## Объявление структуры

Структуру можно объявить следующим образом:

```go
type Person struct {
    FirstName string
    LastName  string
    Age       int
    Email     string
}
```

Здесь мы создали структуру `Person` с четырьмя полями: `FirstName`, `LastName`, `Age` и `Email`.

## Создание экземпляров структуры

Существует несколько способов создания экземпляров структуры.

### Создание с нулевыми значениями

```go
var person Person
// Все поля получат нулевые значения своих типов:
// FirstName: "" (пустая строка)
// LastName: "" (пустая строка)
// Age: 0
// Email: "" (пустая строка)
```

### Создание с использованием литерала структуры

```go
person1 := Person{
    FirstName: "Иван",
    LastName:  "Иванов",
    Age:       30,
    Email:     "ivan@example.com",
}
```

Можно также не указывать имена полей, но тогда нужно указать значения для всех полей в порядке их объявления:

```go
person2 := Person{"Мария", "Петрова", 25, "maria@example.com"}
```

Рекомендуется использовать именованные поля, так как это делает код более читаемым и устойчивым к изменениям структуры.

### Создание с использованием оператора new

```go
person3 := new(Person)
// Все поля получат нулевые значения
// person3 — это указатель на структуру (*Person)
```

## Доступ к полям структуры

Доступ к полям структуры осуществляется через оператор точки (`.`):

```go
person := Person{FirstName: "Иван", LastName: "Иванов", Age: 30}

// Чтение полей
fmt.Println("Имя:", person.FirstName)
fmt.Println("Фамилия:", person.LastName)
fmt.Println("Возраст:", person.Age)

// Изменение полей
person.Age = 31
person.Email = "ivan@example.com"
fmt.Println("Новый возраст:", person.Age)
fmt.Println("Email:", person.Email)
```

## Вложенные структуры

Структуры могут содержать другие структуры в качестве полей, что позволяет создавать сложные иерархические типы данных:

```go
type Address struct {
    Street  string
    City    string
    Country string
    ZipCode string
}

type Employee struct {
    FirstName string
    LastName  string
    Age       int
    Address   Address  // Вложенная структура
    Contact   struct { // Анонимная вложенная структура
        Email string
        Phone string
    }
}
```

Доступ к полям вложенных структур:

```go
employee := Employee{
    FirstName: "Иван",
    LastName:  "Иванов",
    Age:       30,
    Address: Address{
        Street:  "Пушкина",
        City:    "Москва",
        Country: "Россия",
        ZipCode: "123456",
    },
}

// Указываем значения для анонимной вложенной структуры
employee.Contact.Email = "ivan@example.com"
employee.Contact.Phone = "+7(987)654-32-10"

// Доступ к полям вложенной структуры
fmt.Println("Город:", employee.Address.City)
fmt.Println("Email:", employee.Contact.Email)
```

## Методы для структур

В Go можно определять методы, связанные со структурами, что позволяет добавлять функциональность к типам:

```go
type Rectangle struct {
    Width  float64
    Height float64
}

// Метод, вычисляющий площадь прямоугольника
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

// Метод, изменяющий ширину и высоту (с указателем на структуру)
func (r *Rectangle) Resize(factor float64) {
    r.Width *= factor
    r.Height *= factor
}
```

Использование методов:

```go
rect := Rectangle{Width: 10, Height: 5}

// Вызов метода для вычисления площади
area := rect.Area()
fmt.Println("Площадь:", area) // 50

// Вызов метода для изменения размеров
rect.Resize(2)
fmt.Println("Новая ширина:", rect.Width)    // 20
fmt.Println("Новая высота:", rect.Height)   // 10
fmt.Println("Новая площадь:", rect.Area())  // 200
```

## Примеры использования структур

### Пример: Структура книги

```go
package main

import "fmt"

type Book struct {
    Title  string
    Author string
    Year   int
    Price  float64
}

func (b Book) Info() string {
    return fmt.Sprintf("%s, автор: %s (%d), цена: %.2f руб.", 
                      b.Title, b.Author, b.Year, b.Price)
}

func (b *Book) ApplyDiscount(percent float64) {
    b.Price = b.Price * (1 - percent/100)
}

func main() {
    book := Book{
        Title:  "Война и мир",
        Author: "Лев Толстой",
        Year:   1869,
        Price:  599.99,
    }

    fmt.Println(book.Info())

    book.ApplyDiscount(15)
    fmt.Println("После скидки 15%:", book.Info())
}
```

### Пример: Вложенные структуры для адресной книги

```go
package main

import "fmt"

type Address struct {
    Street  string
    City    string
    Country string
}

type Person struct {
    Name    string
    Age     int
    Address Address
}

func main() {
    contacts := []Person{
        {
            Name: "Иван Иванов",
            Age:  30,
            Address: Address{
                Street:  "Ленина 15",
                City:    "Москва",
                Country: "Россия",
            },
        },
        {
            Name: "Мария Петрова",
            Age:  25,
            Address: Address{
                Street:  "Гагарина 21",
                City:    "Санкт-Петербург",
                Country: "Россия",
            },
        },
    }

    for _, person := range contacts {
        fmt.Printf("%s (%d лет) - %s, %s, %s\n",
                  person.Name, person.Age,
                  person.Address.Street,
                  person.Address.City,
                  person.Address.Country)
    }
}
```

## Итоги

- Структуры в Go позволяют группировать связанные данные вместе в один тип.
- Существует несколько способов создания экземпляров структур, включая литералы структуры и оператор new.
- Доступ к полям структуры осуществляется через оператор точки (`.`).
- Структуры могут содержать вложенные структуры для организации иерархических данных.
- К структурам можно добавлять методы, расширяя их функциональность.

Структуры — один из наиболее мощных и гибких инструментов в Go, позволяющий создавать сложные абстракции и модели данных, сохраняя при этом простоту и ясность кода.
