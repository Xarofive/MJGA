<h1 style="text-align: center; color: #2c3e50; margin-top: 0;">📦 Маркировка посылок для доставки</h1>

<div style="background: #e8f5e9; border: 2px solid #4caf50; border-radius: 8px; padding: 14px; margin: 20px 0;">
<h3 style="color: #2e7d32; margin-top: 0;">📌 Цель задачи</h3>

<p style="margin: 0;">Научиться использовать условные операторы для классификации данных на основе нескольких критериев и корректно обрабатывать недопустимые входные значения.</p>
</div>

<div style="background: linear-gradient(to right, #ffeaa7, #fdcb6e); border-radius: 8px; padding: 16px; margin: 20px 0;">
<h3 style="color: #2d3436; margin-top: 0;">💼 Контекст и мотивация</h3>

<p style="margin-bottom: 10px;">В логистических компаниях автоматическая маркировка посылок критически важна для безопасной транспортировки и оптимизации процессов. Правильная классификация по весу и хрупкости позволяет предотвратить повреждения, снизить расходы на страховые выплаты и повысить удовлетворённость клиентов.</p>

<p style="margin: 0;"><strong>Задача:</strong> создать функцию для определения типа обработки посылки при доставке.</p>
</div>

<h2 style="color: #34495e; margin-top: 28px;">💡 Что нужно знать</h2>

<div style="background: #f0f4f8; border-radius: 8px; padding: 16px; margin: 16px 0;">
<h4 style="color: #455a64; margin-top: 0;">Приоритет условий</h4>

<p>При классификации важен порядок проверок:</p>

<pre style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;">
<code class="language-go">// Сначала проверяем критичные условия
if недопустимо {
    return "ошибка"
}
// Затем особые случаи
if хрупкое {
// специальная обработка хрупких товаров
}
// В конце — стандартные случаи
if обычное {
// стандартная обработка
}</code></pre>

<p style="margin-bottom: 0;">Проверяйте от более специфичных к более общим условиям.</p>
</div>

<div style="background: #e3f2fd; border: 1px solid #90caf9; border-radius: 8px; padding: 14px; margin: 16px 0;">
<h4 style="color: #1565c0; margin-top: 0;">🎯 Работа с булевыми параметрами</h4>

<p>Булевый тип <code>bool</code> в условиях:</p>

<ul style="margin: 10px 0;">
	<li><strong>Прямая проверка:</strong> <code>if fragile</code> эквивалентно <code>if fragile == true</code></li>
	<li><strong>Отрицание:</strong> <code>if !fragile</code> проверяет на <code>false</code></li>
	<li><strong>Комбинирование:</strong> можно сочетать с другими условиями через <code>&amp;&amp;</code> и <code>||</code></li>
	<li><strong>Приоритет:</strong> булевые условия часто определяют специальную обработку</li>
</ul>
</div>

<h2 style="color: #34495e; margin-top: 28px;">📋 Техническое задание</h2>

<div style="background: #fff3e0; border: 2px solid #ff9800; border-radius: 8px; padding: 16px; margin: 16px 0;">
<p style="margin-top: 0;"><strong>Создайте функцию для маркировки посылок:</strong></p>

<h4 style="color: #e65100; margin-top: 16px;">🔧 Функция deliveryLabel</h4>

<p><strong>Сигнатура:</strong></p>

<div style="background: #263238; color: #aed581; padding: 12px; border-radius: 4px; margin: 10px 0;"><code>func deliveryLabel(weight int, fragile bool) string</code></div>

<p><strong>Параметры:</strong></p>

<ul style="margin: 10px 0;">
	<li><code>weight</code> (int) &mdash; вес посылки в килограммах</li>
	<li><code>fragile</code> (bool) &mdash; признак хрупкого груза (<code>true</code> &mdash; хрупкий, <code>false</code> &mdash; обычный)</li>
</ul>

<p><strong>Правила маркировки:</strong></p>

<ul style="margin: 10px 0;">
	<li><strong>&quot;invalid&quot;</strong> &mdash; недопустимые данные (вес &le; 0)</li>
	<li><strong>&quot;handle with care&quot;</strong> &mdash; хрупкий груз весом до 10 кг включительно</li>
	<li><strong>&quot;special handling&quot;</strong> &mdash; хрупкий груз весом более 10 кг</li>
	<li><strong>&quot;standard&quot;</strong> &mdash; обычный груз весом до 10 кг включительно</li>
	<li><strong>&quot;bulk&quot;</strong> &mdash; обычный груз весом более 10 кг</li>
</ul>

<p><strong>Требования к реализации:</strong></p>

<ul style="margin: 10px 0;">
	<li>Обязательно использовать условные операторы <code>if</code></li>
	<li>Запрещено использовать циклы (<code>for</code>) и оператор <code>switch</code></li>
	<li>Сначала проверять валидность веса</li>
	<li>Приоритет отдавать хрупким грузам</li>
</ul>

<h4 style="color: #e65100; margin-top: 16px;">📝 Формат решения</h4>

<ul style="margin: 10px 0;">
	<li>Включите в ответ полную сигнатуру функции, например <code>func deliveryLabel(weight int, fragile bool) string { ... }</code></li>
	<li>НЕ указывайте <code>package main</code></li>
	<li>НЕ добавляйте <code>import</code> директивы</li>
</ul>
</div>

<h4 style="color: #c62828; margin-top: 0;">⚠️ Частые ошибки и антипаттерны</h4>

<p><strong>Ошибка 1: Неправильный порядок проверок</strong><br />
❌ Сначала проверять тип груза, потом валидность<br />
✅ Всегда начинайте с проверки валидности данных</p>

<p><strong>Ошибка 2: Неверная граница веса</strong><br />
❌ 10 кг считается тяжёлым грузом<br />
✅ 10 кг &mdash; это ещё лёгкий груз, тяжёлый начинается с 11 кг</p>

<p><strong>Ошибка 3: Игнорирование нуля</strong><br />
❌ Вес 0 кг считается допустимым<br />
✅ Вес должен быть строго больше 0</p>

<p><strong>Ошибка 4: Неправильная обработка хрупкости</strong><br />
❌ Проверять вес до хрупкости<br />
✅ После валидации сначала проверять хрупкость, затем вес</p>

<p><strong>Ошибка 5: Опечатки в строках</strong><br />
❌ <code>&quot;Handle with care&quot;</code>, <code>&quot;BULK&quot;</code>, <code>&quot;standart&quot;</code><br />
✅ Точные значения: <code>&quot;invalid&quot;</code>, <code>&quot;handle with care&quot;</code>, <code>&quot;special handling&quot;</code>, <code>&quot;standard&quot;</code>, <code>&quot;bulk&quot;</code></p>

<p style="margin-bottom: 0;"><strong>Ошибка 6: Неправильная сигнатура</strong><br />
❌ <code>func deliveryLabel(w int, f bool)</code> &mdash; забыт тип возврата<br />
✅ <code>func deliveryLabel(weight int, fragile bool) string</code> &mdash; имена параметров могут быть любыми, главное &mdash; типы</p>
</div>

<h4 style="color: #f57c00; margin-top: 0;">💡 Подсказка</h4>

<p>Представьте конвейер на складе: сначала отбраковываются посылки с некорректными данными. Затем хрупкие грузы направляются в специальную зону (и там уже сортируются по весу). Обычные грузы идут в стандартную зону и также делятся по весу. Порядок проверок критичен &mdash; хрупкость имеет приоритет над весом при выборе типа обработки.</p>
