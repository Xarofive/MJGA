# Java Core, модуль 2, лекция 6
# `static`, `final` и константы

## TL;DR

- Обычное поле принадлежит конкретному объекту.
- `static` делает поле или метод общим для класса.
- К статическому члену обращаются через имя класса.
- Статический контекст выполняется без конкретного объекта, поэтому в нем нет `this`.
- `final` запрещает повторно присвоить переменной или полю другое значение.
- Для ссылки `final` фиксирует ссылку, но не запрещает менять сам объект.
- Константы обычно объявляются как `static final`.

## Что продолжаем из предыдущих лекций

До этого каждое поле описывало состояние конкретного объекта. Но не все значения принадлежат отдельному товару или счету. Некоторые данные общие для всего класса, а некоторые характеристики объекта после создания не должны изменяться.

## Зачем это нужно

`static` позволяет явно отделить общее состояние и операции класса от состояния отдельных объектов. `final` показывает, какие ссылки и значения нельзя переназначать. Вместе они помогают точнее выразить модель и убрать непонятные числовые литералы из расчетов.

Неправильное применение этих модификаторов, наоборот, приводит к общему изменяемому состоянию и неожиданной связи между объектами. Поэтому важно понимать не только синтаксис, но и смысл.

## Члены объекта и члены класса

Поле или метод без `static` принадлежит конкретному объекту.

```java
public class Product {
    // Поля без static принадлежат каждому объекту отдельно.
    private String name;
    private int quantity;

    // Объектный метод работает с состоянием конкретного Product.
    public int getQuantity() {
        return quantity;
    }
}
```

У каждого объекта свои значения:

```java
Product keyboard = new Product("Keyboard", 10);
Product mouse = new Product("Mouse", 25);
```

`keyboard` и `mouse` имеют независимое поле `quantity`.

Член класса - поле или метод с модификатором `static`, который относится к самому классу, а не к отдельному объекту.

## Статическое поле

Статическое поле - одно общее поле класса для всех его объектов.

```java
public class Product {
    // Одно общее поле для всех объектов Product.
    private static int createdCount;

    // Отдельное значение у каждого объекта.
    private String name;

    public Product(String name) {
        this.name = name;

        // Увеличивается общий счетчик класса.
        createdCount++;
    }
}
```

При каждом создании `Product` увеличивается одно общее значение:

```java
new Product("Keyboard");
new Product("Mouse");
new Product("Monitor");
```

`createdCount` станет равно `3`.

У каждого объекта свое поле `name`, но поле `createdCount` общее.

## Обращение к статическому члену

К статическому члену обращаются через имя класса:

```java
// Статический метод вызывается через имя класса.
int count = Product.getCreatedCount();
```

Пример метода:

```java
// Метод static может работать без объекта Product.
public static int getCreatedCount() {
    return createdCount;
}
```

Технически Java иногда разрешает обратиться к статическому члену через объект, но так писать не следует:

```java
product.getCreatedCount();
```

Такая запись создает ложное впечатление, что результат относится к конкретному объекту.

Правильно:

```java
Product.getCreatedCount();
```

## Статический метод

Статический метод принадлежит классу и вызывается без создания объекта:

```java
public class PriceMath {
    public static long calculateTotal(
            long priceInCents,
            int quantity
    ) {
        return priceInCents * quantity;
    }
}
```

Вызов:

```java
long total = PriceMath.calculateTotal(499_900L, 2);
```

Статический метод удобен, когда операция:

- не использует состояние конкретного объекта;
- полностью определяется аргументами;
- логически относится к классу вспомогательных операций.

## Почему статический метод не видит поля объекта

```java
public class Product {
    private String name;

    public static String getName() {
        // Ошибка: не указан конкретный объект,
        // поле name у каждого объекта свое.
        return name; // ошибка компиляции
    }
}
```

При вызове:

```java
Product.getName();
```

не указан конкретный объект. Непонятно, чье поле `name` нужно вернуть.

В статическом контексте нет ссылки `this`, потому что нет текущего объекта.

Статический контекст - код, который выполняется без привязки к конкретному объекту. К нему относятся статические методы и инициализация статических полей.

Статический метод может напрямую обращаться:

- к своим параметрам и локальным переменным;
- к статическим полям;
- к другим статическим методам.

Для обращения к полям объекта ему нужна явная ссылка:

```java
public static String readName(Product product) {
    return product.name;
}
```

Но простой getter все равно естественнее оставить объектным:

```java
public String getName() {
    return name;
}
```

## Когда `static` использовать не нужно

Не делайте поле статическим только для удобного доступа:

```java
public class Product {
    // Плохо: все товары будут делить одно имя и один остаток.
    public static String name;
    public static int quantity;
}
```

Теперь все товары делят одно имя и один остаток. Создать независимые товары невозможно.

Если значение описывает конкретный объект, оно должно быть полем объекта.

Вопрос для выбора:

```text
Это значение одно для всего класса
или свое у каждого объекта?
```

## `final`

`final` - модификатор, который запрещает повторное присваивание после инициализации.

Локальная переменная:

```java
// final запрещает повторное присваивание.
final int maxAttempts = 3;
maxAttempts = 5; // ошибка компиляции
```

Поле объекта:

```java
public class Product {
    // Значение поля задается один раз.
    private final String name;

    public Product(String name) {
        // final-поле инициализируется в конструкторе.
        this.name = name;
    }
}
```

После завершения конструктора этому полю нельзя присвоить другую строку:

```java
this.name = "Other"; // ошибка вне инициализации
```

`final` не означает, что вся переменная автоматически получает значение. Поле нужно инициализировать:

- при объявлении;
- или в каждом конструкторе.

## `final` и ссылки

Для ссылочной переменной `final` запрещает заменить ссылку, но не обязательно запрещает изменить объект.

```java
// Ссылку names нельзя заменить.
final ArrayList<String> names = new ArrayList<>();

// Но содержимое самого списка менять можно.
names.add("Alice");          // разрешено
names.add("Bob");            // разрешено
names = new ArrayList<>();   // ошибка
```

Ссылка `names` всегда указывает на один список. Сам список остается изменяемым.

Изменяемость - возможность изменить состояние объекта после создания.

Поэтому `final` и неизменяемый объект - разные понятия.

## Неизменяемое поле

Если характеристика объекта не должна меняться после создания, поле можно сделать `final`:

```java
public class BankAccount {
    private final long accountNumber;
    private final String owner;
    private long balanceInCents;

    public BankAccount(
            long accountNumber,
            String owner,
            long balanceInCents
    ) {
        this.accountNumber = accountNumber;
        this.owner = owner;
        this.balanceInCents = balanceInCents;
    }
}
```

Номер счета и владелец фиксируются при создании. Баланс изменяется, поэтому он не `final`.

`final` помогает явно выразить модель: какие части состояния постоянны, а какие изменяются.

## Константа

Константа - именованное значение, которое не меняется во время работы программы и имеет общий смысл для класса или приложения.

В Java константу обычно объявляют сочетанием:

```java
static final
```

Пример:

```java
public class DeliveryCalculator {
    // static: значения общие для класса.
    // final: значения нельзя переназначить.
    private static final long BASE_PRICE_IN_CENTS = 20_000L;
    private static final long PRICE_PER_KM_IN_CENTS = 150L;

    public long calculate(int distanceKm) {
        return BASE_PRICE_IN_CENTS
                + PRICE_PER_KM_IN_CENTS * distanceKm;
    }
}
```

Почему оба модификатора:

- `static` - значение одно для всего класса;
- `final` - значение нельзя переназначить.

## Имена констант

Константы по соглашению называют заглавными буквами с подчеркиваниями:

```java
MAX_ATTEMPTS
DEFAULT_CAPACITY
BASE_PRICE_IN_CENTS
```

Это позволяет отличить константу от обычного поля.

## Магические числа

Магическое число - числовой литерал, смысл которого неясен из кода.

Плохо:

```java
// Плохо: смысл чисел не виден из формулы.
return 20_000L + distanceKm * 150L;
```

Читатель должен догадаться, что означают `20_000` и `150`.

Лучше:

```java
// Лучше: имена объясняют назначение значений.
private static final long BASE_PRICE_IN_CENTS = 20_000L;
private static final long PRICE_PER_KM_IN_CENTS = 150L;

return BASE_PRICE_IN_CENTS
        + distanceKm * PRICE_PER_KM_IN_CENTS;
```

Именованное значение объясняет смысл и дает одно место для изменения.

Не каждый литерал нужно превращать в константу. Значения `0`, `1`, `-1` часто понятны из операции:

```java
if (quantity <= 0) {
    return false;
}
```

## Публичные и приватные константы

Если константа является частью публичного контракта, она может быть `public`:

```java
public static final int MAX_NAME_LENGTH = 100;
```

Вызов:

```java
if (name.length() > Product.MAX_NAME_LENGTH) {
    // ...
}
```

Но большинство внутренних настроек лучше оставлять `private`:

```java
private static final long PRICE_PER_KM_IN_CENTS = 150L;
```

Публичная константа создает зависимость внешнего кода от значения и имени. Не нужно открывать ее без причины.

## `static final` для изменяемого объекта

```java
private static final ArrayList<String> NAMES =
        new ArrayList<>();
```

Ссылку `NAMES` нельзя заменить, но список можно изменить:

```java
NAMES.add("Alice");
NAMES.clear();
```

Поэтому запись `static final` не делает изменяемый объект константным по содержимому.

Для настоящих констант проще использовать:

- примитивные значения;
- `String`;
- другие неизменяемые типы.

Коллекции и неизменяемость будут дополнительно рассматриваться в модуле Collections Framework.

## Счетчик созданных объектов

Полный пример совместного использования членов объекта и класса:

```java
public class Product {
    private static int createdCount;

    private final String name;
    private long priceInCents;

    public Product(String name, long priceInCents) {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException(
                    "Name must not be blank"
            );
        }
        if (priceInCents < 0) {
            throw new IllegalArgumentException(
                    "Price must not be negative"
            );
        }

        this.name = name;
        this.priceInCents = priceInCents;
        createdCount++;
    }

    public String getName() {
        return name;
    }

    public long getPriceInCents() {
        return priceInCents;
    }

    public static int getCreatedCount() {
        return createdCount;
    }
}
```

Здесь:

- `name` свое у каждого товара и не переназначается;
- `priceInCents` свое у каждого товара и может изменяться;
- `createdCount` один для всего класса;
- `getCreatedCount` вызывается через `Product`.

## Частые ошибки

### Ошибка 1. Делать поля объекта статическими

Если `name` или `quantity` объявлены `static`, все объекты будут использовать одно общее значение.

### Ошибка 2. Вызывать статический метод через объект

Такая запись скрывает тот факт, что метод относится ко всему классу. Используйте `ClassName.method()`.

### Ошибка 3. Считать ссылку `final` неизменяемым объектом

Ссылку нельзя заменить, но состояние изменяемого объекта по ней все еще можно менять.

### Ошибка 4. Открывать все константы как `public`

Публичной должна быть только та константа, которая действительно входит в контракт класса.
