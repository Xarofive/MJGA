# Java Core, модуль 2, лекция 2
# Конструктор, `this` и валидация

## Результат

После этой лекции вы сможете:

- объявить собственный конструктор;
- создать объект сразу в корректном состоянии;
- использовать `this` при совпадении имен;
- проверить обязательные данные при создании объекта.

## Как в классе появляется конструктор

Конструктор может появиться в классе двумя способами:

1. **Явный конструктор** - программист объявляет его в коде.
2. **Неявный конструктор по умолчанию** - компилятор Java добавляет его
   автоматически.

Явный конструктор виден в исходном коде:

```java
public class Product {
    // Этот конструктор явно объявил программист.
    public Product(String name) {
    }
}
```

Неявный конструктор по умолчанию не записан в исходном коде. Компилятор
добавляет его автоматически, если программист не объявил в классе ни одного
конструктора:

```java
public class Product {
    // Конструкторов в исходном коде нет.
}
```

Для такого класса доступен вызов:

```java
// Компилятор добавил Product() без параметров.
Product product = new Product();
```

Важно: неявный конструктор по умолчанию всегда не имеет параметров. Явный
конструктор без параметров выглядит похоже, но его пишет сам программист,
поэтому конструктором по умолчанию он не является. Если программист объявит
хотя бы один собственный конструктор, компилятор больше не добавит конструктор
по умолчанию.

## Проблема пошагового создания

Даже с закрытыми полями объект можно оставить незаполненным:

```java
public class Product {
    // Закрытые поля пока получают только значения Java по умолчанию.
    private String name;
    private long priceInCents;
    private int quantity;
}
```

Если выполнить:

```java
// Конструктор без параметров не требует обязательных данных.
Product product = new Product();
```

Java присвоит полям значения по умолчанию:

```text
name = null
priceInCents = 0
quantity = 0
```

Если товар без названия недопустим, объект уже создан некорректным.

## Конструктор

**Конструктор** - специальный блок кода, который выполняется при создании объекта
через `new`.

Он:

- называется так же, как класс;
- не имеет возвращаемого типа;
- получает начальные данные через параметры.

```java
public class Product {
    // Поля будут заполнены при создании объекта.
    private String name;
    private long priceInCents;
    private int quantity;

    public Product(String name, long priceInCents, int quantity) {
        // this указывает на поля создаваемого объекта.
        this.name = name;
        this.priceInCents = priceInCents;
        this.quantity = quantity;
    }
}
```

Создание объекта:

```java
// Все обязательные значения передаются в конструктор.
Product keyboard = new Product("Keyboard", 499_900L, 8);
```

Все обязательные значения передаются сразу.

## Зачем нужен `this`

**`this`** - ключевое слово Java, которое обозначает текущий объект. Внутри
конструктора текущим считается объект, который сейчас создается. Внутри
обычного объектного метода - объект, у которого вызвали этот метод.

**Экземпляр класса** - конкретный объект, созданный по описанию класса. В
рамках курса слова «объект» и «экземпляр класса» обозначают одно и то же.
Фраза «экземпляр класса `Product`» дополнительно подчеркивает, по описанию
какого класса создан объект.

**Объектный метод** - метод без ключевого слова `static`, который вызывается у
конкретного экземпляра класса:

```java
// getName() вызывается у конкретного объекта product.
String name = product.getName();
```

Такой метод может обращаться к полям этого экземпляра и использовать `this`.
Статические методы принадлежат классу, а не отдельному экземпляру, поэтому
слово `this` внутри них недоступно. Подробнее `static` будет разобран в
следующей лекции.

`this` само является ссылкой на текущий экземпляр класса. Эту ссылку можно:

- использовать для обращения к полю: `this.name`;
- использовать для вызова метода: `this.getName()`;
- передать в другой метод;
- вернуть из метода.

```java
public class Product {
    private String name;

    public String getName() {
        // Через this обращаемся к полю текущего экземпляра Product.
        return this.name;
    }

    public void printName() {
        // Через this вызываем метод текущего экземпляра Product.
        System.out.println(this.getName());
    }

    public Product getCurrentProduct() {
        // Возвращаем ссылку на тот экземпляр, у которого вызвали метод.
        return this;
    }
}
```

Например:

```java
Product product = new Product();

// Метод вернет ссылку на тот же объект.
Product sameProduct = product.getCurrentProduct();

// Обе переменные ссылаются на один экземпляр Product.
System.out.println(product == sameProduct); // true
```

Чаще всего `this` встречается, когда поле и параметр имеют одинаковое имя.
Рассмотрим две разные переменные:

```java
public class Product {
    // Поле принадлежит объекту и хранится, пока существует объект.
    private String name;

    public Product(String name) {
        // Параметр принадлежит только этому вызову конструктора.
    }
}
```

Поле `name` объявлено в классе. Параметр `name` объявлен в конструкторе.
Несмотря на одинаковые имена, это разные переменные.

Если внутри конструктора написать только `name`, Java выбирает ближайшую
переменную с таким именем - параметр конструктора. Параметр как бы закрывает
поле с тем же именем. Такая ситуация называется **затенением имени**.

Поэтому эта запись не заполняет поле:

```java
public Product(String name) {
    // Обе записи name обозначают параметр конструктора.
    name = name;
}
```

Справа читается параметр `name`, а результат записывается обратно в тот же
параметр. Поле объекта остается равным `null`.

`this.name` явно указывает, что нужно поле текущего объекта:

```java
public Product(String name) {
    // Слева поле объекта, справа параметр конструктора.
    this.name = name;
}
```

Слева находится поле текущего объекта, справа - параметр конструктора.

Можно было бы дать параметру другое имя:

```java
public Product(String productName) {
    // Затенения нет, поэтому к полю можно обратиться просто по имени.
    name = productName;
}
```

Такой код корректен. Но в Java параметры конструктора обычно называют так же,
как соответствующие поля. Запись `this.name = name` сразу показывает связь:
значение параметра `name` сохраняется в поле `name` создаваемого объекта.

Такой стиль является обычным для Java:

```java
// Один и тот же прием используется для всех одноименных полей и параметров.
this.name = name;
this.priceInCents = priceInCents;
this.quantity = quantity;
```

## Попробуйте самостоятельно

Создайте класс `Product` и проверьте разницу между параметром конструктора и
полем объекта.

1. Скопируйте класс с ошибочным присваиванием:

```java
public class Product {
    private String name;

    public Product(String name) {
        // Обе записи name пока обозначают параметр.
        name = name;
    }

    public String getName() {
        // Возвращаем значение поля объекта.
        return name;
    }
}
```

2. Создайте объект в методе `main` и выведите его имя:

```java
public class Main {
    public static void main(String[] args) {
        // Передаем строку в параметр конструктора.
        Product product = new Product("Keyboard");

        // Проверяем, было ли заполнено поле объекта.
        System.out.println(product.getName());
    }
}
```

Программа выведет:

```text
null
```

Параметр получил значение `"Keyboard"`, но поле объекта осталось незаполненным.

3. Исправьте в конструкторе одну строку:

```java
public Product(String name) {
    // this.name обозначает поле, а name - параметр.
    this.name = name;
}
```

4. Снова запустите `Main`. Теперь программа должна вывести:

```text
Keyboard
```

5. Для дополнительной проверки создайте второй объект:

```java
// Каждый объект получает собственное значение поля name.
Product keyboard = new Product("Keyboard");
Product mouse = new Product("Mouse");

System.out.println(keyboard.getName()); // Keyboard
System.out.println(mouse.getName());    // Mouse
```

Так можно увидеть сразу две вещи: `this` направляет присваивание в поле
текущего объекта, а разные экземпляры класса хранят независимое состояние.

## Валидация в конструкторе

**Валидация** - проверка данных на соответствие правилам. Конструктор должен
отклонить данные, с которыми корректный объект создать невозможно.

```java
public Product(String name, long priceInCents, int quantity) {
    // Сначала проверяем все обязательные данные.
    if (name == null || name.isBlank()) {
        throw new IllegalArgumentException("name must not be blank");
    }
    if (priceInCents < 0) {
        throw new IllegalArgumentException("price must not be negative");
    }
    if (quantity < 0) {
        throw new IllegalArgumentException("quantity must not be negative");
    }

    // Поля заполняются только после успешной валидации.
    this.name = name;
    this.priceInCents = priceInCents;
    this.quantity = quantity;
}
```

Если аргумент неверен, конструктор выбрасывает исключение и объект не
создается.

Почему здесь подходит исключение, а не `boolean`? Конструктор не является
обычным методом и не возвращает результат. Его задача - либо создать
корректный объект, либо сообщить, что это невозможно.

## Конструктор по умолчанию

Вернемся к правилу из начала статьи. После объявления собственного
конструктора компилятор больше не добавляет конструктор по умолчанию:

```java
public Product(String name, long priceInCents, int quantity) {
    // Собственный конструктор отключает автоматически добавляемый Product().
}
```

Теперь вызов `new Product()` не скомпилируется. Это полезно, если пустой товар
не имеет смысла.

При необходимости конструктор без параметров можно объявить явно:

```java
public Product() {
    // Явный пустой конструктор следует добавлять только осознанно.
}
```

Но делать это только ради устранения ошибки компиляции не следует. Сначала
решите, допустим ли объект без начальных данных.

## Полный пример

```java
public class Product {
    // Состояние закрыто от прямого изменения извне.
    private String name;
    private long priceInCents;
    private int quantity;

    public Product(String name, long priceInCents, int quantity) {
        // Конструктор не допускает создание некорректного товара.
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("name must not be blank");
        }
        if (priceInCents < 0 || quantity < 0) {
            throw new IllegalArgumentException("numbers must not be negative");
        }

        this.name = name;
        this.priceInCents = priceInCents;
        this.quantity = quantity;
    }

    public boolean sell(int amount) {
        // Неуспешная продажа оставляет объект без изменений.
        if (amount <= 0 || amount > quantity) {
            return false;
        }

        quantity -= amount;
        return true;
    }

    public int getQuantity() {
        // Внешнему коду разрешено только читать остаток.
        return quantity;
    }
}
```

Объект создается корректным, а последующие операции сохраняют его инвариант.

## Словарь

- **Явный конструктор** - конструктор, объявленный программистом в исходном
  коде.
- **Неявный конструктор по умолчанию** - конструктор без параметров, который
  компилятор добавляет, если в классе не объявлено других конструкторов.
- **Конструктор** - специальный блок кода, выполняемый при создании объекта.
- **`this`** - ссылка на текущий экземпляр класса.
- **Экземпляр класса (объект)** - конкретный объект, созданный по описанию
  класса; оба термина в рамках курса обозначают одно и то же.
- **Объектный метод** - метод без `static`, вызываемый у конкретного объекта.
- **Затенение имени** - ситуация, когда параметр или локальная переменная
  скрывает поле с таким же именем.
- **Валидация** - проверка данных на соответствие установленным правилам.
