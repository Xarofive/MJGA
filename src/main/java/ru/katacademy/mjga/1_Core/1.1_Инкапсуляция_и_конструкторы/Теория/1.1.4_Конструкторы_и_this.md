# Конструкторы и `this`

## TL;DR

- Конструктор выполняется при создании объекта через `new`.
- Имя конструктора совпадает с именем класса, возвращаемого типа у него нет.
- Параметры конструктора передают обязательные начальные данные.
- `this` - ссылка на текущий объект.
- `this.field` отличает поле от одноименного параметра.
- Конструктор должен создать корректный объект или завершиться исключением.
- Конструктор по умолчанию появляется только при отсутствии собственных конструкторов.

## Что продолжаем из предыдущей лекции

Мы научились защищать уже существующий объект через закрытые поля, валидацию и осмысленные методы. Но объект, созданный через пустой `new Product()`, все еще может некоторое время оставаться незаполненным.

## Зачем это нужно

Конструктор требует обязательные данные в момент создания. Благодаря этому внешний код не может забыть заполнить важное поле, а объект не существует в промежуточном некорректном состоянии.

Конструктор также собирает начальную валидацию в одном месте. Любой код, который получил ссылку на успешно созданный объект, может рассчитывать на его базовые инварианты.

## Проблема пошагового создания объекта

В Base объект часто создавался пустым, а поля заполнялись после `new`:

```java
Product product = new Product();

// Объект уже создан, но обязательные данные еще не заполнены.
product.name = "Keyboard";
product.priceInCents = 499_900L;
product.quantity = 8;
```

Между первой и последней строкой объект существует в незаполненном состоянии:

```text
name = null
priceInCents = 0
quantity = 0
```

Код может забыть заполнить поле:

```java
Product product = new Product();
product.name = "Keyboard";
// Ошибка: цена забыта, объект остался частично заполненным.
product.quantity = 8;
```

Если поля `private`, внешний код вообще не сможет заполнить их напрямую. Для создания корректного объекта нужен конструктор.

## Что такое конструктор

Конструктор - специальный блок кода, который выполняется при создании объекта через `new`.

Основная задача конструктора - получить начальные данные и привести новый объект в корректное состояние.

### Аналогия из жизни

Конструктор похож на сборку автомобиля перед передачей владельцу. Машину
нельзя считать готовой, пока не установлены обязательные детали и не
проверена их исправность. Так же объект должен получить обязательные данные
и пройти проверки до того, как ссылка на него станет доступна внешнему коду.

Пример:

```java
public class Product {
    private String name;
    private long priceInCents;
    private int quantity;

    // Конструктор требует все обязательные данные сразу.
    public Product(String name, long priceInCents, int quantity) {
        // this.name - поле объекта, name - параметр.
        this.name = name;
        this.priceInCents = priceInCents;
        this.quantity = quantity;
    }
}
```

Создание объекта:

```java
// Аргументы передаются в конструктор при создании объекта.
Product product = new Product("Keyboard", 499_900L, 8);
```

## Чем конструктор отличается от метода

У конструктора:

- имя совпадает с именем класса;
- нет возвращаемого типа;
- он вызывается при создании объекта;
- его нельзя вызвать как обычный метод у уже созданного объекта.

Конструктор:

```java
// Конструктор: имя совпадает с именем класса,
// возвращаемого типа нет.
public Product(String name) {
    this.name = name;
}
```

Обычный метод:

```java
// Обычный метод: есть возвращаемый тип String.
public String getName() {
    return name;
}
```

Ошибка:

```java
// Ошибка: из-за void это обычный метод, а не конструктор.
public void Product(String name) {
    this.name = name;
}
```

Из-за `void` это не конструктор, а обычный метод с именем `Product`.

## Что происходит при `new`

В выражении:

```java
Product product = new Product("Keyboard", 499_900L, 8);
```

происходит несколько шагов:

1. Java выделяет память для нового объекта.
2. Поля получают значения по умолчанию.
3. Выполняется подходящий конструктор.
4. Конструктор записывает начальные значения и проверяет правила.
5. Выражение `new` возвращает ссылку на готовый объект.
6. Ссылка сохраняется в переменной `product`.

Значения по умолчанию все еще существуют, но внешний код получает ссылку только после завершения конструктора.

## Параметры конструктора

Параметры конструктора описывают данные, обязательные для создания объекта:

```java
public Product(String name, long priceInCents, int quantity) {
    // ...
}
```

Аргументы передаются при вызове:

```java
new Product("Keyboard", 499_900L, 8);
```

Как и у методов:

- параметр - переменная в объявлении;
- аргумент - конкретное переданное значение.

## Ключевое слово `this`

`this` - ссылка на текущий объект, внутри которого выполняется конструктор или объектный метод.

### Аналогия из жизни

`this` похоже на слово «я». Когда несколько товаров выполняют один и тот же
метод, каждый из них под словом `this` имеет в виду самого себя. Поэтому
`this.name` означает «поле `name` именно этого текущего объекта».

В конструкторе:

```java
public Product(String name, long priceInCents, int quantity) {
    this.name = name;
    this.priceInCents = priceInCents;
    this.quantity = quantity;
}
```

Слева:

```java
this.name
```

находится поле текущего объекта.

Справа:

```java
name
```

находится параметр конструктора.

## Затенение имени

Затенение - ситуация, когда локальная переменная или параметр имеет такое же имя, как поле.

```java
private String name;

public Product(String name) {
    // Слева поле текущего объекта, справа параметр.
    this.name = name;
}
```

Параметр `name` затеняет поле `name`. Простое имя `name` означает ближайшую переменную, то есть параметр.

`this.name` явно указывает на поле текущего объекта.

Ошибка:

```java
public Product(String name) {
    // Ошибка: параметр присваивается самому себе.
    // Поле объекта не изменяется.
    name = name;
}
```

Обе части относятся к параметру. Поле останется `null`.

## `this` в объектных методах

`this` доступен не только в конструкторе:

```java
public boolean hasSameName(Product other) {
    return this.name.equals(other.name);
}
```

Обычно `this` перед полем не пишут, если нет затенения:

```java
public String getName() {
    return name;
}
```

Обе записи равнозначны:

```java
return name;
return this.name;
```

## Валидация в конструкторе

Конструктор должен защищать инварианты с момента создания объекта:

```java
public class Product {
    private String name;
    private long priceInCents;
    private int quantity;

    public Product(String name, long priceInCents, int quantity) {
        // Валидация выполняется до присваивания полей.
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("Name must not be blank");
        }

        if (priceInCents < 0) {
            throw new IllegalArgumentException(
                    "Price must not be negative"
            );
        }

        if (quantity < 0) {
            throw new IllegalArgumentException(
                    "Quantity must not be negative"
            );
        }

        // Поля заполняются только проверенными значениями.
        this.name = name;
        this.priceInCents = priceInCents;
        this.quantity = quantity;
    }
}
```

Теперь невозможно получить объект `Product` с отрицательной ценой через этот конструктор.

## Почему конструктор не возвращает `boolean`

Обычный метод может сообщить об отказе:

```java
boolean sold = product.sell(10);
```

Конструктор должен либо создать корректный объект, либо не создать объект.

У конструктора нет возвращаемого типа, поэтому он не может вернуть `false`. Если обязательные данные недопустимы, конструктор выбрасывает исключение.

```java
Product product = new Product("", -100, -3);
```

Вызов завершится `IllegalArgumentException`, а ссылка на некорректный `Product` не будет получена.

## Конструктор по умолчанию

Если в классе не объявлено ни одного конструктора, компилятор добавляет конструктор без параметров:

```java
class Product {
    private String name;
    // Конструкторов нет: компилятор добавит Product().
}
```

Можно написать:

```java
Product product = new Product();
```

Добавленный компилятором конструктор называют конструктором по умолчанию.

### Аналогия из жизни

Конструктор по умолчанию похож на стандартную комплектацию, которая
предоставляется только тогда, когда разработчик не указал собственный
вариант сборки. Как только появляется собственная инструкция, Java перестает
догадываться и ожидает, что нужные варианты будут описаны явно.

Его можно представить так:

```java
Product() {
}
```

Это упрощенная модель: конструктор действительно создается компилятором, если разработчик не объявил ни одного собственного конструктора.

## Конструктор без параметров

Конструктор без параметров - любой явно или неявно объявленный конструктор, который не принимает параметры.

```java
public Product() {
    // Явный конструктор без параметров задает начальные значения.
    this.name = "Unknown";
    this.priceInCents = 0;
    this.quantity = 0;
}
```

Важно различать:

- конструктор по умолчанию добавляется компилятором;
- конструктор без параметров может быть написан разработчиком.

Не каждый конструктор без параметров является конструктором по умолчанию.

## Собственный конструктор отключает конструктор по умолчанию

```java
public class Product {
    private String name;

    // После объявления этого конструктора
    // компилятор уже не добавит Product().
    public Product(String name) {
        this.name = name;
    }
}
```

Теперь этот код не скомпилируется:

```java
// Ошибка компиляции: конструктора Product() больше нет.
Product product = new Product();
```

Причина: как только разработчик объявил хотя бы один конструктор, компилятор больше не добавляет конструктор по умолчанию.

Если нужны оба варианта, оба конструктора нужно объявить явно. Перегрузка конструкторов будет разобрана в следующей лекции.

## Модификатор доступа конструктора

Конструктор тоже имеет модификатор доступа:

```java
public Product(String name) {
    this.name = name;
}
```

`public` разрешает создавать объект из других пакетов.

Package-private конструктор доступен только внутри пакета:

```java
Product(String name) {
    this.name = name;
}
```

`private` конструктор доступен только внутри самого класса:

```java
private Product(String name) {
    this.name = name;
}
```

Закрытые конструкторы применяются в специальных способах создания объектов. В этом модуле основной вариант - публичный конструктор для создания корректной сущности.

## Не выполняйте в конструкторе несвязанную тяжелую работу

Конструктор должен инициализировать объект и проверить его начальное состояние.

Инициализация - назначение объекту начальных значений.

Не стоит помещать в простой конструктор:

- чтение пользовательского ввода;
- печать в консоль;
- долгие вычисления;
- загрузку данных из файла;
- обращение к базе данных;
- изменение других несвязанных объектов.

### Аналогия из жизни

При выдаче готового автомобиля логично проверить документы и комплектацию.
Нелогично в этот же момент строить для владельца гараж, прокладывать дорогу и
оформлять страховку всей его семье. Конструктор также должен подготовить
собственный объект, а не запускать долгую несвязанную работу.

Плохо:

```java
public Product() {
    // Плохо: конструктор сам читает пользовательский ввод.
    Scanner scanner = new Scanner(System.in);
    System.out.println("Enter product name:");
    name = scanner.nextLine();
}
```

Такой объект невозможно удобно создать в другом сценарии. Конструктор смешивает модель товара с консольным вводом.

Лучше получить данные снаружи:

```java
// Лучше: ввод выполняется снаружи.
String name = scanner.nextLine();

// Конструктор получает готовые данные.
Product product = new Product(name, priceInCents, quantity);
```

## Полный пример

```java
public class Product {
    private String name;
    private long priceInCents;
    private int quantity;

    public Product(String name, long priceInCents, int quantity) {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException(
                    "Name must not be blank"
            );
        }
        if (priceInCents < 0) {
            throw new IllegalArgumentException(
                    "Price must not be negative"
            );
        }
        if (quantity < 0) {
            throw new IllegalArgumentException(
                    "Quantity must not be negative"
            );
        }

        this.name = name;
        this.priceInCents = priceInCents;
        this.quantity = quantity;
    }

    public String getName() {
        return name;
    }

    public long getPriceInCents() {
        return priceInCents;
    }

    public int getQuantity() {
        return quantity;
    }

    public boolean sell(int amount) {
        if (amount <= 0 || amount > quantity) {
            return false;
        }

        quantity -= amount;
        return true;
    }
}
```

Объект сразу создается с названием, ценой и остатком. Поля нельзя изменить напрямую. Продажа сохраняет инвариант неотрицательного остатка.

## Частые ошибки

### Ошибка 1. Написать возвращаемый тип у конструктора

`void Product(...)` является обычным методом, а не конструктором.

### Ошибка 2. Написать `name = name`

При затенении обе части выражения относятся к параметру. Для поля нужно использовать `this.name`.

### Ошибка 3. Ожидать конструктор без параметров после объявления своего

Компилятор добавляет конструктор по умолчанию только тогда, когда в классе нет ни одного объявленного конструктора.

### Ошибка 4. Выполнять в конструкторе ввод и вывод

Конструктор должен получать готовые данные, проверять их и инициализировать объект.
