# Перегрузка, `static`, `final` и константы

## Результат

После этой лекции вы сможете:

- перегрузить конструктор;
- убрать дублирование через `this(...)`;
- отличить поле объекта от поля класса;
- использовать `final` и объявлять константы.

## Несколько способов создать объект

Иногда один тип объекта можно создавать с разным набором начальных данных.
Например, новый товар может сразу иметь остаток или начинаться с нулевого
остатка:

```java
// Вызываются разные конструкторы одного класса.
Product keyboard = new Product("Keyboard", 499_900L, 8);
Product mouse = new Product("Mouse", 129_900L);
```

Для этого класс может иметь несколько конструкторов.

## Перегрузка

**Перегрузка** - несколько конструкторов или методов с одним именем, но разными
наборами параметров.

```java
public Product(String name, long priceInCents, int quantity) {
    // Полная форма принимает начальный остаток.
}

public Product(String name, long priceInCents) {
    // Сокращенная форма использует остаток по умолчанию.
}
```

Java выбирает нужный конструктор по количеству и типам аргументов:

```java
// Java выбирает конструктор по количеству и типам аргументов.
new Product("Keyboard", 499_900L, 8);
new Product("Mouse", 129_900L);
```

Имена параметров не участвуют в выборе. Поэтому эти конструкторы нельзя
объявить вместе:

```java
public Product(String name) {
    // Для Java это конструктор с одним параметром String.
}

public Product(String code) {
    // Ошибка компиляции: другое имя параметра не создает новый вариант.
}
```

Для Java у них одинаковая **сигнатура**: один параметр типа `String`.

## Цепочка конструкторов

При **цепочке конструкторов** один конструктор вызывает другой. Это позволяет
не копировать проверки и присваивания:

```java
public Product(String name, long priceInCents, int quantity) {
    // Полный конструктор проверяет название и числовые значения.
    if (name == null || name.isBlank()) {
        throw new IllegalArgumentException("name must not be blank");
    }
    if (priceInCents < 0 || quantity < 0) {
        throw new IllegalArgumentException("numbers must not be negative");
    }

    this.name = name;
    this.priceInCents = priceInCents;
    this.quantity = quantity;
}

public Product(String name, long priceInCents) {
    // Те же проверки и присваивания приходится писать повторно.
    if (name == null || name.isBlank()) {
        throw new IllegalArgumentException("name must not be blank");
    }
    if (priceInCents < 0) {
        throw new IllegalArgumentException("price must not be negative");
    }

    this.name = name;
    this.priceInCents = priceInCents;
    this.quantity = 0;
}
```

В обоих конструкторах повторяются:

- проверка `name`;
- проверка `priceInCents`;
- присваивания полей `name` и `priceInCents`.

Если правило проверки изменится, придется не забыть исправить оба
конструктора.

Уберем дублирование: сокращенный конструктор вызовет полный через `this(...)`:

```java
public Product(String name, long priceInCents, int quantity) {
    // Все проверки и присваивания находятся в одном месте.
    if (name == null || name.isBlank()) {
        throw new IllegalArgumentException("name must not be blank");
    }
    if (priceInCents < 0 || quantity < 0) {
        throw new IllegalArgumentException("numbers must not be negative");
    }

    this.name = name;
    this.priceInCents = priceInCents;
    this.quantity = quantity;
}

public Product(String name, long priceInCents) {
    // Передаем работу полному конструктору, задавая остаток 0.
    this(name, priceInCents, 0);
}
```

Теперь вызов `new Product("Mouse", 129_900L)` попадет в сокращенный
конструктор, а затем перейдет в полный конструктор со значением `quantity = 0`.
Проверки выполнятся в одном месте.

Вызов `this(...)` должен быть первой инструкцией конструктора.

## Поля объекта и поля класса

**Поле объекта** принадлежит конкретному экземпляру:

```java
// У каждого объекта Product будет собственное значение quantity.
private int quantity;
```

У каждого товара свой остаток.

**Статическое поле** с `static` принадлежит классу и является общим для всех
его объектов:

```java
// Одно значение createdProducts принадлежит всему классу Product.
private static int createdProducts;
```

Его можно увеличить в конструкторе:

```java
public Product(String name, long priceInCents, int quantity) {
    // После проверок и присваиваний учитываем созданный объект.
    createdProducts++;
}
```

Получить общее значение можно через **статический метод**:

```java
public static int getCreatedProducts() {
    // Статический метод работает с общим полем класса.
    return createdProducts;
}
```

Обращаться к статическому члену принято через имя класса:

```java
// К статическому методу обращаемся через имя класса.
int count = Product.getCreatedProducts();
```

Не делайте поле `static`, если значение должно отличаться у объектов.
`static quantity` означало бы один общий остаток для всех товаров.

## `final`

**Полю `final`** можно присвоить значение только один раз:

```java
// Значение name задается один раз для каждого объекта.
private final String name;
```

Обычно такое поле заполняется в конструкторе:

```java
public Product(String name, long priceInCents, int quantity) {
    // final-поле обязательно получает значение при создании объекта.
    this.name = name;
    this.priceInCents = priceInCents;
    this.quantity = quantity;
}
```

После создания товара его название нельзя переназначить. При этом остальные
поля без `final` могут изменяться через методы.

## Константы

**Константа** - общее неизменяемое значение. В Java ее обычно объявляют как
`static final`:

```java
// Константа общая для класса и не может быть переназначена.
private static final int MAX_NAME_LENGTH = 100;
```

Имена констант записывают заглавными буквами с `_`.

Константа помогает дать смысл числу:

```java
// Именованная константа объясняет смысл ограничения.
if (name.length() > MAX_NAME_LENGTH) {
    throw new IllegalArgumentException("name is too long");
}
```

Внешнему коду константу открывают только тогда, когда она действительно
является частью публичных правил класса.

## Словарь

- **Перегрузка** - несколько конструкторов или методов с одним именем, но
  разными наборами параметров.
- **Сигнатура** - имя метода или конструктора вместе с типами и порядком его
  параметров.
- **Цепочка конструкторов** - вызов одного конструктора из другого через
  `this(...)`.
- **Поле объекта** - отдельное поле, принадлежащее конкретному экземпляру.
- **Статическое поле** - общее поле, принадлежащее классу.
- **Статический метод** - метод класса, который можно вызвать без создания
  объекта.
- **Поле `final`** - поле, которому значение присваивается только один раз.
- **Константа** - общее неизменяемое значение, обычно объявленное как
  `static final`.
