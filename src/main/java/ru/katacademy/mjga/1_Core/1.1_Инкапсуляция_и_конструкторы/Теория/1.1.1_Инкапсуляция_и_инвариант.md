# Инкапсуляция и инвариант объекта

## Результат

После этой лекции вы сможете:

- закрыть поля класса с помощью `private`;
- оставить снаружи только разрешенные операции;
- защитить объект от некорректного состояния;
- решить, когда нужен getter, а когда setter лучше не создавать.

## Состояние и поведение объекта

Из Base вы уже знаете, что объект объединяет состояние и поведение.

**Состояние объекта** - текущие значения его полей. Например, состояние товара
состоит из названия, цены и количества на складе.

**Поведение объекта** - операции, которые выполняют его методы. Метод может:

- прочитать состояние, например вернуть текущий остаток;
- изменить состояние, например добавить товар на склад;
- выполнить расчет на основе состояния, например вычислить стоимость остатка.

Когда далее говорится, что метод изменяет состояние, это означает, что он
изменяет значение одного или нескольких полей объекта.

## Проблема открытых полей

В Base мы создавали простые классы с открытыми полями:

```java
class Product {
    // Открытые поля доступны для прямого чтения и изменения.
    String name;
    long priceInCents;
    int quantity;
}
```

Такой объект легко заполнить:

```java
// Объект создается пустым, а его состояние заполняется по шагам.
Product product = new Product();
product.name = "Keyboard";
product.priceInCents = 499_900L;
product.quantity = 8;
```

Но любой код может записать бессмысленные значения:

```java
// Прямая запись позволяет обойти все правила товара.
product.name = "";
product.priceInCents = -100;
product.quantity = -5;
```

Класс хранит данные, но не управляет ими.

## Инкапсуляция

**Инкапсуляция** означает, что объект скрывает внутреннее состояние и разрешает
работать с ним только через предусмотренные операции.

Поля, которые не должны изменяться напрямую, делают `private`:

```java
public class Product {
    // Закрытые поля доступны только коду самого класса.
    private String name;
    private long priceInCents;
    private int quantity;
}
```

`private` разрешает обращаться к полю только из методов того же класса.
Поэтому такой код снаружи больше не скомпилируется:

```java
// Ошибка компиляции: private-поле недоступно внешнему коду.
product.quantity = -5;
```

**Модификатор доступа** определяет, откуда можно обратиться к элементу класса.
Для этого модуля достаточно трех вариантов:

- `public` - элемент доступен другим классам;
- `private` - элемент доступен только внутри своего класса;
- отсутствие модификатора, или **package-private**, - элемент доступен классам
  того же пакета.

`protected` будет разобран в одном из будущих модулей - «Наследование,
полиморфизм и интерфейсы».

Но просто закрыть поля недостаточно. Класс должен предоставить полезные
операции:

```java
public class Product {
    // Внешний код не может изменить эти поля напрямую.
    private String name;
    private long priceInCents;
    private int quantity;

    public boolean addStock(int amount) {
        // Некорректный аргумент не должен менять состояние.
        if (amount <= 0) {
            return false;
        }

        quantity += amount;
        return true;
    }

    public boolean sell(int amount) {
        // Нельзя продать неположительное количество или больше остатка.
        if (amount <= 0 || amount > quantity) {
            return false;
        }

        quantity -= amount;
        return true;
    }

    public int getQuantity() {
        // Getter разрешает чтение, но не прямое изменение остатка.
        return quantity;
    }
}
```

Теперь изменить остаток можно только по правилам `Product`.

## Инвариант

**Инвариант объекта** - правило, которое должно выполняться у корректного объекта
всегда.

Для товара примем три правила:

- название не пустое;
- цена неотрицательная;
- остаток неотрицательный.

Методы класса должны сохранять эти правила. Например, `sell` сначала проверяет
аргумент и только затем меняет поле. Если операция неуспешна, объект остается
в прежнем состоянии.

Плохо:

```java
public boolean sell(int amount) {
    // Плохо: состояние изменяется до проверки.
    quantity -= amount;

    if (quantity < 0) {
        return false;
    }

    return true;
}
```

Метод вернет `false`, но отрицательное значение уже сохранено.

Лучше:

```java
public boolean sell(int amount) {
    // Сначала проверяем возможность операции.
    if (amount <= 0 || amount > quantity) {
        return false;
    }

    // Изменяем состояние только после успешной проверки.
    quantity -= amount;
    return true;
}
```

## Getter и setter

**Getter** возвращает значение поля:

```java
public long getPriceInCents() {
    // Getter предоставляет контролируемое чтение поля.
    return priceInCents;
}
```

**Setter** устанавливает новое значение:

```java
public void setPriceInCents(long priceInCents) {
    // this отделяет поле объекта от одноименного параметра.
    this.priceInCents = priceInCents;
}
```

Ключевое слово `this` здесь означает текущий объект. Подробно оно будет
разобрано в следующей лекции.

Getter или setter не нужно автоматически создавать для каждого поля.
Открывайте только то, что действительно требуется внешнему коду.

Для остатка товара технический setter опасен:

```java
// Опасно: технический setter позволяет передать бессмысленное значение.
product.setQuantity(-100);
```

Предметные методы понятнее и безопаснее:

```java
// Предметные методы выражают намерение и выполняют проверки внутри класса.
product.addStock(10);
product.sell(3);
```

Их имена объясняют, зачем изменяется состояние, а проверки остаются внутри
класса.

## Главное правило

Класс должен делать правильное использование простым, а неправильное -
невозможным или явно неуспешным.

Для этого:

1. Закройте поля через `private`.
2. Определите правила корректного состояния.
3. Меняйте поля только после успешной проверки.
4. Добавляйте getters и setters только при реальной необходимости.

## Словарь

- **Состояние объекта** - текущие значения полей объекта.
- **Поведение объекта** - операции, которые объект выполняет через методы.
- **Инкапсуляция** - сокрытие внутреннего состояния и работа с ним через
  разрешенные операции.
- **Модификатор доступа** - ключевое слово, определяющее, откуда доступен
  класс, поле, конструктор или метод.
- **Package-private** - доступ только для классов того же пакета; задается
  отсутствием явного модификатора.
- **Инвариант объекта** - правило, которое должно выполняться для корректного
  объекта всегда.
- **Getter** - метод, который возвращает значение поля.
- **Setter** - метод, который устанавливает новое значение поля.
