# Задача 1.0.1: Рефакторинг расчета скидки

## Цель задачи

Закрепить идею рефакторинга: вынести расчет из `main` в отдельный метод с понятным названием.

## Сценарий

В курсе Base такой расчет часто можно было написать прямо внутри `main`. В текущем курсе нужно постепенно привыкать выносить логику в методы.

## Какие навыки пригодятся

- Объявление метода с параметрами.
- Возврат значения через `return`.
- Простая арифметика с `int`.
- Понимание рефакторинга: поведение остается тем же, структура кода становится понятнее.

## Техническое задание

Напишите метод:

```java
static int calculateFinalPrice(int price, int discountPercent)
```

Метод должен вернуть цену после скидки:

```text
price - price * discountPercent / 100
```

## Примеры

```java
calculateFinalPrice(1000, 10) // 900
calculateFinalPrice(1200, 15) // 1020
calculateFinalPrice(500, 0)   // 500
```

## Ограничения

- Не используйте ввод с клавиатуры.
- Не печатайте результат через `System.out.println`.
- Метод должен только вернуть значение.
- Соблюдайте кодстайл: понятное имя метода уже задано, имена параметров не сокращайте до `p` и `d`.

## Где это пригодится

Сегодня: вы учитесь отделять расчет от запуска программы.

В задачах курса: такие методы проще проверять автотестами, потому что они возвращают результат, а не печатают его.

В проектах: расчет скидок, налогов, доставки и итоговой цены обычно живет в отдельных методах или классах, а не внутри `main`.

## Критерии приемки

Решение считается верным, если метод возвращает правильную цену для разных значений цены и скидки.
