# Задача 1.0.3: Карточка товара с полным списком типов

## Цель задачи

Закрепить выбор примитивных типов и отличие `String` от примитивов.

## Сценарий

Нужно вывести простую карточку товара. Значения задаются прямо в коде.

## Какие навыки пригодятся

- Выбор подходящего типа данных.
- Объявление переменных с явными типами.
- Использование суффикса `L` для `long`.
- Вывод значений через `System.out.println`.

## Техническое задание

Внутри `runSolution` объявите переменные:

| Имя | Тип | Значение |
|---|---|---|
| `productId` | `long` | `9_000_000_001L` |
| `title` | `String` | `"Keyboard"` |
| `priceInCents` | `long` | `499_900L` |
| `rating` | `double` | `4.8` |
| `available` | `boolean` | `true` |
| `category` | `char` | `'A'` |

Выведите значения каждое с новой строки в указанном порядке.

## Ожидаемый вывод

```text
9000000001
Keyboard
499900
4.8
true
A
```

## Ограничения

- Не добавляйте подписи вроде `id:`.
- Не используйте ввод с клавиатуры.
- Используйте указанные типы.

## Где это пригодится

Сегодня: вы закрепляете полный список типов на одном понятном примере.

В задачах курса: выбор типа будет влиять на корректность расчетов и прохождение автотестов.

В проектах: карточки товаров, заказов и пользователей почти всегда состоят из значений разных типов.

## Критерии приемки

Решение считается верным, если вывод совпадает с ожидаемым полностью.
