# Задача 1.0.8: Денежные расчеты без переполнения и `double`

## Цель задачи

Закрепить переполнение `int`, расчеты в копейках и правило не использовать `double` для точных денежных значений.

## Сценарий

Цена хранится в копейках. Нужно уметь посчитать итоговую сумму заказа и применить скидку в процентах.

## Какие навыки пригодятся

- Диапазон `int`.
- Тип `long` для денежных значений в копейках.
- Явное преобразование перед умножением.
- Понимание переполнения целых чисел.
- Проверка некорректного процента скидки.

## Техническое задание

Напишите два метода:

```java
static long calculateTotalInCents(int priceInCents, int quantity)
static long calculateDiscountedPrice(long priceInCents, int discountPercent)
```

Метод `calculateTotalInCents` должен вернуть:

```text
priceInCents * quantity
```

но без переполнения `int`.

Метод `calculateDiscountedPrice` должен:

- вернуть цену после скидки;
- считать только через целые числа;
- вернуть исходную цену, если `discountPercent` меньше `0` или больше `100`.

## Примеры

```java
calculateTotalInCents(1000, 3)          // 3000
calculateTotalInCents(2_000_000_000, 3) // 6000000000

calculateDiscountedPrice(10_000L, 10)   // 9000
calculateDiscountedPrice(999L, 15)      // 850
calculateDiscountedPrice(5000L, -1)     // 5000
```

## Ограничения

- Не используйте `double` и `float`.
- Не печатайте результат.
- Преобразуйте один из множителей к `long` до умножения.

## Где это пригодится

Сегодня: вы учитесь предотвращать ошибку, которую Java может не показать явно, и закрепляете денежные расчеты в копейках.

В задачах курса: большие значения будут встречаться в расчетах денег, количества, скидок и идентификаторов.

В проектах: переполнение или `double` в точных деньгах может испортить итоговую сумму заказа, статистику или лимиты.

## Критерии приемки

Решение считается верным, если большие значения считаются корректно, а скидка применяется без дробных типов.
