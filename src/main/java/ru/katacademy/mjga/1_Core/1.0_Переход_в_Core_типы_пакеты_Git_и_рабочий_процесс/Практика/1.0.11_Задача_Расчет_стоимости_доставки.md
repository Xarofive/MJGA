# Задача 1.0.11: Расчет стоимости доставки

## Цель задачи

Закрепить `long`, расчеты в копейках и преобразование перед умножением.

## Сценарий

Стоимость доставки зависит от базовой цены, веса и расстояния.

## Какие навыки пригодятся

- Расчеты в копейках через `long`.
- Умножение `int`-значений без переполнения.
- Вынос формулы в отдельный метод.
- Понятные имена промежуточных переменных.

## Техническое задание

Напишите метод:

```java
static long calculateDeliveryPriceInCents(int weightGrams, int distanceKm)
```

Формула:

```text
20_000 + weightGrams * 3 + distanceKm * 150
```

Все значения - в копейках.

## Примеры

```java
calculateDeliveryPriceInCents(1000, 10)       // 24500
calculateDeliveryPriceInCents(0, 0)           // 20000
calculateDeliveryPriceInCents(2_000_000_000, 2) // 6000020300
```

## Ограничения

- Метод возвращает `long`.
- Не используйте `double`.
- Преобразуйте множители к `long` до умножения, если это нужно для защиты от переполнения.

## Где это пригодится

Сегодня: вы соединяете выбор типа, формулу и защиту от переполнения.

В задачах курса: такие расчеты будут встречаться в итоговых ценах и проверке условий.

В проектах: доставка, комиссии и сервисные сборы часто считаются отдельными методами.

## Критерии приемки

Решение считается верным, если маленькие и большие значения считаются корректно.
