# Java Core, справочный материал
# Карта основных ключевых слов Java

## Как пользоваться картой

**Ключевое слово** - слово, которому язык Java уже назначил специальный смысл.
Его нельзя использовать как обычное имя переменной, метода или класса.

Эту таблицу не нужно заучивать целиком. Возвращайтесь к ней, когда встречаете
незнакомое слово и хотите быстро понять:

- к какой части программы оно относится;
- что оно изменяет;
- где тема разбирается подробно.

Это карта основных слов, встречающихся в курсе, а не полный перечень всех
ключевых слов и контекстных конструкций Java.

## Объявления и структура программы

| Ключевое слово | К чему применяется | Что делает | Где изучается |
|---|---|---|---|
| `package` | Java-файл | Указывает пакет, к которому относится файл | Core 1.0 |
| `import` | Java-файл | Позволяет использовать тип по короткому имени | Core 1.0 |
| `class` | Объявление класса | Создает новый ссылочный тип | Base, классы |
| `interface` | Объявление интерфейса | Описывает контракт поведения | Будущий модуль об интерфейсах |
| `enum` | Объявление перечисления | Создает ограниченный набор именованных значений | Будущий модуль об `enum` |
| `extends` | Класс или интерфейс | Объявляет наследование | Будущий модуль о наследовании |
| `implements` | Класс | Показывает, какие интерфейсы реализует класс | Будущий модуль об интерфейсах |
| `abstract` | Класс или метод | Запрещает создавать объект класса напрямую или оставляет метод без реализации | Будущий модуль о наследовании |

## Доступ и члены класса

| Ключевое слово | К чему применяется | Что делает | Где изучается |
|---|---|---|---|
| `public` | Класс, конструктор, поле, метод, вложенный класс | Открывает доступ внешнему коду | Core 1.1 |
| `private` | Конструктор, поле, метод, вложенный класс | Разрешает доступ только внутри своего класса | Core 1.1 |
| `protected` | Конструктор, поле, метод, вложенный класс | Дает доступ пакету и наследникам | Будущий модуль о наследовании |
| `static` | Поле, метод, вложенный класс, блок инициализации | Связывает элемент с классом, а не с отдельным объектом | Core 1.1 |
| `final` | Переменная, параметр, поле, метод, класс | Запрещает повторное присваивание, переопределение или наследование в зависимости от места | Core 1.1 и следующие модули |
| `void` | Возвращаемый тип метода | Показывает, что метод не возвращает значение | Base, методы |
| `return` | Тело метода | Завершает метод и при необходимости возвращает значение | Base, методы |

## Как работает `static`

`static` связывает объявление с классом, а не с отдельным объектом.

### Статическое поле

```java
public class Product {
    // Одно общее значение для всех объектов Product.
    private static int createdProducts;

    public Product() {
        // Каждый новый объект увеличивает общий счетчик класса.
        createdProducts++;
    }
}
```

У каждого объекта могут быть собственные `name` и `price`, но поле
`createdProducts` существует в одном экземпляре на класс.

### Статический метод

```java
public class Product {
    private static int createdProducts;

    public static int getCreatedProducts() {
        // Статический метод может напрямую обращаться к статическому полю.
        return createdProducts;
    }
}
```

Статический метод вызывают через имя класса:

```java
// Для вызова не требуется создавать объект Product.
int count = Product.getCreatedProducts();
```

В статическом методе нет текущего объекта, поэтому внутри него нельзя
использовать `this` и напрямую обращаться к обычным полям объекта.

`static` не применяется к:

- конструктору;
- обычной локальной переменной метода;
- верхнеуровневому классу.

Вложенный класс и блок инициализации могут быть `static`, но эти варианты
будут разобраны позже, когда понадобятся в коде.

## Как работает `final`

Смысл `final` зависит от места применения.

| Где стоит `final` | Что запрещает |
|---|---|
| Локальная переменная (переменная внутри метода) | Повторно присвоить значение |
| Параметр метода | Переназначить параметр внутри метода |
| Поле | Повторно присвоить поле после начальной инициализации |
| Метод | Переопределить метод в классе-наследнике |
| Класс | Создать класс-наследник |

### `final` у значения

```java
// После присваивания переменной нельзя задать другое значение.
final int maxAttempts = 3;
```

### `final` у ссылки

```java
// Ссылку нельзя направить на другой объект.
final Product product = new Product();

// Сам объект при этом может оставаться изменяемым.
product.setPriceInCents(100_000L);
```

`final` у ссылки не делает объект неизменяемым. Оно запрещает только
перенаправить переменную:

```java
final Product product = new Product();

// Ошибка компиляции: переменной product нельзя присвоить другую ссылку.
product = new Product();
```

### Константа

Общее неизменяемое значение обычно объявляется как `static final`:

```java
public class Product {
    // Константа принадлежит классу и не может быть переназначена.
    public static final int MAX_NAME_LENGTH = 100;
}
```

`static` делает значение общим для класса, а `final` запрещает повторное
присваивание.

## Не ключевые слова

Некоторые специальные элементы Java легко принять за ключевые слова:

- `true` и `false` - логические литералы, которые нельзя использовать как
  имена;
- `null` - литерал отсутствующей ссылки, который нельзя использовать как имя;
- `String` - класс стандартной библиотеки;
- `System` - класс стандартной библиотеки;
- `main` - обычное имя метода с особой ролью при запуске программы.

`String`, `System` и `main` не являются ключевыми словами. Язык технически
позволяет объявлять некоторые одноименные элементы, но такие имена запутывают
код и в курсе не используются.

### Что такое литерал

**Литерал** - значение, записанное непосредственно в исходном коде.

Часто литерал находится справа от `=`:

```java
// 10, 99.95, 'A', true и "Keyboard" - литералы.
int quantity = 10;
double price = 99.95;
char category = 'A';
boolean available = true;
String name = "Keyboard";
```

Но литерал не обязательно стоит после `=`. Он может быть аргументом метода,
частью выражения или условия:

```java
// "Hello" передается методу как строковый литерал.
System.out.println("Hello");

// 0 - целочисленный литерал внутри условия.
if (quantity > 0) {
    System.out.println("Product is available");
}
```

Основные примеры литералов:

| Вид литерала | Примеры |
|---|---|
| Целочисленный | `10`, `0`, `-25`, `1_000`, `100L` |
| Дробный | `3.14`, `0.5`, `4.8f` |
| Символьный | `'A'`, `'7'`, `'\n'` |
| Строковый | `"Hello"`, `"Keyboard"` |
| Логический | `true`, `false` |
| Отсутствующая ссылка | `null` |

Знак `-` в записи `-25` технически является оператором, примененным к
положительному целочисленному литералу `25`. Для чтения учебного кода всю
запись можно воспринимать как явно указанное отрицательное значение.

## Словарь

- **Ключевое слово** - зарезервированное слово со специальным смыслом в Java.
- **Литерал** - значение, записанное непосредственно в исходном коде, например
  `10`, `'A'`, `"Hello"`, `true` или `null`.
- **Статический член** - поле или метод, принадлежащий классу.
- **Поле объекта** - поле, отдельное для каждого экземпляра класса.
- **Константа** - именованное неизменяемое значение, обычно `static final`.
- **Переопределение метода** - новая реализация унаследованного метода.
- **Наследование** - создание нового класса на основе существующего.
