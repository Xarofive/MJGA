# Задача 1.0.4: Проверка оформления заказа

## Уровень сложности

**1 из 5 — начальный.** Требуется собрать три явно сформулированных правила в одно логическое выражение.

## Цель задачи

Закрепить `boolean`, условия и понятные имена логических значений.

## Сценарий

Пользователь может оформить заказ, если:

- ему 18 лет или больше;
- в корзине есть хотя бы один товар;
- аккаунт не заблокирован.

## Какие навыки пригодятся

- Логический тип `boolean`.
- Операторы сравнения: `>=`, `>`.
- Логические операторы `&&` и `!`.
- Понятные имена boolean-переменных.

## Техническое задание

Напишите метод:

```java
static boolean canCheckout(int age, int itemCount, boolean blocked)
```

Метод должен вернуть `true`, если заказ можно оформить, иначе `false`.

## Примеры

```java
canCheckout(18, 1, false) // true
canCheckout(17, 1, false) // false
canCheckout(20, 0, false) // false
canCheckout(20, 2, true)  // false
```

## Ограничения

- Не печатайте результат.
- Используйте `boolean`-выражения с понятным смыслом.

## Где это пригодится

Сегодня: вы учитесь выражать правила задачи через `boolean`.

В задачах курса: такие проверки будут встречаться в валидации входных данных.

В проектах: оформление заказа, доступ пользователя и включение функций часто зависят от нескольких условий сразу.

## Критерии приемки

Решение считается верным, если метод правильно обрабатывает все три условия.
