# Задача 1.0.9: `byte`, `short`, `float` и numeric promotion

## Уровень сложности

**3 из 5 — средний.** Нужно удерживать правила нескольких редко используемых типов, numeric promotion, явный cast и суффикс `f` сразу в пяти методах.

## Цель задачи

Закрепить редкие примитивные типы, суффикс `f` и правило numeric promotion для `byte` и `short`.

## Сценарий

Иногда готовый формат данных или уже написанный код требует использовать не только привычные `int` и `double`, но и `byte`, `short`, `float`.

Нужно написать несколько маленьких методов, которые показывают эти случаи.

## Какие навыки пригодятся

- Типы `byte`, `short`, `float`.
- Суффикс `f` для `float`.
- Понимание, что `byte + byte` и `short + short` дают `int`.
- Явный cast, если результат нужно вернуть обратно в маленький тип.

## Техническое задание

Напишите четыре метода:

```java
static int sumBytes(byte first, byte second)
static int sumShorts(short first, short second)
static byte addByteAndCast(byte value, byte increment)
static float calculateScreenX(float startX, float speed, int seconds)
static float calculateDefaultScreenX(float startX, int seconds)
```

Метод `sumBytes` должен вернуть сумму двух `byte` как `int`.

Метод `sumShorts` должен вернуть сумму двух `short` как `int`.

Метод `addByteAndCast` должен сложить два `byte` и вернуть результат как `byte` через явное преобразование.

Метод `calculateScreenX` должен вернуть координату:

```text
startX + speed * seconds
```

Метод `calculateDefaultScreenX` должен считать координату при скорости `1.5f`:

```text
startX + 1.5f * seconds
```

## Примеры

```java
sumBytes((byte) 10, (byte) 20)             // 30
sumShorts((short) 1000, (short) 2000)      // 3000
addByteAndCast((byte) 10, (byte) 20)       // 30
calculateScreenX(10.5f, 2.25f, 4)          // 19.5f
calculateDefaultScreenX(10.0f, 4)          // 16.0f
```

## Ограничения

- `sumBytes` и `sumShorts` должны возвращать `int`.
- В `addByteAndCast` нужен явный cast к `byte`.
- Для `float`-литералов используйте суффикс `f`, например `1.5f`.
- Не используйте `double`.
- Не печатайте результат.

## Где это пригодится

Сегодня: вы закрепляете, что маленькие целочисленные типы в выражениях поднимаются до `int`.

В задачах курса: это поможет понимать ошибки компиляции вроде `byte sum = a + b`.

В проектах: `byte` и `short` встречаются при работе с компактными форматами данных, а `float` - в координатах, графике и готовых библиотеках.

## Критерии приемки

Решение считается верным, если методы используют правильные типы и возвращают ожидаемые значения.
