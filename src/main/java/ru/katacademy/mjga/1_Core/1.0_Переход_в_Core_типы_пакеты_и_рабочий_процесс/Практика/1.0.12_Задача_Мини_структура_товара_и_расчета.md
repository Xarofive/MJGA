# Задача 1.0.12: Мини-структура товара и расчета

## Уровень сложности

**3 из 5 — средний.** Нужно создать два связанных класса, разделить данные и поведение и сохранить корректный числовой тип расчета.

## Цель задачи

Закрепить идею разделения ответственности: данные отдельно, расчет отдельно.

## Сценарий

В настоящем проекте такие классы позже можно будет разнести по пакетам `model` и `service`. В этой задаче классы пишутся в одном файле, чтобы сосредоточиться на разделении данных и расчета.

## Какие навыки пригодятся

- Простые классы из курса Base.
- Поля объекта.
- Разделение данных и расчета.
- Тип `long` для денежных значений.

## Техническое задание

Создайте два вложенных класса:

```java
static class Product
static class PriceCalculator
```

`Product` должен содержать поля:

```java
String title;
long priceInCents;
```

`PriceCalculator` должен содержать метод:

```java
long calculateTotal(Product product, int quantity)
```

Метод должен вернуть `product.priceInCents * quantity` без переполнения `int`.

## Примеры

```java
Product product = new Product();
product.title = "Keyboard";
product.priceInCents = 499_900L;

new PriceCalculator().calculateTotal(product, 2) // 999800
```

## Ограничения

- Не используйте `System.out.println`.
- Не используйте `double`.
- `Product` хранит данные.
- `PriceCalculator` выполняет расчет.

## Где это пригодится

Сегодня: вы видите первый шаг от одного метода к маленькой структуре классов.

В задачах курса: дальше классы будут получать более строгие правила создания и защиты данных.

В проектах: данные товара и расчет цены обычно живут в разных частях программы, чтобы код было проще поддерживать.

## Критерии приемки

Решение считается верным, если классы созданы, поля доступны тестам, а расчет возвращает правильный `long`.
