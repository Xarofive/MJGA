# 📚 Ревью 1: Переход в Java Core

## 💡 Что такое ревью?

Ревью — это формат проверки знаний, максимально приближенный к реальному техническому собеседованию. Ментор будет задавать вопросы по пройденным темам, просить объяснить концепции своими словами и обсуждать практические кейсы.

Это не экзамен. Это диалог, где вы можете:

- рассуждать вслух и искать решения вместе с ментором;
- признаться, что чего-то не знаете;
- получить обратную связь и рекомендации по развитию;
- узнать, как проходят реальные собеседования в IT.

Формат реальных собеседований мало чем отличается от ревью на курсе. Такая практика помогает подготовиться не только технически, но и психологически.

## 📋 Что будет проверяться

В ревью входит модуль 1.0 Java Core:

- повтор тем Java Base и переход к более структурному коду;
- рефакторинг, кодстайл и ответственность метода;
- все примитивные типы Java и выбор подходящего типа;
- числовые преобразования, numeric promotion и переполнение;
- пакеты, импорты и структура исходников.

Вас попросят не только дать определение, но и объяснить причины поведения программы, найти проблему в коде и предложить исправление.

## 📅 Как проходит ревью

- **Длительность:** около 1 часа.
- **Формат:** онлайн-созвон с ментором.
- **Теория:** вопросы по модулю, 20–30 минут.
- **Live coding:** решение одной или нескольких задач, 20–30 минут.
- **Обратная связь:** рекомендации по итогам ревью, 5–10 минут.

Вопросы из прошлых тем Java Base тоже могут появиться, если они нужны для решения задачи.

## 💻 Задачи для live coding

Ментор выбирает одну или несколько задач. Условия могут быть немного изменены.

### Задача 1. Улучшить код в `main`

```java
public static void main(String[] args) {
    int price = 1000;
    int discount = 15;
    int result = price - price * discount / 100;
    System.out.println(result);
}
```

Нужно:

- объяснить, что можно улучшить;
- вынести расчёт в метод;
- объяснить, почему изменение является рефакторингом.

### Задача 2. Выбрать типы

Выберите тип и объясните выбор для следующих значений:

- возраст пользователя;
- идентификатор заказа;
- цена товара в копейках;
- рейтинг товара;
- признак активного аккаунта;
- один символ категории.

### Задача 3. Найти переполнение

```java
int priceInCents = 2_000_000_000;
int quantity = 3;
long total = priceInCents * quantity;
```

Нужно:

- найти проблему;
- объяснить, почему `long total` не предотвращает ошибку;
- исправить расчёт.

### Задача 4. Разложить классы по пакетам

Даны классы:

- `Main`;
- `Product`;
- `Customer`;
- `DeliveryCalculator`;
- `MoneyFormatter`.

Нужно предложить пакеты для классов и объяснить распределение.

### Задача 5. Объяснить результат

```java
System.out.println(0.1 + 0.2);
```

Нужно:

- объяснить, почему результат может отличаться от `0.3`;
- назвать связанный с этим стандарт хранения дробных чисел;
- объяснить, почему для точных денег лучше не использовать `double`.

## 📏 Правила ревью

- Включённая камера — обязательное требование.
- Работаем по-честному: не списываем, не ищем готовые ответы и не принимаем подсказки со стороны.
- Отвечаем своими словами, а не воспроизводим заученное определение.
- Во время live coding объясняем ход решения вслух.
- Желательно находиться в спокойном месте со стабильным подключением к интернету.
- Желательно иметь под рукой VPN на случай технических проблем.

## ⚠️ Критерии прохождения

Ревью считается не пройденным, если студент:

- неправильно ответил или совсем не ответил на три вопроса;
- не решил задачу live coding;
- не может объяснить написанное решение.

После неудачной попытки студент получает обратную связь, повторяет проблемные темы и может пройти ревью ещё раз.

## ⚠️ Важно понимать

На ревью проверяется не заучивание определений, а понимание концепций. Ментор может спросить:

- «Почему это работает именно так?»
- «Какие есть альтернативы?»
- «Как бы вы применили это на практике?»
- «Что произойдёт на граничных значениях?»
- «С какими проблемами можно столкнуться?»

- Сравнивайте корректные и некорректные варианты преобразований.
- Практикуйтесь объяснять решение другому человеку.
- Задавайте себе вопросы «А что если?» и «Почему именно так?».

- Честно скажите, что не знаете точного ответа.
- Попробуйте рассуждать логически и озвучивайте ход мыслей.
- Задавайте уточняющие вопросы.
