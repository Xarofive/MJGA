# Полный список примитивных типов Java

## TL;DR

- В Java есть 8 примитивных типов: `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`.
- В курсе Base мы уверенно использовали только `int`, `double`, `boolean`, `char`; теперь добавляем остальные.
- Тип выбирают не по привычке, а по смыслу данных: возраст, количество, деньги, идентификатор, флаг, символ.
- У целочисленных типов есть диапазоны. Если выйти за диапазон, можно получить переполнение.
- `String` не примитив, а ссылочный тип. Это важно для сравнения, `null` и памяти.

## Что уже известно из курса Base

В курсе Base вы видели такие объявления:

```java
int age = 25;
double price = 199.99;
boolean active = true;
char grade = 'A';
String name = "Anna";
```

Этого достаточно для первых задач, но в реальной Java-программе встречаются и другие типы. Например:

- `long` для идентификаторов и времени;
- `byte` для байтовых данных;
- `float` в некоторых готовых библиотеках, графике и старом коде;
- `short` в задачах, где важен компактный числовой формат;
- `boolean` для четких флагов;
- `char` для одного UTF-16 code unit, а не для любого видимого символа целиком.

## Полный список примитивных типов

Ниже полный список всех примитивных типов в Java. Их ровно 8.

Тип можно представить как контейнер определенного размера. Маленький контейнер занимает меньше места, но вмещает меньше значений. Большой контейнер вмещает больше, но не всегда нужен.

| Тип | Размер | Что хранит | Пример |
|---|---:|---|---|
| `byte` | 8 бит | целые числа от -128 до 127 | `byte level = 10;` |
| `short` | 16 бит | целые числа от -32768 до 32767 | `short year = 2026;` |
| `int` | 32 бита | основной целочисленный тип | `int count = 1500;` |
| `long` | 64 бита | большие целые числа | `long userId = 9000000000L;` |
| `float` | 32 бита | дробные числа одинарной точности | `float rating = 4.8f;` |
| `double` | 64 бита | основной дробный тип | `double price = 99.99;` |
| `char` | 16 бит | один UTF-16 code unit | `char letter = 'A';` |
| `boolean` | JVM-зависимо | `true` или `false` | `boolean enabled = true;` |

## Целочисленные типы

### `byte`

`byte` хранит маленькие целые числа.

```java
byte temperature = 25;
byte maxLevel = 100;
```

В обычных учебных и рабочих задачах `byte` используют нечасто. Он нужен, когда программа работает с данными на уровне байтов: файлами, сетью, изображениями, архивами или другими бинарными форматами.

```java
byte[] data = {10, 20, 30};
```

Не стоит использовать `byte` только потому, что число маленькое. Для возраста, количества товаров и счетчиков обычно берут `int`, потому что с ним проще работать.

### `short`

`short` больше `byte`, но меньше `int`.

```java
short productionYear = 2026;
```

В большинстве обычных задач `short` не нужен, но есть ситуации, где он уместен. Например, если программа читает бинарный файл, где по формату число занимает ровно 2 байта.

```java
short soundSample = 1200;
short mapTileId = 315;
```

Здесь `short` выбран не потому, что "число маленькое", а потому что внешний формат или уже готовая структура данных ожидает 16-битное число. Если такого требования нет, для целых чисел обычно выбирают `int`.

### `int`

`int` - основной тип для целых чисел.

```java
int age = 30;
int productCount = 42;
int retryLimit = 3;
```

Если нет причины выбрать другой целочисленный тип, обычно выбирают `int`.

### `long`

`long` нужен для больших целых чисел.

```java
long userId = 9_000_000_000L;
long milliseconds = 1_700_000_000_000L;
```

Суффикс `L` важен. Без него Java сначала пытается прочитать литерал как `int`.

Плохой пример:

```java
long value = 9000000000; // ошибка компиляции
```

Правильно:

```java
long value = 9000000000L;
```

Используйте большую `L`, а не маленькую `l`, потому что маленькая похожа на цифру `1`.

## Дробные типы

### `double`

`double` - основной тип для дробных чисел в Java.

```java
double weight = 72.5;
double taxRate = 0.13;
double averageScore = 88.75;
```

В курсе Base вы уже использовали `double`. Теперь важно добавить осторожность: `double` не подходит для точных денежных расчетов, потому что хранит число в двоичном формате с ограниченной точностью.

```java
System.out.println(0.1 + 0.2); // может вывести 0.30000000000000004
```

**Важно:** `float` и `double` в Java хранятся по стандарту IEEE 754. Этот стандарт использует двоичное представление дробных чисел. Некоторые привычные десятичные дроби, например `0.1` и `0.2`, невозможно сохранить в двоичном виде абсолютно точно. Поэтому результат может выглядеть неожиданно: `0.30000000000000004` вместо `0.3`.

Для учебных расчетов скидок и налогов `double` допустим. Для реальных денег позже используют `BigDecimal` или хранят сумму в копейках через `long`.

### `float`

`float` занимает меньше памяти, но менее точен.

```java
float rating = 4.8f;
```

Суффикс `f` обязателен, потому что дробный литерал по умолчанию считается `double`.

Плохой пример:

```java
float rating = 4.8; // ошибка компиляции
```

Правильно:

```java
float rating = 4.8f;
```

В обычных прикладных задачах чаще используют `double`. `float` нужно знать, потому что он встречается в графике, играх, Android, старом коде и готовых библиотеках, где явно требуется число одинарной точности.

Пример ситуации, где `float` может быть нужен:

```java
float x = 10.5f;
float y = 20.25f;
```

Координаты на экране или в простой игре часто хранят как `float`, потому что там важны скорость и компактность, а не точность денежных расчетов.

## `char`

`char` хранит один UTF-16 code unit.

```java
char firstLetter = 'J';
char digit = '7';
char symbol = '#';
```

Важно: `char` пишется в одинарных кавычках, а `String` - в двойных.

```java
char letter = 'A';
String text = "A";
```

В курсе Base тема Unicode уже была. Теперь фиксируем аккуратную формулировку: `char` не всегда равен "одному видимому символу". Некоторые символы, например часть emoji, могут занимать больше одного `char`.

```java
String smile = "🙂";
System.out.println(smile.length()); // может быть 2
```

Для большинства учебных задач с латинскими буквами и цифрами `char` удобен. Для полноценной обработки Unicode нужны специальные методы строки. Эту тему мы будем разбирать отдельно, когда снова вернемся к строкам.

### Почему с `char` возможна арифметика

`char` хранит не "букву как картинку", а числовой код символа в таблице Unicode. Поэтому Java позволяет выполнять с `char` арифметические операции.

```java
char letter = 'A';
int code = letter;

System.out.println(code); // 65
System.out.println((char) (letter + 1)); // B
```

**Важно:** арифметика с `char` работает потому, что за символом стоит число. Это не значит, что любой текст можно безопасно обрабатывать как набор отдельных `char`. Для простых латинских букв это удобно, но для полного Unicode нужны более аккуратные инструменты.

## `boolean`

`boolean` хранит только `true` или `false`.

```java
boolean isActive = true;
boolean hasDiscount = false;
```

У `boolean` есть важная особенность: его размер нельзя объяснять так же просто, как размер `int` или `byte`.

**Спецификация Java** - это официальный документ, который описывает правила языка и виртуальной машины: какие типы есть в Java, как работают выражения, классы, массивы, выполнение программы и другие базовые механизмы. Основные документы - Java Language Specification и Java Virtual Machine Specification. Их можно найти на странице Oracle Java SE Specifications: https://docs.oracle.com/javase/specs/

Упрощенно для старта:

- в массиве `boolean[]` один элемент обычно занимает 1 байт;
- отдельная переменная `boolean` или поле объекта может занимать больше, например 4 байта, из-за выравнивания памяти и внутреннего устройства JVM;
- на разных JVM, настройках запуска и компьютерах фактический размер `boolean` может отличаться;
- причина в том, что JVM и процессору удобнее работать с данными блоками определенного размера, а не всегда с отдельными битами.

Выравнивание памяти означает, что машине удобнее читать данные блоками определенного размера. Иногда из-за этого маленькое значение занимает больше места, чем кажется. Поэтому `boolean` выбирают не ради точной экономии памяти, а потому что значение отвечает на вопрос "да или нет".

Пример массива:

```java
boolean[] flags = new boolean[3];

flags[0] = true;
flags[1] = false;
flags[2] = true;
```

Идея такая: в массиве каждый элемент `boolean` обычно хранится отдельно, примерно как один байт на один элемент.

Хорошее имя boolean-переменной звучит как вопрос:

```java
boolean isAdult = age >= 18;
boolean hasItems = itemCount > 0;
boolean canCheckout = isAdult && hasItems;
```

Плохой пример:

```java
boolean status = true;
boolean check = false;
```

Неясно, что именно означает `true`.

## `String` не примитив

`String` часто стоит рядом с примитивами в первых лекциях, потому что он нужен почти сразу. Но технически это не примитив, а класс.

```java
String name = "Anna";
```

Следствия:

- у `String` есть методы: `length()`, `toUpperCase()`, `substring()`;
- `String` может быть `null`;
- строки сравнивают через `equals`;
- переменная типа `String` хранит ссылку на объект.

```java
String city = "Moscow";

if (city.equals("Moscow")) {
    System.out.println("Found");
}
```

Если переменная может быть `null`, безопаснее писать так:

```java
if ("Moscow".equals(city)) {
    System.out.println("Found");
}
```

## Как выбирать тип

### Возраст

Можно хранить возраст в `byte`, потому что человек не живет до 127 лет в обычной системе. Но на практике чаще выбирают `int`.

```java
int age = 25;
```

Причина: `int` проще в выражениях и является стандартным типом для целых чисел.

### Идентификатор

Идентификатор - это значение, по которому можно отличить один объект от другого. Например, номер заказа, номер пользователя, номер товара в системе.

Для идентификатора часто берут `long`.

```java
long orderId = 12345678901L;
```

Даже если сегодня идентификатор маленький, база данных или другая программа может использовать большие значения.

### Деньги

Для учебного уровня:

```java
double price = 199.99;
```

Для более аккуратного подхода:

```java
long priceInCents = 19999L;
```

Так сумма хранится как целое число копеек или центов.

### Количество

Обычно:

```java
int quantity = 12;
```

Если количество может быть очень большим:

```java
long totalEvents = 5_000_000_000L;
```

### Рейтинг

```java
double rating = 4.7;
```

`float` нужен только если готовая библиотека, задача или уже написанный код явно требует `float`.

### Флаг

```java
boolean enabled = true;
```

Если у значения больше двух состояний, `boolean` может быть плохим выбором.
Например, заказ может быть `NEW`, `PAID`, `SHIPPED`, `CANCELLED`. Позже для
этого будет изучен `enum` - специальный тип Java для ограниченного набора
именованных значений.

## Частые ошибки

### Ошибка 1. Забыли `L`

```java
long id = 9000000000; // ошибка
```

Исправление:

```java
long id = 9000000000L;
```

### Ошибка 2. Забыли `f`

```java
float rating = 4.5; // ошибка
```

Исправление:

```java
float rating = 4.5f;
```

### Ошибка 3. Выбрали `double` для точных денег

```java
double total = 0.1 + 0.2;
```

Для учебного расчета это допустимо, но нужно знать ограничение. Для точных сумм лучше хранить минимальные денежные единицы:

```java
long totalCents = 10L + 20L;
```

### Ошибка 4. Слишком маленький тип ради экономии

```java
byte itemCount = 120;
itemCount++; // может привести к неожиданностям около границы
```

Для счетчиков выбирайте `int`, если нет сильной причины делать иначе.
