# Пакеты и структура исходников

## TL;DR

- `package` задает логическое место класса в проекте.
- Имя пакета должно совпадать с расположением файла внутри `src/main/java`.
- `import` позволяет использовать классы из других пакетов без полного имени.
- В текущем курсе код постепенно делится на `app`, `model`, `service`, `util`, позже появится отдельное место для хранения и поиска данных.
- Пакеты нужны не для красоты, а чтобы проект не превращался в одну папку с десятками классов.

В курсе Base вы чаще писали один класс:

```java
public class Main {

```

Иногда появлялись импорты стандартных классов:

```java
import java.util.Scanner;

import java.util.HashMap;
```

В текущем курсе классов станет больше. Если все держать рядом, проект быстро станет нечитаемым. Поэтому появляется структура.

## Что такое пакет

Пакет - это логическая группа классов.

Пример:

```java
package ru.katacademy.core.model;

public class Product {
    String title;
    long priceInCents;
}
```

Строка `package` должна быть первой значимой строкой файла. До нее могут быть только комментарии.

Если класс лежит в пакете `ru.katacademy.core.model`, файл обычно расположен так:

```text
src/main/java/ru/katacademy/core/model/Product.java
```

Пакет и путь должны соответствовать друг другу.

## Зачем нужны пакеты

### Причина 1. Навигация

Когда классов мало, все кажется простым:

```text
Main.java
Product.java
Order.java
Customer.java
Calculator.java
```

Когда классов становится больше, лучше так:

```text
src/main/java
└── ru
    └── katacademy
        └── core
            ├── app
            │   └── Main.java
            ├── model
            │   ├── Product.java
            │   ├── Order.java
            │   └── Customer.java
            ├── service
            │   └── OrderService.java
            └── util
                └── MoneyFormatter.java
```

По структуре видно, где искать запуск, где данные, где логика, где вспомогательные функции.

### Причина 2. Имена классов

В разных пакетах могут быть классы с одинаковым простым именем.

```text
app.User
model.User
service.UserService
```

Полное имя класса включает пакет:

```text
ru.katacademy.core.model.User
```

### Причина 3. Границы ответственности

Пакет помогает не смешивать разные роли.

- `model` - классы данных задачи: `Product`, `Order`, `Customer`.
- `service` - логика программы: расчет цены, создание заказа, проверка правил.
- `app` - запуск приложения и взаимодействие с пользователем.
- `util` - маленькие вспомогательные классы без состояния.
- отдельные классы для хранения и поиска данных появятся позже.

## `import`

Если класс находится в другом пакете, его нужно импортировать.

```java
package ru.katacademy.core.app;

import ru.katacademy.core.model.Product;

public class Main {
    public static void main(String[] args) {
        Product product = new Product();
    }
}
```

Без `import` можно написать полное имя:

```java
ru.katacademy.core.model.Product product = new ru.katacademy.core.model.Product();
```

Так почти никогда не пишут в обычном коде, потому что строка становится тяжелой.

## Импорт стандартной библиотеки

Классы из `java.lang` импортировать не нужно. Они доступны автоматически:

```java
String text = "Hello";
System.out.println(text);
```

А вот классы из `java.util` импортировать нужно:

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.Scanner;
```

## Импорт через `*`

Можно написать:

```java
import java.util.*;
```

Такую запись часто называют wildcard import, то есть "импорт через звездочку". Она импортирует все классы из пакета `java.util` на уровне записи. В учебном коде лучше писать явные импорты:

```java
import java.util.ArrayList;
import java.util.HashMap;
```

Так проще видеть, какие классы реально используются.

## Пример разделения простой программы

В курсе Base можно было написать все в `Main`:

```java
public class Main {
    public static void main(String[] args) {
        String title = "Keyboard";
        long priceInCents = 499900L;
        int quantity = 2;

        long total = priceInCents * quantity;
        System.out.println(title + ": " + total);
    }
}
```

После улучшения лучше разделить код по смыслу.

В следующем примере классы находятся в разных пакетах. Поэтому у классов,
полей и методов встречается модификатор `public`. В данном примере достаточно
понимать его так: `public` разрешает обратиться к элементу из другого пакета.
Модификаторы доступа и правила выбора между `public` и закрытым доступом будут
подробно изучены в следующем модуле.

`Product.java`:

```java
package ru.katacademy.core.model;

public class Product {
    public String title;
    public long priceInCents;
}
```

Пока поля оставлены открытыми, как в Base. В следующем модуле они станут
закрытыми, а объект будет получать обязательные данные при создании.

`OrderService.java`:

```java
package ru.katacademy.core.service;

import ru.katacademy.core.model.Product;

public class OrderService {
    public long calculateTotal(Product product, int quantity) {
        return product.priceInCents * quantity;
    }
}
```

`Main.java`:

```java
package ru.katacademy.core.app;

import ru.katacademy.core.model.Product;
import ru.katacademy.core.service.OrderService;

public class Main {
    public static void main(String[] args) {
        Product product = new Product();
        product.title = "Keyboard";
        product.priceInCents = 499900L;

        OrderService service = new OrderService();

        long total = service.calculateTotal(product, 2);
        System.out.println(total);
    }
}
```

Такой код длиннее для первого примера, но он лучше масштабируется.

## Как IntelliJ IDEA помогает с пакетами

Обычно не нужно вручную создавать всю вложенность папок.

Порядок:

1. Откройте `src/main/java`.
2. Создайте package, например `ru.katacademy.core.model`.
3. Внутри package создайте class `Product`.
4. IDEA сама добавит строку `package`.
5. При использовании класса из другого пакета IDEA предложит import.

Если package подсвечен красным, проверьте:

- файл лежит в правильной папке;
- строка `package` совпадает с путем;
- папка `src/main/java` помечена как Sources Root;
- имя класса совпадает с именем файла для `public class`.

## Имена пакетов

Пакеты пишут маленькими буквами.

Хороший пример:

```text
ru.katacademy.core.model
ru.katacademy.core.service
ru.katacademy.core.app
```

Плохой пример:

```text
Ru.KatAcademy.Core.Model
my package
service-layer
```

В именах пакетов не используют пробелы и дефисы.

## Частые ошибки

### Ошибка 1. Путь и package не совпадают

Файл лежит здесь:

```text
src/main/java/ru/katacademy/core/model/Product.java
```

А внутри написано:

```java
package ru.katacademy.core.service;
```

Нужно исправить package или переместить файл.

### Ошибка 2. Забыли import

```java
package ru.katacademy.core.app;

public class Main {
    public static void main(String[] args) {
        Product product = new Product();
    }
}
```

Если `Product` в другом пакете, нужен import:

```java
import ru.katacademy.core.model.Product;
```

### Ошибка 3. Слишком общий пакет

```text
src/main/java/core/Product.java
src/main/java/core/Order.java
src/main/java/core/Main.java
src/main/java/core/Calculator.java
```

Лучше разделить по смыслу:

```text
core/model
core/service
core/app
```

### Ошибка 4. Логика не в том пакете

`Product` не должен читать ввод из консоли. `Main` не должен хранить весь расчет заказа. `MoneyFormatter` не должен менять состояние заказа.

Пакеты помогают замечать такие смешения.

## Практика

### Задание 1. Разнести классы

Создайте структуру:

```text
src/main/java/ru/katacademy/core
├── app
├── model
├── service
└── util
```

Создайте классы:

- `app.Main`;
- `model.Product`;
- `model.Customer`;
- `service.DeliveryCalculator`;
- `util.MoneyFormatter`.

### Задание 2. Импорты

В `Main` создайте `Product` и вызовите `DeliveryCalculator`.

Требование: не использовать полные имена классов в коде, подключить их через `import`.

### Задание 3. Ошибка package

Специально переместите класс в другой package и посмотрите ошибку в IDEA. Затем исправьте структуру.

### Задание 4. Мини-структура доставки

Создайте `DeliveryCalculator`:

```java
public long calculatePriceInCents(int weightGrams, int distanceKm) {
    return 20_000L + (long) weightGrams * 3 + (long) distanceKm * 150;
}
```

Вызовите метод из `Main`.
