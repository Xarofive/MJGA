# Числовые преобразования и переполнение

## TL;DR

- Java может автоматически расширять числовой тип, если это безопасно: `int` -> `long`, `int` -> `double`.
- Обратное преобразование требует явного cast: `long` -> `int`, `double` -> `int`.
- При операциях с `byte` и `short` Java часто поднимает значения до `int`.
- Целочисленное деление отбрасывает дробную часть.
- Переполнение не всегда дает ошибку. Иногда Java молча получает неверное число.

## Что продолжаем из курса Base

В курсе Base вы уже писали арифметику:

```java
int a = 10;
int b = 3;
int sum = a + b;
int result = a / b;
```

И видели cast для получения дробного результата:

```java
double average = (double) sum / count;
```

В текущем курсе мы объясняем, почему это работает, где Java помогает автоматически, а где требует явного решения от разработчика.

## Расширяющее преобразование

Расширяющее преобразование безопасно: значение переезжает из меньшего контейнера в больший.

```java
int count = 100;
long totalCount = count;

int price = 999;
double priceAsDouble = price;
```

Java делает это автоматически, потому что `long` может хранить любое значение `int`, а `double` может представить очень большой диапазон чисел.

Пример цепочки:

```java
byte small = 10;
short medium = small;
int normal = medium;
long big = normal;
double decimal = big;
```

## Сужающее преобразование

Сужающее преобразование может потерять данные. Поэтому нужен явный cast.

```java
long big = 1000L;
int value = (int) big;
```

Если значение помещается в `int`, все выглядит нормально. Но если не помещается, результат будет испорчен.

```java
long tooBig = 3_000_000_000L;
int broken = (int) tooBig;

System.out.println(broken); // отрицательное или неожиданное значение
```

Cast говорит компилятору: "Я понимаю риск". Он не делает преобразование безопасным.

## `double` в `int`

При преобразовании `double` в `int` дробная часть отбрасывается.

```java
double price = 99.99;
int rounded = (int) price;

System.out.println(rounded); // 99
```

Это не математическое округление. Это отсечение.

Если нужно округлить:

```java
double value = 99.99;
long rounded = Math.round(value);

System.out.println(rounded); // 100
```

## Numeric promotion

**Numeric promotion** можно перевести как "числовое продвижение". Это правило, по которому Java поднимает маленькие числовые типы до `int` при вычислениях.

Пример:

```java
byte a = 10;
byte b = 20;
// byte sum = a + b; // ошибка
int sum = a + b;
```

Почему ошибка? Потому что выражение `a + b` имеет тип `int`, даже если оба операнда `byte`.

Если очень нужно получить `byte`, нужен cast:

```java
byte sum = (byte) (a + b);
```

Но в обычном коде лучше хранить результат в `int`.

То же касается `short`:

```java
short x = 100;
short y = 200;
int result = x + y;
```

## Тип результата в выражении

Java выбирает тип результата по операндам.

```java
int a = 10;
int b = 4;
System.out.println(a / b); // 2
```

Оба числа `int`, значит результат тоже `int`. Дробная часть отбрасывается.

Чтобы получить дробный результат, хотя бы один операнд должен быть `double`.

```java
int a = 10;
int b = 4;
System.out.println((double) a / b); // 2.5
System.out.println(a / 4.0);        // 2.5
```

Важно, где стоит cast:

```java
int sum = 5;
int count = 2;

double wrong = (double) (sum / count); // 2.0
double right = (double) sum / count;   // 2.5
```

В первом случае деление уже произошло как `int / int`.

## Суффиксы `L` и `f`

### `L` для `long`

```java
long views = 5_000_000_000L;
```

Без `L` компилятор считает целочисленный литерал типом `int`, если возможно.

### `f` для `float`

```java
float rating = 4.9f;
```

Без `f` дробный литерал считается `double`.

## Переполнение

Переполнение происходит, когда результат не помещается в тип.

```java
int max = Integer.MAX_VALUE;
System.out.println(max);      // 2147483647
System.out.println(max + 1);  // -2147483648
```

Java не выбрасывает исключение при переполнении `int`. Значение "оборачивается" по диапазону.

**Важно:** переполнение целых чисел может не остановить программу и не показать ошибку. Код продолжит работать, но с неверным значением. Поэтому большие суммы, идентификаторы и результаты умножения нужно проверять особенно внимательно.

Это опасно в расчетах:

```java
int price = 2_000_000_000;
int quantity = 2;
int total = price * quantity;

System.out.println(total); // неверный результат
```

Исправление:

```java
long price = 2_000_000_000L;
int quantity = 2;
long total = price * quantity;
```

Важно: недостаточно присвоить результат в `long`, если само умножение происходит как `int`.

Плохой пример:

```java
int price = 2_000_000_000;
int quantity = 2;
long total = price * quantity; // переполнение уже случилось
```

Лучше:

```java
int price = 2_000_000_000;
int quantity = 2;
long total = (long) price * quantity;
```

## Подчеркивания в числах

Для читаемости можно использовать `_` внутри числовых литералов.

```java
int oneMillion = 1_000_000;
long userId = 9_000_000_000L;
```

Это не меняет значение, только помогает читать число.

## Практический пример: расчет доставки

Допустим, доставка зависит от веса в граммах и расстояния в километрах.

```java
public static long calculateDeliveryPriceInCents(int weightGrams, int distanceKm) {
    long basePrice = 20_000L;
    long weightPart = (long) weightGrams * 3;
    long distancePart = (long) distanceKm * 150;

    return basePrice + weightPart + distancePart;
}
```

Почему `long`?

- итоговая сумма может стать большой;
- деньги удобнее хранить в минимальных единицах;
- умножение явно переводится в `long`, чтобы избежать переполнения.

## Частые ошибки

### Ошибка 1. Cast после целочисленного деления

```java
double result = (double) (5 / 2); // 2.0
```

Исправление:

```java
double result = (double) 5 / 2; // 2.5
```

### Ошибка 2. Умножение переполнилось до присваивания

```java
int a = 2_000_000_000;
int b = 2;
long result = a * b;
```

Исправление:

```java
long result = (long) a * b;
```

### Ошибка 3. Cast используется как "починка"

```java
int value = (int) 3_000_000_000L;
```

Код компилируется, но значение становится неверным. Cast не проверяет, помещается ли число.

### Ошибка 4. `float` используется без причины

```java
float tax = 0.13f;
```

Если готовая библиотека, задача или уже написанный код не требует `float`, в учебных и обычных прикладных задачах обычно выбирают `double`.
